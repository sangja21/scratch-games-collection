# 💰 복리 카드 게임

자산 1,000원으로 시작해 **10턴 안에 자산을 10배(10,000원)** 로 만들어 보세요. 매 턴 화면에 나타나는 **이자율 카드 3장** 중 하나를 골라 자산을 굴리는데, 카드마다 **이자율 `r`** 과 **성공 확률 `p`** 가 다릅니다. 안전한 +5% 카드만 골라서는 절대 10배에 도달할 수 없습니다 — 어느 정도 위험을 감수해야 한다는 사실을 손으로 체감하는 게임입니다.

> **학습 포인트** — `A = P · (1 + r)^n` 이 매 턴 "× (1+r)" 한 번이라는 사실, 그리고 "10배까지 평균 r 이 얼마나 필요한가?" 가 곧 **n번째 루트 = 분수 지수** 라는 사실을 동시에 학습합니다.

## 🎮 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 학습 / 의사결정 카드 게임 |
| 핵심 메커닉 | 매 턴 카드 1장 선택 → 자산이 곱셈으로 변함 |
| 학습 주제 | 복리 `A = P · (1+r)^n`, 기하평균, 기댓값 |
| 권장 학년 | 중3 ~ 고2 (지수와 로그 단원) |
| 한 판 길이 | 약 1 ~ 2분 (10턴) |

## 🕹 조작법

| 입력 | 동작 |
|------|------|
| 🟢 초록 깃발 | 게임 시작 (자산 1000원, 턴 1/10 로 초기화) |
| 카드 클릭 | 그 카드를 선택해 자산을 굴림 (다른 두 카드는 사라짐) |
| 다시 초록 깃발 | 처음부터 재시작 |

## 📺 화면 구성

```
┌────────────────────────────────────────┐
│ 자산: 1,234        턴: 3/10             │  ← HUD (상단)
│ 목표: 10,000     필요r(%): 25.7         │
│ 메시지: 성공! 자산 × (1+r)               │
├────────────────────────────────────────┤
│                                        │
│   ┌────────┐ ┌────────┐ ┌────────┐     │
│   │r=5% /  │ │r=15% / │ │r=50% / │     │
│   │p=100%/ │ │p=60% / │ │p=30% / │     │  ← 카드 3장
│   │E=+5.0% │ │E=+1.0% │ │E=-10%  │     │     (클론)
│   └────────┘ └────────┘ └────────┘     │
│                                        │
│       [ 다음 턴 ▶ ]   (자동 진행)       │
└────────────────────────────────────────┘
```

각 카드에는 세 가지 정보가 표시됩니다.

- **`r`** — 성공 시 자산 증가율 (예: `r=15%` → 자산 × 1.15)
- **`p`** — 성공 확률 (예: `p=60%` → 100명 중 60명만 성공)
- **`E`** — 기댓값 `E[r] = p · r + (1 − p) · (−50%)` (실패하면 자산이 절반으로!)

## 🃏 카드 풀 (총 8종)

매 턴 아래 풀에서 무작위로 3장을 뽑습니다.

| 이자율 `r` | 확률 `p` | 기댓값 `E[r]` | 성격 |
|-----------|---------|--------------|------|
| +5%  | 100% | +5.0%  | 안전 채권 |
| +8%  |  95% | +5.1%  | 우량 회사채 |
| +12% |  85% | +2.7%  | 안정 주식 |
| +20% |  70% | −1.0%  | 성장 주식 |
| +35% |  55% | −3.5%  | 모험 자산 |
| +50% |  40% | −10.0% | 투기 |
| +100%|  25% | −12.5% | 도박 |
| −3%  | 100% | −3.0%  | 현금 (인플레 패널티) |

> 모든 위험 카드의 기댓값은 음수입니다. 그런데도 10배 달성을 노리려면 어쩔 수 없이 위험 카드를 골라야 하는 순간이 옵니다 — 이게 **복리의 양면성** (큰 r → 큰 성장 / 큰 r → 큰 변동)을 체감하는 핵심입니다.

## 🧮 수학 배경

### 1. 복리는 곱셈의 누적

```
A = P · (1 + r)^n
```

게임에서 매 턴 카드를 고른다 = `× (1 + r)` 를 한 번 적용한다. 10턴 = 곱셈 10번. **거듭제곱은 반복된 곱셈** 이라는 정의가 게임 진행 그 자체입니다.

### 2. "필요 r" HUD 의 비밀 — n번째 루트

화면 상단에 항상 보이는 `필요r(%)` 변수는 **남은 턴 동안 매 턴 평균 몇 % 씩 늘려야 목표에 도달하는지** 를 보여줍니다.

```
필요r = (목표 / 현재자산)^(1 / 남은턴) − 1
```

Scratch 에는 `^` (거듭제곱) 블록이 없으므로, 코드는 다음 항등식으로 계산합니다.

```
x^(1/n) = e^( ln(x) / n )
```

`operator_mathop` 의 `ln` 블록과 `e ^` 블록을 합성한 것이죠. **n번째 루트 = 분수 지수 = 자연로그/지수의 합성** 이라는 사실이 HUD 한 줄에 다 들어 있습니다.

### 3. 기댓값 ≠ 도착값

각 카드의 `E` 값은 한 번 굴렸을 때 평균적으로 자산이 몇 %p 변하는지를 알려줍니다. 하지만 **복리는 평균이 아니라 곱** 입니다. 똑같은 카드 두 번 굴려 한 번 성공(+20%) · 한 번 실패(−50%) 하면:

```
산술평균: ( +20% + (−50%) ) / 2 = −15%
실제 결과: × 1.20 × 0.50 = × 0.60   (−40% 손실!)
```

이 차이가 곧 **기하평균 < 산술평균** 이라는 핵심 부등식의 출발점입니다.

### 4. 안전 카드만 10번 = 약 1.6배

`+5% × 10턴` 의 결과:

```
1000 · (1.05)^10 ≈ 1,629
```

10배가 안 됩니다. **목표 10,000 에 도달하려면 평균 약 25.9% 의 성장률**이 필요합니다 — 게임 시작 직후 HUD 의 `필요r(%)` 가 `25.9` 로 표시되는 이유입니다.

## 🛠 내부 구조 (스프라이트 2개)

| 스프라이트 | 역할 | 주요 동작 |
|-----------|------|----------|
| `Stage` | 배경 + 전역 변수 + HUD 갱신 루프 | 깃발 클릭 시 초기화, 매 0.15초 `필요r(%)` 재계산 |
| `버튼` | 부모 = "다음 턴 ▶" 버튼 / 클론 3개 = 카드 | 클론은 카드 코스튬으로 전환, 클릭 시 자산 갱신 + 다음 턴 trigger |

스프라이트 하나가 **두 코스튬** (`버튼` / `카드`) 을 가지고, 부모는 버튼으로 보이고 클론은 카드로 변신하는 구조로 단순화했습니다.

### 변수 (Stage)

| ID | 한국어 | 의미 |
|----|--------|------|
| `varP001` | 자산 | 현재 자산 |
| `varGoal002` | 목표 | 10,000 |
| `varTurn003` | 턴 | 현재 턴 (1..10) |
| `varMaxTurn004` | 최대턴 | 10 |
| `varState005` | 게임상태 | 1 진행 / 0 종료 |
| `varResult006` | 결과 | 1 승 / −1 패 / 0 진행 |
| `varNeedR007` | 필요r(%) | 매 0.15초 자동 갱신되는 학습 HUD |
| `varLastMsg008` | 메시지 | 마지막 결과 텍스트 |

### 메시지 (broadcast)

- `게임시작` — 깃발 클릭 시
- `새턴` — 새 턴 시작, 카드 3장 spawn 신호
- `카드선택됨` — 카드 클릭 직후, 나머지 카드 클론 삭제
- `게임종료` — 승리 또는 10턴 만료

## 🧪 권장 학습 흐름

1. **첫 판 — 안전만 골라보기.** +5% / +8% 카드만 골라 10턴 끝까지 가본다. 결과는 약 1,500 ~ 1,700. **10,000 에 한참 못 미친다는 사실** 을 직접 확인.
2. **두 번째 판 — `필요r(%)` HUD 를 관찰하기.** 매 턴 이 값이 어떻게 바뀌는지, 카드 선택 후 어떻게 늘어나거나 줄어드는지를 본다.
3. **세 번째 판 — 위험 카드 도전.** +50% / +100% 카드를 골라본다. 한 번 실패 = 자산 절반. 회복하려면 두 배 이상 늘려야 한다 (`× 0.5 × 2 = × 1`).
4. **마지막 — 기댓값 추리.** 모든 카드의 `E` 가 음수인데도 왜 게임이 가능한가? → "기대값이 음수여도 한 번에 큰 보상" 이 복리에서는 통하는 이유를 토론.
