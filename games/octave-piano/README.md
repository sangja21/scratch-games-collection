# 🎹 옥타브 피아노

화면에 C4~B5 흰건반 14개가 펼쳐져 있다. 매 라운드 두 건반 위에
**노란 별** 이 떠오르고 두 음이 차례로 들린다. 두 음이
**같은 음의 다른 옥타브** (예: C4와 C5) 인지 판단해서

- **O** 키 → "옥타브 맞음"
- **X** 키 → "옥타브 아님"

을 누른다. 60초 안에 점수를 쌓아라.

> **학습 포인트** — *옥타브 = 주파수 ×2* 이고, 음악은 본질적으로 **로그 스케일**.
> 화면의 **비율** = f₂/f₁ 가 정확히 2.00 이고 **log2** = log₂(f₂/f₁)
> 가 정확히 1.00 일 때만 한 옥타브. Scratch 의 `log` 블록은 밑 10
> 상용로그뿐이라, 게임 코드 안에서 **밑 변환 공식**
> `log₂(x) = log₁₀(x) / log₁₀(2)` 을 직접 쓴다. 이 한 줄이 "왜 로그를
> 배우나" 의 답이 된다.

## 🎮 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 학습형 (지수/로그) — 음감 + 비율 인식 |
| 핵심 메커닉 | 두 음의 주파수 비를 보고 옥타브 여부 판정 |
| 학습 주제 | `f₂/f₁ = 2^k` (옥타브), `log₂(f₂/f₁)` 의 정수성 |
| 권장 학년 | 고1 (지수와 로그 단원) |
| 플레이 시간 | 60초 / 1판 |

## 🕹 조작법

| 입력 | 동작 |
|------|------|
| `O` 키 | "두 음이 한 옥타브 차이다" 라고 응답 |
| `X` 키 | "옥타브가 아니다" 라고 응답 |

정답: **점수 +1**, 오답: **점수 -1**. 응답 후 0.7초 뒤 다음 라운드.

```
┌──────────────────────────────────────────────┐
│ 점수: 7    비율: 2.00    시간: 42s            │  ← HUD (Stage 모니터)
│ 라운드: 8  log2: 1.00                         │
├──────────────────────────────────────────────┤
│                                              │
│                 ★         ★                  │  ← 두 강조 마커
│  ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐│
│  │C4│D4│E4│F4│G4│A4│B4│C5│D5│E5│F5│G5│A5│B5││  ← 14 흰건반
│  └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘│
```

- HUD 상단: `점수`, `비율` (= f₂/f₁), `시간`
- HUD 좌측: `라운드`, `log2` (= log₂(f₂/f₁))

## 🎯 라운드 진행

- **K1** (왼쪽 건반 인덱스 1..7) 무작위 선택
- 50% 확률로 **옥타브 라운드** (K2 = K1 + 7, 정확히 한 옥타브 위)
- 50% 확률로 **비옥타브 라운드** (K2 = K1 + Offset, Offset ∈ {1..8}\{7}, 14 로 clamp)
- 따라서 한 라운드는 정답이 **O** 또는 **X** 둘 중 하나가 균등하게 등장

## 🧮 수학 배경

### 옥타브의 정의

음악에서 **한 옥타브** 는 주파수가 정확히 2배가 되는 음정.

```
  f₂ / f₁ = 2     (한 옥타브)
  f₂ / f₁ = 4 = 2² (두 옥타브)
  일반적으로 k 옥타브 차이  ⇔  f₂ / f₁ = 2^k
```

게임에서 `K2 - K1 = 7` (흰건반 7개 차이 = 한 옥타브) 일 때만 정확히
`비율 = 2.00`. 다른 거리에서는 `비율` 이 1.12, 1.50, 1.78 같은 *무리수*
가까운 값으로 나온다 (12음 평균율의 본질).

### 로그로 옥타브 보기

`f₂ / f₁ = 2^k` 양변에 밑 2 로그를 씌우면

```
  log₂(f₂ / f₁) = k
```

즉 **log₂ 값이 정수면 옥타브, 아니면 옥타브 아님**. 게임 HUD 의 `log2`
가 정확히 1.00, 2.00, 3.00 일 때만 정답이 **O**. 이 직관을 한 번
손으로 확인하면 *옥타브는 곱셈, 음정 거리는 로그* 라는 음악 수학이 몸에 박힌다.

### 밑 변환 공식 (이 게임의 진짜 학습 포인트)

Scratch 의 `log` 블록은 **상용로그(밑 10)** 만 가지므로, 밑 2 로그는
직접 계산해야 한다.

```
  log₂(x)  =  log₁₀(x) / log₁₀(2)
```

이 한 줄이 `build.py` 의 새 라운드 진입 블록에 그대로 들어있다 — 이
계산이 끝나야 `log2` HUD 가 갱신된다. "log 의 밑을 바꾸려면 같은 밑의
로그를 분모에 깔면 된다" 는 사실이 게임 메커닉의 골격.

## 🔊 사운드

| 파일 | 용도 | 합성 방식 |
|------|------|----------|
| `assets/note_low.wav` | 왼쪽 건반 (낮은 음) | C4 기준 사인파 + 배음, PITCH 효과로 옥타브 조절 |
| `assets/note_high.wav` | 오른쪽 건반 (높은 음) | C5 기준 사인파 + 배음 |
| `assets/correct.wav` | 정답 ✓ 화음 (C5 + E5) | 부드러운 사인 차임 |
| `assets/wrong.wav` | 오답 ✗ 저음 버즈 | 150/155Hz 디튠 스퀘어 |

**PITCH 효과**: Scratch 의 `sound_seteffectto PITCH` 는 *반음 × 10 단위*
이므로, K번째 흰건반의 반음 거리(0,2,4,5,7,9,11,12,...,23) 를 곱해 10 한 값을
세팅하면 정확한 음을 낸다. 두 사운드는 각각 *C4 / C5* 를 기준점으로
합성됐기 때문에 PITCH=0 일 때 그 음이 그대로 들린다.

## 🛠 내부 구조 (스프라이트 5개)

| 스프라이트 | 역할 |
|-----------|------|
| `Stage` | 피아노 배경, 변수 초기화, 60초 타이머, 라운드 로직 (K1/K2 선택, 비율/log2 계산), 정답 판정, 게임 종료 |
| `Highlight` | 클론 2개로 두 별 마커를 띄움. Side(=1 왼쪽, 2 오른쪽) 변수로 위치 분기 |
| `Sound` | PITCH 효과로 두 음 재생 + 정/오답 효과음. K1/K2 → 반음수 매핑 (if-블록 14개 × 2) |
| `Result` | 정/오답 직후 0.5초간 ✓ / ✗ 아이콘 표시 |
| `GameOver` | 시간 0 도달 시 중앙에 "게임 종료" 배너 |

### 데이터 흐름

```
▶ 깃발
  ├─ Stage: 변수 초기화 + L_FREQ (14개 주파수), L_NAME (14개 음이름) 채움
  └─ broadcast "시작"
       ├─ Stage: broadcast "새라운드" (첫 라운드)
       └─ Stage: 1초마다 시간--; 시간=0 → broadcast "게임종료"

  [새라운드 처리 (Stage)]
       1. K1 = random 1..7
       2. 동전 던지기로 옥타브 vs 비옥타브 결정
       3. F1=L_FREQ[K1], F2=L_FREQ[K2]
       4. 비율 = round(F2/F1 * 100) / 100
       5. log2 = round(log10(비율) / log10(2) * 100) / 100
       6. 정답 = (K2-K1 = 7 ? 1 : 0)
       7. wait 0.2 → broadcast 저음재생 → wait 0.45 → broadcast 고음재생
                                 ↓
                ├─ Sound: K1 → Half (반음수) → PITCH 세팅 → note_low 재생
                ├─ Sound: K2 → Half → (Half-12)*10 PITCH → note_high 재생
                └─ Highlight: 클론 2개 생성, 별을 두 키 위에 띄움

  [키 입력]
       O → broadcast "응답옥타브" → Stage: 정답==1 ? +1 ✓ : -1 ✗
       X → broadcast "응답아님"   → Stage: 정답==0 ? +1 ✓ : -1 ✗

  [응답 후]
       Stage: wait 0.7 → broadcast 새라운드 (상태=1 이면)
```

## 🔧 다시 빌드하기

```bash
cd games/octave-piano
python3 build.py
# → 옥타브_피아노.sb3 갱신
```

사운드 파일이 `assets/` 에 없으면 빌드 시점에 **사인파 합성으로 자동 생성**
된다 (Python 표준 `wave` 모듈만 사용, 외부 라이브러리 X).

`build.py` 안에서 만져볼 수 있는 값:

- `KEY_FREQS` — 14개 흰건반 주파수 표 (12-TET 평균율). 순정률로 바꾸면 옥타브가 정확히 2.0, 5도가 정확히 1.5 가 되는 비교 실험 가능
- `KEY_SEMITONES` — 반음 거리. PITCH 매핑에 사용
- `GAME_TIME` — 게임 한 판 시간(초, 기본 60)
- `BG_SVG`, `HIGHLIGHT_SVG`, `CORRECT_SVG`, `WRONG_SVG`, `GAME_OVER_SVG` —
  SVG 텍스트로 직접 들어있음. 색·크기 자유롭게 수정
