# 🎧 데시벨 DJ

두 트랙의 강도(I₁, I₂)가 화면에 표시되면, **슬라이더로 두 소리의 합산 dB** 를 추측해 ±1.0 dB 이내로 맞추는 로그 학습 게임입니다. 60초 안에 가장 많이 통과시키세요.

> **학습 포인트** — 데시벨은 로그 스케일이라 `dB₁ + dB₂ ≠ dB_total` 입니다. 60 dB + 60 dB 는 120 dB 가 아니라 **63 dB** 일 뿐입니다. 슬라이더로 직접 맞춰 보며 이 사실을 손에 익힙니다.

## 🎮 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 학습 게임 / 슬라이더 퀴즈 |
| 핵심 메커닉 | 두 강도(I₁, I₂)의 합산 dB 추측 |
| 학습 주제 | `dB = 10·log₁₀(I/I₀)`, 로그의 합과 비합산성 |
| 제한 시간 | 60초 |
| 권장 학년 | 고등학교 1학년 (지수·로그 단원, 로그의 응용) |

## 🕹 조작법

| 입력 | 동작 |
|------|------|
| `내답dB` 슬라이더 (좌측 하단) | 합산 dB 추측값 입력 (0 ~ 100, 0.1 단위) |
| `Space` | 통과 시도 — 차이가 1.0 dB 이내면 정답 (+1점, 새 라운드) |
| `힌트 ▼` 버튼 | 현재 정답 dB 노출 (학습용, 점수 감점 없음) |

- 왼/오른쪽 음표(♪/♫)가 통통 튀며 두 트랙이 동시에 재생되는 느낌
- 화면 하단 가운데에 판정 메시지(`looks_say`)가 떠다님
- 힌트 버튼을 누르면 화면 중간에 "정답: XX.X dB" 가 표시됨

## 🎯 라운드 / 시나리오

| 라운드 | I₁ | I₂ | 정답 dB | 학습 의도 |
|--------|----|----|----------|----------|
| 1 | 10,000 | 10,000 | ≈13.0 | 같은 dB(=10) 두 개 → +3 만 추가 |
| 2 | 100,000 | 100,000 | ≈23.0 | 위와 같은 패턴, 다른 절대값 |
| 3 | 1,000,000 | 100 | ≈30.0 | 강도 비 10⁴ → 큰 쪽이 거의 결정 |
| 4 | 100,000 | 50,000 | ≈21.8 | 2:1 비율 → +1.8 dB |
| 5 | 1,000 | 1,000 | ≈3.0 | dB=0 두 개 합쳐도 0 이 아님! |
| 6+ | 10^(랜덤 3..7) | 10^(랜덤 3..7) | 매번 다름 | 자유 라운드 |

기준 강도 `I₀ = 1000` 으로 고정.

## 🧮 학습 포인트 — 데시벨이 더해지지 않는 이유

### 1. 데시벨의 정의

$$ \text{dB} = 10 \cdot \log_{10}\!\left(\dfrac{I}{I_0}\right) $$

- 강도(I)가 10배 → dB는 +10
- 강도가 100배 → dB는 +20
- 강도가 2배  → dB는 +3.01 (왜냐하면 `log₁₀(2) ≈ 0.301`)

### 2. 두 소리의 합산

$$ \text{dB}_{\text{total}} = 10 \cdot \log_{10}\!\left(\dfrac{I_1 + I_2}{I_0}\right) $$

이건 `dB₁ + dB₂` 가 **아닙니다**. 강도는 합쳐지지만, 로그는 그렇지 않습니다. 같은 dB 두 개를 더하면 강도만 2배 → +3 dB.

| 합칠 두 dB | 결과 dB | 이유 |
|------------|---------|------|
| 60 + 60 | ≈ 63 | 강도 2배 → +3 |
| 80 + 60 | ≈ 80.04 | 강도 비 100:1, 큰 쪽이 거의 결정 |
| 70 + 70 + 70 | ≈ 74.77 | 같은 dB 3개 → +10·log₁₀(3) ≈ +4.77 |

이게 라이브 사운드 엔지니어가 무대 위 마이크 8개를 켜도 "그냥 8배 시끄러워지는 게 아닌" 이유입니다.

### 3. Scratch 의 `log` 블록

이 게임은 `operator_mathop` 블록의 `OPERATOR: "log"` 를 3 번 사용합니다 — dB₁, dB₂, 그리고 합산 정답 dB. Scratch 의 `log` 는 **상용로그(밑 10)** 이라 데시벨 정의에 그대로 맞습니다.

## 🔧 다시 빌드하기

```bash
cd games/decibel-dj
python3 build.py
```

`데시벨_DJ.sb3` 가 생성됩니다. https://scratch.mit.edu 의 "내 작업실" 에 업로드하거나 Scratch 데스크탑에서 열어주세요.

`build.py` 안에서 바꿔보면 재미있는 것들:

- `FIXED` 리스트 — 라운드 1~5 의 (I₁, I₂) 직접 편집
- 허용 오차 1.0 dB (`if V_DIFF < 1.0`) — 더 엄격하게(0.5) 또는 더 너그럽게(2.0)
- `V_DB_USER` monitor 의 `sliderMin`/`sliderMax` — 슬라이더 범위 변경
- 제한 시간 60초 (Stage 의 V_TIME 초기값)

## 🧪 권장 학습 흐름

1. **라운드 1·2**: 같은 강도 두 개를 합치는 경우. dB 가 +3 만 늘어남을 발견하세요.
2. **라운드 3**: 한쪽이 압도적일 때 작은 쪽은 거의 무시됨을 확인.
3. **라운드 5**: dB=0 인 트랙을 두 개 합쳐도 dB=0 이 아니라 ≈3.0 입니다. "dB 0 ≠ 무음" 이 핵심.
4. **라운드 6+**: 무작위 강도에서도 같은 직관이 통하는지 시험.
5. 막힐 때마다 `힌트 ▼` 버튼을 누르면 정답이 표시됩니다 — 답을 보고 슬라이더와 비교해 "왜 그 값이 되는지" 머릿속으로 식을 떠올려 보세요.
