# 용사 진격전

> **용사 한 명으로 오른쪽 적 성을 향해 진격한다.** 앞으로 밀고 나가면 `진격도`가 차오르고, 적 병사들이 오른쪽에서 **웨이브로 몰려온다**. `스페이스`(또는 마우스 클릭)로 검을 휘둘러 베고(잡졸은 1~2타), `X`로 광역 스킬(쿨다운 게이지가 다 차야 발동)을 쓴다. 밀리면 `←`로 후퇴하며 싸울 수도 있다. 진격도가 가득 차면 **적 성문이 밀려 들어오고 → 성문을 부수는 공성 피날레 → 스테이지 클리어 → 강화 택1**(공격력/체력/이동속도/스킬쿨). 스테이지가 오를수록 적이 **곱하기로**(지수) 세진다. 용사 체력이 0이면 GAME OVER, 처음부터 다시. **도달한 스테이지가 곧 점수다.**

로그 나이트·마법 생존·캐슬 디펜스 계보의 1인 진격형 횡스크롤 액션 로그라이트다. 이 게임의 진짜 재미는 **"코드의 숫자를 직접 바꿔보며 내 게임으로 만드는 것"** 이다. 용사의 공격력·체력·이동속도·검격/스킬 쿨과 범위, 진격 속도와 스테이지 길이, 적 3종의 체력/속도/공격력, 그리고 **난이도 곡선을 통째로 정하는 `적성장배율`** 까지 **모든 값이 한글 변수 35개로 정리**되어 있어, 숫자 하나만 바꿔도 게임이 확 달라진다. 아래 [개조 손잡이](#️-개조-손잡이-바꿔보세요)와 [개조 미션 사다리](#-개조-미션-사다리)를 따라 하나씩 도전해 보자.

## 조작법

| 입력 | 동작 |
|------|------|
| `←` / `→` | 좌우 이동. **앞(진격기준X 너머)에서 `→`를 누르면 진격도가 차오른다**(적 성으로 밀고 가기). 밀리면 `←`로 후퇴하며 싸운다 |
| `↑` | 점프(회피). 중력으로 낙하하며, 착지하면 다시 점프 가능 |
| `Space` 또는 **마우스 클릭** | **검격** — 용사 앞에 반달 베기 히트박스가 잠깐 나타나 닿은 적·성문에 `공격력`만큼 피해. 잡졸은 1~2타에 쓰러진다 |
| `X` | **광역 스킬** — 용사 중심에 회전베기 링이 퍼져 범위 안 적·성문 전부에 `스킬공격력`. **스킬 게이지가 가득 찼을 때만**(좌상단 ⚡ 바 READY) 발동 |
| `1` / `2` / `3` / `4` | 스테이지 클리어 후 강화 카드 고르기 (`1` 공격력+ · `2` 체력+회복 · `3` 이동속도+ · `4` 스킬쿨↓) — 고르는 즉시 다음 스테이지 |
| 초록 깃발 | 시작 / 다시 시작 (능력치·스테이지 기본값으로 리셋) |

> **📊 상태가 항상 눈에 보인다**: 좌상단에 **체력바**(❤️ 11단 게이지)·**스킬게이지**(⚡ 충전율, 가득이면 READY)·**진격 진행바**(🏰 적 성까지 남은 거리) 3종이 상시 표시되고, `스테이지`(점수) 모니터가 하나 있다. 마우스 검격은 **클릭 폴링**으로 감지해(`이 스프라이트를 클릭했을 때` 미사용) 다른 스프라이트가 가려도 동작한다.
>
> **⚔️ 모든 공격이 눈에 보인다**: 검격은 검을 휘두르는 모션 + 반달 이펙트, 스킬은 회전베기 링, 적 처치는 폭발, 피해는 흰 데미지 숫자(골드는 금색)로 뜬다.

## 게임 흐름

1. **시작** — 초록 깃발을 누르면 모든 능력치가 기본값으로 맞춰지고, 들판과 멀리 검은 성이 그려진다. 묵직한 뿔피리와 함께 1스테이지가 시작되고, 곧 오른쪽에서 적 잡졸이 몰려온다.
2. **진격 (treadmill 모델)** — 배경을 실제로 스크롤하는 대신 추상 `진격도`로 진행을 표현한다. 용사가 화면 앞쪽(`진격기준X`=60 너머)에서 `→`를 누르면 진격도가 `진격속도`만큼 차오르고, **상단 진격 진행바가 채워진다**. 적은 항상 오른쪽 끝에서 스폰돼 용사를 향해 전진하므로 "앞에서 밀려오는" 진격감이 난다. 이 방식은 Scratch가 확실히 잘하는 것 안에서 진행을 그려 스크롤 버그를 원천 회피한다.
3. **전투 (전선 감각)** — 적이 접촉하면 용사가 피해를 입고 잠깐 무적이 된다. 검격(`Space`/클릭)으로 앞의 적을 베고, 위기엔 광역 스킬(`X`)로 무리를 한 번에 쓸어버린다. 밀리면 `←`로 후퇴하며 싸울 수 있어 전선을 밀고 당기는 감각이 산다. 적 3종은 색·크기로 구분된다 — 잡졸(작고 빠름·1~2타), 전사(중간·약 3타), 기사(크고 단단·느림).
4. **공성 피날레** — 진격도가 `스테이지길이`(900)에 닿으면 **적 성문이 오른쪽에서 슬라이드 인**하고 웨이브 스폰이 멈춘다(공성 단계). 이제 검격·스킬로 성문을 두들긴다 — 성문은 체력이 깎이며 **온전 → 균열 → 잔해** 코스튬으로 무너지고, 체력이 0이 되면 '쾅!' 파괴 피날레와 함께 **스테이지 클리어**. 짧고 굵은 클라이맥스다.
5. **강화 → 다음 스테이지** — 클리어하면 화면 가운데 강화 카드가 뜬다. `1 공격력+`, `2 체력+`(최대체력 증가 + 완전 회복), `3 이동속도+`, `4 스킬쿨↓` 중 하나를 키로 고르면 **곧바로** 다음 스테이지가 시작된다(대기 없음).
6. **점점 어려워짐** — 스테이지가 오를수록 적 종류가 늘고(1=잡졸만 → 2~3=잡졸·전사 → 4+=잡졸·전사·기사 섞임), 적 체력과 성문 체력이 **× 적배율**로 곱해진다. 웨이브 간격도 짧아져 후반엔 우르르 몰려온다.
7. **죽으면 처음부터** — 용사 체력이 0이 되면 GAME OVER 배너가 뜨고 도달한 스테이지(점수)가 보인다. 초록 깃발을 다시 누르면 1스테이지부터 다시 시작한다.

> **왜 적이 안 버벅일까(설계 핵심)**: 적 클론은 **다른 클론을 전혀 참조하지 않는다** — 오직 "용사를 향해 전진 + 접촉하면 피해 + 검격/스킬 히트박스에 닿으면 죽음"만 한다. 여러 유닛이 서로를 인식하며 자율 전투하는 다대다 로직은 Scratch의 한계 밖(멀뚱거리거나 느려짐)이라 의도적으로 버렸다. 그래서 **리스트 0개, 클론끼리 상호작용 0개** — 단순함이 곧 부드러운 손맛이다.

## ⚙️ 개조 손잡이 (바꿔보세요!)

게임을 바꾸는 모든 숫자는 **무대(Stage)의 초록 깃발 스크립트 한 곳**에 모여 있다. 코드 어디에도 숨겨진 숫자가 없으니, 여기 **35개** 손잡이만 돌리면 게임이 달라진다.

> 찾는 법: Scratch 에서 **무대**를 클릭 → 코드 탭 → 맨 위 `초록 깃발을 클릭했을 때` 묶음을 보면 `공격력 을(를) 3 로 정하기` 같은 블록이 쭉 있다. 그 숫자만 바꾸면 끝! (노란 코멘트 "🛠️ 개조 손잡이" 가 붙어 있다.)

### ⭐ 게임의 심장 — 난이도 곡선 & 챕터 템포

| 손잡이 (변수) | 기본값 | 무슨 뜻? / 바꾸면 일어나는 일 |
|---------------|:------:|-------------------------------|
| **적성장배율** | 1.22 | **★ 게임의 심장.** 스테이지가 1 오를 때마다 적 체력·성문 체력이 이만큼 **곱하기**로 세진다. 실제 = `기본 × 적성장배율^(스테이지−1)`. `1.1`=순한 맛, `1.5`=지옥 — **한 숫자로 난이도 곡선 전체가 바뀐다!** |
| **진격속도** | 1.8 | `→`를 누를 때 한 틱에 차는 `진격도`. **한 스테이지의 길이감(챕터 템포)** — 올리면 성문에 빨리 도달, 내리면 더 오래 진격 |

| **최대체력** | 6 | 용사 시작 체력. 올리면 훨씬 오래 버틴다 |

## 🎯 개조 미션 사다리

쉬운 것부터 어려운 것까지. **하나씩 깨보면서 "코드를 망가뜨리고 고치는" 재미**를 느껴보자. 망가져도 괜찮다 — 초록 깃발만 다시 누르면 된다!

### 0층 — 숫자 하나 바꾸기
- **세진 용사**: `공격력` 을 `3` → `8` 로. 잡졸이 한 방에!
- **번개 이동**: `이동속도` 를 `6` → `10` 으로. 순식간에 전선을 오간다.
- **두부 멘탈**: `최대체력` 을 `6` → `2` 로. 한두 대만 맞아도 위험(난이도 급상승).
- **속사 검**: `검격쿨` 을 `8` → `3` 으로. 검을 따발총처럼 휘두른다.
- **광역 지배자**: `스킬범위` 를 `130` → `220` 으로, `스킬쿨` 을 `200` → `80` 으로. 스킬을 자주·넓게.
- **거북이 기사**: `기사_속도` 를 `1.0` → `0.3` 으로. 기사가 더 느릿.
- **긴 챕터 / 짧은 챕터**: `스테이지길이` 를 `900` → `1400`(길게) 또는 `400`(짧게) 으로. **스테이지가 너무 빨리 끝나면 올리자!**
- **순한 적 / 지옥 적**: `적성장배율` 을 `1.22` → `1.1`(쉬움) 또는 `1.5`(지옥) 로. **한 숫자로 난이도 곡선 전체가 바뀐다!**
- **물량 지옥**: `웨이브간격` 을 `1.2` → `0.4` 로, `최대적수` 를 `8` → `16` 으로.

### 1층 — 값을 식으로 묶기
- **진격할수록 세지는 검**: 용사 검격에서 `공격력` 대신 `공격력 + floor(진격도 / 300)` — 밀고 나갈수록 강해지게.
- **후반 보상**: 적 처치 골드를 `처치골드` → `처치골드 × 스테이지` 로.
- **묵직 점프**: `중력` 을 `(0 − 점프력) / 12` 로 점프력에 비례하게.

### 2층 — if 조건 더하기
- **최후의 발악**: 용사 피격에 `if 체력 = 1 : set 무적 = 60` — 위기 시 잠깐 무적.
- **광전사**: `if 체력 <= 2 : set 이동속도 = 10`(위험하면 빨라짐), 회복 시 복구.
- **엘리트 웨이브**: 스포너에 `if (스테이지 mod 4 = 0) : set 적생성종류 = 3` — 4의 배수 스테이지는 기사 러시.

### 3층 — 규칙 바꾸기
- **2단 점프**: `점프남음` 리필을 `1` → `2` 로(용사 착지·초기화), 공중 회피 강화.
- **랜덤 강화**: 강화 선택을 `pick random 1 to 4` 자동으로.
- **스킬로 성문 즉파**: 성문 피해에서 스킬이면 `스킬공격력 × 2`를 적용하는 조건 추가.

### 4층 — 새 메커닉
- **원거리 적(궁수)**: 적 4번째 종류로 화살 쏘는 궁수 추가(스프라이트 1개·적사격 방송) — 클론끼리는 여전히 상호작용 없음.
- **보스 스테이지**: `스테이지 mod 5 = 0` 이면 성문 대신 거대 보스(성문 스프라이트 재활용·체력 크게).
- **연속 콤보**: 검격을 짧게 연속 입력하면 3타째 데미지 2배(콤보 카운터 변수 추가).
- **골드 상점**: 스테이지 사이 골드로 능력치 구매(강화패널 확장).

## 📚 수업 포인트

직관적인 진격 액션 게임으로 포장돼 있지만, **핵심 가치는 "코딩 학습"** 이다. 모든 능력치가 한글 변수로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (4층) 새 적/메커닉을 추가하며 **변수·연산·조건·클론·에지 디텍션·타이머·broadcast** 를 게임을 망가뜨리고 고치며 체득한다. 부수적으로 **메커닉의 결과로** 자연스럽게 만나는 것:

- **지수 스케일링 (곱셈 성장)**: 적은 `기본 × 적성장배율^(스테이지−1)` 로 **곱하기**로 커지는데, 내 강화(공격력 +2씩)는 조금씩 따라간다. **직선으로는 못 따라잡고, 좋은 강화를 골라야 넘는 벽**을 몸으로 느낀다. `적성장배율` 한 숫자가 난이도 곡선 전체를 정한다는 걸 실험으로 발견한다(1.1이면 후반이 심심, 1.5면 몇 스테이지 못 가 벽). Scratch엔 지수 블록이 없어 `적배율`을 **repeat 곱셈 루프**로 계산하는 것도 볼거리.
- **"챕터 템포" 손잡이 (진격속도 × 스테이지길이)**: 한 스테이지에 걸리는 시간은 `스테이지길이 ÷ 진격속도`로 정해진다(진격도가 매 틱 진격속도만큼 차서 스테이지길이에 닿을 때까지). 그래서 **템포가 너무 빠르게 느껴지면 `스테이지길이`를 올리거나(예 900→1400) `진격속도`를 내리면**(1.8→1.2) 한 챕터가 길어져 적을 더 오래 상대하게 된다. 반대로 지루하면 스테이지길이를 줄인다. 두 숫자의 **나눗셈 관계**를 조절해 자기 취향의 게임 길이를 만드는 것이 이 게임의 튜닝 포인트다.
- **비율/게이지 (0~1 정규화)**: 체력바(`체력/최대체력`)·스킬게이지(`(스킬쿨−스킬타이머)/스킬쿨`)·진격바(`진격도/스테이지길이`)는 값을 0~1로 정규화한 뒤 `×10`으로 11단 코스튬을 골라 그린다 — 비율을 바로 눈에 보이는 막대로 바꾸는 법.
- **타이머·쿨다운·에지 디텍션**: 검격/스킬/무적이 매 틱 줄어드는 타이머로 관리되고, 키 입력은 "이번엔 눌렸는데 직전엔 안 눌렸다"(에지)를 봐서 한 번만 반응한다 — 꾹 눌러도 한 번만 나가는 이유.

> **🏗️ 아키텍처 눈여겨보기 — 단순함이라는 설계**: 이 게임은 **리스트 0개**로 돌아간다. 적은 각자 독립된 클론이고, 자기 체력·속도·공격력을 **자기 로컬 변수**로 들고 "용사에게 다가가 부딪히기"만 한다 — 다른 클론을 쳐다보지도 않는다. 데미지 판정은 용사 본체가 아니라 **`검판정`·`스킬판정` 별도 히트박스 스프라이트**가 맡아(휘두를 때만 나타남), 적·성문은 `touching 검판정/스킬판정`만 보면 된다. 복잡한 다대다 전투를 억지로 구현하는 대신 **Scratch가 확실히 잘하는 것 안에서 설계**해 부드러운 손맛을 얻는 것 — "무엇을 안 만들지 정하는 것"도 프로그래밍이다.
