# 🧪 pH 적정 실험실

> 비커 안 H⁺ 농도를 보고 산(↓) / 염기(↑) 한 방울씩 추가해 목표 pH 에 도달시키는 로그 학습 게임. 한 방울 = H⁺ ×2 또는 ×0.5 → pH 약 ±0.30 변화. "10배 변해야 pH 1 변함" 을 손으로 체감한다.

## 조작

| 입력 | 동작 |
|------|------|
| **빨간 ↓ 산 버튼 클릭** | H⁺ 농도를 2배로 증가 (= pH 약 0.30 ↓) |
| **파란 ↑ 염기 버튼 클릭** | H⁺ 농도를 0.5배로 감소 (= pH 약 0.30 ↑) |
| **보라 힌트 버튼 클릭** | 현재 라운드의 목표 pH 를 텍스트로 표시 |

## 게임 흐름

1. **초록 깃발** 클릭 → 라운드 1 시작.
2. 화면 상단 HUD 에 **현재 pH / 목표 pH / 사용한 방울 수 / 점수 / 라운드** 표시. 화면 좌측에 색상 pH 막대(0~14)와 두 마커(녹색 = 현재, 빨강 = 목표) 가 y 좌표로 위치를 알려줌. 중앙 비커는 pH 구간에 따라 색이 바뀜 (빨강 강산 → 노랑 약산 → 초록 중성 → 파랑 염기).
3. 산/염기 버튼을 클릭해 H⁺ 농도를 곱셈으로 조정. **`|현재 pH − 목표 pH| < 0.31`** 이면 정답 처리되어 점수 (= max(20 − 방울수, 1)) 가 가산되고 다음 라운드.
4. 라운드 5 까지 진행. 끝나면 게임오버 메시지로 누적 점수 표시.

라운드 별 시나리오:

| 라운드 | 초기 pH | 목표 pH | 의도 |
|--------|---------|---------|------|
| 1 | 3.0 | 6.0 | 강산 → 약산 (염기 약 10번) |
| 2 | 9.0 | 7.0 | 약염기 → 중성 (산 약 7번) |
| 3 | 1.0 | 5.0 | 강산 → 약산 (염기 약 13번) |
| 4 | 11.0 | 8.0 | 강염기 → 약염기 (산 약 10번) |
| 5 | 5.0 | 7.0 | 미세 조정 (염기 약 7번) |

## 학습 포인트

핵심 공식

```
pH = -log₁₀([H⁺])
```

는 Scratch 의 `(log ▼) of (H농도)` 블록을 부호 뒤집어 그대로 구현했다 (`build.py` 의 `build_stage_blocks()` → BR_RECALC 핸들러).

**왜 한 방울이 pH 를 0.3 만 바꿀까?**

산 한 방울 = `H농도 × 2` 이므로

```
pH_new = -log₁₀(2 · [H⁺]) = -log₁₀(2) - log₁₀([H⁺]) = pH_old - 0.301
```

즉 **곱셈** 으로 H⁺ 농도를 두 배 늘려도 **로그** 인 pH 는 약 0.30 만 줄어든다. pH 가 1 줄어들려면 H⁺ 농도가 정확히 10배 (= 두 배 곱하기 ≈ 3.3번) 늘어나야 한다. 라운드 3 의 "pH 1 에서 pH 5 까지" 는 H⁺ 농도를 10000 배 (= 0.5 곱하기 약 13.3번) 묽혀야 하는데, 게임을 플레이하면 정말로 약 13~14 번 클릭해야 도달한다. 이게 **곱셈 → 덧셈 변환** 이라는 로그의 본질이다.

또한 화면 좌측의 pH 막대는 눈금이 균등(등간격)이지만 **한 칸 = 10배 농도 차이** 이다. pH 3 과 pH 7 마커가 4칸 떨어져 있다고 농도 차이가 4배가 아니라 **10⁴ = 10000배** 라는 사실을 마커 두 개의 거리로 직관적으로 보여준다.
