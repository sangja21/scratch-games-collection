# ☢ 반감기 광산

방사능 광물이 매 `T` 초마다 **질량이 절반으로** 줄어드는 광산.
"지금 캘까, 더 기다릴까" — 클릭 한 번이 점수가 되지만, 너무 늦으면
질량은 절반의 절반의 절반… 으로 사라진다. 60초 안에 최대 질량 채굴.

> **학습 포인트** — 지수 *감쇠* 의 직관. 광물 하나하나가 자기만의
> 반감기 `T` 와 시작 질량 `N₀` 를 가지고 매 0.1초마다
> `N(Δt) = N₀ · (1/2)^(Δt/T)` 로 갱신된다. 박테리아 디펜스의
> *증가* 와 정반대 — 같은 지수함수 가족, 부호만 뒤집힌 모습이다.

## 🎮 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 지수 감쇠 채굴 (클릭 액션 + 수학) |
| 핵심 메커닉 | 광물 4종이 각자 다른 반감기 `T (1~5s)` 로 동시에 감쇠. 클릭 = 현재 질량을 점수로 |
| 학습 주제 | 지수 감쇠 `N(t) = N₀·(1/2)^(t/T)`, `0.5^x` 의 `e^` 합성, 밑 변환 로그 (보너스 HUD) |
| 권장 학년 | 고1 (지수와 로그 단원 — 반감기 응용) |
| 플레이 시간 | 60초 × 원하는 만큼 반복 |

## 🕹 조작법

| 입력 | 동작 |
|------|------|
| 마우스 클릭 (광물) | 광물 채굴 — 현재 질량 `round(N)` 만큼 점수에 가산, 처치 카운트 +1 |

광물은 시간이 지날수록 작아지고 어두워진다. 작아질수록 클릭 hit-box 도
같이 작아지므로 *늦으면 클릭조차 힘들어진다*. 5% 이하로 감쇠하면
자동 소멸 (점수 0). 빈자리는 1.5초 안에 새 광물이 채움.

## 🧮 수학 배경

### 지수 감쇠식

반감기가 `T` 인 물질의 시각 `t` 에서의 질량은

```
  N(t) = N₀ · (1/2)^(t/T)
```

`T = 2초`, `N₀ = 100` 이라면:
- t = 2s → 50 (절반)
- t = 4s → 25 (절반의 절반)
- t = 6s → 12.5
- t = 10s → 3.125 — 곧 사라진다

같은 시간을 기다려도, **반감기가 작은 광물일수록 더 급격히 줄어든다**. T=1 짜리는 4초만에 1/16, T=5 짜리는 4초가 지나도 약 57% 보존.

### `0.5^x` 를 Scratch 에서 합성하기

Scratch 의 `operator_mathop` 블록에는 `e^`, `10^`, `log`, `ln`, `sqrt`,
`abs`, `floor`, `ceiling`, `sin/cos/tan`, … 만 있고 **임의의 밑 지수
함수는 직접 합성** 해야 한다.

```
  0.5^x  =  e^( x · ln(0.5) )
```

`ln(0.5) ≈ -0.6931471805599453` 을 Stage 변수 `V_LN_HALF` 에 박아두고,
광물 클론마다 매 0.1초:

```
  Δt        = sensing_timer - V_T_BIRTH
  exp_arg   = (Δt / V_T_HALFLIFE) · V_LN_HALF
  V_MASS_NOW = V_N0 · (e^ exp_arg)
```

이걸 한 줄로 풀면 위의 `N(t) = N₀ · (1/2)^(t/T)` 와 같다.

### 보너스: "남은 수명" 의 로그

광물이 임계 `N_min = 0.05·N₀` 에 닿기까지 남은 시간은

```
  N · (1/2)^(k/T) = 0.05·N₀
  (1/2)^(k/T)    = 0.05·N₀ / N
  k/T            = log₂(N / 0.05·N₀)
  k              = T · log₂(N / 0.05·N₀)
```

Scratch 의 `log` 는 상용로그(밑 10)이므로 **밑 변환**:

```
  log₂(x) = log₁₀(x) / log₁₀(2)
```

이 식이 `V_LIFE_LEFT` 모니터 값. 박테리아 디펜스의 `log_r(N_max/N)` 과
형태가 *데칼코마니* — 그쪽은 임계까지 *몇 번 분열* 인지, 이쪽은
임계까지 *몇 초* 인지.

### 시각 크기는 `sqrt`

```
  size = 100 · sqrt(V_MASS_NOW / V_N0)
```

면적 비례로 보이게 하려는 의도. 질량이 절반이면 한 변은 √0.5 ≈ 70%.
*수치 절반 ≠ 시각 절반* — sqrt 가 끼어드는 학습 포인트가 부수적으로 등장.

## 🛠 내부 구조 (스프라이트 3개)

| 스프라이트 | 역할 |
|-----------|------|
| `Stage` | 광산 배경, 카운트다운 (1초마다 -1), 광물 스폰 디스패처 (1.5초마다 `CloneCount < 4` 일 때 좌표/`T`/`N₀` 무작위 세팅 후 스폰 broadcast), 게임오버 판정 |
| `광물` | 클론으로 동시 최대 4마리. 각자 클론 로컬 변수 `N0` `T_half` `출생시간` `질량` `수명` 으로 독립 감쇠. 매 0.1초 갱신. 5% 이하 자동 소멸. 클릭 시 현재 질량 점수에 가산 후 삭제 |
| `결과` | 60초 종료 시 정중앙에 띄우는 배너 |

### 데이터 흐름

```
▶ 깃발
  ├─ Stage 초기화 (점수=0, 남은시간=60, ln(0.5)=-0.693…)
  ├─ reset timer  (광물의 출생시간 기준)
  └─ broadcast "시작"
       ├─ Stage A: 매 1초 남은시간-- (0 → 게임오버)
       ├─ Stage B: 매 1.5초 (광물 < 4) 면 좌표/T/N₀ 세팅 후 broadcast "스폰"
       └─ 광물 (원본): broadcast "스폰" 수신 → create clone of myself
            └─ 클론: 출생시간 = timer, 매 0.1초 N(Δt) 갱신, 크기/색/수명 갱신,
                     5% 이하면 자가 소멸, 클릭되면 점수 가산 후 소멸
```

## 🔧 다시 빌드하기

```bash
cd games/radioactive-mine
python3 build.py
# → 반감기_광산.sb3 갱신
```

`반감기_광산.sb3` 파일이 생성됩니다. https://scratch.mit.edu 의 "내 작업실" 에 업로드하거나 Scratch 데스크탑에서 열어주세요.

`build.py` 안에서 만져볼 수 있는 값:
- `GAME_DURATION` (기본 60) → 라운드 길이 (초)
- `MAX_ORE` (기본 4) → 동시 화면 광물 수
- 스폰 디스패처의 `random(10,50)/10` → `T` 범위 (현재 1.0~5.0초)
- 스폰 디스패처의 `random(50,200)` → `N₀` 범위
- `ORE_FRESH` / `ORE_DECAYED` 의 색상 → 광물 비주얼
- 광물 loop 의 `wait 0.1` → 갱신 주기 (작을수록 부드러움, CPU↑)
