# 탱크 배틀 · Tank Battle (서바이벌)

> 톱다운 솔로 서바이벌. 내 탱크 1대로 끝없이 몰려오는 빨간 적 탱크들을 상대한다. 적은 **한 방에 터지고**(처치할 때마다 점수 +1), 내 체력이 0이 되면 게임오버. 흩어진 엄폐물 뒤에 숨어 최대한 오래 버티는 게 목표다.

## 조작
- **↑ / ↓ / ← / →**: 누른 방향으로 바로 이동 (탱크가 그 방향을 보고, 포신도 그쪽을 향함)
- **space**: 포탄 발사 (쿨다운 있음, 연사 불가)

## 게임 흐름
- 시작: 내 탱크는 화면 아래 중앙(0,−120). 화면 곳곳에 부서지지 않는 엄폐물 8개가 고정 배치된다.
- 플레이: 좌상단에 내 체력(♥3)과 점수(처치 수)가 표시된다. 적 탱크는 화면 위쪽 랜덤 위치에서 시간차로 계속 등장(동시 최대 5대)해 나를 추적하고 간헐적으로 포탄을 쏜다. 적을 포탄으로 맞히면 한 방에 폭발하며 점수 +1.
- 피해: 적 포탄에 맞거나 적 탱크에 부딪히면 내 체력 −1(잠시 무적). 내 체력 0 → **GAME OVER** 배너. 초록 깃발을 다시 누르면 재도전.

승리 조건은 없다. 순수하게 **오래 버티기 / 많이 처치하기** 게임이다.

## 구현 메모
- 적탱크·엄폐물·포탄·적포탄은 모두 클론으로 생성.
- **적 스포너**: `적탱크` 원본이 `게임시작` 후 forever 루프로 `적수 < 5` 일 때 랜덤 좌표(x −200~200, y 110~170)에 클론을 0.6~1.6초 간격으로 생성. `적수` 변수로 동시 생존 수를 제한한다.
- **적은 한 방에 죽는다**: 각 적 클론이 `touching 포탄` 이면 즉시 `점수+1`, `적수−1`, 폭발음 후 클론 삭제. (이전의 적체력+적무적 프레임 구조를 없애 "마지막 한 방이 잘 안 맞는" 문제를 제거.)
- **포탄은 두 스프라이트로 분리**: `포탄`(플레이어) / `적포탄`(적). 발사 탱크가 자기 포탄에 자폭하지 않으며 friendly-fire가 차단된다.
- 플레이어 피해: `touching (적포탄 OR 적탱크) AND 무적=0` → 내체력−1, 잠시 무적. 적 탱크는 플레이어에 닿으면(돌격) 스스로 터진다.
- 게임오버: Stage 워처가 `내체력<1` 을 감시해 게임상태=0 으로 바꾸고, `게임오버` 스프라이트가 배너를 띄운다. 살아있던 적 클론들은 게임상태=0 을 보고 스스로 삭제된다.
- **아트 방향**: 탱크·포탄 SVG는 위를 향하게 그린 뒤 `rotate(90)` 으로 오른쪽(=Scratch "전방향 회전"의 기준 방향 90)을 보게 한다. 안 그러면 스프라이트가 90° 틀어져 포탄이 옆으로 날아가는 것처럼 보인다.

## 검증 (headless scratch-vm)
구조 검증과 함께 scratch-vm으로 실제 실행해 확인:

- **4방향 이동** — ↑/↓/←/→ 각각 해당 방향으로만 이동하고 탱크 방향(0/180/−90/90)이 맞음.
- **발사 방향 = 탱크가 보는 방향** (`포탄방향` = 플레이어 direction).
- **적 스폰** — 시간이 지나며 적 클론이 화면 위쪽 밴드(y≥100)에 생성되고, 동시 최대 5대 제한이 지켜짐.
- **적 추적** — 생성된 적이 플레이어(아래쪽)를 향해 실제로 내려옴.
- **게임오버** — 내체력 0 → 게임상태=0, 결과=2(패배). 게임오버 후 잔여 적 클론 전부 정리됨.

> 렌더러 없는 headless 에서 `touching` 은 false 라, 터치 기반 격파(점수 가산)는 직접 실행 검증이 불가하다. 해당 로직은 실제 Scratch 에서 동작하던 피격 감지와 동일한 패턴이며, 한 방 격파 구조라 판정이 단순하다.
