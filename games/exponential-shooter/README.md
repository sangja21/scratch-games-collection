# 📈 지수함수 슈터

지수함수 `y = a · b^x` 의 두 계수 **a**, **b** 를 실시간으로 조절하면서, 그 곡선을 따라 로켓을 발사해 풍선을 터뜨리는 학습 게임입니다. **「로그 보기」** 버튼을 누르면 좌표축이 로그 스케일로 바뀌어 지수곡선이 **직선** 으로 변하는 마법을 직접 확인할 수 있습니다.

> **학습 포인트** — 지수에서 밑 `b` 가 1보다 큰지 작은지에 따라 모양이 어떻게 갈리는지, 그리고 로그 변환이 왜 "지수의 가파름을 풀어주는" 도구인지를 *수식 / 곡선 / 로켓 비행* 세 가지로 동시에 보여줍니다.

## 🎮 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 슈팅 / 학습 게임 |
| 핵심 메커닉 | 곡선 미리보기 + 로그 토글 + 발사 |
| 학습 주제 | 지수함수 `y = a·b^x`, 로그 스케일의 의미 |
| 권장 학년 | 고등학교 1학년 (지수와 로그 단원 도입 직후) |
| 라운드 | 5 라운드 (5라운드는 로그 모드 권장) |

## 🕹 조작법

| 입력 | 동작 | 변화량 |
|------|------|--------|
| `a` 슬라이더 | 계수 `a` 직접 조절 | 0.5 ~ 5.0 |
| `b` 슬라이더 | 밑 `b` 직접 조절 | 0.3 ~ 3.0 |
| `↑` / `↓` | `a` 미세 조정 | ±0.1 |
| `→` / `←` | `b` 미세 조정 | ±0.1 |
| `[로그 보기 ▼]` 버튼 클릭 | 선형 ↔ 로그 스케일 토글 | — |
| `Space` | 로켓 발사 | — |

```

│ b ▭━━●━━━  ┃    🎈                                    │

│             ┃                                         │
│             ┃    ╭━━━━━━━━╮   🎈                      │
│  🚀━━━━━━━━┄┄┄┄                                       │
│ ═════════════════════════════════════════════════════ │
└──────────────────────────────────────────────────────┘
   분홍 곡선 = 미리보기      풍선 = 표적 (라운드별 패턴)
```

- **좌상단**: 점수 / 라운드 / a, b 슬라이더 / 로그 모드 표시
- **상단 중앙**: 현재 식 (예: `y = 2.0 · 1.5^x`)
- **분홍 곡선**: 현재 a, b 의 곡선 미리보기 (실시간 갱신, 약 0.08초)
- **오른쪽 위 버튼**: 로그 스케일 ↔ 선형 스케일 토글
- **풍선**: 라운드별로 정해진 위치(x, y)에 출현

## 🎯 라운드 / 풍선 배치

| 라운드 | 풍선 위치 `(x, y)` | 정답 a, b | 학습 의도 |
|--------|---------------------|-----------|-----------|
| 1 | (0,1) (1,2) (2,4) (3,8) | a=1, b=2 | `2^x` 의 두 배씩 증가 |
| 2 | (0,1) (1,0.5) (2,0.25) (3,0.125) | a=1, b=0.5 | `b<1` → 감소 |
| 3 | (-2,0.25) (-1,0.5) (0,1) (1,2) (2,4) | a=1, b=2 | 음수 x 에서 `b^x = 1/b^|x|` |
| 4 | (-1,0.83) (0,1.5) (1,2.7) (2,4.86) | a=1.5, b=1.8 | a, b 모두 추정 |
| 5 | (-3,0.5) (-1,2) (1,8) (3,32) | a=4, b=2 | **로그 모드** 에서 직선! |

> 5라운드는 y 값이 0.5 ~ 32 로 폭이 커서 선형 모드에서는 풍선이 화면 밖으로 나갑니다. **「로그 보기」 를 켜야** 모든 풍선이 한 화면에 보이고, 곡선이 직선이 되어 풀기 쉬워집니다.

## 🧮 수학 배경

### 1. `a` 와 `b` 의 역할

발사체와 곡선은 다음 식을 따릅니다.

$$ y = a \cdot b^x $$

- **`a`** — x=0 일 때 값 (y절편). 슬라이더 0.5 ~ 5.0
- **`b`** — 성장률 (밑). 슬라이더 0.3 ~ 3.0
  - `b > 1` → 증가 (예: `2^x`)
  - `b = 1` → 상수 (직선! — 슬라이더를 정확히 1.0 으로 맞춰 보세요)
  - `b < 1` → 감소 (예: `0.5^x = 1/2^x`)

### 2. 좌표 변환 (직접 확인 가능한 핵심)

게임 내 논리 좌표 `x_g ∈ [-6, 6]` 를 화면 좌표로 옮기는 두 가지 방법:

**선형 모드 (`로그 = 0`)**:

```
screen_x = 40 · x_g
screen_y = 30 · (a · b^x_g) - 120
```

**로그 모드 (`로그 = 1`)**:

```
screen_x = 40 · x_g
screen_y = 30 · log₁₀(a · b^x_g) - 30
        = 30 · (log₁₀(a) + x_g · log₁₀(b)) - 30   ← 직선!
```

지수곡선에 `log` 를 씌우면 `x` 에 대한 **1차식** 이 나오므로, 로그 스케일에서 보면 직선이 됩니다. 이게 로그 그래프 종이의 비밀입니다.

### 3. Scratch 에 `^` 가 없다 — `b^x` 는 어떻게 계산?

Scratch 블록에는 `^` (거듭제곱) 연산자가 없습니다. 대신:

$$ b^x = 10^{x \cdot \log_{10}(b)} $$

이 항등식을 그대로 블록으로 구현합니다 (코드 안 `pow_b_x()` 함수). 이것 자체도 학습 포인트입니다 — **임의의 밑은 10 (또는 e) 의 거듭제곱으로 환원** 할 수 있다는 사실.

## 🛠 내부 구조 (스프라이트 5개)

| 스프라이트 | 역할 | 주요 블록 |
|-----------|------|----------|
| `Stage` | 두 코스튬 — `linear` (균등 격자) / `log` (로그 격자) | 토글 시 switch backdrop |
| `로켓` 🚀 | 발사체 | 화살표/스페이스 입력, `발사` 수신 시 곡선 따라 글라이드 |
| `풍선` 🎈 | 표적 (클론) | 라운드별 위치 테이블 + 충돌 감지 |
| `곡선` | 보이지 않음, 펜으로 곡선 미리보기 | `pen` 확장, a/b 변할 때마다 재그림 |
| `로그버튼` | 우상단 클릭 버튼 | 클릭 시 `로그` 변수 토글 + 배경 교체 |

펜 확장(`pen`)을 사용합니다.

## 🔧 다시 빌드하기

```bash
python3 build.py
# → 지수함수_슈터.sb3 갱신
```

`build.py` 안에서 바꿔보면 재미있는 것들:

- `ROUND_LAYOUTS` — 라운드별 풍선 위치 직접 설계
- 슬라이더 `min/max` (`monitors` 섹션) — `a`, `b` 범위 변경
- `BG_LINEAR_SVG` / `BG_LOG_SVG` — 좌표축 디자인 교체
- `pow_b_x()` 함수 — 거듭제곱 구현 — `ln`/`e^` 로 바꿔도 동등

## 🧪 권장 학습 흐름

1. 라운드 1 — `b` 슬라이더만 만지면서 "두 배씩 커지는" 감각을 잡는다.
2. 라운드 2 — `b` 를 1보다 작게. 감소 곡선의 모양.
3. **`b` 를 정확히 1.0** 에 맞춰 본다. 곡선이 수평 직선이 되는 순간!
4. 라운드 4 — `a` 도 같이 만져 일반화.
5. 라운드 5 — 선형 모드로는 거의 풀 수 없다. **「로그 보기」** 를 켜고 직선이 되는 마법을 본다.
