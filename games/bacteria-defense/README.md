# 🦠 박테리아 디펜스

페트리 접시에 박테리아 한 마리가 떨어진다. 그냥 두면 **`T`초마다 `r`배로** 분열한다.
1 → 2 → 4 → 8 → 16 → … **임계치 `N_max`** 에 닿으면 게임 오버.
마우스로 박테리아를 클릭해 항생제를 살포하라.

> **학습 포인트** — 지수함수와 로그함수를 *체감*하기. 화면 상단에 표시되는
> **남은분열** 은 `log_r(N_max / N)` 이다. Scratch 의 `log` 블록은
> 상용로그(밑 10) 만 가지므로, 게임 코드 안에서 **밑 변환 공식**
> `log_r(x) = log₁₀(x) / log₁₀(r)` 을 직접 사용한다.

## 🎮 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 지수 증가 디펜스 (클릭 액션 + 수학) |
| 핵심 메커닉 | 박테리아가 `T` 초마다 `r` 배로 분열, 클릭으로 제거 |
| 학습 주제 | 지수함수 `N(t) = r^t`, 로그 `log_r`, 밑 변환 공식 |
| 권장 학년 | 고1 (지수와 로그 단원) |
| 플레이 시간 | 라운드별 ~1~2분, 총 5라운드 |

## 🕹 조작법

| 입력 | 동작 |
|------|------|
| 마우스 클릭 | 박테리아 1마리 제거 (점수 +1, 처치 +1) |

## 📋 화면 구성 (480×360)

```
┌──────────────────────────────────────────┐
│  N · 남은분열 · 라운드  │   점수 · R · T · N_max │   ← HUD
├──────────────────────────────────────────┤
│           ⊙ (페트리 접시)               │
│        🦠   🦠                          │
│     🦠   🦠   🦠   🦠                   │
│        🦠      🦠                        │
└──────────────────────────────────────────┘
```

- 좌측 모니터: `N` (개체수), `남은분열` (= log_r(N_max/N)), `라운드`
- 우측 모니터: `점수` (누적 처치), `R` (분열률), `T` (분열 주기), `N_max` (임계치)

## 🎯 라운드 표

| 라운드 | R (분열률) | T (주기) | N_max | 처치 목표 |
|--------|------------|---------|-------|----------|
| 1 | 1.5 | 2.5s | 128 | 30 |
| 2 | 2.0 | 2.0s | 256 | 60 |
| 3 | 2.0 | 1.5s | 512 | 120 |
| 4 | 2.5 | 1.5s | 768 | 200 |
| 5 | 3.0 | 1.5s | 1024 | sudden death |

라운드를 클리어하면 `R` / `T` / `N_max` 가 표 대로 갱신되고, 박테리아는
다시 1마리부터 시작합니다 (`N=1`).

## 🧮 수학 배경

### 지수 증가식
박테리아 한 마리가 매 `T` 초마다 `R` 마리로 분열한다면, 시각 `t` 초 뒤의
개체수는

```
  N(t) = R ^ (t / T)         (t = 0 일 때 N = 1)
```

`R=2`, `T=2s` 라면 10초 뒤 `2^5 = 32` 마리, 20초 뒤 `1024` 마리.
얼마나 빠른지 손으로 느껴보자.

### 로그로 "남은 시간" 보기
임계치 `N_max` 까지 *분열이 몇 번 더 필요한지* 는

```
  N · R^k = N_max
  R^k     = N_max / N
  k       = log_R (N_max / N)
```

이 `k` 가 화면의 **남은분열** 값. 단, Scratch 의 `log` 는 밑 10. 그래서
**밑 변환 공식** 을 쓴다:

```
  log_R(x)  =  log₁₀(x) / log₁₀(R)
```

이 식이 게임 안에 그대로 들어 있다 (`build.py` 의 `# HUD update loop` 부분).
"왜 로그를 배워야 하나" 의 답이 게임 HUD 한 줄에 응축된 셈.

### 분열률이 소수일 때 (`R = 1.5`)
"한 마리가 1.5 마리가 된다" 는 비현실적이므로, 게임에서는 각 박테리아가
매 틱마다 **0.5 확률로 추가 클론 1마리** 를 만드는 식으로 평균을 맞춘다.

```
  자식 수 = floor(R-1) + (random < (R-1 - floor(R-1)) ? 1 : 0)
```

라운드 1 (`R=1.5`) 에서는 각 박테리아가 매 틱마다 50% 확률로 1마리를
복제. 라운드 2 (`R=2`) 부터는 매 틱마다 100% 복제. 라운드 4 (`R=2.5`)
는 1마리 확정 + 50% 확률로 1마리 추가. 큰 수의 법칙 (대수의 법칙) 도
은근슬쩍 학습 포인트가 됨.

## 🛠 내부 구조 (스프라이트 3개)

| 스프라이트 | 역할 |
|-----------|------|
| `Stage` | 페트리 접시 배경, 분열 타이머, HUD 로그 계산, 라운드 진행, 게임 오버 판정 |
| `박테리아` | 클론으로 다수 등장. 코스튬 3종 (idle / splitting / dying). 분열 broadcast 수신 시 자기 자신을 복제. 클릭 시 사라짐 |
| `게임오버` | `N >= N_max` 도달 시 정중앙에 띄우는 배너 |

### 데이터 흐름

```
▶ 깃발
  ├─ Stage 초기화 (N=1, R=1.5, T=2.5, N_max=128, 라운드=1)
  └─ broadcast "시작"
       ├─ 박테리아: 첫 클론 1마리 생성
       └─ Stage: 3개 동시 루프
            ├─ A. 매 T 초마다 broadcast "분열"  → 각 박테리아 클론이 자기복제
            ├─ B. 매 0.1초마다 HUD 갱신 + 80% 도달 시 alarm 재생
            │     + N >= N_max 면 broadcast "게임종료"
            └─ C. 매 0.2초마다 처치 카운트 검사, 목표 달성 시 broadcast "다음라운드"
```

### 클론 한도 (Scratch 300개 제한) 대응

Scratch 는 클론을 최대 300개까지만 만들 수 있다. 라운드 4~5 에서 N 이
이 한도를 넘으면, 시각적인 클론 생성은 멈추되 **변수 `N` 은 계속
증가** 한다 (`CLONE_CAP = 120` 에서 끊는다). 임계치까지의 카운트다운
정확도는 유지하면서 Scratch 의 클론 폭주를 막는 절충안.

## 🔊 사운드

| 파일 | 용도 |
|------|------|
| `assets/pop.wav` | 박테리아 제거 (whack-a-prime 에서 재사용) |
| `assets/split.wav` | 분열 효과음 (220→140 Hz 글라이드, 0.12 s) — `build.py` 작성 중 1회 합성 |
| `assets/alarm.wav` | `N >= 0.8 · N_max` 도달 시 1회 경고 (880/660 Hz 알람, 0.4 s) |

## 🔧 다시 빌드하기

```bash
python3 build.py
# → 박테리아_디펜스.sb3 갱신
```

`build.py` 안에서 만져볼 수 있는 값:
- `ROUNDS` 리스트 → 라운드별 `(R, T, N_max, 처치 목표)` 직접 수정
- `CLONE_CAP` → 시각적 박테리아 최대 클론 수 (기본 120)
- `BACT_IDLE` / `BACT_SPLIT` / `BACT_DYING` 의 SVG fill → 박테리아 색
- HUD 모니터의 `mode` (`"default"` ↔ `"large"`) → 모니터 크기
