# 핑퐁 (Pong)

위/아래 화살표로 왼쪽 패들을 움직여 공을 받아치고, 오른쪽 컴퓨터(AI) 패들과 점수 대결을 한다. 공은 위/아래 벽에서 튕기고, 패들에 맞으면 진행 방향이 반대로 꺾인다. 패들의 어디에 맞느냐(중심에서 떨어진 거리)에 따라 공이 날아가는 각도가 달라진다. 랠리가 길어질수록 공이 조금씩 빨라진다. 먼저 5점에 도달하면 승리. 1972 아타리 PONG의 클래식 메커닉이다.

## 조작

| 키 | 동작 |
|----|------|
| ↑ | 왼쪽 패들 위로 이동 |
| ↓ | 왼쪽 패들 아래로 이동 |

- 키를 누르고 있으면 패들이 계속 움직입니다(연속 이동).
- 패들은 위/아래 끝(y = ±130)을 넘어가지 않습니다.
- 패들의 **위쪽 끝**에 맞히면 공이 위로 가파르게, **아래쪽 끝**에 맞히면 아래로, **중앙**에 맞히면 거의 수평으로 곧게 날아갑니다. 받아치는 위치로 각도를 만드는 것이 손맛의 핵심입니다.

## 룰

- 공이 내 뒤(왼쪽 벽 x < -235)로 나가면 컴퓨터 +1, 컴퓨터 뒤(오른쪽 벽 x > 235)로 나가면 나 +1.
- 먼저 5점에 도달한 쪽이 승리합니다.
- 패들에 맞을 때마다 공이 빨라지고(공속도 +0.4), 새 랠리(서브)가 시작되면 속도가 4.5로 초기화됩니다.
- 오른쪽 컴퓨터 패들은 공의 Y를 따라오지만 한 번에 움직일 수 있는 거리(AI속도 = 4)에 제한이 있어, 빠르고 가파른 각으로 받아치면 따라잡지 못하고 놓칩니다. 집중하면 이길 수 있는 난이도입니다.

## 게임 흐름

초록 깃발 클릭 → 점수 0:0으로 초기화, 공이 가운데에서 출발 → 화살표로 공을 받아치며 랠리 → 공이 좌/우 벽 밖으로 나가면 득점하고 공이 가운데에서 다시 서브 → 한쪽이 5점에 먼저 도달하면 "YOU WIN!"(승) 또는 "YOU LOSE"(패) 배너가 뜨고 공·패들 정지 → 깃발 다시 클릭으로 즉시 재시작.

## 핵심 메커닉 (각도 컨트롤 + 속도 램프)

- **각도 컨트롤**: 패들 반사 시 `오프셋 = (공의 y − 패들의 y) / 패들반높이(35)`로 충돌 위치를 −1..+1로 정규화하고, `공dy = 오프셋 × 공속도`로 세로 속도를 정합니다. 패들 높이가 70(반높이 35)이라 패들 양끝에서 오프셋이 정확히 ±1이 됩니다.
- **속도 램프**: 패들에 맞을 때마다 `공속도 += 0.4`. 공dx/공dy가 공속도를 기준으로 다시 계산되므로 랠리가 길수록 공이 빨라지고, 서브 때 4.5로 리셋되어 무한 가속을 막습니다.
- **벽 반사**: 공의 y가 ±165를 넘으면 `공dy × −1`로 뒤집고 벽 안쪽으로 살짝 밀어 끼임을 방지합니다.
- **AI 추적 제한**: 오른쪽 패들은 `(공의 y) − (자기 y)` 차이만큼 따라가되 ±AI속도(4)로 제한해, 평균 공 속도(4.5)보다 살짝 느려 플레이어가 이길 여지를 만듭니다.

## 실행

1. `핑퐁.sb3` 다운로드
2. [scratch.mit.edu](https://scratch.mit.edu) → 만들기 → 파일 → 컴퓨터에서 가져오기
3. 초록 깃발 ▶ 클릭

## 구조

- 스프라이트 5: Stage / 좌패들(플레이어) / 우패들(AI) / 공 / 결과배너
- 총 블록 255개 (Stage 43 / 좌패들 28 / 우패들 59 / 공 109 / 결과배너 16)
- 변수 13개: 점수(내점수·컴점수)·게임상태·목표점수·공 물리(공dx·공dy·공속도·서브방향)·AI속도·패들반높이·결과 + 임시(오프셋·AI차)
- 자산: 인라인 SVG 6종(코트·좌패들·우패들·ball·win·lose) + bounce.wav / score.wav
- 클론·리스트 없이 단일 인스턴스 공 하나가 매 틱 dx/dy로 움직이며 벽·패들과 물리 반사하는 첫 게임
