# 풍선 펌프 (Balloon Pump)

화면 가운데 풍선에 펌프로 바람을 넣는다. **스페이스를 꾹 누르면** 풍선이 점점 커지고 점수도 함께 오른다. 크게 불수록 점수가 가파르게 오르지만, 풍선마다 터지는 한계가 다르고 **보이지 않는다**. 너무 욕심내면 **펑!** 터져서 그 라운드는 0점. 터지기 직전에 스페이스를 떼면 그때까지 모은 점수가 내 것이 된다. 한계에 가까워지면 풍선이 **떨리고 색이 빨개지며** 슬쩍 경고한다. 5라운드 합계로 베스트 기록을 노린다. 욕심과 안전 사이의 도박 같은 액션 게임이다.

## 조작

| 키 | 동작 |
|----|------|
| 스페이스 (꾹 누름) | 펌프질 — 누르고 있는 동안 풍선이 계속 커지고 점수가 오름 |
| 스페이스 (뗌) | 점수 확정 — 그때까지 모은 이번 점수를 합계에 더함 |

- 스페이스 한 키만 쓰면 됩니다. 꾹 누르면 불고, 떼면 멈춥니다.
- 풍선이 떨리고 색이 빨개지기 시작하면 위험 신호 — 곧 터질 수도 있습니다.
- 한 번도 펌프하지 않고 떼면(시작 직후) 라운드가 넘어가지 않습니다. 최소 한 번은 불어야 합니다.

## 룰

- 스페이스를 누르는 동안 풍선 크기가 커지고 점수가 **가속해서** 오릅니다(크게 불수록 1틱당 점수 증가폭이 큼).
- 풍선마다 터지는 한계 크기(120~220)가 무작위이고 **화면에 표시되지 않습니다**.
- 한계에 도달하면 **펑!** 터지고 파편이 사방으로 흩어지며 그 라운드는 0점.
- 터지기 전에 스페이스를 떼면 그때까지의 점수가 합계에 확정됩니다.
- 5라운드 합계가 베스트보다 높으면 신기록입니다.
- 떨림/색 경고가 시작되는 시점은 한계치의 60~80% 사이 무작위 지점이라, 떨린다고 곧 터지는 것도 아니고 안 떨린다고 안전한 것도 아닙니다. 라운드마다 패턴이 달라 외울 수 없습니다.

## 게임 흐름

초록 깃발 클릭 → 1라운드 시작(풍선 크기 80으로 리셋, 라운드별 색, 보이지 않는 한계치/떨림 시작점 새로 추첨) → 스페이스를 꾹 눌러 풍선을 분다 → 풍선이 커지며 점수가 가파르게 오름 → 어느 순간부터 풍선이 떨리고 색이 빨개짐(위험 경고) → **여기서 떼면** 안전 배너 + 점수 확정(+합계) / **계속 누르면** 한계에서 펑! 0점(파편 분출 + 펑 소리) → 다음 라운드는 한계치·떨림 시작점이 매번 달라짐 → 5라운드 후 합계/베스트 배너(신기록이면 "신기록!") → 깃발 다시 클릭으로 즉시 재시작.

화면 상단에는 라운드 / 이번점수(지금 부는 중 점수) / 합계 / 베스트가 표시됩니다. **한계치와 떨림 시작점은 일부러 숨겨져** 있습니다(보이면 게임이 무의미해지므로).

## 핵심 메커닉 (욕심 vs 안전 트레이드오프)

- **비선형 점수(가속)**: 이번점수 = `반올림((크기-80) × √(크기-80))`. 즉 부푼 양의 1.5제곱입니다. 작게 멈추면 점수가 적고, 크게 불수록 점수가 가파르게 오릅니다 — "조금만 더…"의 욕심을 유발합니다.
- **숨겨진 한계치**: 라운드마다 `무작위(120~220)`로 정해지고 화면에 표시되지 않습니다. 크기가 한계치에 도달하면(`크기 ≥ 한계치`, 내부적으로 `(크기 < 한계치)의 반대`로 판정) 터집니다.
- **누설 방지 떨림/색 경고**: 떨림·색 경고는 한계치가 아니라 `떨림시작 = 반올림(한계치 × 무작위(60~80)%)` 지점부터 시작합니다. 한계치 자체를 역산할 수 없게 라운드마다 비율도 달라집니다. 떨림은 `set x to 무작위(-2~2)`로 제자리에서만 흔들려(누적 이동 없음), 색은 위험 구간 진입분만큼 빨개집니다.
- **터짐 시 0점 보장**: 터지는 틱에 이번점수가 갱신되더라도, 합계 합산은 라운드결과가 "안전 확정(1)"일 때만 일어나고 터짐은 "터짐(2)"으로 표시되므로 점수가 합계에 새지 않습니다.

### 밸런스 (왜 욕심에 보상이 있는가)

점수 공식과 한계치 분포로 기대값을 계산해 보면:

- **무조건 안전**(항상 크기 119에서 멈춤, 절대 안 터짐): 라운드당 244점 → 5라운드 1,220점.
- **적당한 욕심**(크기 ~164에서 멈추기로 결정): 기대값이 라운드당 약 427점으로 가장 높음 → 안전 플레이보다 약 75% 높음. **욕심에 보상이 있습니다.**
- **과한 욕심**(크기 200 이상까지): 터질 확률이 높아져 기대값이 안전 플레이 밑으로 떨어짐 → 과욕은 손해.

즉 무작정 안전하지도, 무작정 욕심내지도 않은 **중간 지점**에 최적 전략이 있어, 매 라운드 "여기서 멈출까 더 갈까"의 긴장된 도박 결정이 게임의 핵심이 됩니다.

## 실행

1. `풍선펌프.sb3` 다운로드
2. [scratch.mit.edu](https://scratch.mit.edu) → 만들기 → 파일 → 컴퓨터에서 가져오기 (또는 Scratch 데스크탑에서 열기)
3. 초록 깃발 ▶ 클릭 → 스페이스를 꾹 눌러 풍선을 분다

## 다시 빌드

```bash
cd games/balloon-pump
python3 build.py
# → 풍선펌프.sb3 갱신
```

## 구조

- 스프라이트 5: Stage(배경 + 전역 변수 + 라운드 진행/한계치 생성/합계·베스트 판정) / 풍선(입력+점수+떨림·색 경고+터짐·확정 판정) / 펌프(손잡이 들썩 연출) / 파편(터질 때 사방 분출 클론) / 결과배너(펑/안전/끝/신기록 코스튬)
- 총 블록 239개 (Stage 53 / 풍선 96 / 펌프 19 / 파편 28 / 결과배너 43)
- 변수 11개(글로벌): 게임상태·라운드·총라운드·이번점수·합계·베스트·현재크기·한계치·떨림시작·라운드결과·펌프중 + 파편 sprite-local 1개(속도)
  - **한계치·떨림시작은 모니터 숨김**(누설 방지)
- 방송 6개: 게임시작 / 라운드시작 / 터짐 / 확정 / 라운드끝 / 결과
- 자산: 인라인 SVG 9종(축제 배경 + 풍선 + 펌프 2코스튬 + 파편 + 결과배너 4코스튬) + pop.wav(피치 변주로 터짐/안전음/신기록음 재사용)
- 비선형 점수(√)와 숨겨진 한계치로 만드는 욕심 vs 안전 트레이드오프가 핵심. 추상 학습 개념 없는 순수 반응·도박 심리 액션 게임
