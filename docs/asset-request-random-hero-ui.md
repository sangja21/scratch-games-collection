# 🖼️ UI 에셋 요청서 — 「낭만 고양이」 (Grok 이미지 생성용)

> 캐릭터/이펙트는 별도 문서(`asset-request-random-hero.md`)에서 다룹니다. 이 문서는 **인터페이스 장식 아트**만.
>
> ## ⚠️ 먼저 읽어주세요 (중요 원칙 2가지)
> 1. **글자는 넣지 마세요.** 이미지 생성기는 한글(그리고 대부분의 텍스트)을 깨뜨립니다. 그래서 여기 요청하는 건 **"텍스트 없는 빈 틀/엠블럼"**뿐이고, "낭만 고양이"·"레벨업"·숫자·스킬 이름 같은 **모든 글자는 제가 스크래치에서 얹습니다.**
> 2. **움직이는 UI는 요청 안 함.** HP/경험치 바의 "채워지는 부분", 데미지 숫자, 쿨다운 게이지, 카드 안의 아이콘/설명은 전부 스크래치가 실시간으로 그립니다. Grok엔 그 **바깥 틀·배경 판·장식**만 부탁해요(속은 비어 있게).

---

## 📐 공통 스타일 가이드 (Grok에 먼저 알려주기)
캐릭터 세트와 **같은 세계관**이라야 붙였을 때 안 튑니다:
```
스타일: 「낭만 고양이」 게임 UI 아트. 감성적인 밤 무드의 2D 모바일 게임 인터페이스,
깊은 남보라 밤하늘 팔레트에 은은한 달빛/별빛, 따뜻한 주황 포인트, 부드러운 셀 셰이딩,
굵고 매끈한 외곽선, 둥근 모서리. 귀엽지만 살짝 쓸쓸한 낭만적 감성.
배경 완전 투명(PNG 알파). 글자·숫자·텍스트 절대 넣지 말 것. 워터마크 없음, 사진풍 금지.
```

## 🎨 UI 에셋 목록 (총 8종) — 우선순위 순

### ⭐ 1. 타이틀 엠블럼 `ui_title.png` (강추)
- **용도**: 시작화면 로고. **"낭만 고양이" 글자는 스크래치에서 얹음** → 여기선 글자 없는 **엠블럼/장식만**.
- **프롬프트**:
```
[공통 스타일] 게임 타이틀용 장식 엠블럼, 글자 없음.
둥근 보름달 앞에 목도리 두른 고양이 실루엣, 주변에 흩날리는 별과 작은 물고기 뼈 장식,
아래에 텅 빈 리본/현판(그 안에 글자는 나중에 넣을 것). 감성적인 밤 색감, 투명 배경.
```

### ⭐ 2. 스킬 버튼 베이스(링) `ui_skillbtn.png` (강추)
- **용도**: 하단 스킬 버튼 3개의 **둥근 받침/테두리**. 위에 발톱/꼬리/생선 아이콘(이미 있음)과 쿨 게이지(스크래치)가 올라감.
- **프롬프트**:
```
[공통 스타일] 게임 스킬 버튼용 빈 원형 베이스 1개, 안쪽은 비어 있고(투명/반투명 유리 느낌)
가장자리는 은은한 별빛 테두리와 부드러운 발광. 심볼/글자 없음. 투명 배경, 정사각 중앙.
```
- **참고**: 한 개만 주면 제가 3개로 복제해서 씁니다.

### ⭐ 3. 레벨업 카드 프레임 (기본) `ui_card.png` (강추)
- **용도**: 레벨업 때 뜨는 강화 카드 3장의 **빈 카드 틀**. 안의 아이콘·강화 이름·수치는 스크래치가 그림.
- **프롬프트**:
```
[공통 스타일] 세로로 긴 둥근 모서리 게임 카드 프레임 1장, 속은 비어서 은은한 밤하늘 질감,
테두리엔 별과 달빛 장식. 심볼/글자 없음(안은 나중에 채움). 투명 배경.
```

### 4. 레벨업 카드 프레임 (잭팟/골드) `ui_card_gold.png`
- **용도**: 3번과 같은 모양의 **희귀(잭팟) 버전** — 금빛으로 화려하게.
- **프롬프트**:
```
[공통 스타일] 3번 카드 프레임과 같은 형태·같은 크기지만 희귀 등급 버전:
금빛 테두리, 반짝이는 별가루, 은은한 후광. 속은 비고 글자 없음. 투명 배경.
```

### 5. 경험치 보석 `ui_gem.png`
- **용도**: 처치 시 튀는 경험치 픽업 아이콘(작게)
- **프롬프트**:
```
[공통 스타일] 반짝이는 청보라 보석/크리스탈 아이콘 1개, 별빛 하이라이트,
작고 또렷하게. 투명 배경.
```

### 6. 체력바 프레임 `ui_hpbar.png`
- **용도**: 고양이 체력바의 **빈 껍데기(캡슐 틀)**. 안의 빨간 채움은 스크래치가 그림 → **속은 투명**해야 함.
- **프롬프트**:
```
[공통 스타일] 가로로 긴 둥근 캡슐형 게임 체력바 프레임, 왼쪽 끝에 작은 하트 장식,
캡슐 안쪽은 완전히 비어 투명(테두리만), 부드러운 금속/유리 테. 글자·눈금 없음. 투명 배경.
```
- **참고**: 안이 비어 있어야 스크래치가 그 안에 체력을 채워 넣습니다.

### 7. 경험치바 프레임 `ui_xpbar.png`
- **용도**: 6번과 같되 경험치용(가늘고, 별 모티프). 속은 투명.
- **프롬프트**:
```
[공통 스타일] 가로로 긴 얇은 둥근 경험치바 프레임, 왼쪽 끝에 작은 별 장식,
안쪽은 완전히 비어 투명(테두리만). 글자·눈금 없음. 투명 배경.
```

### 8. 결과/게임오버 현판 `ui_panel.png`
- **용도**: 게임오버·결과 표시용 **장식 판**(그 위에 점수·최고기록 글자는 스크래치가 얹음).
- **프롬프트**:
```
[공통 스타일] 둥근 모서리의 아늑한 밤 무드 게임 패널/현판 1장, 은은한 별과 달빛 테두리,
가운데는 비어 있어 나중에 글자를 얹을 공간. 심볼/글자 없음. 투명 배경.
```

## 🧩 스크래치가 직접 그리는 것 (요청 불필요)
아래는 Grok에 요청하지 않습니다 — 실시간으로 바뀌어야 해서 코드로 그려요:
- HP/경험치 **채워지는 막대**(색·길이 실시간), 데미지 **숫자 팝업**, 스킬 **쿨다운 게이지**
- 카드 안의 **강화 아이콘/이름/수치**, 모든 **텍스트·숫자·모니터**

## ✅ 받은 뒤 처리
- 투명 PNG로 `games/random-hero/assets/` 에 파일명대로 넣어주세요.
- 제가 각 틀 위에 스크래치로 바·숫자·글자를 정렬해 얹습니다. **바 프레임(6·7)은 속이 투명**이라야 정렬이 맞아요 — 속이 채워져 오면 그 파일만 다시 뽑아주세요.
- **우선순위**: 1·2·3(타이틀·스킬버튼·카드)이 체감 효과가 제일 커요. 6·7(바 프레임)은 정렬이 까다로우니 나중에 붙여도 됩니다. 시간 없으면 1·2·3·5만 먼저!
