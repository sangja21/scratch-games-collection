# 체스 대전쟁 (chess-war) — Plan

> 화면 **왼쪽 하얀 킹 vs 오른쪽 검은 킹**. 하단 버튼으로 **폰·비숍·나이트·룩·퀸**을 소환하면, 유닛이 킹 앞에서 튀어나와 **자동으로 오른쪽으로 전진**한다. 검은 군단(검은 폰/비숍/나이트/룩, 후반 검은 퀸 보스)도 검은 킹 앞에서 나와 왼쪽으로 밀고 온다. 양쪽 유닛이 만나면 멈춰서 **자동 전투**(폰=싸고 빠른 칼잡이 / 나이트=강한 근접 / 비숍=원거리 단일 / 룩=장사거리 포격·광역 / 퀸=**비숍+룩 공격을 둘 다** 퍼붓는 여왕 전사). 골드는 **초당 자동으로 쌓이고**, 그 골드로 유닛을 소환한다. **검은 킹의 체력을 0으로 만들면(체크메이트) 스테이지 클리어 → 강화 택1**(초당골드+/유닛공격력+/유닛체력+/킹수리) → 다음 스테이지. 스테이지가 오를수록 검은 군단은 **지수(곱셈)로 강해지고**(`적배율 = 적성장배율^(스테이지−1)`), **하얀 킹의 체력이 0이 되면 GAME OVER** → 깃발 재클릭으로 1스테이지부터. **점수 = 도달한 스테이지.**
> **디자인은 체스 기물 모양이 아니라 체스 이름·역할을 딴 판타지 전사들**이다(폰=칼잡이 보병, 비숍=로브 입은 사제, 나이트=말 탄 기사, 룩=포병, 퀸=여왕 전사). 킹은 본진(기존 성 체력 메커닉을 이름·비주얼만 킹으로).
> 베이스: `games/castle-defense/build.py` (한글 튜닝 변수 **일괄 초기화**·매직넘버 0 / 클론 스포너 + `복제됨` 가드 / 강화 택1 패널 / 게임오버 배너 / 플로팅 숫자 팝업(say 미사용, 흰=데미지/금=골드) / 전용 합성 효과음 `_wav_bytes`·`synth_*` / `add_comment` 가이드 투어). 리스트 조작은 `games/snake/build.py`·`games/othello/build.py`의 `data_addtolist`·`data_replaceitemoflist`·`data_itemoflist` 패턴 참조.
> 학습 콘셉트 없음(직관적 액션 전쟁 게임 — 추상 학습개념 금지, [[feedback-game-design]] 준수). **진짜 가치는 castle-defense/magic-survivor와 똑같이 "아이가 코드의 숫자·규칙을 직접 바꾸며 노는 것"** — 유닛 5종·적 5종의 체력/공격력/공속/속도/사거리/코스트, 초당골드, `적성장배율`, 소환쿨 등 **모든 조절 값을 한글 전역 변수로 노출**, 코드 어디에도 매직넘버 없음. 시후(초등·성격 급함)를 위해 **템포를 빠르게**: 소환은 버튼 클릭 즉시 반응(마우스 폴링), 초당골드로 계속 뭔가 뽑게 되고, 스테이지 클리어 시 강화 카드를 1/2/3/4 키로 골라 즉시 다음 스테이지. ★★★★☆.

- **주제**: 가로 라인 배틀 전쟁 게임 (하얀 군단 vs 검은 군단 — 체스 이름·역할의 판타지 전사)
- **카테고리**: 액션
- **난이도**: ★★★★☆
- **폴더**: `games/chess-war/`
- **출력 파일**: `체스_대전쟁.sb3`
- **빌드**: `python3 games/chess-war/build.py`

## 1. 게임 한 줄

하얀 킹과 검은 킹이 가로로 마주 본다. 하단 버튼을 눌러 체스 전사를 소환하면 알아서 검은 킹 쪽으로 전진해 적을 만나면 싸운다. 골드는 초당 자동으로 차오르고, 아이는 **어떤 전사를 언제 얼마나 뽑을지**만 정하면 된다. 검은 킹의 체력을 0으로 만들면(체크메이트) 스테이지 클리어, 강화 카드를 1·2·3·4 키로 골라 점점 강해진다. 스테이지가 오를수록 검은 군단이 **곱하기로** 세지므로(직선이 아니라 지수), 좋은 강화를 골라야 벽을 넘는다. 하얀 킹의 체력이 0이 되면 GAME OVER, 깃발을 다시 누르면 1스테이지부터(도달 스테이지가 곧 점수).

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

- HUD: 좌상단에 `하얀킹체력` `골드` `스테이지`(점수) `검은킹체력` 모니터 4종. 튜닝 변수(코스트·유닛 스탯 등)는 **모니터 숨김**.
- 평면 사이드뷰 — 중력·점프 없음. **모든 유닛은 한 줄 레인(단일 Y≈`레인Y`)** 위에서 X좌표만으로 전진/전투. 전투 판정은 순수하게 **X거리** 기준(아래 §7 매니저). 슬롯별로 Y를 아주 조금씩 흔들어(±) 겹침만 줄인다(전투 판정은 X만 사용).
- 하얀 킹은 왼쪽 끝(`하얀킹X≈-200`), 검은 킹은 오른쪽 끝(`검은킹X≈200`). 아군은 `하얀킹X+25`에서 생성돼 오른쪽으로, 적은 `검은킹X-25`에서 생성돼 왼쪽으로 전진.
- 하단 소환 버튼 바는 **5칸**이고 항상 보이며, 해금 안 된 버튼(나이트·룩·퀸)엔 🔒(잠금 코스튬). 스테이지 진행으로 해금되면 버튼 활성화.

> **왜 클론이 아니라 리스트가 시뮬레이션의 진실인가(핵심 아키텍처)**: 클론 대 클론 전투(clone↔clone 충돌·데미지)는 Scratch에서 경쟁 조건·순서 문제로 매우 까다롭다. 그래서 **아군/적 각각을 병렬 리스트(X·HP·타입·살아있음·쿨)로 관리**하고, 단일 스레드인 `전투매니저`의 forever 루프가 리스트를 읽어 이동·전투·죽음을 **순차적·결정적**으로 계산한다. 클론(아군유닛/적군유닛)은 자기 `슬롯`으로 리스트 값을 읽어 **위치·모습만 렌더**한다. 이렇게 하면 (1)전투 판정이 경쟁 없이 한 곳에서 결정되고 (2)죽음은 클론 삭제가 아니라 `살아있음[슬롯]=0`이라 **인덱스가 절대 밀리지 않는다**.

## 4. 사운드 — 전용 합성(build.py `_wav_bytes` + `synth_*`), orphan 0

castle-defense처럼 **결정적 합성 WAV**를 만들어, 각 스프라이트에 **실제로 재생하는 사운드만** 등록(고아 0). 재생은 대부분 `play sound`(대기 안 함, 빠른 손맛). 전투 타격음은 **처치·킹피격·포격에만** 재생(매 틱 브롤마다 울리면 시끄럽다 — 아래 규칙).

| 합성 함수 | 이름 | 음색 스펙(결정적) | 등록 스프라이트 | 트리거 |
|-----------|------|-------------------|------------------|--------|
| `synth_summon` | summon | 밝은 "뿅" 상승 처프(400→900Hz, 0.08초). 타입별 `pitch` 살짝 변주(폰 0 / 비숍 +20 / 나이트 −20 / 룩 −40 / 퀸 +40) | 소환버튼 | 유닛 소환 성공 |

> **가성비(수업거리)**: 폰 `4÷18≈0.22` · 나이트 `9÷45=0.20`(고단타·근접 위험) · 비숍 `5÷30≈0.17`(안전 원거리) · 룩 `6÷70≈0.086`(대신 광역 다중타) · 퀸(단일+광역 둘 다·최고 체력) — **공격력÷코스트**를 비교하며 "무엇을 몇 개 뽑을지" 저울질하게 된다.

### 전투매니저 (시뮬레이션 두뇌)

> **공통 헬퍼(빌더 참조)**: `속도(타입)`·`공격력(타입)`·`공속(타입)`·`사거리(타입)` 은 아군/적 각각 **5갈래 if/else**로 5.1 변수를 반환. 아군 실제 데미지엔 `× 유닛공격력배수`, 적은 그대로. 거리 = `abs(a − b)`(같은 레인이라 X만).

**(A) 시뮬 forever — 최전방 찾기 → 이동 → 공격 → 죽음**
```
when I receive 게임시작
  hide
  forever
    if (게임상태 = 1)
        # ── PASS A: 최전방 찾기 ──
        set 적선두X = 검은킹X ; set 적선두슬롯 = 0            # 기본 목표=검은 킹
        set 루프i = 1
        repeat (length of 적군X)
            if (item 루프i of 적군살아있음 = 1) and ((item 루프i of 적군X) < 적선두X)
                set 적선두X = (item 루프i of 적군X) ; set 적선두슬롯 = 루프i
            change 루프i by 1
        set 아군선두X = 하얀킹X ; set 아군선두슬롯 = 0         # 기본 목표=하얀 킹
        set 루프i = 1
        repeat (length of 아군X)
            if (item 루프i of 아군살아있음 = 1) and ((item 루프i of 아군X) > 아군선두X)
                set 아군선두X = (item 루프i of 아군X) ; set 아군선두슬롯 = 루프i
            change 루프i by 1

        # ── PASS B: 아군 이동 + 공격 ──
        set 루프i = 1
        repeat (length of 아군X)
            if (item 루프i of 아군살아있음 = 1)
                set 계산타입 = (item 루프i of 아군타입)
                if ( (적선두X - (item 루프i of 아군X)) > 사거리아군(계산타입) )    # 아직 사거리 밖 → 전진(오른쪽)
                    replace item 루프i of 아군X with ( (item 루프i of 아군X) + 속도아군(계산타입) )
                    if ( (item 루프i of 아군X) > (검은킹X - 킹공격거리) )          # 킹보다 앞으로 못 감
                        replace item 루프i of 아군X with (검은킹X - 킹공격거리)
                else                                                              # 사거리 안 → 공격
                    replace item 루프i of 아군쿨 with ( (item 루프i of 아군쿨) - 시뮬틱 )
                    if ( (item 루프i of 아군쿨) <= 0 )
                        set 계산뎀 = (공격력아군(계산타입) * 유닛공격력배수)
                        # ── 공격 방식 분기 ──
                        if (계산타입 = 4)                                          # 룩 = 광역만
                            call 광역공격_아군이적을(중심 = item 루프i of 아군X, 반경 = 사거리아군(4))
                        else if (계산타입 = 5)                                     # 퀸 = 단일 + 광역 둘 다!
                            call 단일공격_적(적선두슬롯, 계산뎀, 팝업x = 적선두X)
                            call 광역공격_아군이적을(중심 = item 루프i of 아군X, 반경 = 사거리아군(5))
                        else                                                       # 폰/비숍/나이트 = 단일
                            call 단일공격_적(적선두슬롯, 계산뎀, 팝업x = 적선두X)
                        replace item 루프i of 아군쿨 with 공속아군(계산타입)
            change 루프i by 1

        # ── PASS C: 적 이동 + 공격 (대칭: 왼쪽으로, 아군선두 목표) ──
        set 루프i = 1
        repeat (length of 적군X)
            if (item 루프i of 적군살아있음 = 1)
                set 계산타입 = (item 루프i of 적군타입)
                if ( ((item 루프i of 적군X) - 아군선두X) > 사거리적(계산타입) )
                    replace item 루프i of 적군X with ( (item 루프i of 적군X) - 속도적(계산타입) )
                    if ( (item 루프i of 적군X) < (하얀킹X + 킹공격거리) )
                        replace item 루프i of 적군X with (하얀킹X + 킹공격거리)
                else
                    replace item 루프i of 적군쿨 with ( (item 루프i of 적군쿨) - 시뮬틱 )
                    if ( (item 루프i of 적군쿨) <= 0 )
                        set 계산뎀 = 공격력적(계산타입)        # 적은 배수 없음(적배율은 스폰 시 체력에만)
                        if (계산타입 = 4)                                          # 검은룩 = 광역
                            call 광역공격_적이아군을(중심 = item 루프i of 적군X, 반경 = 사거리적(4))
                        else if (계산타입 = 5)                                     # 검은퀸 = 단일 + 광역
                            call 단일공격_아군(아군선두슬롯, 계산뎀, 팝업x = 아군선두X)
                            call 광역공격_적이아군을(중심 = item 루프i of 적군X, 반경 = 사거리적(5))
                        else
                            call 단일공격_아군(아군선두슬롯, 계산뎀, 팝업x = 아군선두X)
                        replace item 루프i of 적군쿨 with 공속적(계산타입)
            change 루프i by 1
    wait 시뮬틱
```

**단일공격_적(슬롯, 뎀, 팝업x)** — 아군이 최전방 적(또는 검은 킹)을 때림
```
if (슬롯 = 0)                                   # 목표=검은 킹
    change 검은킹체력 by (0 - 뎀) ; broadcast 검은킹피격
    call 데미지팝업(값=뎀, x=검은킹X, y=0, 종류=0)
else
    replace item 슬롯 of 적군HP with ( (item 슬롯 of 적군HP) - 뎀 )
    call 데미지팝업(값=뎀, x=팝업x, y=레인Y, 종류=0)
    if ( (item 슬롯 of 적군HP) <= 0 )
        replace item 슬롯 of 적군살아있음 with 0
        change 적군수 by -1
        change 골드 by 처치골드
        play sound coin
        call 데미지팝업(값=처치골드, x=팝업x, y=(레인Y+20), 종류=1)   # 금색 +골드
```
**단일공격_아군(슬롯, 뎀, 팝업x)** — 대칭(슬롯=0이면 `하얀킹체력−뎀`·`하얀킹피격`, 아니면 `아군HP[슬롯]−뎀`, 0이하 시 `아군살아있음=0`·`아군수−1`. 적 처치엔 골드 없음)

**광역공격_아군이적을(중심, 반경)** — 룩·퀸: 반경 안 살아있는 적 전부에 `계산뎀` + 포격음
```
play sound cannon
set 루프j = 1
repeat (length of 적군X)
    if (item 루프j of 적군살아있음 = 1) and ( abs((item 루프j of 적군X) - 중심) <= 반경 )
        replace item 루프j of 적군HP with ( (item 루프j of 적군HP) - 계산뎀 )
        call 데미지팝업(값=계산뎀, x=item 루프j of 적군X, y=레인Y, 종류=0)
        if ( (item 루프j of 적군HP) <= 0 )
            replace item 루프j of 적군살아있음 with 0 ; change 적군수 by -1
            change 골드 by 처치골드 ; play sound coin
    change 루프j by 1
# 반경 안 적이 하나도 없고 검은킹이 반경 안이면(무리 다 뚫음) 검은킹에 데미지
if (적군수 = 0) and ( abs(검은킹X - 중심) <= 반경 )
    change 검은킹체력 by (0 - 계산뎀) ; broadcast 검은킹피격
```
> **광역공격_적이아군을** 은 대칭(검은룩·검은퀸). `루프j` 는 광역 전용 인덱스라 바깥 `루프i` 순회와 안 겹침.
> **★ 퀸의 이중 공격 판정**: 퀸(타입 5)은 한 번의 공격 쿨(`퀸_공속`)에 **`단일공격_적`(비숍처럼 최전방 하나) + `광역공격_아군이적을`(룩처럼 사거리 안 전부)** 을 **연달아 호출**한다 → 최전방 적은 단일 데미지 + 광역 데미지를 **둘 다** 맞아 특히 아프고, 뒤의 무리는 광역만 맞는다. 검은퀸 보스도 대칭으로 아군을 이중 타격. (교대 방식 변형은 §9 미션 사다리.)

**(B) 적 스포너 forever**
```
when I receive 게임시작
  forever
    if (게임상태 = 1)
        change 적소환타이머 by (0 - 0.1)
        if (적소환타이머 <= 0) and (적군수 < 적최대유닛수)
            # ── 스테이지 조합으로 적 타입 결정 ──
            if (스테이지 <= 1) : set 적생성타입 = 1                                    # 검은폰만
            else if (스테이지 <= 3) : set 적생성타입 = (1 + (pick random 0 to 1))       # 폰·비숍
            else if (스테이지 <= 5) : set 적생성타입 = (pick from {1,2,3,4})            # +나이트·룩
            else                                                                       # 6+ : 가끔 퀸 보스
                if (pick random 1 to 8 = 1) : set 적생성타입 = 5                        # 검은퀸 보스
                else : set 적생성타입 = (pick from {1,2,3,4})
            # ── 적 리스트에 append ──
            add 검은킹X to 적군X                                       # 검은 킹 앞에서 등장
            add (체력적(적생성타입) * 적배율) to 적군HP               # 지수 스케일!
            add 적생성타입 to 적군타입
            add 1 to 적군살아있음
            add 0 to 적군쿨
            set 새적슬롯 = (length of 적군X)
            change 적군수 by 1
            broadcast 적소환 and wait                                 # 적군유닛 클론이 새적슬롯 캡처
            # ── 다음 스폰까지 대기(스테이지 오를수록 짧게) ──
            set 적소환타이머 = ( 적소환간격 - ((스테이지 - 1) * 적소환간격감소) )
            if (적소환타이머 < 적소환최소간격) : set 적소환타이머 = 적소환최소간격
    wait 0.1
```
> 빌더 노트: `pick from {1,2,3,4}` 는 `(1 + (pick random 0 to 3))`. 스테이지 4~5 조합에 나이트·룩 포함.

### 소환버튼 (마우스 폴링 — `이 스프라이트를 클릭했을 때` 금지, 5칸)

```
when green flag clicked
  show ; goto (0,-150) ; set size 100 ; go to front ; set rotation style [don't rotate]

when I receive 게임시작
  forever
    # 해금 상태 코스튬(시작잠금/나이트해금/룩해금/모두해금 — 순차)
    if (나이트해금 = 0) : switch costume to [시작잠금]
    else if (룩해금 = 0) : switch costume to [나이트해금]
    else if (퀸해금 = 0) : switch costume to [룩해금]
    else : switch costume to [모두해금]
    # 소환 쿨타이머 감소
    change 폰쿨타이머 by (0 - 0.02) ; change 비숍쿨타이머 by (0 - 0.02) ; change 나이트쿨타이머 by (0 - 0.02)
    change 룩쿨타이머 by (0 - 0.02) ; change 퀸쿨타이머 by (0 - 0.02)
    # ── 마우스 클릭 폴링 (버튼 바 5등분: -240..-144..-48..48..144..240) ──
    if (게임상태 = 1) and (mouse down?) and (mouse y < -120)
        if (mouse x < -144) : call 소환시도(1, 폰코스트, 폰쿨타이머, 폰_소환쿨, 해금=1)
        else if (mouse x < -48) : call 소환시도(2, 비숍코스트, 비숍쿨타이머, 비숍_소환쿨, 해금=1)
        else if (mouse x < 48) : call 소환시도(3, 나이트코스트, 나이트쿨타이머, 나이트_소환쿨, 해금=나이트해금)
        else if (mouse x < 144) : call 소환시도(4, 룩코스트, 룩쿨타이머, 룩_소환쿨, 해금=룩해금)
        else : call 소환시도(5, 퀸코스트, 퀸쿨타이머, 퀸_소환쿨, 해금=퀸해금)
        wait until (not (mouse down?))                                 # 디바운스: 뗄 때까지 대기
    wait 0.02
```
**소환시도(타입, 코스트, 쿨타이머, 소환쿨, 해금)**
```
if (해금 = 1) and (골드 >= 코스트) and (쿨타이머 <= 0) and (아군수 < 최대유닛수)
    change 골드 by (0 - 코스트)
    # ── 아군 리스트 append ──
    add (하얀킹X + 25) to 아군X                        # 하얀 킹 앞 등장
    add (체력아군(타입) * 유닛체력배수) to 아군HP
    add 타입 to 아군타입
    add 1 to 아군살아있음
    add 0 to 아군쿨
    set 소환타입 = 타입 ; set 새아군슬롯 = (length of 아군X)
    change 아군수 by 1
    # 소환음(타입별 pitch)
    if (타입=1): pitch 0  else if (타입=2): pitch 20  else if (타입=3): pitch -20  else if (타입=4): pitch -40  else: pitch 40
    play sound summon
    broadcast 아군소환 and wait                         # 아군유닛 클론이 새아군슬롯 캡처
    set (해당 쿨타이머) = 소환쿨                        # 타입별 폰~퀸쿨타이머 중 맞는 것
else
    pitch 0 ; play sound error
```
> 빌더 노트: `해당 쿨타이머` 는 타입별로 `폰쿨타이머`…`퀸쿨타이머` 중 맞는 것에 set(소환시도를 5갈래로 펼치거나 타입 인자로 분기). **`이 스프라이트를 클릭했을 때` 금지** — 다른 스프라이트가 버튼을 가려도 `mouse down?` 폴링이 동작(castle-defense 교훈). `wait until (not mouse down?)` 디바운스로 한 클릭당 1소환. x영역 5등분 경계는 소환버튼 SVG의 5칸 그림과 정확히 맞출 것.

### 아군유닛 (렌더러 — 위치·모습만)

```
when green flag clicked
  hide ; set rotation style [don't rotate] ; set 복제됨 = 0 ; point in direction 90

when I receive 아군소환
  if (복제됨 = 0) : create clone of [myself]

when I start as a clone
  set 복제됨 = 1
  set 슬롯 = 새아군슬롯                                  # 캡처(and wait 덕에 안정)
  # 타입별 코스튬·크기
  if (item 슬롯 of 아군타입 = 1) : switch costume to [폰] ; set size 55
  else if (item 슬롯 of 아군타입 = 2) : switch costume to [비숍] ; set size 58
  else if (item 슬롯 of 아군타입 = 3) : switch costume to [나이트] ; set size 70
  else if (item 슬롯 of 아군타입 = 4) : switch costume to [룩] ; set size 68
  else : switch costume to [퀸] ; set size 72
  goto ((item 슬롯 of 아군X), (레인Y + (슬롯 mod 3) - 1)) ; show    # 슬롯별 ±1 Y 흔들기(겹침 완화)
  forever
    if (게임상태 != 1) : delete this clone                          # 강화/게임오버 → 전장 정리
    if (item 슬롯 of 아군살아있음 = 0)                              # 죽음 → 폭죽
        switch costume to [폭죽] ; play sound death
        repeat 5 : change size by 8 ; change [ghost] effect by 20 ; wait 0.02
        delete this clone
    goto ((item 슬롯 of 아군X), (레인Y + (슬롯 mod 3) - 1))          # 매니저가 갱신한 X로 렌더
    wait 0.02
```
> 빌더 노트: 클론은 **goto로만 렌더**(move-steps 미사용 → `b_movesteps` 변수 id 함정 회피). 좌표 계산은 `item 슬롯 of 아군X` 리포터로 넘김(리터럴 상수 금지). `아군소환`이 **broadcast and wait** 라 클론 start가 `새아군슬롯`을 덮어쓰기 전에 캡처된다.

### 적군유닛 (렌더러)

> 아군유닛과 **대칭**. `적소환` 수신 → 클론, `슬롯=새적슬롯`. 코스튬 `검은폰/검은비숍/검은나이트/검은룩/검은퀸`(왼쪽 바라봄) + `폭죽`. `item 슬롯 of 적군X/타입/살아있음`으로 렌더. `게임상태≠1`이면 삭제. `don't rotate`+`point in direction 90` 유지, 코스튬만 좌향.

### 숫자팝업 (플로팅 데미지/골드 — say 미사용)

> castle-defense 숫자팝업과 **동일**. `팝업종류=0`이면 흰 숫자(코스튬 0~9), `=1`이면 금 숫자(코스튬 10~19).

```
when green flag clicked
  hide ; set 복제됨 = 0 ; set rotation style [don't rotate]

when I receive 데미지표시
  if (복제됨 = 0)
      set 데미지글자수 = (length of 데미지표시값) ; set 데미지자리 = 1
      repeat 데미지글자수
          set 데미지숫자 = (letter 데미지자리 of 데미지표시값)
          set 데미지오프셋 = ((데미지자리 - 1) * 14) - ((데미지글자수 - 1) * 7)
          create clone of [myself] ; change 데미지자리 by 1

when I start as a clone
  set 복제됨 = 1 ; go to front
  switch costume to (데미지숫자 + (팝업종류 * 10) + 1)     # 1~10 흰 / 11~20 금 (코스튬 1-기반)
  goto ((데미지표시x + 데미지오프셋), 데미지표시y) ; set [ghost] effect to 0 ; show
  repeat 12 : change y by 4 ; change [ghost] effect by 8 ; wait 0.02
  delete this clone
```
> `데미지팝업(값,x,y,종류)` 헬퍼(매니저): `set 데미지표시값=값; set 데미지표시x=x; set 데미지표시y=y; set 팝업종류=종류; broadcast 데미지표시`. 코스튬 이름을 숫자 문자열로 두고 흰/금 두 세트로 매칭(castle-defense 방식).

### 강화카드

```
when green flag clicked
  hide ; goto (0,20) ; set size 100 ; go to front

when I receive 스테이지클리어
  show
  wait until (key [1] or key [2] or key [3] or key [4])
  if key [1] : change 초당골드 by 강화골드증가
  if key [2] : set 유닛공격력배수 to (유닛공격력배수 * 강화공격배수)
  if key [3] : set 유닛체력배수 to (유닛체력배수 * 강화체력배수)
  if key [4] : change 하얀킹최대체력 by 강화킹수리 ; set 하얀킹체력 = 하얀킹최대체력
  play sound upgrade
  hide ; wait 0.15
  # ── 다음 스테이지 준비 ──
  change 스테이지 by 1
  set 적배율 = 1 ; repeat (스테이지 - 1) : set 적배율 = (적배율 * 적성장배율)   # ★ 지수 스케일(곱셈 루프)
  set 검은킹체력 = (검은킹기본체력 * 적배율)
  if (스테이지 >= 나이트해금스테이지) : set 나이트해금 = 1
  if (스테이지 >= 룩해금스테이지) : set 룩해금 = 1
  if (스테이지 >= 퀸해금스테이지) : set 퀸해금 = 1
  # 전장 리셋(클론은 게임상태=2일 때 이미 자기 삭제됨)
  delete all of 아군X ; ... (10개 리스트 전부 delete all) ...
  set 아군수 = 0 ; set 적군수 = 0 ; set 적소환타이머 = 0 ; set 골드누적 = 0
  set 게임상태 = 1
  broadcast 스테이지시작                                # 뿔피리
  broadcast 강화완료
```
> `적성장배율^(스테이지−1)` 은 Scratch에 지수 블록이 없으므로 **repeat 곱셈 루프**로 계산(위). 빌더는 이 한 줄을 루프로 펼칠 것.

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식 1층", "조건 2층", "새 메커닉 4층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화만 건드린다.

### 0층 — 숫자 하나 바꾸기 (무대 초록 깃발 스크립트)
- **부자 영주**: `시작골드` 를 `200` → `800` 으로. 시작부터 군대를 쏟아붓는다.
- **황금 폭포**: `초당골드` 를 `8` → `20` 으로. 유닛을 쉴 새 없이 뽑는다.
- **싸진 폰**: `폰코스트` 를 `18` → `5` 로. 폰을 끝없이 도배!
- **강철 나이트**: `나이트_체력` 을 `22` → `80` 으로. 나이트가 벽이 된다.
- **날쌘 폰**: `폰_속도` 를 `2.3` → `4` 로. 폰이 순식간에 적진으로.
- **한 방 퀸**: `퀸_공격력` 을 `7` → `30` 으로. **단일+광역이 둘 다 세지는지**(맞을 때 데미지 숫자) 확인!
- **묵직한 룩**: `룩_사거리` 를 `130` → `220` 으로. 화면 절반을 포격으로 쓸어버린다.
- **튼튼한 킹**: `하얀킹최대체력` 을 `100` → `300` 으로. 훨씬 오래 버틴다.
- **순한 적 / 사나운 적**: `적성장배율` 을 `1.25` → `1.1`(쉬움) 또는 `1.5`(지옥) 로. **한 숫자로 난이도 곡선 전체가 바뀐다!**
- **빨리 해금**: `나이트/룩/퀸해금스테이지` 를 `2,3,4` → `1,1,1` 로. 처음부터 다섯 유닛 전부.

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **후반 보상 폭발**: Stage (D) 클리어 골드를 `스테이지클리어골드` → `스테이지클리어골드 × 스테이지` 로. 갈수록 골드가 팍팍.
- **점점 몰아치는 적**: `적소환간격감소` 를 `0.12` → `0.3` 으로. 후반 스테이지에 검은 군단이 물밀듯이.
- **강화 더 세게**: `강화공격배수` 를 `1.15` → `1.4` 로. 공격 강화 한 번이 훨씬 크다(적 지수 성장을 따라잡기).

### 2층 — if 조건 더하기
- **퀸 러시 보스전**: 스포너에 `if (스테이지 mod 5 = 0) : set 적생성타입 = 5` 추가 — 5의 배수는 검은 퀸이 쏟아지는 보스전.
- **위기 보너스**: Stage (D)에 `if (하얀킹체력 < 20) : change 초당골드 by 5` 로 벼랑 끝 역전 장치.
- **강화 5번(즉시 군대)**: 강화카드에 `if key [5] : change 골드 by 300` 선택지를 더하고 카드에 5칸을 그린다.

### 3층 — 규칙 자체를 바꾸기 (분기)
- **퀸의 교대 공격**: 퀸을 매 쿨 단일+광역 동시 대신 **번갈아** 쏘게 — 클론-로컬 `교대` 변수로 홀수 쿨엔 단일, 짝수 쿨엔 광역만(브리프의 "동시/교대" 변형).
- **강화로 유닛 해금**: 강화카드에 `if key [5] : set 퀸해금 = 1` 을 더해 강화로 직접 해금.
- **광역 폰**: 폰도 룩처럼 광역이 되게 매니저에서 `계산타입=1` 도 `광역공격` 을 부르도록(전열이 한꺼번에 쓸림).

### 4층 — 새 메커닉 만들기 (구조)
- **여섯 번째 유닛(성기사)**: 아군 스탯 6개(성기사_*)와 코스트·소환 버튼 6번째 칸, 매니저 `속도/공격력/…(타입)` 분기 6번째 갈래를 추가.
- **킹의 반격**: 하얀 킹이 스스로 최전방 적을 쏘게, 매니저에 "하얀킹도 하나의 공격자" 규칙을 추가(하얀킹X 기준 사거리·공격력 변수 신설).
- **두 줄 전선**: `레인Y` 를 슬롯 짝/홀로 두 값 중 선택하게 바꿔 위·아래 두 전선을 만든다(전투 판정은 여전히 X).
- **황금 기물**: 스포너가 가끔(`pick random 1 to 10 = 1`) 뽑는 반짝 적을 잡으면 `처치골드 × 5`(새 타입/색).

## 10. 학습 포인트

보너스 직관 액션 전쟁 게임 — 수학/추상 학습 콘셉트를 전면에 내세우지 않음([[feedback-game-design]] 준수: 데시벨/음파/산성 같은 추상 콘셉트 금지). **castle-defense/magic-survivor와 똑같이 "코딩 학습"이 핵심 가치**: 모든 능력치(초당골드·코스트·유닛 5종 및 적 5종의 체력/공격력/공속/속도/사거리·`적성장배율`·소환쿨·강화량)가 한글 튜닝 변수·리스트로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (4층) 새 유닛/전선을 추가하며 **변수·리스트·병렬 배열·인덱스·연산·조건·클론·broadcast and wait** 를 게임을 망가뜨리고 고치며 체득한다.
부수적으로 **메커닉의 결과로** 자연스럽게 만나는 수학:
- **지수 스케일링(곱셈 성장)**: `적 = 기본 × 적성장배율^(스테이지−1)`. 적은 곱하기로 커지는데 내 강화는 조금씩 → **직선으로는 못 따라잡고 지수(공격배수 ×1.15 반복)로 따라가야 하는 벽**을 몸으로 느낀다. `적성장배율` 한 숫자가 난이도 곡선 전체를 정한다는 것을 실험으로 발견.
- **가성비(나눗셈 직관)**: 폰·비숍·나이트·룩·퀸의 **공격력÷코스트·체력÷코스트**를 비교하며 "무엇을 몇 개 뽑을지"를 저울질(3~4스테이지부터 조합 고민). 퀸은 단일+광역을 둘 다 하는 대신 최고 코스트 — "비싼 게 정말 이득인가?"를 계산하게 된다.
- **최댓값/최솟값 찾기**: 매니저가 매 틱 최전방(최대 X 아군 / 최소 X 적)을 리스트 순회로 찾는다.
- **자원 관리·누적**: 초당골드 누적과 소비의 균형.

어디까지나 **직관적인 라인 배틀 전쟁 게임**으로 포장된 채, 코드 곳곳의 **노란 코멘트(가이드 투어)** 가 각 부분을 설명한다.
