# 풍선 슈터 (balloon-shooter) — Plan

> 60초 동안 화면 아래에서 떠오르는 다양한 색·크기 풍선을 마우스 클릭으로 격파하는 캐주얼 사격 게임.
> 베이스: `games/duck-hunt/build.py` (마우스 추적 십자선 + 클릭 + 섬광 클론) + `games/whack-a-prime/build.py` (60초 타이머 + 클론 무한 스폰).
> 학습 콘셉트 절대 없음. 순수 액션 (조준 정확도 + 빠른 반응).

## 1. 컨셉 / 메커닉

- 화면 하단에서 풍선이 무작위 위치에서 떠올라 천천히 위로 상승 (속도는 크기에 반비례).
- 풍선은 5가지 색상 (빨강·파랑·노랑·초록·보라) 중 랜덤, 3가지 크기 (작음·중간·큼) 중 랜덤.
- 풍선은 살짝 좌우 흔들림 (sin 흔들림).
- 플레이어는 마우스로 십자선을 조준하고 **클릭** 시 사격 — 클릭 위치에 노란 섬광이 0.12초 표시.
- 섬광이 풍선에 닿으면 풍선이 "터짐" 코스튬으로 바뀌고 0.1초 뒤 사라짐 (+점수).
- 크기별 점수: **작은 풍선 30점, 중간 20점, 큰 10점**. (작을수록 맞히기 어려움)
- 풍선이 화면 위 (y>180) 로 사라지면 점수 X, 그냥 클론 삭제.
- 60초 타이머. 시간 0 되면 게임 종료, "TIME UP!" 배너 + 최종 점수 표시.

라운드 없음. 한 판 = 60초 + 최종 점수. 단순.

## 2. 화면 레이아웃 (Scratch 480×360)

```
y=180  ─── (풍선이 여기 도달하면 사라짐) ───
                    ↑ 풍선 천천히 상승
y=0    🎈🎈    🎈        🎈  🎈
                    ↑ 좌우 흔들림
y=-180 ───── 풍선 스폰 라인 (y=-180) ─────
       (랜덤 x: -200..200)
```

배경: 하늘 그라데이션 (위 연한 보라 → 아래 연한 분홍) + 태양/구름.
HUD: 좌상단에 점수 / 시간 변수 모니터.

## 6. 스프라이트별 블록 흐름

### Stage

**A) 깃발 클릭 → 초기화 + 시작**
1. 점수=0, 시간=60, 게임상태=1
2. 1초 대기
3. `게임시작` 방송

**B) 게임시작 받으면 (스폰 루프)**
- forever:
  - if 게임상태 = 1:
    - 풍선스폰X = random(-200, 200)
    - 풍선색 = random(1, 5)
    - r = random(1, 3): 크기 매핑 → 1=40(작), 2=70(중), 3=100(큰). 단순화: `풍선크기 = pick random 1..3`, 풍선 스프라이트가 이걸 보고 분기.
    - `풍선스폰` 방송
    - wait random(0.4, 0.8)
  - else: wait 0.1 (게임 종료 후 idle)

**C) 게임시작 받으면 (타이머)**
- repeat 60:
  - wait 1
  - 시간 -1
  - if 시간 = 0: `게임종료` 방송, 게임상태=0, stop this script

(Stage 에 핸들러가 두 개 — 둘 다 `게임시작` 받으면 — Scratch에서 동시 실행됨.)

### 조준선 (crosshair) — duck-hunt 와 거의 동일
1. **깃발** → 크기 80, 맨앞, 보이기, forever: 마우스X/Y로 이동.
2. **이 스프라이트 클릭** → if 게임상태=1:
   - 섬광X = mouseX, 섬광Y = mouseY
   - "섬광" 클론 생성
   - pop 사운드 (피치 200)

### 섬광 (flash) — duck-hunt 와 동일
1. **깃발** → 숨김, 크기 65.
2. **클론으로 시작** → 섬광X/Y로 이동 → 보이기 → 0.12초 대기 → 숨김 → 클론 삭제.

### 풍선 (balloon) — 클론 메인

1. **깃발 클릭** → 숨김, 크기 70.
2. **풍선스폰 받으면** → 본체는 클론 생성만:
   - 본체는 보이지 않음. `create clone of myself` 만.
3. **클론으로 시작**:
   - 코스튬을 풍선색 변수에 맞춰 설정 (1=red, 2=blue, ..., 5=purple)
     - 5번 if 분기 또는 `switch costume to (풍선색)` (Scratch 의 변수 → 코스튬 번호로 직접 가능).
     - 단순화: `switch costume to (풍선색)` 로 번호 직접 사용. 코스튬 순서를 red→blue→yellow→green→purple 로 맞춤. (코스튬 6번 pop_fx 는 격파시만 사용.)
   - 크기 분기: 풍선크기 변수가 1/2/3 → 크기 40/70/100 으로 설정. 점수도 30/20/10 매핑.
     - 로컬 변수 `내크기` 와 `내점수` 두 개 사용 (스프라이트 로컬).
   - 위치: x = 풍선스폰X, y = -180
   - 보이기, 맨앞 레이어
   - 로컬 `흔들위상` = random(0, 360)
   - **상승 루프** repeat until (y position > 180) OR (게임상태 = 0):
     - change y by (작=1.8, 중=1.2, 큰=0.7) — 단순화: `y by (1.8 - (내크기 / 100))` 같은 식.
       - 명확하게: `y by (2.5 - 내크기 * 0.018)` → 크기 40 → 1.78, 70 → 1.24, 100 → 0.7. OK.
     - 좌우 흔들림: 흔들위상 += 8; change x by (sin(흔들위상) * 0.8)
     - if touching 섬광:
       - 점수 += 내점수
       - 코스튬 = pop_fx
       - pop 사운드 피치 (작=300, 중=100, 큰=-100) — 단순화: 피치 = (4 - 내크기 매핑 인덱스) * 200 류. 명확하게: `set pitch to (300 - 내크기 * 2)` → 크기 40→220, 70→160, 100→100. 다소 변화 있으면 충분.
       - wait 0.12
       - 클론 삭제
     - wait 0.03
   - 루프 탈출 후 (놓침 또는 게임종료): 클론 삭제.

4. **게임종료 받으면 (클론에서)** → 즉시 클론 삭제 (선택 사항: 빠른 정리).

### 게임오버 (timeup banner)
- duck-hunt 의 게임오버 패턴 그대로:
- 깃발 → 숨김 → x=0,y=0,크기=100,맨앞
- wait until 게임상태 = 1
- wait until 게임상태 = 0
- show
