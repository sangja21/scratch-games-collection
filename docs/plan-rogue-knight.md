# 로그 나이트 (rogue-knight) — Plan

> 한 화면짜리 전투 방을 연달아 클리어하는 **횡스크롤 액션 로그라이트**(Dead Cells 라이트). 점프·중력 플랫포머 위에서 근접 검 공격으로 적을 베고, 대시/2단 점프로 회피한다. 적은 **약·중·강 3종**으로, 방이 깊어질수록(방1=약, 방2=약+중, 방3+=중+강) 더 단단하고 빠른 적이 나온다. 방의 적을 전부 잡으면 **강화 택1(공격력+ / 체력+ / 이동속도+)** 을 먹고 다음 방으로. 죽으면 처음부터. 매 판 강해지는 맛.
> 베이스: `games/geometry-dash/build.py`(VY·중력·점프·바닥 클램프 패턴) + `games/doodle-jump/build.py`(좌우 이동·VY 처리) + `games/tank-battle/build.py`(클론 스포너·복제됨 가드·체력/무적·폭발 연출·게임오버 배너).
> 학습 콘셉트 없음(보너스 액션 게임). **이 게임의 진짜 존재 이유는 "아이가 코드의 숫자·규칙을 직접 바꾸며 노는 것"** — 그래서 모든 조절 가능한 값을 한글 전역 변수로 노출하고, 매직넘버를 코드에 흩뿌리지 않는다. ★★★.

- **주제**: 방 클리어형 검투사 로그라이트
- **카테고리**: 액션
- **난이도**: ★★★☆☆
- **폴더**: `games/rogue-knight/`
- **출력 파일**: `로그_나이트.sb3`
- **빌드**: `python3 games/rogue-knight/build.py`

## 1. 게임 한 줄

기사 한 명으로 한 화면짜리 방에 갇혀, 다가오는 적을 검으로 베어 전멸시킨다. 적은 약·중·강 3종이며 방이 깊어질수록 센 적이 섞여 나온다. 방을 비우면 강화 셋 중 하나를 골라 더 세진 채 다음 방으로 넘어간다. 내 체력이 0이 되면 1번 방부터 다시 시작(로그라이트). 미리 정의된 방 3개를 순환하며 돌수록 적이 많아지고 강해진다.

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

                                          (1/2/3 키로 선택 → 다음 방)
  체력 0 → 화면 중앙에 GAME OVER 배너 → 깃발 재클릭으로 1방부터.

- HUD: 좌상단에 `체력` `방번호` 변수 모니터 + `처치현황`(처치수/방목표) 모니터. 튜닝 변수(공격력 등)는 **모니터 숨김**(코드에서만 본다)이되, 아이가 무대에서 직접 체크해 켤 수 있다.
- 바닥선 y=-130 고정(기사 발 기준 정지 y ≈ -90, size 따라 조정). 공중 발판은 방마다 다른 고정 좌표(절차생성 금지 — 방 3종 하드코딩).

> **아트 방향**: 기사·적은 **오른쪽을 보게** 그린다(rotationStyle `left-right`이므로 방향 -90이면 자동 좌우반전). 검판정/베기 이펙트도 오른쪽 기준. 기사 발바닥이 정지 y와 맞도록 rotationCenterY 를 발 근처로.
> **검판정 분리 패턴(tank-battle 의 포탄 분리와 동형)**: 데미지 판정을 기사 본체가 아니라 별도 `검판정` 스프라이트가 담당한다. 적은 `touching 검판정`만 보면 되고, friendly-fire 분기가 필요 없다. 검을 안 휘두를 땐 `검판정`이 숨겨져 있어 충돌이 없다.
> **적 3종 외형 구분**: 클론은 `적종류`(1=약/2=중/3=강)에 따라 색(color 이펙트)과 크기(size)를 다르게 해 한눈에 구분되게 한다. 능력치는 `약한적_체력/속도`·`중간적_체력/속도`·`강한적_체력/속도` 6개 튜닝 변수에서 읽어 `내체력`/`내속도`에 세팅(매직넘버 없음).
> **데미지 숫자 코스튬 렌더(say 미사용)**: 데미지 팝업은 말풍선 대신 `데미지` 스프라이트의 숫자 코스튬 0~9 를 자릿수별 클론으로 표시한다. 폰트·말풍선 위치 편차 없이 항상 같은 자리에 또렷이 뜬다.

| 데미지숫자 | `varDmgDigit` | 0 | 데미지 코스튬 렌더용: 현재 클론이 표시할 한 자리 숫자(0~9) |
| 데미지오프셋 | `varDmgOff` | 0 | 데미지 코스튬 렌더용: 자릿수별 클론의 가로 위치 오프셋(자리 정렬) |

총 진행 전역 변수 21개. (전역 변수 합계 = 18 + 21 = **39개**)

> 모니터로 띄울 것: `체력` `방번호` `처치수`/`방목표`(처치현황) — 3~4개만. 나머지는 모니터 숨김.

### 5.3 클론-로컬 변수 — ID `V_*ISC`

`복제됨` 플래그(원본=0, 클론=1)로 클론 증식 가드(tank-battle 패턴).

| 스프라이트 | 한국어 | ID | 초기값 |
|------------|--------|----|--------|
| 적 | 복제됨 | `varEnemyIsClone` | 0 | 클론 증식 가드(원본=0, 클론=1) |
| 적 | 내체력 | `varEnemyHP` | 2 | 이 클론 개체의 남은 체력(`적종류`에 따라 종류별 체력 변수로 시작) |
| 적 | 피격쿨 | `varEnemyHit` | 0 | >0 동안 같은 검질에 중복 피격 방지(틱) |
| 적 | 적종류 | `varEnemyType` | 1 | 이 클론의 종류(1=약/2=중/3=강). `적생성종류`에서 복사 |
| 적 | 내속도 | `varEnemySpd` | 1.2 | 이 클론의 추적/순찰 이동량(`적종류`에 따라 종류별 속도 변수로 시작) |
| 발판 | 복제됨 | `varPFIsClone` | 0 |
| 데미지 | 복제됨 | `varDmgIsClone` | 0 | 데미지 팝업 클론 증식 가드(원본=0, 클론=1) |

> 적 클론-로컬 변수 5개(복제됨/내체력/피격쿨/적종류/내속도), 발판 1개, 데미지 1개.

### 5.4 메시지(방송) — ID `BR_*` / `br*`

| 한국어 | ID | 트리거 → 동작 |
|--------|----|---------------|
| 게임시작 | `brStart01` | Stage 깃발 초기화 직후 → 첫 방 빌드 |
| 방빌드 | `brBuild02` | 새 방 진입 시 → 발판 배치 + 적 방목표 수만큼 스폰 |
| 강화등장 | `brUpgrade03` | 방목표 처치 달성 → 강화패널 표시(게임상태=2) |
| 다음방 | `brNext04` | 강화 선택 완료 → 방번호+1, 처치수 0, 게임상태=1, 방빌드 재호출 |
| 데미지표시 | `brDmg05` | 적이 검에 맞는 순간(`데미지표시값`=공격력, `데미지표시x/y`=적 위치 세팅 후 발송) → 데미지 스프라이트가 그 자리에 데미지 숫자 팝업 클론 생성 |

총 방송 5개.

## 6. 씬 / 상태머신

```
[깃발 클릭]
   │  Stage: 튜닝 변수 18개(적 종류별 6 포함) + 진행 변수 전부 초기화(체력=최대체력, 방번호=1, 게임상태=1 …)
   │  기사: 위치(-160,-90), VY=0, 점프남음=점프횟수상한
   ▼
[게임시작 방송] → 방번호=1 로 [방빌드]
   ▼
[방빌드 방송]  (게임상태=1)
   │  방목표 = 2 + 방번호      ← 방마다 적 수 증가(튜닝 가능 규칙)
   │  발판: 방번호별 고정 좌표 세트 배치(방 3종 순환)
   │  적 스포너: 방목표 수만큼 좌우 랜덤 위치에 클론 스폰. 매 클론마다 등장 규칙
   │           (방1=약 / 방2=약·중 번갈아 / 방3+=중·강 번갈아)으로 `적생성종류` 세팅, 적수=방목표
   ▼
[전투중 forever]  (게임상태=1)
   ├ 기사: ←→ 이동 / 점프(2단) / 중력+VY / 발판·바닥 착지 / Z 검 / X 대시 / 적 접촉→체력-1(무적)
   ├ 검판정: Z 입력 시 기사 앞 0.18초 나타남 → 닿은 적에 데미지
   ├ 적 클론: 순찰/추적 + 접촉 데미지 + touching 검판정 → 내체력 -공격력 / 0이하면 폭발(처치수+1, 적수-1)
   └ Stage 클리어 감시: 처치수 ≥ 방목표 → 게임상태=2, [강화등장]
   ▼
[강화선택중]  (게임상태=2)
   │  강화패널 show. 1/2/3 키 입력:
   │   1 → 공격력 += 강화량
   │   2 → 최대체력 += 강화량 ; 체력 += 강화량(회복)
   │   3 → 이동속도 += 강화량
   │  선택 후 [다음방] 방송
   ▼
[다음방 방송]
   │  방번호 += 1 (3 넘으면 1로 순환하되 난이도 누적: 방목표는 계속 증가)
   │  처치수=0, 게임상태=1, 기사 위치 리셋 → [방빌드]
   ▼  (위 전투중 루프로 복귀)
[종료]  체력 ≤ 0 → 게임상태=0
   │  게임오버 배너 show / 잔여 적 클론 자기 삭제
   ▼
[다시 시작]  초록 깃발 재클릭(1방부터, 능력치 기본값으로 — 로그라이트)
```

> 방 순환 규칙(빌더 주의): 발판 좌표는 `방번호 mod 3`(1/2/0)으로 3세트를 순환. 단 **방목표 적 수는 `2 + 방번호`로 계속 증가**(돌수록 어려워짐). 방번호가 4,5,6… 으로 커져도 발판 레이아웃만 재사용.

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait` 넣어 CPU 폭주 방지. 이동·충돌 틱은 0.025초(≈40fps). **매직넘버 금지** — 항상 튜닝 변수를 참조.

### Stage

**(A) 깃발 클릭 → 변수 전부 초기화(한 곳에 모음) → 게임시작**
```
when green flag clicked
  # ── 개조 손잡이(튜닝) 기본값: 아이가 여기 숫자만 바꾸면 게임이 바뀐다 ──
  set 공격력 = 1 ; set 최대체력 = 5 ; set 이동속도 = 4
  set 점프력 = 12 ; set 중력 = -1 ; set 점프횟수상한 = 2
  set 대시거리 = 90 ; set 대시쿨 = 30 ; set 무적시간 = 25
  # ── 적 종류별 능력치(약/중/강) ──
  set 약한적_체력 = 1 ; set 약한적_속도 = 2.0
  set 중간적_체력 = 2 ; set 중간적_속도 = 1.3
  set 강한적_체력 = 4 ; set 강한적_속도 = 0.8
  set 강화량 = 1
  # ── 진행 상태 ──
  set 체력 = 최대체력
  set 방번호 = 1 ; set 처치수 = 0 ; set 적수 = 0 ; set 총처치 = 0
  set 게임상태 = 1 ; set VY = 0
  set 점프남음 = 점프횟수상한 ; set 점프이전키 = 0 ; set 대시이전키 = 0
  set 대시타이머 = 0 ; set 무적 = 0
  wait 0.3
  broadcast 게임시작
```

**(B) 게임시작 → 첫 방 빌드**
```
when I receive 게임시작
  set 방번호 = 1
  broadcast 방빌드
```

**(C) 방빌드 받으면 방목표 계산**
```
when I receive 방빌드
  set 방목표 = 2 + 방번호      ← 방 1=3마리, 2=4마리, 3=5마리 …
  set 처치수 = 0
  (발판·적 스프라이트가 같은 방빌드를 받아 각자 배치)
```

**(D) 타이머 감소 forever (대시쿨/무적)**
```
when green flag clicked
  forever
    if 대시타이머 > 0 : change 대시타이머 by -1
    if 무적 > 0 : change 무적 by -1
    wait 0.025
```

**(E) 클리어 감시 forever**
```
when green flag clicked
  wait until (게임상태 = 1)
  forever
    if (처치수 >= 방목표) and (게임상태 = 1)
        set 게임상태 = 2
        broadcast 강화등장
    if (체력 < 1) and (게임상태 = 1)
        set 게임상태 = 0          ← 게임오버
    wait 0.05
```

**(F) 다음방 받으면 진행**
```
when I receive 다음방
  change 방번호 by 1
  set 게임상태 = 1
  broadcast 방빌드
```

### 기사

**(A) 깃발 초기화**
```
when green flag clicked
  show ; set size 60 ; set rotation style [left-right]
  point in direction 90 ; goto (-160,-90) ; go to front
```

**(B) 방빌드/다음방 시 위치 리셋** (방 시작마다 좌측에서)
```
when I receive 방빌드
  goto (-160,-90) ; point in direction 90 ; set VY = 0
  set 점프남음 = 점프횟수상한
```

**(C) 이동·점프·중력 메인 forever**  (전투중일 때만)
```
when green flag clicked
  forever
    if (게임상태 = 1)
      # ── 좌우 이동 ──
      if key [right arrow] : point in direction 90  ; change x by 이동속도
      if key [left arrow]  : point in direction -90 ; change x by (0 - 이동속도)
      clamp x to ±220
      # ── 점프 에지 디텍션(2단) ──
      set 점프키 = (key [up arrow] or key [space])
      if (점프키) and (점프이전키 = 0) and (점프남음 > 0)
          set VY = 점프력
          change 점프남음 by -1
          set sound effect [pitch] to 80 ; play sound pop
      set 점프이전키 = 점프키
      # ── 중력 + 수직 이동 ──
      change VY by 중력
      change y by VY
      # ── 발판 위 착지(내려오는 중에만) ──
      if (touching [발판]) and (VY < 0)
          change y by 5 ; set VY = 0 ; set 점프남음 = 점프횟수상한
      # ── 바닥 착지 ──
      if (y position < -90)
          set y to -90 ; set VY = 0 ; set 점프남음 = 점프횟수상한
    wait 0.025
```
> 점프 가능 조건은 `점프남음 > 0`(착지하면 점프횟수상한으로 리필) → 2단 점프 자연 구현. `점프횟수상한`을 바꾸면 단/2단/3단 점프가 즉시 바뀐다(미션 사다리용).

**(D) 검 입력 forever**
```
when green flag clicked
  forever
    if (key [z]) and (게임상태 = 1)
        switch costume to [knight_attack]
        set sound effect [pitch] to 30 ; play sound pop
        broadcast (검판정에게 알림은 검판정이 직접 key z 폴링으로 처리 — 아래 검판정 (B) 참조)
        wait 0.18
        switch costume to [knight_idle]
        wait 0.12              ← 연타 간 짧은 쿨
    wait 0.02
```
> 빌더 노트: 검판정 스프라이트가 `key z` 를 직접 폴링하므로 별도 방송 없이도 동작. 위 broadcast 줄은 생략 가능(검판정이 자체 폴링). 코드 단순화를 위해 **방송 없이 검판정 자체 폴링**으로 구현.

**(E) 대시 입력 forever**
```
when green flag clicked
  forever
    if (key [x]) and (대시이전키 = 0) and (대시타이머 = 0) and (게임상태 = 1)
        # 바라보는 방향으로 순간 이동(대시거리)
        if (direction = 90) : change x by 대시거리
        else                : change x by (0 - 대시거리)
        clamp x to ±220
        set 대시타이머 = 대시쿨
        set sound effect [pitch] to 150 ; play sound pop
    set 대시이전키 = (key [x])
    wait 0.025
```

**(F) 피격 감시 forever**
```
when green flag clicked
  forever
    if (touching [적]) and (무적 = 0) and (게임상태 = 1)
        change 체력 by -1
        set 무적 = 무적시간
        set sound effect [pitch] to -300 ; play sound pop
    wait 0.04
```
> 무적 동안 살짝 깜빡임(선택): `무적 > 0` 이면 ghost 효과 토글. 미션과 무관한 연출이므로 옵션.

### 검판정 (공격 히트박스)

```
when green flag clicked
  hide ; set size 60 ; set rotation style [left-right]

when green flag clicked
  forever
    if (key [z]) and (게임상태 = 1)
        # 기사 앞쪽에 정렬
        point in direction (direction of 기사)
        if (기사 direction = 90) : goto ((기사 x) + 35, 기사 y)
        else                     : goto ((기사 x) - 35, 기사 y)
        show
        wait 0.18              ← 베기 노출 시간(데미지는 적이 touching 으로 감지)
        hide
        wait 0.12
    wait 0.02
```
> 검판정이 보이는 0.18초 동안 적이 `touching 검판정`을 감지해 데미지. 검판정은 데미지 수치를 모르고, 적이 `공격력`을 깎는다(단순화).

### 적 (스포너 + 클론 본체)

**(A) 깃발 초기화**
```
when green flag clicked
  hide ; set size 55 ; set rotation style [left-right] ; set 복제됨 = 0
```

**(B) 방빌드 → 방목표 수만큼 스폰** (원본만)
```
when I receive 방빌드
  if (복제됨 = 0)
      wait 0.2                       ← 발판 배치와 채널 경쟁 회피(발판이 발판X/Y, 적은 적생성X/Y로 채널 분리되어 사실 안전)
      set 적수 = 0
      repeat 방목표
          set 적생성X = pick random -200 to 200
          set 적생성Y = -90           ← 바닥 위 스폰(공중 발판 위 스폰은 옵션)
          # ── 등장 규칙으로 적생성종류 결정 ──
          if (방번호 = 1)
              set 적생성종류 = 1                          ← 방1: 약만
          else if (방번호 = 2)
              set 적생성종류 = (1 + (적수 mod 2))         ← 방2: 약(1)·중(2) 번갈아
          else
              set 적생성종류 = (2 + (적수 mod 2))         ← 방3+: 중(2)·강(3) 번갈아
          change 적수 by 1
          create clone of [myself]
          wait 0.15
```

**(C) 클론 본체**
```
when I start as a clone
  set 복제됨 = 1
  set 적종류 = 적생성종류        ← 스폰 채널에서 이 클론의 종류 복사
  # ── 종류별 능력치·외형 분기(매직넘버 없음: 종류 튜닝 변수 참조) ──
  if (적종류 = 1)               ← 약
      set 내체력 = 약한적_체력 ; set 내속도 = 약한적_속도
      set size 45 ; set [color] effect to 0
  else if (적종류 = 2)          ← 중
      set 내체력 = 중간적_체력 ; set 내속도 = 중간적_속도
      set size 55 ; set [color] effect to 70
  else                          ← 강
      set 내체력 = 강한적_체력 ; set 내속도 = 강한적_속도
      set size 70 ; set [color] effect to 140
  set 피격쿨 = 0
  goto (적생성X, 적생성Y) ; show
  forever
    # 1) 게임오버/상태 정리
    if (게임상태 = 0) : hide ; delete this clone
    # 2) 검 피격(중복 방지 쿨 적용)
    if (touching [검판정]) and (피격쿨 = 0)
        change 내체력 by (0 - 공격력)
        set 피격쿨 = 10            ← 같은 베기에 여러 틱 중복 차감 방지
        set sound effect [pitch] to 0 ; play sound pop
        # ── 플로팅 데미지 팝업(맞은 자리에 공격력 숫자) ──
        set 데미지표시값 = 공격력      ← 매직넘버 아님: 현재 공격력을 그대로 표시
        set 데미지표시x = x position ; set 데미지표시y = y position
        broadcast 데미지표시
    if 피격쿨 > 0 : change 피격쿨 by -1
    # 3) 처치(폭발 연출 후 삭제)
    if (내체력 < 1)
        change 처치수 by 1 ; change 총처치 by 1 ; change 적수 by -1
        switch costume to [폭발]
        repeat 5 : change size by 12 ; change [ghost] effect by 20 ; wait 0.02
        delete this clone
    # 4) 추적(전투중일 때만 다가옴)
    if (게임상태 = 1)
        if ((x position) > (기사 x)) : point in direction -90
        else                         : point in direction 90
        change x by ( (내속도) * (방향부호) )    ← 방향부호: 90→+1, -90→-1 (내속도=종류별 속도)
        # 바닥 고정(공중 발판 무시: 지상 적)
        set y to -90
        clamp x to ±220
    wait 0.025
```
> 빌더 노트: `방향부호`는 별도 변수 없이 `if direction=90 then change x by 내속도 else change x by (0-내속도)` 로 인라인.
> 접촉 데미지(기사 체력 -1)는 기사 (F) 피격 감시가 처리 — 적은 자기 위치만 유지.

### 발판 (방마다 공중 발판 배치)

**(B) 방빌드 → 방번호 mod 3 으로 3세트 중 하나 배치** (원본만)

when I receive 방빌드
  if (복제됨 = 0)
      # 기존 클론 정리는 stop 으로: 새 방 진입 전 모든 발판 클론 삭제 신호가 필요
      # → 간단화: 발판도 게임상태/방번호 변화 감지해 클론이 스스로 재배치되도록,
      #   여기서는 "방빌드 받으면 기존 클론 삭제 후 새로 배치" 패턴 사용(아래 (D) 참조)
      broadcast (사용 안 함)         ← 실제로는 (D)의 자가삭제로 처리

**(C) 클론 본체** (제자리 발판)

### 데미지 (플로팅 데미지 팝업)

> 적이 검에 맞을 때마다 그 자리에 데미지 숫자(`데미지표시값`=공격력)가 떠올랐다 사라진다. 적 머리 위 상시 체력 표시 대신 도입한 연출 — 공격력을 바꾸면 뜨는 숫자도 따라 바뀐다. **말풍선(say) 대신 숫자 코스튬(0~9)을 자릿수별 클론으로 렌더**해 항상 같은 자리에 또렷하게 표시한다.

**(A) 깃발 초기화**
```
when green flag clicked
  hide ; set 복제됨 = 0 ; set rotation style [don't rotate]
```

**(B) 데미지표시 받으면 자릿수만큼 클론 생성** (원본만)
```
when I receive 데미지표시
  if (복제됨 = 0)
      # 데미지표시값의 각 자리를 왼쪽→오른쪽으로 클론 생성
      set 글자수 = (length of 데미지표시값)
      set i = 1
      repeat 글자수
          set 데미지숫자 = (letter i of 데미지표시값)        ← 이 자리 숫자(0~9)
          set 데미지오프셋 = ((i - 1) * 14) - (((글자수) - 1) * 7)  ← 가운데 정렬용 가로 오프셋
          create clone of [myself]
          change i by 1
```

**(C) 클론 본체** (숫자 코스튬으로 렌더 → 떠올랐다 페이드 후 삭제)
```
when I start as a clone
  set 복제됨 = 1
  go to front
  switch costume to (데미지숫자)          ← 숫자 코스튬 0~9 중 하나 (say 미사용)
  goto ((데미지표시x) + 데미지오프셋, 데미지표시y)
  set [ghost] effect to 0
  show
  repeat 12
      change y by 4          ← 위로 떠오름
      change [ghost] effect by 8   ← 점점 투명
      wait 0.02
  delete this clone
```
> 빌더 노트: 핵심은 `switch costume to (데미지숫자)` — 적 스프라이트가 `데미지표시값=공격력` 으로 채운 뒤 발송하면, (B)가 자릿수만큼 클론을 만들고 각 클론이 한 자리 숫자 코스튬으로 렌더한다. 강화로 `공격력`이 오르면 떠오르는 숫자도 같이 커진다(미션 사다리 "공격력 바꾸기" 와 자연 연동). 코스튬 이름은 `"0"`~`"9"` 로 숫자 문자열과 그대로 매칭.

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식·조건 1~2층"으로. 아이가 코드를 망가뜨리며 배우게 한다. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화 스크립트만 건드린다.

### 0층 — 숫자 바꾸기 (한 곳만 고친다)
- **세진 기사**: Stage 깃발에서 `공격력`을 `1` → `3` 으로. 적이 훨씬 빨리 죽는다.
- **두부 멘탈**: `최대체력`을 `5` → `1` 로. 한 대만 맞아도 위험! (난이도 급상승)
- **슈퍼 점프**: `점프력`을 `12` → `20` 으로. 천장까지 날아오른다.
- **느릿느릿 적**: `약한적_속도`를 `2` → `0.3` 으로. 약한 적이 거북이가 된다.
- **강한적 무력화**: `강한적_체력`을 `4` → `1` 로. 강한 적도 한 방에 가깝게.
- **빠른 약골**: `약한적_속도`를 `2` → `5` 로. 약한 적이 떼로 빠르게 몰려온다.
- **3단 점프**: `점프횟수상한`을 `2` → `3` 으로. 공중에서 세 번 뛴다.

### 1층 — 관계식 (값을 다른 값으로 계산)
- **잡을수록 세지는 검**: 기사 검 입력에서 `공격력` 대신 `1 + (총처치 / 3)` 로 데미지를 계산하게 바꿔, 많이 잡을수록 강해지게.
  - 적 (C)-2 의 `change 내체력 by (0 - 공격력)` → `change 내체력 by (0 - (1 + (총처치 / 3)))`.
- **달밤 체조**: `이동속도 = 4 + 방번호` 로 바꿔(방빌드 시 set), 깊은 방일수록 빨라지게.
- **묵직 점프**: `중력`을 `점프력`에 묶어 `set 중력 = (0 - 점프력) / 12` 로. 점프력을 키우면 중력도 비례.

### 2층 — 조건 (if 추가)
- **마지막 발악**: 기사 피격 감시에 `if 체력 = 1 : set 무적 = 60` 추가 — 체력 1 남으면 잠깐 무적.
- **광전사 모드**: 기사 (C)에 `if 체력 = 1 : set 이동속도 = 8` (체력 위험하면 빨라짐) / 회복 시 되돌리기.
- **보스 방**: 방빌드에서 `if (방번호 mod 3) = 0 : set 강한적_체력 = 강한적_체력 * 2` — 매 3번째 방의 강한 적을 더 단단하게.

### 3층 — 분기 (강화 규칙 바꾸기 / 선택지 추가)
- **회복 강화 추가**: 강화패널에서 `if key [4] : change 체력 by (최대체력)` (풀회복 선택지)를 추가하고 카드 SVG에 4번 칸 그리기.
- **랜덤 강화**: 강화 선택을 키 입력 대신 `pick random 1 to 3` 으로 자동 결정하게 바꿔, 운에 맡기기.

### 4층 — 구조 (새 적·새 메커닉)
- **점프 적 추가**: 적 클론에 VY/중력을 달아 발판을 뛰어다니는 점프형 적으로 개조(기사 (C) 중력 로직 복붙).
- **원거리 적**: 적이 `tank-battle`의 적포탄 스프라이트를 빌려와 화살을 쏘게(스프라이트 1개 추가 + 적사격 방송).
- **새 방 추가**: 발판 (B)의 `방번호 mod 3` 을 `mod 4` 로 늘리고 4번째 좌표 세트를 추가.

## 10. 학습 포인트

보너스 액션 게임 — 수학 학습 콘셉트 없음. **대신 "코딩 학습"이 핵심 가치**: 모든 능력치가 한글 튜닝 변수로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (4층) 새 스프라이트를 추가하며, 변수·연산·조건·클론이라는 프로그래밍 개념을 게임을 망가뜨리고 고치며 체득한다. 강화 선택지 자체가 `변수 += 강화량` 이라 "강해지는 규칙"이 곧 코드다.
