# 캐슬 디펜스 (castle-defense) — Plan

> 길을 따라 성으로 몰려오는 고블린·오크·트롤을 **포탑을 지어** 막아내는 **클래식 로그라이트 타워디펜스.** 몬스터는 고정 웨이포인트(S자 길)를 따라 성까지 행진하고, 성에 닿으면 `성체력`이 1씩 깎인다. 마우스로 **팔레트에서 포탑을 고르고 잔디 위에 설치**(길 위·성 위는 금지)하면, 각 포탑이 **사거리 안 가장 가까운 몬스터**에게 자동으로 발사한다(화살탑=싸고 빠름 / 대포탑=느리고 광역 / 마법탑=강함). 몬스터를 잡으면 `골드`가 들어오고, 그 골드로 포탑을 더 짓는다. **웨이브를 클리어할 때마다 강화 카드 셋 중 하나**(포탑공격력+/사거리+/연사+/골드+)를 1·2·3·4 키로 즉시 고른다. 웨이브가 올라갈수록 몬스터 수·체력·속도가 비례해 늘고, `성체력`이 0이 되면 GAME OVER → 깃발을 다시 누르면 1웨이브부터 다시. **점수 = 도달한 웨이브.**
> 베이스: `games/magic-survivor/build.py` (한글 튜닝 변수 **일괄 초기화**·매직넘버 0 / `조준요청` **broadcast-and-wait 최솟값 리덕션** 자동 조준 / `타격` 광역 데미지 / 플로팅 숫자 팝업(say 미사용) / 강화 택1 패널 / 게임오버 배너 / 클론 스포너 + `복제됨` 가드 / 폭발 연출 / 전용 합성 효과음 `_wav_bytes`·`synth_*` / `add_comment` 가이드 투어).
> 학습 콘셉트 없음(직관적 액션 — 추상 학습개념 금지, [[feedback-game-design]] 준수). **진짜 가치는 마법 생존과 똑같이 "아이가 코드의 숫자·규칙을 직접 바꾸며 노는 것"** — 모든 조절 값을 한글 전역 변수로 노출, 코드 어디에도 매직넘버 없음. 시후(초등·성격 급함)를 위해 **템포를 빠르게**: 웨이브 간 대기 없음(강화 고르면 즉시 다음 웨이브), 포탑 사격 손맛(타입별 발사음+발사체), 강화는 1/2/3/4 키 즉시 선택. ★★★☆☆.

- **주제**: 클래식 로그라이트 타워디펜스 (성/기사 vs 몬스터)
- **카테고리**: 액션
- **난이도**: ★★★☆☆
- **폴더**: `games/castle-defense/`
- **출력 파일**: `캐슬_디펜스.sb3`
- **빌드**: `python3 games/castle-defense/build.py`

## 1. 게임 한 줄

가장자리에서 S자 길을 따라 성으로 행진하는 몬스터 무리를, 길 옆 잔디에 포탑을 지어 막아낸다. 포탑은 사거리 안 가장 가까운 몬스터에게 알아서 쏘므로 아이는 **어디에 무슨 포탑을 지을지**만 정하면 된다. 몬스터를 잡으면 골드가 쌓이고, 골드로 포탑을 더 짓는다. 한 웨이브를 막아내면 강화 카드 넷 중 하나를 1·2·3·4 키로 골라 점점 강해진다. 웨이브가 오를수록 몬스터가 더 많이·세게·빠르게 몰려오고, 성체력이 0이 되면 GAME OVER, 깃발을 다시 누르면 1웨이브부터(도달 웨이브가 곧 점수).

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

```
 (-240,180)                                            (240,180)
   ┌──────────────────────────────────────────────────────┐
   │ 🌊웨이브:１  💰골드:１５０  🏰성:２０                     │  ← 좌상단 HUD 모니터 3종
   │ ◀몬스터═════╗                                          │  몬스터: 왼쪽 가장자리에서 스폰
   │   (고블린)   ║                                          │
   │             ╚════════════╗            🏰성             │  길(S자 돌길): 웨이포인트 따라 행진
   │                          ║         (목표·성체력)        │  성: 길 끝(오른쪽). 몬스터 닿으면 −1
   │              [화살탑]      ╚═══════════╗                 │  포탑: 잔디 위 클릭 설치(길 위 금지)
   │                  ◦사거리원              ║                │  사거리원: 설치 커서가 미리보기
   │           [대포탑]💥광역      [마법탑]   ║                │
   │  ┌────────────────────────────────────────────────┐   │
   │  │ [１ 화살탑 50] [２ 대포탑 100🔒] [３ 마법탑 150🔒] │   │  ← 하단 팔레트(포탑 선택 버튼)
   └──┴────────────────────────────────────────────────┴───┘
 (-240,-180)                                          (240,-180)

  설치: 팔레트 버튼 클릭 → 선택포탑 세팅 → 잔디 클릭(길/성/팔레트/다른포탑 위 금지 + 골드≥가격) → 설치
  웨이브 클리어 → 화면 중앙 [강화카드]: 1.포탑공격력+  2.사거리+  3.연사+  4.골드+  (1/2/3/4 키 → 즉시 다음 웨이브)
  성체력 0 → 화면 중앙 GAME OVER 배너(도달 웨이브 표시) → 깃발 재클릭으로 1웨이브부터(능력치 리셋)
```

- HUD: 좌상단에 `웨이브`(점수) `골드` `성체력` 모니터 3종. 튜닝 변수(가격·사거리 등)는 **모니터 숨김**.
- 평면 탑다운 — 중력·점프 없음. 길은 무대 배경에 그린 **돌길 S자 띠**(단일 색 `길색`), 나머지는 잔디. 포탑 설치 유효성은 설치 커서가 `touching color [길색]?` 로 판정.
- 몬스터는 길 시작점(왼쪽 가장자리)에서 스폰되어 웨이포인트 1→2→…→끝 순으로 행진, 끝에서 성에 닿으면 `성체력 −1` 후 자기 삭제.
- 하단 팔레트는 항상 보이며, 해금 안 된 포탑 버튼엔 🔒(잠금 코스튬). 웨이브 진행으로 해금되면 버튼이 활성화.

> **웨이포인트(길) 좌표 — Scratch 좌표 기준 6점 S자.** Stage 깃발에서 리스트 `경로X`/`경로Y`에 순서대로 add. 몬스터는 `현재점`(1부터)으로 다음 점을 향해 이동.
> | 점 | X | Y | 비고 |
> |----|----|----|------|
> | 1 | -240 | 70 | 스폰(왼쪽 가장자리 밖) |
> | 2 | -120 | 70 | 우로 |
> | 3 | -120 | -50 | 아래로 |
> | 4 | 40 | -50 | 우로 |
> | 5 | 40 | 90 | 위로 |
> | 6 | 170 | 90 | 우로(성 앞) |
> | (성) | 205 | 90 | `현재점 > 6` 이면 성 도달 처리 |

> **자동 조준 핸드셰이크(핵심 패턴, 마법 생존 재사용)**: 포탑이 발사 직전 전역 락 `조준중`을 잡고 `조준탑X/Y`(자기 위치)·`조준탑사거리`(`내사거리+사거리보너스`)를 세팅 후 `조준요청`을 **broadcast and wait** 로 보낸다. Scratch 단일 스레드 협력형이라, wait 없는 몬스터 수신 스크립트가 **클론 순서대로 원자 실행**된다 → 각 몬스터가 "요청 포탑까지 거리가 사거리 안이고 지금까지 최소면 `조준X/Y` 갱신"하는 **최솟값 리덕션**이 경쟁 조건 없이 성립. 락(`조준중`)으로 **포탑들이 한 번에 한 기씩 순차 조준**해 서로의 리덕션 변수를 침범하지 않는다.
> **광역/단일 데미지 통일(`타격` 방송)**: 데미지 판정을 포탑·탄 본체가 아니라 `타격` **broadcast and wait** 로 처리. 탄이 폭발 지점에서 `폭발X/Y`·`폭발데미지`(내공격력)·`폭발반경`(화살탑·마법탑=작음, 대포탑=큼)을 세팅 후 broadcast → 반경 안 모든 몬스터가 원자적으로 `내체력−폭발데미지`. 단일/광역이 한 코드로 통일되고 friendly-fire 분기 없음. **데미지 숫자 팝업은 폭발당 1개**(탄이 폭심에서 띄움 → 다중 피해 시 좌표 경쟁 없음).
> **플로팅 숫자(say 미사용)**: 마법 생존과 동일하게 숫자 코스튬으로 렌더. 흰색 0~9는 데미지, 금색 0~9는 골드 획득(`+골드`). `팝업종류`로 코스튬 오프셋만 다름.

## 4. 사운드 — 전용 합성(build.py `_wav_bytes` + `synth_*`), orphan 0

마법 생존처럼 **결정적 합성 WAV**를 만들어, 각 스프라이트에 **실제로 재생하는 사운드만** 등록(고아 사운드 0). 모든 합성 함수는 `synth_zap`/`synth_boom`(magic-survivor)와 같은 형태: 샘플 리스트 → `_wav_bytes(samples)` → md5 저장. 재생은 `play sound [이름]`(빠른 손맛 위해 대부분 `play`(대기 안 함)). 필요 시 pitch 이펙트 병용.

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait` 넣어 CPU 폭주 방지(이동/조준 틱 0.025초). **매직넘버 금지** — 항상 튜닝 변수 참조. "좌표 향하기"는 Scratch에 직접 블록이 없어 atan 공식 사용(아래 공통 헬퍼).

> **공통: 좌표(tx,ty) 향하기 / 좌표까지 거리** (몬스터·포탑탄에서 사용)
> ```
> point in direction ( (atan((tx - x position)/(ty - y position))) + ((y position > ty) * 180) )
> dist = sqrt( ((x position - tx)*(x position - tx)) + ((y position - ty)*(y position - ty)) )
> ```

  else                   ← 트롤
      set 내체력 = (트롤_체력 + ((웨이브 - 1) * 웨이브체력증가))
      set 내속도 = (트롤_속도 + ((웨이브 - 1) * 웨이브속도증가))
      set 내골드 = 트롤_골드 ; switch costume to [트롤] ; set size 90
  set 현재점 = 1
  goto (item 1 of 경로X, item 1 of 경로Y) ; show
  forever
    if (게임상태 = 0) : change 적수 by -1 ; delete this clone
    if (게임상태 = 1)
        # 1) 경로 행진 (현재점 향해 이동)
        point in direction (atan(((item 현재점 of 경로X) - x)/((item 현재점 of 경로Y) - y))
                            + ((y > (item 현재점 of 경로Y)) * 180))
        move 내속도 steps
        # 2) 웨이포인트 도착 → 다음 점, 끝이면 성 도달
        if ( sqrt(((x-(item 현재점 of 경로X))^2)+((y-(item 현재점 of 경로Y))^2)) < 도달반경 )
            change 현재점 by 1
            if (현재점 > (length of 경로X))
                change 성체력 by -1
                broadcast 성피격
                change 적수 by -1
                delete this clone
        # 3) 처치(타격으로 내체력 깎임 → 0이하면 골드+폭발+삭제)
        if (내체력 < 1)
            change 골드 by 내골드
            set 데미지표시값 = 내골드 ; set 데미지표시x = x ; set 데미지표시y = y
            set 팝업종류 = 1                    ← 금색 +골드 팝업
            broadcast 데미지표시
            set sound effect [pitch] to 0 ; play sound kill
            play sound coin
            change 적수 by -1
            switch costume to [폭발]
            repeat 5 : change size by 10 ; change [ghost] effect by 20 ; wait 0.02
            delete this clone
    wait 0.025
```

**(D) 타격 받으면 반경 안일 때 데미지** — wait 없는 원자 실행
```
when I receive 타격
  if (복제됨 = 1) and (게임상태 = 1)
      if ( sqrt(((x - 폭발X)^2) + ((y - 폭발Y)^2)) <= 폭발반경 )
          change 내체력 by (0 - 폭발데미지)
          set sound effect [pitch] to 0 ; play sound hit
```
> 빌더 노트: 이 스크립트에 **wait/yield 절대 없음** → `타격 and wait`가 모든 몬스터를 원자 실행해 광역 피해가 경쟁 없이 적용. 처치 판정·골드·폭발은 (C)-3 본체 루프가 처리(중복 폭발 방지).

**(E) 조준 보고(최솟값 리덕션)** — wait 없는 원자 실행
```
when I receive 조준요청
  if (복제됨 = 1) and (게임상태 = 1)
      set d = sqrt(((x - 조준탑X)^2) + ((y - 조준탑Y)^2))
      if (d <= 조준탑사거리) and (d < 조준거리)
          set 조준거리 = d
          set 조준X = x ; set 조준Y = y
          set 조준있음 = 1
```
> `d`는 인라인 수식으로 직접 비교(임시 변수 불요). **wait 없음** 필수. 사거리 밖 몬스터는 후보에서 제외 → 포탑이 사거리 안 가장 가까운 적만 조준.

### 포탑 (설치 클론 본체)

**(A) 깃발 초기화**

  hide ; set rotation style [don't rotate] ; set 복제됨 = 0
```

**(B) 포탑설치 → 클론 1기** (원본만, 내 타입일 때)

```

**(C) 클론 본체** (종류 세팅 + 자동 조준 발사 루프)
```
when I start as a clone
  set 복제됨 = 1
  set 내타입 = 설치타입
  if (내타입 = 1)
      set 내사거리=화살탑_사거리 ; set 내공격력=화살탑_공격력 ; set 내간격=화살탑_간격 ; set 내폭발반경=화살탑_폭발반경
      switch costume to [화살탑]
  else if (내타입 = 2)
      set 내사거리=대포탑_사거리 ; set 내공격력=대포탑_공격력 ; set 내간격=대포탑_간격 ; set 내폭발반경=대포탑_폭발반경
      switch costume to [대포탑]
  else
      set 내사거리=마법탑_사거리 ; set 내공격력=마법탑_공격력 ; set 내간격=마법탑_간격 ; set 내폭발반경=마법탑_폭발반경
      switch costume to [마법탑]
  goto (설치X, 설치Y) ; set size 70 ; show ; set 발사쿨 = 0
  forever
    if (게임상태 = 0) : delete this clone
    if (게임상태 = 1)
        if (발사쿨 <= 0)
            wait until (조준중 = 0)               ← 포탑별 순차 조준 락
            set 조준중 = 1
            set 조준탑X = x ; set 조준탑Y = y
            set 조준탑사거리 = (내사거리 + 사거리보너스)
            set 조준거리 = 99999 ; set 조준있음 = 0
            broadcast 조준요청 and wait            ← 몬스터들이 최솟값 리덕션
            if (조준있음 = 1)
                set 발사X = 조준X ; set 발사Y = 조준Y
                set 발사타입 = 내타입
                # 타입별 발사음
                if (내타입 = 1) : play sound arrow
                else if (내타입 = 2) : play sound cannon
                else : play sound magic
                broadcast 포탑발사
                set 발사쿨 = (내간격 * 연사보너스)
            set 조준중 = 0
        change 발사쿨 by -0.025
    wait 0.025
```
> 빌더 노트: `조준중` 락 덕에 **한 번에 한 포탑만** 조준 핸드셰이크를 수행 → 여러 포탑이 `조준거리/조준X/Y` 리덕션 변수를 동시에 건드리지 않음. `broadcast and wait`가 핵심(마법 생존 패턴). 사거리 밖만 있으면 `조준있음=0`이라 발사 생략(탄·소리 낭비 없음).

### 포탑탄 (발사체)

**(A) 깃발 초기화**

```

**(B) 포탑발사 → 탄 클론 1개** (원본만)

**(C) 클론 본체** (쏜 포탑 위치→목표 직진 → 충돌/도착 시 타격 + 데미지 팝업)
```
when I start as a clone
  set 복제됨 = 1
  set 탄타입 = 발사타입
  set 탄공격력 = (발사타입 공격력 + 공격력보너스)     ← 타입 공격력은 아래 노트대로 산출
  set 탄반경 = (발사타입 폭발반경)
  set 목표X = 발사X ; set 목표Y = 발사Y
  goto (조준탑X, 조준탑Y)                            ← 쏜 포탑 위치에서 출발
  if (탄타입 = 1) : switch costume to [화살]
  else if (탄타입 = 2) : switch costume to [포탄]
  else : switch costume to [마법구슬]
  point in direction (atan((목표X - x)/(목표Y - y)) + ((y > 목표Y) * 180))
  go to front ; show
  repeat until (touching [몬스터]?) or (touching edge?) or (게임상태 = 0)
       or ( sqrt(((x-목표X)^2)+((y-목표Y)^2)) < 도달반경 )
      move 탄속도 steps
      wait 0.01
  if (게임상태 != 0)
      # ── 폭발: 광역/단일 통일 ──
      set 폭발X = x ; set 폭발Y = y
      set 폭발데미지 = 탄공격력 ; set 폭발반경 = 탄반경
      broadcast 타격 and wait                         ← 반경 안 몬스터 모두 피해
      # ── 데미지 숫자 팝업 1개(폭심) ──
      set 데미지표시값 = 탄공격력 ; set 데미지표시x = 폭발X ; set 데미지표시y = 폭발Y
      set 팝업종류 = 0                                ← 흰색 데미지 숫자
      broadcast 데미지표시
  delete this clone
```
> 빌더 노트: `발사타입 공격력`은 분기로 산출 — `탄타입=1→화살탑_공격력, =2→대포탑_공격력, =3→마법탑_공격력`에 각각 `+공격력보너스`. `탄반경`도 타입별 폭발반경 분기. (C) 상단의 한 줄 의사코드를 if/else if/else 세 갈래로 펼칠 것. 단일(화살/마법, 반경 작음)과 광역(대포, 반경 큼)이 `타격` 한 코드로 처리됨.

### 건설커서 (설치 미리보기 + 배치)

```
when green flag clicked

  forever
    if (선택포탑 > 0) and (게임상태 = 1)
        show ; go to front
        goto (mouse x, mouse y)
        # 선택 포탑 사거리만큼 미리보기 원 크기 (사거리 지름 ≈ size)
        if (선택포탑 = 1) : set size ((화살탑_사거리 + 사거리보너스) * 2 / 기준지름)   # 표현용 비례
        ...(대포/마법 동일)
        # 유효성: 길/성/팔레트/다른 포탑 위면 불가
        if (touching color [길색]?) or (touching [성]?) or (touching [팔레트]?) or (touching [포탑]?)
            set [color] effect to ... (빨강 틴트)              ← 설치 불가
        else
            set [color] effect to ... (초록 틴트)              ← 설치 가능
    else
        hide
    wait 0.02

when this sprite clicked            ← (또는 forever 안에서 mouse down + 위 유효성 재확인)
  # 권장: 아래처럼 별도 클릭 처리
```
> 빌더 노트(설치 클릭): 커서 스프라이트가 마우스를 덮고 있으므로 `when this sprite clicked`로 클릭 포착이 간단. 클릭 시 ① `선택포탑>0` ② 위 유효성 통과(길/성/팔레트/포탑 비접촉) ③ `골드 ≥ 선택포탑 가격` 모두 만족이면:
> ```
> set 골드 by (0 - 가격)            # 가격 = 선택포탑별(화살탑가격/대포탑가격/마법탑가격)
> set 설치X = mouse x ; set 설치Y = mouse y ; set 설치타입 = 선택포탑
> play sound build
> broadcast 포탑설치
> ```
> 실패(불가 위치/골드 부족)면 `play sound error`. 색 감지 `touching color [길색]?`의 `길색`은 배경 돌길 단일 색(예 `#B8A878`)과 정확히 일치시킬 것. 사거리 미리보기 원은 `cursor.svg`를 사거리=size로 스케일(정확한 픽셀 매칭보다 직관 우선; 기준지름 상수만 인라인 허용).

### 숫자팝업 (플로팅 데미지/골드 숫자)

> 마법 생존 데미지 스프라이트와 **동일 구현 + 팝업종류 색 분기**. `팝업종류=0`이면 흰 숫자(코스튬 0~9), `=1`이면 금 숫자(코스튬 10~19). **say 미사용.**

**(A) 깃발 초기화**
```
when green flag clicked
  hide ; set 복제됨 = 0 ; set rotation style [don't rotate]
```

**(B) 데미지표시 받으면 자릿수만큼 클론** (원본만)
```
when I receive 데미지표시
  if (복제됨 = 0)
      set 데미지글자수 = (length of 데미지표시값)
      set 데미지자리 = 1
      repeat 데미지글자수
          set 데미지숫자 = (letter 데미지자리 of 데미지표시값)
          set 데미지오프셋 = ((데미지자리 - 1) * 14) - ((데미지글자수 - 1) * 7)   ← 가운데 정렬
          create clone of [myself]
          change 데미지자리 by 1
```

**(C) 클론 본체** (색=팝업종류, 숫자 코스튬 렌더 → 떠올랐다 페이드)
```
when I start as a clone
  set 복제됨 = 1
  go to front
  switch costume to (데미지숫자 + (팝업종류 * 10))     ← 0~9 흰 / 10~19 금
  goto ((데미지표시x) + 데미지오프셋, 데미지표시y)
  set [ghost] effect to 0 ; show
  repeat 12 : change y by 4 ; change [ghost] effect by 8 ; wait 0.02
  delete this clone
```
> 코스튬 순서: 인덱스 1~10 = 흰 "0"~"9", 11~20 = 금 "0"~"9". `switch costume to (숫자 + 팝업종류*10)`은 0-기반 계산이므로 빌더는 코스튬 번호 매핑에 맞춰 `+1` 보정(또는 코스튬 이름을 숫자 문자열로 두고 흰/금 두 세트를 이름 규칙으로 전환). 마법 생존의 이름-매칭 방식을 따르되 색 세트만 추가.

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식 1층", "조건 2층", "새 메커닉 4층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화/경로 리스트만 건드린다.

### 0층 — 숫자 하나 바꾸기 (무대 초록 깃발 스크립트)
- **부자 영주**: `기본골드` 를 `150` → `500` 으로. 시작부터 포탑을 잔뜩 지을 수 있다.
- **싸진 화살탑**: `화살탑가격` 을 `50` → `10` 으로. 화살탑을 길에 도배!
- **빠른 손**: `화살탑_간격` 을 `0.45` → `0.15` 로. 화살이 따발총처럼.
- **묵직한 대포**: `대포탑_폭발반경` 을 `60` → `120` 으로. 한 방에 무리를 쓸어버린다.
- **세진 마법탑**: `마법탑_공격력` 을 `4` → `10` 으로. **맞을 때 뜨는 데미지 숫자가 바뀌는지** 확인!
- **거북이 트롤**: `트롤_속도` 를 `0.9` → `0.3` 으로. 트롤이 더 느릿느릿.
- **튼튼한 성**: `성최대체력` 을 `20` → `60` 으로. 훨씬 많이 뚫려도 버틴다.
- **물량 지옥**: `몬스터간격` 을 `0.8` → `0.2` 로. 몬스터가 우르르 몰려온다(난이도 급상승).
- **포탑 빨리 해금**: `대포탑해금웨이브`·`마법탑해금웨이브` 를 `2`,`4` → `1`,`1` 로. 처음부터 세 포탑 다 사용!

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **웨이브 비례 보상**: Stage (D) 클리어 보너스를 `웨이브클리어골드` 대신 `웨이브클리어골드 × 웨이브` 로 — 후반일수록 골드가 팍팍.
- **점점 단단해지는 적**: `웨이브체력증가` 를 `2` → `5` 로. 웨이브가 오를수록 적 체력이 가파르게 오른다(실제 체력 = 종류체력 + (웨이브−1)×이 값).
- **연사 강화 더 세게**: 강화카드 3번을 `연사보너스 × 0.85` → `× 0.7` 로. 한 번 고를 때마다 연사가 훨씬 빨라진다.

### 2층 — if 조건 더하기
- **보스 웨이브**: Stage (B) 매니저에 `if (웨이브 mod 5 = 0) : set 생성타입 = 3` 추가 — 5의 배수 웨이브는 트롤만 몰려오는 보스전.
- **막판 보너스**: Stage (D)에 `if (성체력 = 1) : change 골드 by 100` 류로 위기 보너스(원하면).
- **강화 5번 추가(성 수리)**: 강화카드에 `if key [5] : change 성체력 by 5` 선택지를 더하고 카드에 5번 칸을 그린다.

### 3층 — 규칙 자체를 바꾸기 (분기)
- **랜덤 강화**: 강화 선택을 키 입력 대신 `pick random 1 to 4` 자동 결정으로 — 운에 맡기기.
- **마법탑 즉시 해금 카드**: 강화카드에 `if key [5] : set 마법탑해금 = 1` 을 더해, 강화로 직접 포탑을 해금(브리프의 "강화로 해금" 변형).
- **분열 고블린**: 몬스터 (C) 처치 시 `if (내타입 = 1) : (작은 고블린 2마리를 같은 경로 다음 점에서 스폰)` — 약한 적이 죽으며 둘로 쪼개지게(스폰 채널 재사용).

### 4층 — 새 메커닉 만들기 (구조)
- **길 바꾸기**: Stage (A)의 `경로X`/`경로Y` 리스트 좌표를 바꿔(또는 점을 추가) 더 구불구불한 길을 만들어 보자. 몬스터가 새 길로 행진한다!
- **느려지는 마법탑**: 마법탑 `타격` 에 닿은 몬스터의 `내속도` 를 잠시 줄이는 슬로우 효과(새 클론-로컬 `둔화` 변수)를 추가.
- **황금 몬스터**: 가끔(`pick random 1 to 10 = 1`) 스폰되는 반짝이는 몬스터를 만들어 잡으면 골드 5배(새 타입/색).
- **화려한 골드 숫자**: 숫자팝업 골드 클론의 떠오르는 속도·페이드를 키우거나 `× 2` 로 표시만 두 배 연출.

## 10. 학습 포인트

보너스 직관 액션 게임 — 수학/추상 학습 콘셉트 없음([[feedback-game-design]] 준수: 데시벨/음파/산성 같은 추상 콘셉트 금지). **마법 생존과 똑같이 "코딩 학습"이 핵심 가치**: 모든 능력치(골드·가격·포탑 사거리/공격력/간격·몬스터 체력/속도·웨이브 스케일·길 좌표)가 한글 튜닝 변수·리스트로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (4층) 길/새 포탑을 추가하며 **변수·리스트·연산·조건·클론·broadcast and wait** 를 게임을 망가뜨리고 고치며 체득한다. 코드 곳곳의 **노란 코멘트(가이드 투어)** 가 각 부분을 설명한다. 부수적으로 **경로/인덱스**(웨이포인트 따라가기), **최솟값 찾기**(자동 조준), **자원 관리**(골드 경제), **증가 곡선**(웨이브 스케일링)을 게임 행동으로 자연스럽게 만난다 — 어디까지나 직관적 타워디펜스로 포장된 채.
