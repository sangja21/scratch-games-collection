# 데모: 라인 시즈 (lane-siege)

> **데모 스코프** — 풀게임 아님. "냥코대전쟁식 라인 배틀이 Scratch에서 손맛이 나는가"만 검증받기 위한 빌드. 폴리시·밸런싱·컨텐츠(스테이지/보스/유닛 다양성) 전부 생략. 목표 규모 300~500 블록. 폴더 `demos/lane-siege/`, 출력 `라인_시즈.sb3`.

> ⚠️ **존재 이유 = chess-war 실패 재발 방지.** 과거 chess-war 는 **per-unit 듀얼**(유닛마다 모든 적을 각자 탐색·전투)로 **O(n²)** 가 되어 게임 루프가 붕괴 → 3차 수정 끝에 폐기됐다. 이 데모는 그 함정을 **구조적으로** 피하는 **"최전방 교전(front-line only)" O(n) 모델**이 실제로 손맛이 나는지 확인하는 것이 전부다. 아래 §성능 제약이 이 plan 의 1급 원칙이며, 빌더는 이를 절대 위반하지 말 것.

## 콘셉트
세로 라인 1줄. **화면 아래 = 내 성(아군 기지), 화면 위 = 적 성.** 아군 유닛은 아래→위로, 적 유닛은 위→아래로 전진한다. **플레이어는 조준하지 않는다** — 하단 소환 버튼 3개로 유닛을 배치하면 유닛들이 라인에 줄을 서서 알아서 싸운다. 돈(지갑)이 시간에 따라 차오르고 유닛마다 코스트+쿨다운이 있어, **언제 무엇을 뽑을지가 유일한 전략**이다. 적 성 HP 0 → 승리, 아군 성 HP 0 → 패배.

## ★ 핵심 설계 = "최전방 교전" 모델 (이 구조를 반드시 그대로 구현)
per-unit 듀얼을 금지하고, **각 진영의 선두(최전방) 유닛끼리만** 전투를 판정한다.

- 아군/적 유닛은 각각 **한 줄로 줄을 서서 전진만** 한다. 뒤 유닛은 앞 유닛을 추월/관통하지 않는다 → **앞 유닛 y 뒤에서 대기(전진 정지)**.
- 실제 근접 전투 판정은 **양 진영 선두 유닛이 서로 사거리 안에 들어왔을 때만** 발생. 선두 vs 선두가 서로 HP를 깎는다(쿨다운 간격으로). 선두가 죽으면 그 뒤 유닛이 새 선두가 되어 전진 재개.
- **뒤 유닛은 어떤 적도 탐색하지 않는다.** 전역 O(n²) 이중 루프·`touching [적]` 전체 스캔·리스트 순회 전투 **전부 금지.** 전투는 항상 **선두 2기 사이의 상수 시간 판정**이다 → 프레임당 비용 O(n)(각 유닛 전진 1회) + O(1)(선두 전투).
- **원거리 유닛**은 예외적으로 사거리가 길지만, 여전히 "사거리 안의 **최전방 적**"만 노린다(적 전체 스캔 아님 — 적 선두 유닛 1기만 참조). 발사 주기 제한 + 투사체 클론 화면 밖 즉시 삭제 + 투사체 클론 총량 캡.

> **구현 힌트(빌더 재량, 이 중 택1)**: 선두 판정을 값싸게 하는 방법 —
> (a) **선두 좌표 채널 변수**: 매 틱 아군 유닛들이 자기 y 를 비교해 "가장 앞선 아군 y" 를 공유 변수 `아군선두y`(적은 `적선두y`)에 기록(min/max 갱신). 전투는 `아군선두y` 와 `적선두y` 가 사거리 안이면 발생. 개별 클론은 자기가 선두인지(`y == 아군선두y` 근사)만 판정 → 선두만 데미지 판정.
> (b) 더 단순: **전투 판정을 Stage(또는 전용 심판 스프라이트) 1곳**에서 `아군선두y`/`적선두y`/선두 HP 두 개만 보고 처리(선두 HP도 공유 변수로 승격). 클론은 전진·정렬·자기 HP 표시만.
> 어느 쪽이든 **"모든 유닛이 모든 적을 본다"가 절대 나오면 안 된다.** (b)가 chess-war 재발 위험이 가장 낮음 — 권장.

## 성능 제약 (plan 1급 원칙 — 빌더가 수치로 못박아 구현)
1. **한 진영 동시 유닛 캡 = 8기** (`유닛캡 8`). 아군/적 각각 살아있는 클론 수를 카운트(`아군수`/`적수`)해 캡 도달 시 소환 거부(버튼 회색·적 AI 스폰 스킵).
2. **전투 판정은 선두 위주.** 전역 O(n²) 탐색·유닛 전체 `touching` 스캔·리스트 전투 순회 **0개.** 근접 전투는 선두 2기(또는 선두 HP 공유 변수) 사이 **상수 시간** 판정.
3. **투사체 클론 총량 캡 = 12** (`투사체캡 12`). 초과 시 발사 스킵. 발사 주기 제한(`원거리쿨` 틱). **화면 밖(y > 185 또는 y < -185) 즉시 `delete this clone`.** 명중 시에도 즉시 삭제.
4. **유닛 3종만**(아래). 스테이지/진화/다양성 없음.

## 화면 레이아웃 (480×360)

## 스프라이트 (7개 + Stage)
- **Stage**: 배경(세로 라인 + 양쪽 성). **모든 전역 변수 초기화 한 곳** + **돈 자동 충전 루프** + **적 AI 스폰 타이머** + **승패 감시**(성 HP≤0). 선택지 (b) 채택 시 **선두 전투 심판 루프**도 여기(또는 전용 심판 스프라이트).
- **아군유닛**: 스포너+클론 본체. `아군소환` 받으면 클론 1기(`소환종류`로 3종 분기). 클론은 종류별 HP/속도/공격/사거리 세팅 후 **위로 전진(선두 정렬로 앞 유닛 뒤 대기)**. 선두이고 적 선두가 사거리 안이면 근접 데미지(쿨). 원거리(궁수)면 사거리 안 최전방 적에 투사체 발사(쿨·캡). 아군 성 도달 방어는 불필요(적 유닛이 아군 성 판정 담당). HP≤0 → 팝 이펙트·`아군수`−1·삭제.
- **적유닛**: 위 아군유닛의 미러(아래로 전진, `적소환`/`적수`). 같은 3종 로직 재활용(방향/타깃 진영만 반전).
- **화살**: 투사체 클론(궁수 전용). 발사 진영 방향으로 직진, **화면 밖/명중 즉시 삭제**, 총량 캡 `투사체캡`. 명중 시 대상 선두 HP 차감.
- **소환버튼**: 하단 3버튼(1 스프라이트에 3클론 또는 3스프라이트). 클릭 폴링+디바운스. `돈≥코스트 & 소환쿨끝 & 아군수<유닛캡` 이면 스폰 방송·`돈`−코스트·쿨 리셋. 불가 시 회색(비활성 코스튬).
- **아군성HP바 / 적성HP바**: 11단 costume-fill 각 1개(상단·하단).
- **지갑바**: `돈/돈상한` 11단 costume-fill(최하단). (성 스프라이트는 배경에 그려 스프라이트 절약 가능 — HP바만 스프라이트로.)

## 핵심 변수 (한글 손잡이 ~20개)
`돈` 0 / `돈상한` 200 / `돈충전율` 4(초당) / `유닛캡` 8 / `투사체캡` 12 / `아군성HP` 100 / `적성HP` 100 / `성타격` 5(유닛이 상대 성에 닿을 때 초당 딜) / `적스폰간격` 3.0(초) / `라인X` 0 / `아군스폰y` -110 / `적스폰y` 110 / `아군선두y` / `적선두y` / `아군수` / `적수` / `게임상태`(1=플레이 0=종료) / 유닛 종류별(코스트·HP·속도·공격·사거리·소환쿨) 3종 → **필요 시 종류별 리터럴은 클론 세팅 스크립트 안에 표로.** 매직넘버 최소화하되 데모라 종류 스탯은 클론 세팅부에 모아도 됨.

> 클론-로컬(유닛): `복제됨`(증식 가드) / `내종류` / `내HP` / `내속도` / `내공격` / `내사거리` / `공격쿨` / `발사쿨`. 클론끼리 **참조 금지**(선두 판정은 공유 변수 경유).

## 방송 (~4개)
`게임시작`(초기화 후 루프 가동) / `아군소환`(종류는 `소환종류`) / `적소환`(종류는 `적소환종류`) / `게임끝`(승/패 배너). 승패 결과는 `게임상태`+배너 스프라이트로.

## 템포 (손맛 = 데모의 전부)
- 시작 즉시 돈이 조금 있어(`돈` 40 시작) 첫 유닛을 5초 안에 뽑을 수 있음.
- 첫 교전(아군 선두 vs 적 선두 격돌)까지 10초 내 — "줄 서서 밀고 밀리는" 라인 배틀 감이 빨리 보여야 함.
- 방패병으로 버티고 궁수로 뒤에서 쏘는 **조합의 맛**이 느껴지게(궁수 화살이 방패병 너머 적 선두에 날아감).
- 척후병 러시로 순식간에 적 성에 도달하는 **한 방 전략**도 가능하게(빠름·저코스트).
- 한 판 30~90초. 이기거나 지거나 확실히 끝남.

## 사운드/연출 (데모 최소)
- 합성 효과음 3~4종: 소환(pop), 근접 타격(hit 틱), 화살(pew), 유닛 격파/성 타격(crack). `_wav_bytes`+`synth_*`, 고아 0.
- 유닛 격파 시 팝(size↑+ghost) 이펙트, 성 HP 감소 시 성 흔들림/플래시 정도. 그 이상 폴리시 없음.

## 검증 (행동 시나리오 — headless scratch-vm 실측)
1. **소환 게이트**: 버튼 클릭 시 `돈≥코스트 & 소환쿨끝 & 아군수<유닛캡` 일 때만 클론 스폰 + `돈`−코스트 + 쿨 리셋. 돈 부족/쿨 중/캡 도달 시 스폰 안 됨(버튼 회색).
2. **★ O(n) 최전방 교전(chess-war 회피 — 핵심)**: 유닛 클론 스크립트에 **다른 유닛 클론을 직접 참조/스캔하는 코드 0개**(유닛끼리 `touching`·리스트 전투 순회·이중 루프 없음). 전투는 **선두 공유 변수(`아군선두y`/`적선두y`+선두 HP) 경유 상수 시간**임을 코드 구조로 확인. **뒤 유닛은 전진/정렬만** 한다(전투 판정 없음).
3. **선두 격돌**: 아군 1기 + 적 1기 스폰 → 라인에서 만나면 서로 HP를 쿨 간격으로 깎고, HP 0 된 쪽이 삭제되며 카운트 감소. 뒤 유닛이 새 선두로 전진 재개.
4. **원거리·투사체 캡**: 궁수 선두가 사거리 안 **최전방 적 1기**에만 화살 발사(전체 스캔 아님), 발사 주기 준수, 화면 밖(y>185/<-185) 즉시 삭제, 동시 투사체 ≤ `투사체캡`(초과 발사 스킵).
5. **유닛 캡**: 아군/적 각각 동시 유닛 ≤ `유닛캡 8`(초과 소환·스폰 거부).
6. **경제 루프**: `돈`이 `돈충전율`로 차오르고 `돈상한`에서 멈춤, 소환마다 코스트 차감, 지갑바 costume-fill 반영.
7. **승패**: 유닛이 상대 성에 도달하면 성 HP 감소, 적성HP≤0 → 승리 배너·`게임끝`, 아군성HP≤0 → 패배 배너. 성 HP바 costume-fill 반영.
8. **자동 플레이 60초 안정성**: 양측 유닛·화살이 계속 스폰/삭제되는 동안 프레임 붕괴/클론 누수 없이 승패 결착. (캡·삭제가 실제로 총량을 억제하는지 클론 수 관찰.)

## 기술 규칙
클론끼리 상호작용 금지(선두 판정은 공유 변수) / 전투는 선두 상수 시간(O(n²) 금지) / 투사체 화면 밖·명중 즉시 삭제 + 총량 캡 / 유닛·투사체 동시 캡 / vrep 리포터(`change x/y by 변수`) / 클릭 폴링+디바운스(`이 스프라이트 클릭했을 때` 지양) / say 금지 / `복제됨` 증식 가드 / add_comment 3~5개(선두 교전 모델·성능 캡·경제 루프 설명).
