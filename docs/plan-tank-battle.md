# 탱크 배틀 (tank-battle) — Plan

> 톱다운 탱크 슈터. 화살표키로 회전·전후진 + 스페이스로 포탄 발사. 엄폐물 뒤로 숨어 적 탱크를 격파하고 라운드를 진행.
> 베이스: `games/zombie-shooter/build.py` (마우스→키 회전·발사 패턴) + `games/alien-invasion/build.py` (라운드제·웨이브). 단, 마우스 조준 X, **회전키 조작**으로 변경.
> 학습 콘셉트 없음. 톱다운 시점이므로 포탄은 직선(포물선 아님). 만화풍 귀여운 디자인.

## 1. 컨셉 / 메커닉

- 톱다운 시점, 480×360 무대.
- **플레이어 탱크 조작**:
  - 왼쪽/오른쪽 화살표 = 좌/우 회전 (틱당 ±4°)
  - 위 화살표 = 전진 (틱당 +2.2 steps)
  - 아래 화살표 = 후진 (틱당 −1.6 steps)
  - 스페이스 = 포탄 발사 (쿨다운 0.45초)
- **포탄**: 발사 시점 탱크 위치/방향에서 클론 생성 → 직선 이동(10 steps/틱) → 화면 가장자리 OR 적 탱크 OR 엄폐물 닿으면 사라짐.
- **적 탱크 AI**: 클론으로 1~3대 동시 등장. 단순 follow AI:
  - 매 틱 플레이어 방향으로 회전(틱당 ±2°, 즉시 회전이 아닌 점진적 추적)
  - 매 틱 1.0~1.3 steps 전진 (라운드↑ 시 속도↑)
  - 일정 확률(매 60틱 중 1회)로 포탄 발사
- **엄폐물(블록)**: 라운드 시작 시 4~6개 클론, 무작위 위치 + 미니 회피 영역(중앙 100×100은 비움) → 포탄 닿으면 사라짐(체력 1).
- **충돌**:
  - 플레이어 포탄 ↔ 적 탱크 → 적 사망, 점수 +20, 포탄 삭제
  - 적 포탄 ↔ 플레이어 탱크 → 플레이어 체력 −1, 포탄 삭제, 무적 0.8초
  - 포탄 ↔ 엄폐물 → 엄폐물 사라짐, 포탄 삭제
  - 탱크끼리 직접 부딪힘은 무시(단순화)
- **라운드제**:
  - 라운드 1 = 적 1대
  - 라운드 2 = 적 2대
  - 라운드 3 = 적 3대 (이후 라운드도 3대 유지하되 적 속도 ×1.10)
  - 적 모두 격파 시 1.5초 대기 → 다음 라운드 시작.
- **플레이어 체력 3**, 0 되면 게임오버 배너.

> 적탱크는 클론이지만 Scratch 에서 클론별 로컬 변수가 일반 변수로는 불가. 적 클론마다 발사 타이밍을 분리하려면 `repeat random N→ broadcast 적사격`. 단순화: 각 적 클론이 자기 forever 안에서 `if random(1..120)=1 → 포탄종류=1 → 포탄 좌표/방향 set → broadcast 적사격`. 적쿨다운 변수는 사용 안 함 (제거).

✱ 최종 변수: 14개 (적쿨다운 V_ECD15 제거)

## 6. 스프라이트별 블록 흐름

### Stage

1. **깃발 클릭** → 모든 변수 초기화 → 0.5초 대기 → `게임시작` 방송.
2. **게임시작 받으면** → `라운드시작` 방송.
3. **라운드시작 받으면**:
   - 잔여적 = 라운드와 라운드 수만큼 (round 1=1, round 2=2, round 3+=3) — 단순화: `min(라운드, 3)`
     - Scratch 에는 min 블록이 없으므로 if/else 로 3 클램프:
       - if 라운드 < 3 → 잔여적 = 라운드, else 잔여적 = 3
   - 엄폐물 5개 배치 루프:
     - repeat 5:
       - 스폰X = random(-200..200) 단, abs(SX) > 60 보장 단순화: 두 영역 중 골라서. 더 단순: `random(1..2) = 1 → SX=random(-200..-70), else random(70..200)`
       - 스폰Y = random(-140..140) — 단, 플레이어 시작점(0,-120) 근처 회피: `random(1..2)=1 → SY=random(-140..-30), else random(30..140)` — 단순화: 그냥 random(-140..140), 플레이어가 알아서 피해도 됨. 본 plan 채택: **random(-200..200) × random(-140..140), 단 abs(SY) > 25 보장 (간단 if 절)**
       - `엄폐물생성` 방송
   - 적 스폰 루프:
     - repeat 잔여적:
       - 스폰X = random(-220..220)
       - 스폰Y = random(80..160) (적은 화면 위쪽에서 등장)
       - `적스폰` 방송
       - wait 0.3
4. **forever (게임 진행 카운터)** — 별도 스크립트:
   - if 게임상태=1:
     - if 무적 > 0 → 무적 −1
     - if 쿨다운 > 0 → 쿨다운 −1
   - wait 0.04
5. **forever (라운드 진행 감시)** — 별도 스크립트:
   - if 게임상태=1 AND 잔여적 = 0:
     - wait 1.5
     - 라운드 += 1
     - 적속도 *= 1.10
     - `라운드시작` 방송
   - wait 0.2

### 플레이어탱크

1. **깃발 클릭** → 보이기, 크기 75, 위치 (0, -120), 방향 0 (위), 회전 스타일 = `all around`, 코스튬 = tank.
2. **forever (조작)**:
   - if 왼쪽 화살표 → turn ccw 4
   - if 오른쪽 화살표 → turn cw 4
   - if 위 화살표 → move 2.2 steps
   - if 아래 화살표 → move −1.6 steps
   - 가장자리 clamp: x ±220, y ±160
   - if (touching 엄폐물): 한 칸 뒤로 (반대 방향 move −2.5)
   - wait 0.02
3. **forever (사격 입력)** — 별도 스크립트:
   - if (space pressed) AND 쿨다운 = 0 AND 게임상태=1:
     - 포탄X = x, 포탄Y = y, 포탄방향 = direction, 포탄종류 = 0
     - pop pitch 100 sound
     - create clone of 포탄
     - 쿨다운 = 12 (≈ 0.45초)
   - wait 0.05
4. **forever (피격 감시)** — 별도 스크립트:
   - if (touching 포탄) AND 무적 = 0:
     - (단, 자기 발사한 포탄을 즉시 맞지 않도록: 발사 시점 포탄이 0.1초 이상 살아 있어야 충돌 카운트 되도록 단순화 — Scratch 에서 즉시 분리하긴 어려우므로 발사 직후 플레이어가 즉시 자기 포탄에 닿지 않게 **발사 직후 1틱 동안 무적=4** 처리. 본 plan 채택.)
     - 체력 −1, 무적 = 25, pitch −300 pop
     - if 체력 ≤ 0 → 게임상태 = 0
   - wait 0.05

### 포탄

1. **깃발 클릭** → 숨김, 크기 50.
2. **클론으로 시작**:
   - 포탄X/Y 로 이동, 포탄방향으로 point, 보이기.
   - repeat until (out of bounds OR touching 적탱크 OR touching 엄폐물 OR touching 플레이어탱크):
     - move 10 steps
     - wait 0.02
   - 숨김 → 클론 삭제
3. 충돌 결과(점수·체력·엄폐물 사라짐 등)는 각 대상 스프라이트의 forever 가 감지하여 처리. 포탄 자신은 충돌 시 그냥 사라지기만 함.
   - 다만 플레이어 자기 포탄에 즉사 방지: 발사 시점 무적=4 가 플레이어 쪽에서 적용되므로 첫 4틱은 플레이어 충돌해도 무시.

### 적탱크

1. **깃발 클릭** → 숨김, 크기 75, 코스튬 = enemy, 회전 스타일 = all around.
2. **적스폰 받으면** → 클론 생성.
3. **클론으로 시작**:
   - 스폰X/Y 로 이동, 보이기, 방향 = 180 (아래 향함)
   - forever (until 클론 사망):
     - point towards 플레이어탱크 — 단, 즉시 회전이 아닌 점진:
       - 단순화: turn cw 2 (방향 추적은 zombie 처럼 즉시 point. 본 plan 채택: 즉시 `point towards 플레이어탱크`로 단순화, 단 적 속도가 느려서 충분히 회피 가능)
     - move 적속도 steps
     - if (touching 엄폐물): move −2.5 (뒤로) + turn cw 30 (방향 틀기)
     - if (touching 포탄) AND 포탄종류 = 0 (플레이어 발사한 포탄):
       - 점수 += 20, 잔여적 −1, pitch 0 pop, 숨김, 클론 삭제, exit
       - ✱ Scratch 에서 클론 안의 if 분기로 break 하려면 `stop this script`. plan: 충돌 검사를 forever 첫 줄에서 하고, true 면 처리 후 `stop this script` + `delete this clone`.
       - 충돌 시 포탄종류=0 조건만 두면 자기 포탄에 맞아 자폭하지 않음.
     - 발사 확률: if random(1..120) = 1 → 포탄X = 자기 x, 포탄Y = 자기 y, 포탄방향 = 자기 direction, 포탄종류 = 1, broadcast 적사격
     - wait 0.04

### 엄폐물

1. **깃발 클릭** → 숨김, 크기 80, 코스튬 = cover.
2. **엄폐물생성 받으면** → 클론 생성.
3. **클론으로 시작**:
   - 스폰X/Y 로 이동, 보이기.
   - forever:
     - if (touching 포탄): pitch -100 pop, 숨김, 클론 삭제 (stop this script).
     - wait 0.04
4. **라운드시작 받으면** (메인 인스턴스만):
   - 모든 이전 라운드 클론 정리: `delete this clone`은 클론 자신이 호출해야 함. 단순화: 라운드 시작 시 `엄폐물숨김` broadcast 추가 → 모든 엄폐물 클론이 자기 자신 hide+delete.
   - ✱ 단순화 채택: 매 라운드 시작 시 stage 가 `엄폐물생성` 5번을 호출하는데, 그 전에 **stage 가 직접 라운드 시작 첫 액션으로 0.05초 대기 후 엄폐물 클론들이 새로 생성**됨. 이전 라운드 엄폐물은 사실상 라운드가 짧으면 거의 남아있지 않음. 본 plan 에서는 이전 클론 정리 broadcast 를 추가하지 않고 그대로 누적 허용 — 5라운드 정도까지는 누적 25개 정도라 시각적/성능 부담 작음.

### 적사격 받으면 (포탄 스프라이트)
- 포탄 스프라이트가 `사격`/`적사격` 둘 다 처리: 핸들러에서 그냥 `create clone of myself`.
