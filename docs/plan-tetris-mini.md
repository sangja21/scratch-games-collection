# 테트리스 미니 (tetris-mini) — Plan

> 7가지 블록(테트로미노)이 보드 위로 떨어진다. 좌/우 화살표로 옮기고, 위 화살표로 회전, 아래 화살표로 빨리 내리고, 스페이스로 한 번에 바닥까지 떨군다(하드드롭). 한 가로줄이 빈칸 없이 꽉 차면 그 줄이 사라지고 위 줄들이 한 칸씩 내려온다. 줄을 많이 지울수록 블록이 빨라진다. 블록이 맨 위까지 쌓이면 게임오버. 1984 Alexey Pajitnov "Tetris" 의 간소화 버전(보드 8열×14행).
> 베이스: `games/snake/build.py`(**리스트로 보드/상태를 표현 + 리스트 블록 opcode** — 테트리스의 보드 리스트가 snake 의 궤적 리스트와 같은 패턴 / 클론이 리스트 항목을 읽어 위치를 잡는 패턴) + `games/car-race/build.py`(격자 스냅 이동 `change x/y by ±칸` + 키 에지 입력 `wait until NOT key` + 게임상태 broadcast + 깃발 재시작 + 게임오버 배너) + `games/breakout/build.py` / `games/alien-invasion/build.py`(클론을 격자에 배치하고 per-clone 좌표를 복사하는 패턴 → 보드 셀 렌더 클론).
> **차이점**: snake 는 1차원 궤적 리스트지만 tetris 는 **2차원 격자를 1차원 리스트(112항목)로 평탄화**해서 표현한다(인덱스 = `행*8 + 열 + 1`). 회전은 완전한 회전행렬 대신 **7피스×4회전 상태의 셀 오프셋을 미리 계산한 룩업 리스트**로 처리한다(Scratch 에서 행렬 연산보다 훨씬 안전). 충돌·라인검출·라인제거는 모두 보드 리스트 인덱스 조회/치환으로 한다.
> 학습 콘셉트 없음. 초등학생 대상 직관적 액션. 추상 학습 콘셉트 금지(MEMORY.md → feedback-game-design 준수).

## 1. 한 줄 룰

떨어지는 블록을 좌/우로 옮기고(←/→), 위(↑)로 돌리고, 아래(↓)로 빨리 내려서 가로줄을 빈칸 없이 채운다. 꽉 찬 줄은 사라지고 점수 +. 블록이 천장까지 쌓이면 게임오버.

## 2. 화면 / 좌표 (480×360)

- 무대 480×360 (Scratch 표준). 좌표 -240..240 / -180..180.
- **보드(grid)**: **8열 × 14행**. 한 칸 = **22px**(셀 코스튬은 20px + 2px 틈). 보드 크기 = 176×308px, 화면 약간 왼쪽에 배치하고 오른쪽에 HUD/다음블록.
  - 보드 픽셀 폭 = 8×22 = 176, 높이 = 14×22 = 308.
  - **보드 좌상단 칸(열0,행0) 중심 좌표**: `(BOARD_X0, BOARD_Y0) = (-150, 143)`.
    - 즉 보드는 화면 왼쪽 영역 x≈-161..+15, y≈+154..-154 를 차지.
  - **칸 중심 ↔ 격자 인덱스 변환 (핵심 공식)**:
    - 열 col(0..7), 행 row(0..13, 0=맨 위, 13=맨 아래) 인 칸의 무대 좌표:
      - `X = BOARD_X0 + col*22 = -150 + col*22`
      - `Y = BOARD_Y0 - row*22 = 143 - row*22`
    - 1차원 리스트 인덱스(1-base): `idx = row*8 + col + 1`  (행 우선 평탄화, 8=열 수)
      - 역변환: `col = (idx-1) mod 8`, `row = floor((idx-1)/8)`. (블록 흐름에선 보통 col/row → idx 방향만 쓰면 됨.)
- **SVG(0~480, 0~360) ↔ Scratch(-240~240, -180~180) 변환**: `Scratch_x = SVG_x - 240`, `Scratch_y = 180 - SVG_y`. 배경 SVG 에서 보드 사각형은 SVG 좌상단 (79, 26)~(255, 334) 근처(= Scratch (-161, 154)~(15, -154)). 빌더는 배경 그리드 라인을 이 영역에 맞춰 그린다.
- 배경: 짙은 남색. 왼쪽에 8×14 보드 외곽 흰 테두리 + 옅은 격자선. 오른쪽 패널에 "NEXT" 미리보기 박스 + 점수/줄수/레벨 모니터.

```
+----------------------------------------------------+  y=+180
| +------------------------+                          |
| | . . . . . . . .        |   NEXT:                 |  점수: 1200
| | . . . . . . . .        |   +------+              |  줄: 6
| | . . . ■ ■ . . .  ←현재 |   | ■■   |              |  레벨: 2
| | . . . . ■ . . .   블록 |   | ■■   |              |
| | . . . . . . . .        |   +------+              |
| | . . . . . . . .        |                         |
| | . ■ . . . . . .        |                         |
| | ■ ■ ■ . . . ■ ■  ←쌓인 |                         |
| | ■ ■ ■ ■ . ■ ■ ■   블록 |                         |
| +------------------------+                          |
| 보드: 8열×14행, 칸 22px                              |
+----------------------------------------------------+  y=-180
x=-240   col0중심 x=-150 ... col7중심 x=+4         x=+240
         row0중심 y=+143 ... row13중심 y=-143
```

> 클론 수 산정: 보드 셀 렌더 클론은 **최대 112개**(8×14 전부 채워졌을 때). 현재 떨어지는 블록은 별도 4셀 sprite(4 클론). NEXT 미리보기 4셀(4 클론). 합쳐도 **약 120개** ≪ Scratch 클론 한계 300. stamp(펜) 대신 **클론 렌더**를 채택해도 안전(빌더가 클론 방식 권장 — 색/코스튬 표현이 쉽고 재배치가 명확).

## 7. 메커닉 상세

### 7.1 좌표 ↔ 인덱스 헬퍼 (모든 곳에서 사용)

- 칸 (col, row) 의 무대 좌표: `X = -150 + col*22`, `Y = 143 - row*22`.
- 칸 (col, row) 의 보드 리스트 인덱스: `idx = row*8 + col + 1`.
- 보드 경계 검사: `col < 0` 또는 `col > 7` 또는 `row > 13` → 벽/바닥 밖. (`row < 0` 은 천장 위 — 스폰 직후 일시적으로 허용하되 그 셀은 그리지 않음, 7.4 참조.)

### 7.2 입력 (블록판정기) — 좌/우/회전/소프트드롭/하드드롭

`car-race` 의 키 에지 입력(`wait until NOT key`)을 채택해 한 번 누름 = 한 칸 이동/한 번 회전이 되게 한다(소프트드롭·하드드롭 제외). 빠른 forever 로 입력만 받고, 실제 중력 하강은 7.5 의 `브틱` 이 담당.

```
when receive 게임시작:        # 입력 forever (틱 0.03s)
  repeat until 게임상태 = 0:
    # ← 왼쪽 이동
    if key ←:
      시험열 ← 현열 - 1 ; 시험행 ← 현행 ; 시험회전 ← 현회전
      call 충돌?            # 충돌됨 = 1/0 결과
      if 충돌됨 = 0:
        현열 ← 시험열
        broadcast 현재블록그리기
      wait until NOT key ←   # 에지: 한 번 누름 = 1칸
    # → 오른쪽 이동
    if key →:
      시험열 ← 현열 + 1 ; 시험행 ← 현행 ; 시험회전 ← 현회전
      call 충돌?
      if 충돌됨 = 0:
        현열 ← 시험열 ; broadcast 현재블록그리기
      wait until NOT key →
    # ↑ 회전(시계방향): 회전 = (현회전 + 1) mod 4
    if key ↑:
      시험열 ← 현열 ; 시험행 ← 현행 ; 시험회전 ← (현회전 + 1) mod 4
      call 충돌?
      if 충돌됨 = 0:
        현회전 ← 시험회전 ; broadcast 현재블록그리기
      wait until NOT key ↑
    # ↓ 소프트드롭(누르고 있으면 빨리 내림 — 에지 아님)
    if key ↓:
      시험열 ← 현열 ; 시험행 ← 현행 + 1 ; 시험회전 ← 현회전
      call 충돌?
      if 충돌됨 = 0:
        현행 ← 시험행 ; broadcast 현재블록그리기
      else:
        call 고정       # 더 못 내려가면 즉시 고정
      wait 0.04         # 소프트드롭 속도(꾹 누르면 연속)
    # 스페이스 하드드롭(에지): 막힐 때까지 한 칸씩 내린 뒤 고정
    if key space:
      repeat until 충돌됨 = 1:
        시험열 ← 현열 ; 시험행 ← 현행 + 1 ; 시험회전 ← 현회전
        call 충돌?
        if 충돌됨 = 0: 현행 ← 현행 + 1
      broadcast 현재블록그리기
      call 고정
      wait until NOT key space
    wait 0.03
```

> **회전 mod 4**: `(현회전 + 1) mod 4` = `operator_mod`. Scratch 에 mod 블록 있음(`operator_mod`).
> **벽킥 생략**: 초등학생 간소화 버전이라 벽킥(회전 시 벽에 막히면 한 칸 밀어주기) 없음. 회전이 벽/블록과 겹치면 그냥 회전 무시(충돌됨=1 → 아무것도 안 함). I/O/T 등은 보드 가운데에서 충분히 회전 가능.

### 7.3 충돌검사 `충돌?` 서브루틴 (블록판정기) — **핵심**

`시험열`/`시험행`/`시험회전` 으로 정의된 가상 블록의 4셀이 보드 밖이거나 이미 찬 칸과 겹치는지 검사. 결과를 `충돌됨`(1/0)에 둔다. (Scratch 의 "custom block(my block) + run without screen refresh" 로 만들면 깔끔. broadcast+wait 로도 가능하나 my block 권장.)

```
define 충돌? (커스텀 블록, run without screen refresh)
  충돌됨 ← 0
  오프베이스 ← ((현피스 - 1) * 4 + 시험회전) * 4    # 이 (피스,회전)의 첫 셀 앞 인덱스
  i ← 1
  repeat 4:                                       # 4셀 검사
    셀col ← 시험열 + (item (오프베이스 + i) of 오프셋C)
    셀row ← 시험행 + (item (오프베이스 + i) of 오프셋R)
    # 벽/바닥 밖?
    if (셀col < 0) OR (셀col > 7) OR (셀row > 13):
      충돌됨 ← 1
    else:
      if 셀row ≥ 0:                                # 천장 위(row<0)는 빈칸 취급
        셀idx ← 셀row * 8 + 셀col + 1
        if (item (셀idx) of 보드) ≠ 0:             # 이미 찬 칸과 겹침
          충돌됨 ← 1
    i ← i + 1
```

> **오프베이스 인덱싱 검산**: 피스 p(1..7), 회전 r(0..3), 셀 k(1..4) → 리스트 인덱스 = `((p-1)*4 + r)*4 + k`. 예) I피스(p=1) 회전0 4셀 = 인덱스 1..4. L피스(p=7) 회전3 4셀 = `((7-1)*4+3)*4 + (1..4)` = 108+1..4 = 109..112. (리스트 길이 112 와 정확히 일치.)
> **run without screen refresh**: 4셀 루프가 한 프레임에 끝나 깜빡임 없음. my block 미지원이면 일반 broadcast 동기 호출로 대체하되, 같은 sprite 내 호출이므로 my block 이 가장 안전.

### 7.4 새 블록 스폰 `새블록` (블록판정기)

```
when receive 새블록:
  현피스 ← 다음피스
  다음피스 ← pick random 1 to 7
  현회전 ← 0
  현열 ← 3
  현행 ← 0
  현색 ← 현피스            # (색=피스번호. 코스튬으로 사용)
  # 스폰 자리가 이미 막혔으면(천장까지 쌓임) 게임오버
  시험열 ← 현열 ; 시험행 ← 현행 ; 시험회전 ← 현회전
  call 충돌?
  if 충돌됨 = 1:
    게임상태 ← 0
  broadcast 현재블록그리기
  broadcast 보드그리기      # NEXT 미리보기 갱신용(현재블록 sprite 가 같이 그림)
```

> 스폰 기준칸 (3,0). 일부 피스는 오프셋에 `row=-1` 셀이 있어 천장 살짝 위에서 시작하지만, 그 셀(row<0)은 그리지 않고 충돌검사에서도 빈칸 취급(7.3). 한 칸만 내려오면 전부 보드 안으로 들어온다.

### 7.5 중력(브틱) 하강 (블록판정기)

```
when receive 브틱:
  if 게임상태 = 1:
    시험열 ← 현열 ; 시험행 ← 현행 + 1 ; 시험회전 ← 현회전
    call 충돌?
    if 충돌됨 = 0:
      현행 ← 현행 + 1
      broadcast 현재블록그리기
    else:
      call 고정           # 한 칸 더 못 내려감 → 그 자리에 굳힘
```

Stage 의 타이머가 `브틱` 을 발생시킨다:

```
when receive 게임시작:        # 중력 타이머 (Stage)
  repeat until 게임상태 = 0:
    broadcast 브틱
    wait 낙하간격

when receive 게임시작:        # 레벨/속도 ramp (Stage)
  repeat until 게임상태 = 0:
    레벨 ← floor(줄수 / 5) + 1
    낙하간격 ← 0.8 - (레벨 - 1) * 0.1
    if 낙하간격 < 0.12: 낙하간격 ← 0.12
    wait 0.2
```

> `floor` = `operator_mathop`("floor", ...). 줄 5개마다 레벨 +1, 레벨마다 낙하간격 0.1초 단축(하한 0.12).

### 7.6 고정 + 라인검출 + 라인제거 `고정` (블록판정기) — **핵심**

현재 블록의 4셀을 보드 리스트에 색값으로 써넣고, 가득 찬 행을 찾아 지운 뒤 위 행들을 한 칸 내린다.

```
define 고정 (커스텀 블록, run without screen refresh)
  # (1) 현재 4셀을 보드에 굳힌다 (replace item idx of 보드 = 현피스)
  오프베이스 ← ((현피스 - 1) * 4 + 현회전) * 4
  i ← 1
  repeat 4:
    셀col ← 현열 + (item (오프베이스 + i) of 오프셋C)
    셀row ← 현행 + (item (오프베이스 + i) of 오프셋R)
    if 셀row ≥ 0:                                  # 천장 위 셀은 보드에 안 씀
      셀idx ← 셀row * 8 + 셀col + 1
      replace item (셀idx) of 보드 with 현피스
    i ← i + 1

  # (2) 아래 행부터 위로 훑으며 꽉 찬 행 제거 + 위 행 끌어내림
  지운줄카운트 ← 0
  시험행 ← 13                                       # 맨 아래 행부터
  repeat until 시험행 < 0:
    # 이 행이 꽉 찼는지 검사
    행꽉참 ← 1
    j ← 0
    repeat 8:
      if (item (시험행*8 + j + 1) of 보드) = 0: 행꽉참 ← 0
      j ← j + 1
    if 행꽉참 = 1:
      지운줄카운트 ← 지운줄카운트 + 1
      # 시험행 위의 모든 행을 한 칸씩 아래로 복사 (r = 시험행 .. 1)
      i ← 시험행
      repeat until i < 1:                          # i = 시험행 → 1
        j ← 0
        repeat 8:
          replace item (i*8 + j + 1) of 보드 with (item ((i-1)*8 + j + 1) of 보드)
          j ← j + 1
        i ← i - 1
      # 맨 윗행(row0)은 새로 빈칸
      j ← 0
      repeat 8:
        replace item (0*8 + j + 1) of 보드 with 0
        j ← j + 1
      # 같은 행을 다시 검사해야 함(끌어내린 행이 또 꽉 찰 수 있음) → 시험행 그대로 둠
    else:
      시험행 ← 시험행 - 1                            # 이 행은 통과 → 위로

  # (3) 점수/줄수
  if 지운줄카운트 = 1: 점수 ← 점수 + 100
  if 지운줄카운트 = 2: 점수 ← 점수 + 300
  if 지운줄카운트 = 3: 점수 ← 점수 + 500
  if 지운줄카운트 = 4: 점수 ← 점수 + 800
  줄수 ← 줄수 + 지운줄카운트
  if 점수 > 최고기록: 최고기록 ← 점수
  if 지운줄카운트 > 0: play sound clear

  # (4) 렌더 갱신 + 다음 블록
  broadcast 보드그리기
  broadcast 새블록
```

> **라인제거 "끌어내림" 직관**: 꽉 찬 행을 따로 지우는 대신, **그 행 위의 모든 칸을 한 칸 아래로 덮어쓰고 맨 윗행을 0으로 비운다**. 이러면 "지운 줄이 사라지고 위가 내려온" 효과가 한 번에 난다. 같은 `시험행` 을 다시 검사하는 이유: 위에서 내려온 행이 또 꽉 차 있을 수 있으므로(여러 줄 동시 제거 케이스를 한 행씩 안전 처리).
> **왜 보드를 리스트로?**: snake 가 궤적을 리스트로 둔 것과 같다. 격자 전체 상태를 한 리스트에 담고, 인덱스 산수(`row*8+col+1`)로 임의 칸을 읽고 쓴다. 이게 Scratch 에서 2D 격자를 다루는 정석이며 충돌·라인 로직이 전부 단순 인덱스 조회가 된다.

### 7.7 렌더 (현재블록 / 보드셀)

**현재블록 sprite**(4셀 클론). 판정기는 그리기 직전 `현셀X`/`현셀Y` 리스트(길이 4)를 채운다:

```
# 판정기 안에서 broadcast 현재블록그리기 직전, 또는 현재블록그리기 수신 시 판정기가 좌표 계산.
# 단순화: 현재블록 sprite 가 직접 오프셋을 읽어 4셀 좌표를 계산하게 한다(판정기 변수 재사용 X).
```

현재블록 sprite 자체가 그리는 단순안(판정기 변수에 의존하지 않게):

```
when flag clicked: hide
when receive 게임시작:
  hide
  # 4개의 셀 클론 생성
  셀번호 ← 0
  repeat 4:
    셀번호 ← 셀번호 + 1
    create clone of _myself_

when I start as clone:
  show

when receive 현재블록그리기:    # 모든 클론이 자기 셀번호 칸을 갱신
  # 자기 셀(셀번호 1..4)의 오프셋으로 좌표 계산
  오프베이스 ← ((현피스 - 1) * 4 + 현회전) * 4
  셀col ← 현열 + (item (오프베이스 + 셀번호) of 오프셋C)
  셀row ← 현행 + (item (오프베이스 + 셀번호) of 오프셋R)
  switch costume to 현피스        # 색
  if 셀row < 0:
    hide                          # 천장 위 셀은 숨김
  else:
    show
    goto ( -150 + 셀col*22 , 143 - 셀row*22 )
```

> **클론이 글로벌 오프셋 리스트를 읽음**: 현재블록 sprite 클론도 `오프셋C`/`오프셋R`(글로벌) 과 `현피스/현회전/현열/현행`(글로벌) 을 읽어 자기 위치를 잡는다. `오프베이스`/`셀col`/`셀row` 는 충돌의도와 안 겹치게 **현재블록 sprite-local** 로 따로 두거나(권장), 글로벌이라도 판정기 입력 forever 와 시점이 달라 보통 안전. 빌더는 현재블록 sprite-local 임시 변수 3개를 따로 만들어도 됨.

**보드셀 sprite**(고정 블록 렌더). `보드그리기` 수신 시 전체 재렌더:

```
when flag clicked: hide
when receive 보드그리기:
  delete this clone            # (본체엔 영향 없음; 아래 패턴으로 기존 클론 제거)
  # 기존 클론 정리: 표준은 "본체가 보드그리기 수신 → 모든 클론에 '지워' 신호" 또는
  #   각 클론이 보드그리기 수신 시 스스로 delete this clone.
  # 그 뒤 본체가 보드 리스트를 훑어 새로 그림.
```

보드셀 권장 구현(클론 자가삭제 + 본체 재생성):

```
when receive 보드그리기:        # 본체만 실행되게: '본체 전용' 가드 없이도, 클론은 아래 핸들러로 자삭
  # 1) 기존 클론 모두 삭제 신호
  broadcast 보드지우기
  wait 0                        # 클론들이 삭제될 시간 양보
  # 2) 보드 리스트 훑어 값≠0 칸마다 클론 생성
  그릴idx ← 1
  repeat 112:
    그릴색 ← item (그릴idx) of 보드
    if 그릴색 ≠ 0:
      create clone of _myself_   # 클론이 그릴idx/그릴색 읽어 배치
    그릴idx ← 그릴idx + 1

when I start as clone:
  switch costume to 그릴색
  # 그릴idx → (col,row) → 좌표
  goto ( -150 + ((그릴idx-1) mod 8)*22 , 143 - (floor((그릴idx-1)/8))*22 )
  show

when receive 보드지우기:        # 클론만 반응(본체는 show 안 하므로 무시 가능)
  delete this clone
```

> **클론 재렌더 주의**: `보드그리기` 가 자주 호출되지 않게(고정·라인제거 시에만) 한다 — 매 브틱마다 부르면 클론 생성/삭제가 과해짐. 떨어지는 블록은 `현재블록그리기`(클론 4개만 이동)로 가볍게 갱신, 보드 전체 재렌더는 블록이 굳을 때만. **이것이 클론 수와 성능을 안전하게 유지하는 핵심.**
> **`그릴idx`/`그릴색` 복사 타이밍**: breakout/alien-invasion 의 "글로벌(여기선 보드셀 sprite-local) set → create clone → 클론이 start 에서 읽기" 패턴. 본체가 `그릴idx`/`그릴색` 을 set 한 직후 create clone 하면 클론이 그 값을 집는다. 빠른 루프에서 값이 덮어써지기 전에 클론이 읽도록, 필요하면 `create clone` 뒤 미세 wait 또는 my-block 동기화. 안전하게는 **본체가 좌표·색까지 다 계산해 보드셀 sprite-local 에 둔 뒤 create clone**.

### 7.8 게임오버 / 재시작

car-race/snake 패턴. 배너 sprite `wait until 게임상태=1 → wait until 게임상태=0 → show`. 깃발 재클릭 → 변수 리셋(점수0/게임상태1/줄수0/레벨1/낙하간격0.8) + 보드 리스트 재초기화(112×0) + 모든 클론 삭제(Scratch 깃발이 자동 삭제) → 새 게임. 게임오버 시 pop 사운드.

## 6.4 피스 오프셋 데이터 전체 (7피스 × 4회전 × 4셀) — **빌더가 그대로 임베드**

각 셀은 **기준칸(현열, 현행)** 대비 (열오프셋 dc, 행오프셋 dr). dr 은 아래로 갈수록 +(행 인덱스 증가 = 화면 아래). 리스트 인덱스 = `((피스-1)*4 + 회전)*4 + 셀(1..4)`. 아래 표의 셀 순서대로 `오프셋C`/`오프셋R` 에 add.

> 좌표계: dc 오른쪽 +, dr 아래쪽 +. 기준칸은 각 피스 회전상태의 "왼쪽 위 근처" 한 칸으로, 7.4 의 스폰 (현열=3, 현행=0) 기준 가운데 부근에 뜨도록 잡았다. 회전은 시계방향(0→1→2→3).

**build.py 임베드용 평탄 배열** (오프셋C = 각 셀 dc 를 위 표 순서대로 112개, 오프셋R = dr 112개):

```python
# (피스1..7) × (회전0..3) × (셀1..4) 의 (dc, dr). 위 표를 그대로 평탄화.
OFFSETS = [
 # I
 [(0,1),(1,1),(2,1),(3,1)], [(2,0),(2,1),(2,2),(2,3)], [(0,2),(1,2),(2,2),(3,2)], [(1,0),(1,1),(1,2),(1,3)],
 # O
 [(1,0),(2,0),(1,1),(2,1)], [(1,0),(2,0),(1,1),(2,1)], [(1,0),(2,0),(1,1),(2,1)], [(1,0),(2,0),(1,1),(2,1)],
 # T
 [(1,0),(0,1),(1,1),(2,1)], [(1,0),(1,1),(2,1),(1,2)], [(0,1),(1,1),(2,1),(1,2)], [(1,0),(0,1),(1,1),(1,2)],
 # S
 [(1,0),(2,0),(0,1),(1,1)], [(1,0),(1,1),(2,1),(2,2)], [(1,1),(2,1),(0,2),(1,2)], [(0,0),(0,1),(1,1),(1,2)],
 # Z
 [(0,0),(1,0),(1,1),(2,1)], [(2,0),(1,1),(2,1),(1,2)], [(0,1),(1,1),(1,2),(2,2)], [(1,0),(0,1),(1,1),(0,2)],
 # J
 [(0,0),(0,1),(1,1),(2,1)], [(1,0),(2,0),(1,1),(1,2)], [(0,1),(1,1),(2,1),(2,2)], [(1,0),(1,1),(0,2),(1,2)],
 # L
 [(2,0),(0,1),(1,1),(2,1)], [(1,0),(1,1),(1,2),(2,2)], [(0,1),(1,1),(2,1),(0,2)], [(0,0),(1,0),(1,1),(1,2)],
]
offC = [c for grp in OFFSETS for (c,r) in grp]   # 길이 112
offR = [r for grp in OFFSETS for (c,r) in grp]   # 길이 112
```

> 빌더는 `offC`/`offR` 를 게임시작 시 `오프셋C`/`오프셋R` 리스트에 `add` 하거나, project.json 의 리스트 초기 contents 로 직접 임베드한다(후자가 블록 수 절약 — 권장). 회전 형태가 살짝 비표준(SRS 정식과 다른 단순 회전)이지만, 8×14 보드에서 모양·회전·라인제거가 자연스럽게 동작하면 충분하다.

  현피스 ← pick random 1 to 7
  다음피스 ← pick random 1 to 7
  현회전 ← 0 ; 현열 ← 3 ; 현행 ← 0
  # 오프셋 리스트 채우기(또는 임베드)

  # 보드 112칸 0으로
  delete all of 보드
  repeat 112: add 0 to 보드
  broadcast 게임시작
  broadcast 새블록            # 첫 블록(판정기가 처리)

when receive 게임시작:        # 중력 타이머
  repeat until 게임상태 = 0:

## 11. 재사용 코드 (builder 가 참조할 부분)

- **리스트로 게임 상태 표현 + 리스트 블록 opcode**: `games/snake/build.py` — snake 의 `궤적X/궤적Y`(insert/item/delete/length/deleteall)와 동일한 헬퍼(`make_list_helpers`)를 그대로 쓴다. tetris 의 `보드` 는 snake 궤적과 같은 "리스트에 게임 상태를 담고 인덱스로 읽고 쓴다" 패턴의 확장(1D→2D 평탄화). snake 의 list helper 함수(add_to/insert_at/delete_of/delete_all/item_of/length_of)를 그대로 import/복사.
- **클론이 리스트 항목을 읽어 자기 위치를 잡음**: `games/snake/build.py` 의 꼬리 클론(`item(세그번호) of 궤적`)과 동일 — tetris 의 현재블록 4셀 클론이 `item(오프베이스+셀번호) of 오프셋C/R` 로 위치 계산.
- **격자 스냅 좌표 + 키 에지 입력(`wait until NOT key`)**: `games/car-race/build.py` / `games/snake/build.py` — 좌/우/회전을 1회 입력=1칸/1회전으로. 소프트드롭/하드드롭만 에지 아님.
- **N칸 클론 격자 렌더 + per-clone 좌표·색 복사**: `games/breakout/build.py`(8×4 벽돌) / `games/alien-invasion/build.py`(cols×rows spawn + per-clone 값 복사) — tetris 보드셀의 "리스트 훑어 값≠0 칸마다 클론 생성, 클론이 idx→좌표/색 복사" 패턴.
- **게임상태 broadcast + 깃발 재시작 + 게임오버 배너**: `games/car-race/build.py` / `games/snake/build.py` 그대로.
- **속도 ramp(레벨↑→간격↓)**: `games/snake/build.py`(점수 기반 스텝간격 단축) / `games/breakout/build.py`(라운드 속도+1) 패턴.

**필요한 블록 opcode (빌더 확인/추가)**:
- 리스트: `data_addtolist` / `data_deletealloflist` / `data_itemoflist` / `data_lengthoflist` — **snake build.py 에 이미 있음**.
- **`data_replaceitemoflist`(inputs ITEM, INDEX ; fields LIST)** — **snake 에 없음. 빌더가 build.py 의 `make_list_helpers` 에 `replace_at(name, lid, item, index)` 추가 필요** (insert_at 과 동일 구조, opcode 만 `data_replaceitemoflist`). 보드 칸 치환·라인 끌어내림의 핵심.
- `operator_mod`(회전 mod 4, idx→col 역변환) / `operator_mathop`("floor")(레벨, idx→row 역변환) — 기존 게임들에 mathop 존재(snake abs 등 없으면 추가 간단).
- 커스텀 블록(my block): `procedures_definition` / `procedures_prototype` / `procedures_call` + `argumentreporter` — **기존 게임에 없으면 빌더가 build.py 에 추가**. 없으면 `충돌?`/`고정` 을 broadcast+`wait until 처리완료=1` 동기 패턴으로 대체 가능(다소 장황). **my block 권장**.
- 나머지(`motion_gotoxy`, `looks_switchcostumeto`, `sensing_keypressed`, `control_repeat`/`repeat_until`/`if`/`if_else`/`create_clone_of`/`start_as_clone`/`delete_this_clone`, `operator_*`, `data_setvariableto`/`changevariableby`, `event_broadcast`)는 snake/breakout/car-race 에 모두 존재.

빌더 권장 베이스: **snake 의 build.py 를 골격으로**(리스트 헬퍼 + 클론-리스트 추종 + 키 에지 입력 + 게임상태/배너) 잡고, (1) `make_list_helpers` 에 `replace_at` 추가, (2) my block(`충돌?`/`고정`) 추가, (3) 보드/오프셋 리스트 임베드, (4) 보드셀 렌더는 breakout 의 격자 클론 spawn 을 "리스트 훑어 생성" 으로 변형.

   - 경계검사 `(셀col<0) OR (셀col>7) OR (셀row>13)` + `셀row≥0 일 때 item(셀row*8+셀col+1) of 보드 ≠ 0` → 충돌됨=1
10. `브틱` 수신에 `현행+1` 충돌검사 → 가능하면 하강+`현재블록그리기`, 불가하면 `고정` 호출
11. `고정`(my block) 에:
    - 4셀을 `replace item (셀row*8+셀col+1) of 보드 with 현피스`(`data_replaceitemoflist`)
    - 아래행부터 꽉참 검사(`repeat 8` 로 행 8칸 0 여부) + 꽉찬 행 위를 한 칸 끌어내림(replace 루프) + row0 비움
    - 지운줄 1/2/3/4 → 점수 +100/300/500/800, 줄수 += 지운줄, `broadcast 보드그리기`+`broadcast 새블록`

14. 보드셀 sprite: `보드그리기` 수신 시 기존 클론 삭제 후 `repeat 112` 로 `item(그릴idx) of 보드 ≠ 0` 칸마다 클론 생성 + 클론이 idx→좌표/색 복사
15. Stage `브틱` 타이머(`wait 낙하간격`) + 레벨 ramp(`floor(줄수/5)+1`, 낙하간격 하한 0.12) 존재
