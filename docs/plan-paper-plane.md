# 종이비행기 멀리 던지기 (paper-plane) — Plan

> 화면 왼쪽 발사대에서 종이비행기를 **각도 → 힘** 2단 타이밍 게이지로 던진다. 던진 비행기는 중력으로 천천히 하강하며(글라이딩), 매 던지기마다 바뀌는 **바람(순풍/역풍)** 에 밀려 더 멀리 가거나 짧게 떨어진다. 지면에 닿으면 날아간 거리를 미터로 환산해 점수로 기록. 총 3번 던져 합계와 베스트를 겨룬다.
> 베이스: `games/endless-runner/build.py`·`games/car-race/build.py` (배경/마커 좌측 스크롤 + 거리 누적) + `games/flappy-bird/build.py` (중력 dy 적분 + 지면 충돌) + `games/cowboy-duel/build.py` (라운드/턴 진행 + 합계/베스트 배너). 발사 게이지는 신규(타이밍 왕복 바 — 슬라이더 UI 가 아닌 자동 왕복+스페이스 고정 방식).
> 학습 콘셉트 **없음**. 순수 액션 + 약간의 타이밍/바람 읽기 전략. 초등학생 대상 직관적(스페이스 1키만 사용). 추상 개념·수식 노출 절대 금지. (MEMORY.md → feedback-game-design 준수)

> **rocket-trajectory 실패 교훈 반영**: 슬라이더(드래그) UI 대신 **자동으로 왕복하는 타이밍 게이지 + 스페이스로 고정** 방식 채택. 마우스 드래그/슬라이더 위젯을 일절 쓰지 않는다. 입력은 스페이스 키 단 하나.

---

## 1. 한 줄 룰

1단계: 각도 바늘이 **0°~80°** 사이를 자동으로 왕복한다 → 원하는 각도에서 **스페이스**를 눌러 고정.
2단계: 힘 게이지가 **0~100** 사이를 자동으로 왕복(차오르락 내리락)한다 → 원하는 세기에서 **스페이스**를 눌러 발사!
비행기는 던진 각도·힘으로 날아가고, 이번 턴의 바람(→ 순풍이면 멀리, ← 역풍이면 짧게)을 받으며 천천히 미끄러져 내려온다. 지면에 닿은 곳까지가 거리(m). 3번 던져서 합계를 키우자.

## 2. 화면 / 좌표

- 무대 480×360 (Scratch 표준). 좌표 -240..240 / -180..180.
- SVG 좌표(0..480 / 0..360) ↔ Scratch 좌표 변환: `sx = svgX - 240`, `sy = 180 - svgY`.
- **발사 위치(피벗)**: x = -180, y = -100 (화면 왼쪽, 언덕 위). 비행기는 항상 여기서 출발한다.
- **지면 높이**: y = -140 (지면선). 비행기 y 가 -140 이하로 내려오면 착지 판정.
- **비행 표현(중요·아래 7.3에서 결정)**: 비행기는 **화면 왼쪽 고정 밴드(x ≈ -180 → 최대 +40 까지만 실제로 이동)** 안에서 날고, 그 이상 멀리 간 거리는 **배경·거리마커가 좌측으로 스크롤**되어 표현한다. 즉 "비행기가 화면을 벗어나 사라지는" 일이 없도록 카메라가 따라간다. (endless-runner 스크롤 패턴 재사용)
- **HUD(상단)**: 거리(현재 던지기 m) / 베스트 / 합계 / 남은던지기. **바람 표시(화살표 + 세기)** 는 상단 중앙.
- **게이지 바**: 발사 단계에서만 화면 좌하단에 표시(각도 바늘 + 힘 게이지). 발사 후 숨김.

```
+----------------------------------------------------------+ y=+180
| 거리:0m  베스트:120m  합계:240m   남은던지기:2            |  ← 변수 모니터 (상단)
|                  바람:  → 1.2  (순풍)                     |  y=+150  바람 화살표+숫자
|              ☁         ☁              ☁                  |  구름(장식, 살짝 스크롤)
|                                                          |
|                       ✈ (비행기, 기수 방향으로 회전)       |  ← 비행 중
|        ╱80°                                              |
|      ╱  ← 각도 바늘 왕복(0~80°)        🚩       🚩         |  ← 거리마커 깃발(스크롤)
|    [발사대/언덕]                                          |
|====●===================================================== |  y=-140 지면선
|   x=-180                                                 |
| [힘 ▓▓▓▓▓░░░░░ 50]  ← 힘 게이지(왕복)                      |  y=-165
+----------------------------------------------------------+ y=-180
  x=-240                                              x=+240
              [ 스페이스 = 각도 고정 → 다시 스페이스 = 발사 ]
```

## 6. 씬 / 상태머신

```
[깃발]
  → init(거리0, 베스트0, 합계0, 남은던지기3, 게임상태0)
  → broadcast 게임시작
  → broadcast 던지기준비 (첫 던지기)
   │
   ▼
[던지기준비]  (게임상태=0)
  - 바람 ← pick random -1.5..1.5 (소수 1자리)
  - 바람화살표 갱신(방향/세기)
  - 비행기 goto 발사대(-180,-100), 방향 90, 보이기
  - 거리 ← 0, 스크롤량 ← 0, 거리마커/배경 원위치
  - 발사단계 ← 0 (각도 게이지 시작)
   │
   ▼
[발사 게이지]  (비행기 스프라이트가 관리)
  ┌─ 발사단계 0 (각도) ────────────────────────────────┐
  │  게이지값 0↔80 자동 왕복(각도바늘 회전 연출)         │
  │  스페이스 누름? → 발사각도 ← 게이지값 → 발사단계 1   │
  └────────────────────────────────────────────────────┘
  ┌─ 발사단계 1 (힘) ──────────────────────────────────┐
  │  게이지값 0↔100 자동 왕복(힘게이지 채움 연출)        │
  │  스페이스 누름? → 발사힘 ← 게이지값                  │
  │     → dx,dy 초기화(각도·힘으로) → broadcast 발사     │
  │     → 발사단계 2, 게임상태 1                         │
  └────────────────────────────────────────────────────┘
   │
   ▼
[비행]  (게임상태=1)  — 매 틱:
  - dy ← dy - 0.25            (중력, 천천히 = 글라이딩)
  - dx ← dx + 바람            (순풍 가속 / 역풍 감속)
  - 비행기를 dx,dy 만큼 이동 (단, 화면 밴드 넘으면 그만큼 배경/마커 스크롤 = 카메라)
  - point in direction (dx,dy 방향)  ← 기수 방향 연출(하강 시 기수 아래로)
  - 거리 ← (실제 전진 누적 / 4) 정수
  - 비행기 y ≤ -140? → broadcast 착지 → 게임상태 2
   │
   ▼
[착지]  (게임상태=2)
  - 거리 확정 표시
  - 합계 += 거리, if 거리>베스트 {베스트=거리; broadcast 신기록}
  - 남은던지기 -= 1
  - if 남은던지기 > 0: wait 1.2 → broadcast 던지기준비 (다음 던지기)
  - else: 게임상태 3 → broadcast 결과
   │
   ▼ [깃발 재클릭] → 처음으로
```

## 7. 메커닉 상세

### 7.1 발사 게이지 (2단 타이밍 — 핵심)

비행기 스프라이트가 발사 전 게이지 단계를 forever 로 돌린다. **슬라이더 없이** 게이지 값이 자동 왕복하고, 스페이스 1키로 두 번 멈춘다(각도 고정 → 발사).

```
when receive 던지기준비:
  goto (-180, -100)
  point in direction 90        # 오른쪽(수평) 기준
  show
  발사단계 ← 0
  게이지값 ← 0
  게이지방향 ← 1
  게임상태 ← 0
  # 각도바늘/힘게이지에게도 "표시" 신호는 발사단계 값으로 각자 판단

when flag clicked:  (별도 스크립트 — 게이지 왕복 + 입력)
  forever:
    if 게임상태 = 0:
      # ── 단계 0: 각도 왕복(0~80) ──
      if 발사단계 = 0:
        게이지값 ← 게이지값 + (게이지방향 * 2)     # 왕복 속도 2/틱
        if 게이지값 ≥ 80: 게이지값 ← 80; 게이지방향 ← -1
        if 게이지값 ≤ 0:  게이지값 ← 0;  게이지방향 ← 1
        if (key space pressed?):
          발사각도 ← 게이지값
          발사단계 ← 1
          게이지값 ← 0; 게이지방향 ← 1
          wait until (not key space pressed?)      # 같은 누름이 발사로 이어지지 않게
      # ── 단계 1: 힘 왕복(0~100) ──
      else if 발사단계 = 1:
        게이지값 ← 게이지값 + (게이지방향 * 3)     # 힘은 조금 빠르게
        if 게이지값 ≥ 100: 게이지값 ← 100; 게이지방향 ← -1
        if 게이지값 ≤ 0:   게이지값 ← 0;   게이지방향 ← 1
        if (key space pressed?):
          발사힘 ← 게이지값
          # dx,dy 초기화: 각도·힘 → 속도 벡터
          dx ← (cos of 발사각도) * (발사힘 / 100) * 12
          dy ← (sin of 발사각도) * (발사힘 / 100) * 12
          발사단계 ← 2
          게임상태 ← 1
          broadcast 발사
          wait until (not key space pressed?)
    wait 0.02
```

> **각도→속도**: Scratch 각도 0°=수평(오른쪽), 80°=거의 위쪽. `dx = cos(각도)·v`, `dy = sin(각도)·v`. 힘은 `v = (힘/100)·12` 로 최대 12px/틱. 빌더는 12 계수를 조정해 잘 던지면 화면을 시원하게 가로지르도록 튜닝.
> **이중 입력 방지**: 각 스페이스 입력 후 `wait until (not key space pressed?)` 로 키를 뗄 때까지 기다려, 한 번 누름이 두 단계를 동시에 통과하지 않게 한다(rocket-trajectory 류의 입력 꼬임 방지).

### 7.2 각도바늘 / 힘게이지 (게이지 시각화)

```
# 각도바늘 — 발사대(-180,-100)에 붙어 각도 표시
when flag clicked: hide
when receive 던지기준비:
  goto (-180, -100); show
  forever:
    if 발사단계 = 0:   # 왕복 중: 게이지값 각도로 바늘 회전
      show
      point in direction (90 - 게이지값)   # 0°→수평(90), 80°→위쪽(10)
    else if 발사단계 = 1:  # 각도 고정: 발사각도로 멈춤
      show
      point in direction (90 - 발사각도)
    else:  # 비행 중/결과: 숨김
      hide
      stop this script
    wait 0.02
```
> `point in direction (90 - 각도)`: Scratch 에서 90=오른쪽, 작아질수록 위로 향함. 각도 0°(수평)→방향 90, 80°→방향 10(거의 위). 발사 물리의 `cos/sin(발사각도)` 와 방향이 일치한다(둘 다 "수평 기준 위로 각도").

```
# 힘게이지 — 좌하단, 발사단계 1에서만 채움 표시
when flag clicked: hide

  goto (-150, -165)
  forever:
    if 발사단계 = 1:
      show
      set size to (60 + 게이지값)          # 힘 클수록 커짐(또는 stretch 폭)
    else:
      hide

    wait 0.02
```
> 정확한 막대 stretch 가 어려우면 size 변화로 대체(7.3 결정 참조). 수치는 변수 모니터(발사힘/게이지값)로 정확히 전달.

### 7.3 비행 물리 + 카메라 (dx/dy + 스크롤) — 핵심 결정

**카메라/거리 방식 결정**: Scratch 에서 가장 단순하고 안정적인 방식 = **비행기는 화면 왼쪽~중앙 밴드(x: -180 → +40)에서만 실제로 이동**하고, 비행기가 밴드 오른쪽 끝(+40)을 넘으려 하면 그 초과분만큼 **비행기를 +40에 고정한 채 배경/거리마커를 좌측으로 스크롤**한다(카메라가 비행기를 따라감). 이렇게 하면 비행기가 화면 밖으로 사라지지 않고, 거리는 무한히 늘어날 수 있다. (endless-runner/car-race 의 "캐릭터 고정 + 세계 스크롤" 패턴 그대로.)

```
when receive 발사:        # 비행기 물리 비행 시작 (게임상태=1)
  point in direction 90
  forever:
    if 게임상태 ≠ 1: stop this script
    # 1) 물리 적분
    dy ← dy - 0.25                 # 중력(작게 = 종이비행기 글라이딩)
    dx ← dx + 바람                 # 순풍(+)/역풍(-) 매 틱 가속
    if dx < 0.3: dx ← 0.3          # 너무 느려도 앞으로는 가게(역풍 보호, 선택)
    # 2) 가로 이동 → 카메라 처리
    next_x ← (x position) + dx
    if next_x > 40:                # 밴드 오른쪽 끝 초과 → 카메라 스크롤
      ekstra ← next_x - 40
      set x to 40
      스크롤량 ← 스크롤량 + ekstra  # 세계가 ekstra 만큼 왼쪽으로 밀림
      broadcast 스크롤  (또는 마커/배경이 스크롤량 변수를 직접 참조해 매 틱 이동)
    else:
      set x to next_x
    # 3) 세로 이동
    change y by dy
    # 4) 기수 방향 연출 (속도 벡터 방향으로)
    point in direction (90 - (atan of (dy / dx)))   # 올라갈 땐 위, 내려갈 땐 아래로 기수
    # 5) 거리 환산: 실제 전진(밴드 내 이동분 + 스크롤량) / 4
    거리 ← round( ((x position) - (-180) + 스크롤량) / 4 )
    # 6) 착지 판정
    if (y position) ≤ -140:
      set y to -140
      게임상태 ← 2
      broadcast 착지
      stop this script
    wait 0.02
```

> **기수 방향**: `point in direction (90 - atan(dy/dx))`. dx>0 가정(항상 앞으로 감). dy>0(상승)이면 기수 위로, dy<0(하강)이면 기수 아래로 자연스럽게 기운다. dx=0 분모 0 방지를 위해 위에서 dx 최소값 0.3 보장. (atan 은 `operator_mathop` 의 atan.)
> **거리 환산 계수 ÷4**: 12px/틱 최대 속도로 ~3초 글라이딩하면 수백 px 전진 → ÷4 로 100~200m 대 점수가 나온다. 빌더가 체감 조정 가능. **거리는 항상 정수(round)로 표시**.
> **스크롤 전달**: 거리마커/구름/배경 요소는 매 틱 `스크롤량` 변수를 참조해 `x = 원위치 - (스크롤량 mod 화면폭)` 식으로 좌측 이동(아래 7.5). 별도 `스크롤` 방송 없이 변수 참조만으로 충분(더 단순). 방송은 생략 가능.
> **밴드 폭/계수**는 빌더 튜닝 대상. 핵심은 "비행기는 안 사라지고, 거리는 누적된다".

### 7.4 Stage — 턴(3번 던지기) 진행 / 바람 / 판정

```
when flag clicked:
  거리 ← 0; 베스트 ← 0; 합계 ← 0
  남은던지기 ← 3
  게임상태 ← 0; 발사단계 ← 0
  broadcast 게임시작
  broadcast 던지기준비

when receive 던지기준비:
  # 매 던지기마다 바람 새로 뽑기
  바람 ← (pick random -15 to 15) / 10        # -1.5 ~ +1.5 (0.1 단위)
  거리 ← 0
  스크롤량 ← 0
  게임상태 ← 0
  발사단계 ← 0
  # (비행기/마커/배경 원위치는 각 스프라이트가 던지기준비에서 처리)

when receive 착지:
  # 거리 확정 + 합계/베스트
  합계 ← 합계 + 거리
  if 거리 > 베스트:
    베스트 ← 거리
    broadcast 신기록
  남은던지기 ← 남은던지기 - 1
  play sound land
  wait 1.2                                    # 착지 거리 보여주는 시간
  if 남은던지기 > 0:
    broadcast 던지기준비                       # 다음 던지기
  else:
    게임상태 ← 3
    broadcast 결과
```

> **바람 변경 타이밍**: `던지기준비` 마다 새로 뽑으므로 "던질 때마다 바람이 바뀐다"가 자동 충족. 순풍(+)이면 멀리, 역풍(-)이면 짧게 → 플레이어가 화살표를 보고 각도/힘을 조절하는 전략.
> **순풍/역풍 텍스트**: 바람화살표 스프라이트가 바람 부호로 방향·라벨 처리(7.5).


## 8. 스프라이트별 블록 트리 (의사코드 요약)

> 7장이 이미 스프라이트별 의사코드 수준이므로 핵심 분기만 압축(빌더 1:1 매핑용). 상세 수치는 7장 참조.

### Stage
- `when flag` → init 변수 + broadcast 게임시작 + 던지기준비
- `when 던지기준비` → 바람=rand(-15..15)/10; 거리0; 스크롤량0; 게임상태0; 발사단계0

### 비행기 (메인)
- `when 던지기준비` → goto(-180,-100); point dir 90; show; 발사단계0; 게이지값0; 게이지방향1; 게임상태0

- `when flag` hide

## 11. 재사용 코드 (builder 가 참조할 부분)

- **중력 dy 적분 + 지면 충돌(`dy -= 중력` → `change y by dy` → `y ≤ 지면 → 멈춤`)**: `games/flappy-bird/build.py`(새 낙하/점프) · `games/endless-runner/build.py`(점프 VY 적분). 비행기 글라이딩이 이 패턴 — 단지 중력 계수를 작게(0.25) 해 천천히 떨어지게 함.
