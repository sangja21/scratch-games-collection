# 우주정거장 도킹 (station-docking) — Plan

> 우주선을 4방향 분사로 정밀하게 움직여 우주정거장의 도킹 포트에 살포시 닿아야 하는 정밀 조작 게임. 너무 빠르게 닿으면 충돌. 천천히 정확하게 닿으면 도킹 성공.
> 베이스: `games/asteroids/build.py` 의 관성·분사 패턴(VX/VY 누적) 을 4방향 분사(회전 없음) 로 단순화. + 정밀 일치 판정(속도 임계) + 연료 자원 + 라운드제(포트 위치 무작위).
> 학습 콘셉트 없음. 초등학생 대상 직관적 액션.

## 1. 컨셉 / 메커닉

- 480×360 톱다운 우주 시점. 좌측에 우주선 시작, 우측에 우주정거장 고정.
- **우주선 조작 (4방향 분사, 회전 없음)**:
  - 위 화살표 = VY += 0.3
  - 아래 화살표 = VY -= 0.3
  - 오른쪽 화살표 = VX += 0.3
  - 왼쪽 화살표 = VX -= 0.3
  - 분사할 때마다 연료 -1
- **관성**:
  - 매 틱 `x += VX`, `y += VY`. 마찰 없음(우주). 자연 감속 없음.
  - 속도 clamp: `|VX|, |VY| <= 4.5`
- **화면 경계**: wrap 없음. 화면 밖으로 나가면 setx/sety 로 끝에 clamp (벽에 부딪힌 듯 멈춤 — VX/VY 도 0 으로). 단순화: x>240→x=240, VX=0 등.
- **우주정거장** (스프라이트 `정거장`):
  - 화면 우측 고정 (x=160 근처). 라운드마다 y 위치가 변경 (-100 ~ +100 random).
  - 도킹 포트(SVG 안에 강조된 사각형) 가 정거장 중앙 좌측.
- **도킹 판정**:
  - 우주선이 정거장에 닿음 (`touching 정거장`)
  - **그리고** `|VX| < 1.5` 그리고 `|VY| < 1.5` → 도킹 성공
  - 너무 빠르게 닿으면 (속도 어느 한쪽이 1.5 이상) → 충돌 → 게임오버
- **연료 자원**:
  - 시작 100. 분사할 때마다 -1. 0 이 되면 더 분사 불가 (게임오버는 X. 그냥 떠다니다 멈춤 — 단, 도킹 못하면 사실상 진행 불가).
  - 단순화: 연료 0 + 우주선이 정지(VX=VY=0 근사) 상태로 5초 → 게임오버(연료부족).
  - 더 단순화: 연료 0 되면 즉시 게임오버.
- **점수**:
  - 도킹 성공 시 점수 += 100 + 남은연료
  - 라운드 +1, 우주선 위치 초기화(좌측 (-180, 0)), VX/VY=0, 연료 = max(60, 100-라운드*5) (라운드 진행시 연료 약간 감소 — 보너스 조정용. 단순화: 매 라운드 연료=100 고정)
  - 단순화: 연료 = 100 매 라운드.
  - 정거장 y 좌표 random(-100..100), x = 160 (고정).
- **체력 없음. 충돌 1회 → 즉시 게임오버**. (asteroids 와 동일 패턴)

## 6. 스프라이트별 블록 흐름

### Stage

1. **깃발 클릭** → 변수 초기화(점수=0, 라운드=1, 게임상태=1, 연료=100, VX=0, VY=0, 정거장Y=0) → 0.5초 대기 → `게임시작` broadcast.
2. **게임시작 받으면** → `라운드시작` broadcast.
3. **라운드시작 받으면**:
   - VX=0, VY=0
   - 연료 = 100
   - 정거장Y = random(-100..100)
4. **도킹성공 받으면**:
   - 점수 += 100 + 연료
   - 라운드 += 1
   - 1.5초 대기
   - `라운드시작` broadcast

### 우주선

1. **깃발 클릭** → 보이기, 크기 60, 위치 (-180, 0), 방향 90, 회전스타일=don't rotate.
2. **forever (조작 + 물리)**:
   - if up arrow AND 연료 > 0 AND 게임상태=1 → VY += 0.3, 연료 -= 1
   - if down arrow AND 연료 > 0 AND 게임상태=1 → VY -= 0.3, 연료 -= 1
   - if right arrow AND 연료 > 0 AND 게임상태=1 → VX += 0.3, 연료 -= 1
   - if left arrow AND 연료 > 0 AND 게임상태=1 → VX -= 0.3, 연료 -= 1
   - VX clamp: ±4.5
   - VY clamp: ±4.5
   - change x by VX, change y by VY
   - 벽 멈춤:
     - if x > 235 → set x to 235, VX = 0
     - if x < -235 → set x to -235, VX = 0
     - if y > 175 → set y to 175, VY = 0
     - if y < -175 → set y to -175, VY = 0
   - wait 0.02
3. **forever (도킹/충돌 판정)**:
   - if (touching 정거장) AND 게임상태=1:
     - if |VX| < 1.5 AND |VY| < 1.5:
       - VX=0, VY=0 (정지)
       - broadcast 도킹성공
       - pop @ pitch 400 (성공 사운드)
       - wait 1.5 (다음 라운드 broadcast 까지 대기)
     - else:
       - 게임상태 = 0
       - broadcast 충돌
       - pop @ pitch -400
   - wait 0.05
4. **라운드시작 받으면**:
   - 위치 (-180, 0)
   - (VX/VY 는 Stage 에서 0 으로 초기화)

### 정거장

1. **깃발 클릭** → 보이기, 크기 100, x=160, y=0.
2. **라운드시작 받으면**:
   - go to x=160, y=정거장Y
3. **충돌 받으면** → (그대로 — 시각 효과 없음)

### 도킹성공

1. **깃발 클릭** → hide, 위치 (0,0), size 100, front.
2. **도킹성공 받으면**:
   - show
   - wait 1.0
   - hide

### 게임오버

- asteroids 와 동일 패턴: 깃발 → hide + (0,0) + size 100 + front → wait until 게임상태=1 → wait until 게임상태=0 → show.
