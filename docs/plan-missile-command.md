# 미사일 커맨드 (missile-command) — Plan

> 1980 아타리 Missile Command 클래식 클론.
> 도시 3곳을 지키며 떨어지는 적 미사일을 마우스 클릭으로 요격한다.
> 베이스: `games/alien-invasion/build.py` (클론 발사·라운드 시스템·게임오버 배너 재사용).

## 1. 컨셉 / 메커닉

- 화면 하단에 도시 3곳이 일직선으로 배치.
- 화면 상단에서 적 미사일이 임의 X 좌표에서 도시 중 하나를 향해 비스듬히 떨어진다.
- 플레이어가 화면 임의 지점을 마우스로 **클릭**하면 그 좌표에서 폭발(범위 폭발)이 일어남.
- 폭발 범위 내 적 미사일은 모두 격추된다.
- 도시가 적 미사일에 맞으면 파괴되고 라이프 1 감소. 도시 3개가 모두 파괴되면 게임오버.
- 라운드 클리어 = 그 라운드의 적 미사일 N개를 모두 격추. 라운드↑ → 미사일 수↑, 속도↑.
- 학습 컨셉 절대 없음. 순수 액션 (마우스 클릭 반응속도).

## 2. 화면 레이아웃 (Scratch 480×360)

```
y=180   ─────────────────────────────────────
                                                  ← 적 미사일 스폰 라인
                                                     (y=170, x: -200..200 랜덤)

y=0     · · · · · 빈 공간 · · · · ·              ← 미사일이 비스듬히 떨어짐

y=-140  [도시1]      [도시2]      [도시3]         ← 도시 3개, x=-140, 0, 140
y=-180   바닥선
```

## 6. 스프라이트별 블록 흐름

### Stage
1. **깃발 클릭** → 점수=0, 라이프=3, 라운드=1, 게임상태=1, 적속도=1.6, 라운드미사일수=8, 격추수=0 → `도시생성` 방송 → 0.3초 대기 → `게임시작` 방송.
2. **게임시작 받으면** → `적생성` 방송 → `repeat until 게임상태=0`: (게임상태 1이면 계속) wait 0.1초.
3. **다음라운드 받으면** → 라운드 +1, 적속도 +0.3, 라운드미사일수 +2, 격추수=0 → 1.2초 대기 → `적생성` 방송.

### 조준선 (crosshair)
1. **깃발 클릭** → 보이기, 크기 80, `forever`: 마우스 X/Y 로 이동.
2. **무대 클릭 (event_whenstageclicked) 또는 클릭 키 (마우스 다운 폴링)** → if 게임상태=1: 폭발X=마우스X, 폭발Y=마우스Y → `폭발` 클론 생성.
   * Scratch에서 마우스 클릭 감지는 `event_whenthisspriteclicked` 또는 `<mouse down?>` 센싱. 십자선이 마우스를 따라다니므로 `event_whenthisspriteclicked` 가 자연스럽다.

### 폭발 (explosion)
1. **깃발 클릭** → 숨기기.
2. **클론으로 시작** → 폭발X/Y 좌표로 이동 → 크기 20에서 시작 → 보이기 → repeat 12회 (크기 +14, wait 0.02s) → repeat 8회 (크기 -10, wait 0.02s) → 숨기기 → 클론 삭제.
   * 적미사일은 자신의 forever 루프에서 `폭발과 닿았는가` 검사 → 격추 처리. 이 폭발 클론은 단지 시각·충돌 영역 제공.

### 적미사일
1. **깃발 클릭** → 숨기기, 크기 60.
2. **적생성 받으면** → repeat `라운드미사일수` 회: `myself` 클론 생성, 1.0/적속도 ~ 1.4/적속도 초 대기.
3. **클론으로 시작** → 시작 좌표 (랜덤 -200..200, 175) → 목표X = 랜덤 [-140, 0, 140] (도시 3개 중 하나) → 그 방향 점 (목표X, -140) 으로 `point towards`(motion_pointtowardsxy 또는 두 점 차이로 atan2 대용) 처리.
   * 단순화: 매 틱 dx, dy 를 미리 계산해서 `change x/y by`.
   * dx = (목표X - 시작X) / 단계수, dy = (-140 - 175) / 단계수 (음수).
   * 단계수 = floor(60 / 적속도) → 적속도 클수록 적은 단계로 도달.
4. 클론 루프: `repeat until 게임상태=0 or 폭발에 닿음 or y<-140`:
   - change x by dx, change y by dy
   - if 폭발과 닿음: 점수 +25, 격추수 +1, if 격추수=라운드미사일수: `다음라운드` 방송. delete clone.
   - if y<-140: 도시에 도달 — 라이프 -1, 가까운 도시 클론에 메시지(또는 sensing) 로 파괴 처리. 단순화: 적미사일이 y<-140 되면 그냥 라이프 -1 처리하고 delete clone. (도시 시각 파괴는 도시 스프라이트가 `touching 적미사일?` 으로 판정해서 처리.)
   - wait 0.04 초.
5. if 라이프 ≤ 0 → 게임상태=0.

### 도시 (city)
1. **깃발 클릭** → 숨기기, 크기 100, 코스튬=intact.
2. **도시생성 받으면** → 자신 위치 변수 (스프라이트 로컬) 0 → 클론3 만들고 보이기, 위치 (-140,-160) (0,-160) (140,-160) 설정.
   * 단순화: stage 가 도시 3 클론을 생성, 각 클론은 시작 시 자신의 위치를 X 인덱스 기준으로 결정 (스프라이트 로컬 변수 `도시번호` 1/2/3).
3. **클론으로 시작** → 위치 설정 후 forever: if `touching 적미사일?` → 코스튬=ruined, 0.4초 깜빡, 라이프 -1, broadcast 적미사일 처리 (이미 적미사일이 자기 자신 delete). → wait until 게임상태=0 → 숨기기.

### 게임오버
- alien-invasion 과 동일 패턴: 깃발→숨김→state=1 까지 대기→state=0 까지 대기→show.
