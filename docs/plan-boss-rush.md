# 보스 러시 (boss-rush) — Plan

> 1대 보스 vs 플레이어. 화면 하단 플레이어는 좌/우 화살표로 이동하면서 스페이스로 위쪽으로 직선 사격. 화면 상단의 거대한 보스는 좌↔우로 왕복하면서 체력에 따라 3개 페이즈로 점점 강해지는 탄막을 발사. 플레이어는 보스 탄을 피하며 사격해 보스 체력 100을 깎아 격파. 페이즈마다 탄막 패턴이 바뀌어 외워야 한다.
> 베이스: `games/dogfight/build.py` 의 회전+사격+체력+wrap+배너 패턴 + `games/alien-invasion/build.py` 의 상단 적/하단 플레이어 + 직선 발사 패턴. **차이점**: 적이 1대(보스)로 거대화, 보스 체력 100/체력바, 체력 구간별 페이즈 분기, 페이즈마다 다른 탄막 패턴.
> 학습 콘셉트 없음. 초등학생 대상 직관적 액션 (Touhou/Cuphead 보스전의 캐주얼 버전).

## 1. 컨셉 / 메커닉

- 480×360 무대. 배경 = 보스전 분위기의 어두운 보라/남색 그라데이션 + 작은 별 도트들.
- **플레이어 조작 (화면 하단)**:
  - 왼쪽 화살표 = 좌로 이동 (틱당 −4 px), 단 x ≤ −210 이면 멈춤(벽)
  - 오른쪽 화살표 = 우로 이동 (틱당 +4 px), x ≥ 210 이면 멈춤(벽)
  - 스페이스 = 위쪽으로 직선 총알 발사 (쿨다운 0.18초)
  - 회전 없음, direction=0 (위쪽) 고정
  - 초기 위치 (0, −140)
- **보스 (화면 상단)**:
  - 거대 스프라이트 (size 160). 초기 위치 (0, 110), direction=180 (아래).
  - **수평 왕복**: 좌↔우 끝점 (−170, 110) ↔ (170, 110) 사이를 일정 속도로 왕복. 페이즈에 따라 속도 변함.
  - **체력 100**. 페이즈 분기 (Stage 의 `현재페이즈` 변수가 매 틱 보스체력으로 갱신):
    - **페이즈 1**: 보스체력 67~100. 속도 1.5 px/tick. **부채꼴 3발 직선 탄막**, 발사 간격 1.0초. 보스 정면(아래) ± 20° 로 −20/0/+20 방향 3발.
    - **페이즈 2**: 보스체력 34~66. 속도 2.5 px/tick. **부채꼴 5발 직선 탄막**, 발사 간격 0.8초. 정면 ± 30° 로 5발 (−30/−15/0/+15/+30).
    - **페이즈 3**: 보스체력 1~33. 속도 3.5 px/tick. **원형 16발 탄막** + 빠른 이동. 발사 간격 1.2초. 정면 기준 360°/16 간격 16발.
  - **체력바**: 보스 머리 위 (실제로는 화면 최상단 y=165, x 중앙)에 가로 200 px 빈 바 + 채워진 바. 채워진 길이 = 200 × 보스체력/100. 별도 스프라이트 (체력바 배경 + 채움) 2개.
- **플레이어 체력**:
  - 체력 5 (HP). 보스 탄에 닿으면 −1, 무적 0.5초.
  - 체력 0 → 패배.
  - 좌상단 변수 모니터 ("내체력")로 직관적으로 표시.
- **승패 조건**:
  - 보스체력 0 → 승리 배너 + 게임상태=2
  - 내체력 0 → 패배 배너 + 게임상태=0
- **탄막 메커니즘** (페이즈별 발사 방식):
  - Stage 가 매 틱 `보스쿨다운`을 −1 카운트. 0 도달 + 게임상태=1 + 페이즈 N 이면, Stage 가 보스 위치/방향을 변수로 set 한 뒤 `보스사격N` 방송.
  - 각 페이즈별 사격 패턴은 보스 스프라이트의 `보스사격N 받으면` 핸들러가 처리:
    - 페이즈 1: 3회 반복 (i=0..2): `보스탄방향 = 보스방향 + (i-1)*20`, 클론 1개 생성
    - 페이즈 2: 5회 반복: `보스탄방향 = 보스방향 + (i-2)*15`, 클론 1개 생성
    - 페이즈 3: 16회 반복: `보스탄방향 = (i * 22.5)`, 클론 1개 생성 (전방향)
  - 보스탄 스프라이트는 단일. 매 클론마다 `보스탄X/보스탄Y/보스탄방향` 변수를 읽어 위치·방향 세팅 후 직선 이동.
- **플레이어 사격**:
  - 스페이스 + 쿨다운=0 + 게임상태=1 → `플총X=self.x, 플총Y=self.y+15, 플총방향=0` 세팅 후 `플사격` 방송. 플쿨다운=11(≈0.18초).
  - 플레이어총알 클론: 매 틱 move 12, 수명 35틱 (보스까지 닿는 시간). 보스에 닿으면 보스체력 −2 (페이즈 진행을 적당히), 클론 삭제. 화면 위로 나가면(y>180) 클론 삭제.
- **격파 시각 효과**:
  - 보스체력 ≤ 0 → 게임상태=2, 보스/체력바 hide.
  - 플레이어 체력 ≤ 0 → 게임상태=0, 플레이어 hide.
- **시작**: 깃발 클릭 → 변수 초기화 → wait 0.5 → broadcast `게임시작` → 게임 진행.

## 6. 스프라이트별 블록 흐름

### Stage

1. **깃발 클릭** → 변수 초기화 (내체력=5, 보스체력=100, 게임상태=1, 플쿨다운=0, 보스쿨다운=30, 현재페이즈=1, 보스방향=180, 보스X=0, 보스Y=110, 보스속도=1.5, 플총X=0, 플총Y=-125, 보스탄X=0, 보스탄Y=95, 보스탄방향=180, 무적=0) → wait 0.5 → broadcast `게임시작`.

2. **forever (페이즈 갱신)**:
   - 페이즈 계산: 보스체력 > 66 → 현재페이즈=1, 보스속도=1.5
   - 보스체력 ≤ 66 AND 보스체력 > 33 → 현재페이즈=2, 보스속도=2.5
   - 보스체력 ≤ 33 → 현재페이즈=3, 보스속도=3.5
   - wait 0.05
   - 구현: if 보스체력>66 → set 1/1.5 (else stack) — 단순 3-way 분기를 chained if 로
3. **forever (보스 사격 쿨다운)**:
   - if 플쿨다운 > 0 → 플쿨다운 −1
   - if 보스쿨다운 > 0 → 보스쿨다운 −1
   - if 무적 > 0 → 무적 −1
   - if 보스쿨다운 = 0 AND 게임상태 = 1: (페이즈별 분기)
     - 보스탄X = 보스X, 보스탄Y = 보스Y − 15 (보스 아래쪽에서 스폰)
     - 현재페이즈 = 1 → broadcast `보스사격1` + 보스쿨다운 = 25
     - 현재페이즈 = 2 → broadcast `보스사격2` + 보스쿨다운 = 20
     - 현재페이즈 = 3 → broadcast `보스사격3` + 보스쿨다운 = 30
   - wait 0.04 (≈25fps)

### 플레이어

1. **깃발 클릭** → 보이기, size 70, 위치 (0, −140), 방향 0(위), 회전스타일=don't rotate, front.
2. **forever (좌우 이동)**:
   - if left arrow pressed AND x position > −210 AND 게임상태=1 → change x by −4
   - if right arrow pressed AND x position < 210 AND 게임상태=1 → change x by 4
   - wait 0.03
3. **forever (스페이스 발사)**:
   - if space AND 플쿨다운=0 AND 게임상태=1:
     - 플총X = self.x, 플총Y = self.y + 15
     - play pop @ pitch 200
     - broadcast `플사격`
     - 플쿨다운 = 11
   - wait 0.04
4. **forever (피격 감시: 보스탄 OR 보스)**:
   - if (touching 보스탄 OR touching 보스) AND 무적=0 AND 게임상태=1:
     - 내체력 −1
     - 무적 = 12 (≈0.5초)
     - play pop @ pitch -200
     - if 내체력 ≤ 0:
       - 게임상태 = 0
       - hide
   - wait 0.05

### 보스

1. **깃발 클릭** → 보이기, size 160, 위치 (0, 110), 방향 90 (시작은 우측 이동), 회전스타일=don't rotate, front.
2. **forever (좌우 왕복)**:
   - if 게임상태=1:
     - 보스X = self.x (탄막 스폰용)
     - 보스Y = self.y
     - move (보스속도) → 일정 속도로 전진
     - if (방향=90 AND x position > 170) → 방향=−90
     - if (방향=−90 AND x position < −170) → 방향=90
   - wait 0.04
3. **forever (피격 감시)**:
   - if touching 플레이어총알 AND 게임상태=1:
     - 보스체력 −2
     - play pop @ pitch -100
     - if 보스체력 ≤ 0:
       - 게임상태 = 2
       - hide
     - wait 0.08 (무적 — 한 번에 여러 번 카운트 방지)
   - wait 0.03
4. **보스사격1 받으면** (페이즈 1: 부채꼴 3발 −20/0/+20):
   - repeat 3 (변수 `i` 사용):
     - 보스탄방향 = 180 + (i − 1) × 20 (i=0 → 160, i=1 → 180, i=2 → 200)
     - play pop @ pitch -50
     - broadcast `없음` 아닌 → 직접: 보스탄 클론 생성을 Stage 대신 보스에서? **단순화: 보스 자신이 매 발 직접 보스탄 클론을 생성하지 않고, 보스가 보스탄방향 변수만 세팅한 뒤 보스탄 스프라이트에게 "스폰" 방송**.

   **재설계**: 발마다 broadcast 가 비효율적이므로, 보스 스프라이트가 직접 보스탄 스프라이트의 클론을 만들지 못함(다른 스프라이트의 클론은 못 만듦) → 그러므로 다음 방식으로:
   - 보스가 `보스사격1` 받으면 → repeat 3 으로 i in 0..2:
     - 보스탄방향 = 180 + (i-1)*20
     - broadcast `보스탄생성` (보스탄 스프라이트가 `보스탄생성` 받으면 → create clone of myself)
     - **문제**: 같은 틱에 3번 broadcast 하면 보스탄 변수가 덮어쓰기. 해결: 한 발씩 작은 wait(0.02) 끼움.

   **최종**: i in 0..2 반복:
   - 보스탄방향 = 180 + (i − 1) * 20
   - broadcast `보스탄생성` (단순화: 페이즈별 broadcast 이름 `보스사격N` 자체가 보스탄 스프라이트의 `클론 생성` 핸들러로도 동시 수신 → 보스탄 스프라이트가 `보스사격1 받으면` 도 처리)

   **확정 구조**: 보스탄 스프라이트가 `보스사격1`, `보스사격2`, `보스사격3` 세 방송 각각에 대해 핸들러를 가지고 직접 N발의 클론을 만든다.
   - 보스 스프라이트는 페이즈별 발사 핸들러를 가지지 않음. 페이즈 분기는 **Stage** 가 하고, 보스탄 스프라이트가 각 방송에 대해 자기 클론을 N개 만드는 핸들러를 가진다.
   - 보스탄 스프라이트의 `보스사격1 받으면`: repeat 3, 매 발 보스탄방향 세팅 + create clone + wait 0.04
   - `보스사격2 받으면`: repeat 5, 매 발 보스탄방향 세팅 + create clone + wait 0.03
   - `보스사격3 받으면`: repeat 16, 매 발 보스탄방향 세팅 + create clone + wait 0.02 (원형 16발)

### 플레이어총알

1. **깃발 클릭** → 숨김, size 80, 회전스타일=don't rotate (위쪽 고정).
2. **plFire 받으면** → create clone of myself.
3. **클론으로 시작**:
   - goto (플총X, 플총Y)
   - point in direction 0 (위)
   - show
   - repeat 35:
     - change y by 12 (직선 위로)
     - if y position > 180 → hide + delete this clone
     - if touching 보스 → hide + delete this clone (보스의 피격 감시가 데미지 처리)
     - wait 0.02
   - hide + delete this clone

### 보스탄

1. **깃발 클릭** → 숨김, size 100, 회전스타일=all around.
2. **보스사격1 받으면** (페이즈 1):
   - repeat 3 (변수 i):
     - 보스탄방향 = 180 + (i − 1) * 20  ← 160/180/200
     - create clone of myself
     - wait 0.04
3. **보스사격2 받으면** (페이즈 2):
   - repeat 5 (변수 i):
     - 보스탄방향 = 180 + (i − 2) * 15  ← 150/165/180/195/210
     - create clone of myself
     - wait 0.04
4. **보스사격3 받으면** (페이즈 3, 원형 16발):
   - repeat 16 (변수 i):
     - 보스탄방향 = i * 22.5
     - create clone of myself
     - wait 0.02
5. **클론으로 시작**:
   - goto (보스탄X, 보스탄Y)
   - point in direction 보스탄방향
   - show
   - repeat 50:
     - move 4 (보스탄 속도)
     - if (|x|>240 OR |y|>180) → hide + delete this clone — 단순화: x>240/x<-240/y>180/y<-180 어느 하나라도 → 삭제
     - if touching 플레이어 → hide + delete this clone
     - wait 0.03
   - hide + delete this clone

**단순화 참고**: i 변수를 클론마다 따로 가질 필요는 없음. 보스탄방향만 매 발 갱신되면 됨. repeat 안에서 변수를 ID `i` 가 아닌 한 변수 `varI17` 로 공용 사용. → **새 변수 추가**.
