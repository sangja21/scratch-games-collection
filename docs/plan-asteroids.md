# 소행성 (asteroids) — Plan

> 우주를 떠다니는 소행성을 우주선으로 격파하는 클래식 액션. 회전·분사로 관성 이동, 스페이스로 발사. 화면 가장자리는 반대편으로 이어진다(wrap). 1979 아타리 Asteroids 의 직관적 재해석.
> 베이스: `games/tank-battle/build.py` (회전+사격+클론 풀) + `games/alien-invasion/build.py` (라운드제·웨이브). **차이점**: 관성(VX/VY 누적), 화면 wrap, 소행성 분열(큰→중→작).
> 학습 콘셉트 없음. 초등학생 대상 직관적 액션.

## 1. 컨셉 / 메커닉

- 톱다운 우주 시점, 480×360 무대.
- **우주선 조작**:
  - 왼쪽/오른쪽 화살표 = 좌/우 회전 (틱당 ±5°)
  - 위 화살표 = 분사. 매 틱 현재 방향으로 가속도 누적: `VX += sin(dir)*0.18`, `VY += cos(dir)*0.18`
  - 스페이스 = 총알 발사 (쿨다운 0.4초)
- **관성**:
  - 우주선은 매 틱 위치를 `x += VX`, `y += VY` 로 갱신. 분사 시에만 가속.
  - 마찰: 매 틱 `VX *= 0.99`, `VY *= 0.99` (매우 약한 감속, 거의 무한 관성)
  - 최대 속도 clamp: `|VX|, |VY| <= 4.5`
- **화면 wrap**: 우주선·총알·소행성 모두 적용.
  - `if x > 240 → x = -240`
  - `if x < -240 → x = 240`
  - `if y > 180 → y = -180`
  - `if y < -180 → y = 180`
- **총알**:
  - 발사 시점 우주선 위치/방향에서 클론 생성 → 직선 이동(8 steps/틱) → 1.2초 후 자동 소멸 OR 소행성에 닿으면 소멸
  - 총알도 화면 wrap (가장자리에서 사라지지 않음)
- **소행성** (`소행성` 스프라이트, 3 코스튬: 큰/중/작):
  - 라운드 시작 시 큰 소행성 N개 (라운드 N = 라운드 + 2, 최대 6) 화면 가장자리 근처 스폰
  - 각 클론은 자기만의 속도 벡터(SX_local, SY_local) → 어렵다. 단순화: **스폰 시점에 stage 변수 `소행X속도`, `소행Y속도` 를 set한 직후 클론이 시작하며 자기 로컬 motion direction + 일정한 move 패턴으로 흉내**
    - 더 단순: 클론 시작 시 random direction 잡고, forever 안에서 `move 1~2 steps`. 즉 각 클론은 직선 일정 속도.
    - 큰 소행성: 1.2 steps/틱
    - 중간 소행성: 1.7 steps/틱
    - 작은 소행성: 2.2 steps/틱
  - 클론이 총알에 닿으면:
    - 큰 → 점수 +20, 중간 2개로 분리 (분리 위치=자기 위치, 분리 broadcast 발생). 큰 본인 소멸.
    - 중간 → 점수 +50, 작은 2개로 분리. 중간 본인 소멸.
    - 작은 → 점수 +100, 그냥 소멸 (분열 없음). 잔여 −1.
  - 클론이 우주선에 닿으면 → 게임상태=0 (게임오버).
- **잔여 카운트**: 폭발 시 줄어드는 건 작은 소행성 (분열은 잔여 증감 없음? plan 단순화: **잔여=화면에 남아 있는 모든 소행성 수**).
  - 스폰 시 +1, 큰→중분리 시 +1 (큰 −1, 중 2개 +2 → 순증 +1), 중→소분리 시 +1, 작은 소멸 시 −1.
  - 잔여=0 → 다음 라운드.
- **라운드제**: 라운드 1 = 큰 소행성 3개, 라운드 2 = 4개, ..., 최대 6개. 라운드↑ 시 소행성 기본 속도 ×1.10.
- **체력 없음. 충돌 1회 → 즉시 게임오버**.

## 6. 스프라이트별 블록 흐름

### Stage

1. **깃발 클릭** → 변수 초기화 → 0.5초 대기 → `게임시작` 방송.
2. **게임시작 받으면** → `라운드시작` 방송.
3. **라운드시작 받으면**:
   - 잔여 = 0
   - 큰 소행성 개수 계산: `if 라운드+2 > 6 → 6 else 라운드+2` (clamp 6)
   - repeat (큰 소행성 개수):
     - 소행X = 랜덤 픽: random(1..2)=1 → random(-220..-100), else random(100..220)
     - 소행Y = 랜덤 픽: random(1..2)=1 → random(-160..-80), else random(80..160)
     - 소행방향 = random(0..359)
     - 소행크기 = 1 (큰)
     - 잔여 += 1
     - `소행성스폰` broadcast
     - wait 0.2
4. **forever (쿨다운 카운터)**:
   - if 쿨다운 > 0 → 쿨다운 −1
   - wait 0.04
5. **forever (라운드 진행 감시)**:
   - if 게임상태=1 AND 잔여=0:
     - wait 1.5
     - 라운드 += 1
     - 속도배수 *= 1.10
     - `라운드시작` broadcast
   - wait 0.2

### 우주선

1. **깃발 클릭** → 보이기, 크기 70, 위치 (0,0), 방향 0, 회전스타일=all around.
2. **forever (조작)**:
   - if left arrow → turn ccw 5
   - if right arrow → turn cw 5
   - if up arrow:
     - 우주선VX = 우주선VX + sin(direction) * 0.18
     - 우주선VY = 우주선VY + cos(direction) * 0.18
   - VX clamp: if VX > 4.5 → 4.5, if VX < -4.5 → -4.5
   - VY 동일
   - 위치 갱신: x = x + VX, y = y + VY
   - 마찰: VX = VX * 0.99, VY = VY * 0.99
   - wrap:
     - if x > 240 → set x to -240
     - if x < -240 → set x to 240
     - if y > 180 → set y to -180
     - if y < -180 → set y to 180
   - wait 0.02
3. **forever (사격)**:
   - if space pressed AND 쿨다운=0 AND 게임상태=1:
     - 총알X = 자기 x, 총알Y = 자기 y, 총알방향 = 자기 direction
     - pop @ pitch 200
     - create clone of 총알
     - 쿨다운 = 10 (≈0.4초)
   - wait 0.05
4. **forever (충돌 감시)**:
   - if (touching 소행성) AND 게임상태=1:
     - 게임상태 = 0
     - pop @ pitch -400
   - wait 0.05

### 총알

1. **깃발 클릭** → 숨김, 크기 60.
2. **클론으로 시작**:
   - 총알X/Y 로 이동, 방향 = 총알방향, 보이기.
   - repeat 60 (≈1.2초):
     - move 8 steps
     - wrap (x>240→-240, x<-240→240, y>180→-180, y<-180→180)
     - if touching 소행성: exit (브레이크 처리)
     - wait 0.02
   - 숨김 + delete this clone.
   - 단순화: repeat N (60) 안에서 `if (touching 소행성) → hide → delete this clone (stop this script)` 추가.

### 소행성

1. **깃발 클릭** → 숨김, 크기 100, 회전스타일=don't rotate.
2. **소행성스폰 받으면** → 클론 생성.
3. **분열중 받으면** → 클론 생성 2회 (큰 사망 자리에 중간 2개 — 클론 시작 시 자기 코스튬 결정).
4. **분열소 받으면** → 클론 생성 2회 (중간 사망 자리에 작은 2개).
5. **클론으로 시작**:
   - 위치: 소행X, 소행Y
   - 코스튬: 소행크기에 따라 분기 — if 소행크기=1 → costume "rock_big", 크기 100; if =2 → "rock_med", 크기 70; if =3 → "rock_small", 크기 45
   - 방향: 소행방향 + random(-30..30) (분열 시 다른 방향으로 흩어지도록)
   - 자기 속도(이동량) 결정: if 소행크기=1 → 1.2*속도배수, =2 → 1.7*속도배수, =3 → 2.2*속도배수
     - 단순화: 클론 로컬 변수가 없으므로 코스튬 이름으로 판단. forever 안에서 매 틱 코스튬 이름 비교.
     - 또는 분열 직후 자기 스프라이트의 "size" 로 판단: 100=큰, 70=중, 45=작
   - 보이기
   - forever:
     - if (touching 총알):
       - 점수 변경: 크기에 따라 큰=+20, 중=+50, 작=+100 (size 로 분기)
       - if size = 100 (큰):
         - 소행X = 자기 x, 소행Y = 자기 y
         - 소행크기 = 2, 소행방향 = 자기 direction + 45
         - broadcast 분열중 (1번째 중간)
         - 소행방향 = 자기 direction - 45
         - broadcast 분열중 (2번째 중간)
         - 잔여 += 1 (큰 -1, 중 2개 +2 → 순증 +1)
       - else if size = 70 (중):
         - 소행X = 자기 x, 소행Y = 자기 y
         - 소행크기 = 3, 소행방향 = 자기 direction + 45
         - broadcast 분열소
         - 소행방향 = 자기 direction - 45
         - broadcast 분열소
         - 잔여 += 1
       - else (작):
         - 잔여 -= 1
       - pop @ pitch (큰=-100, 중=0, 작=100) — 단순화: pitch 0 고정
       - hide + delete this clone + stop this script
     - move (size 별 속도):
       - 단순화: 코스튬에 무관, 클론 시작 직후 forever 들어가기 전에 "이번 클론의 속도" 를 stage 변수 `속도배수`만 가지고 결정해도 무방하나, 클론마다 다른 크기를 가지려면 size 분기 필요.
       - if size=100 → move 1.2 * 속도배수, elif size=70 → move 1.7 * 속도배수, else move 2.2 * 속도배수
     - wrap (x>240→-240 등)
     - wait 0.04
6. ✱ Scratch 의 if/elif 단순화: 3개의 독립적 if (size=100, size=70, size=45) 로 처리.

### 게임오버

- tank-battle 와 동일 패턴: 깃발 → hide + (0,0) + size 100 + front → wait until 게임상태=1 → wait until 게임상태=0 → show.
