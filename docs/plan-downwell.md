# 다운웰 (downwell) — Plan

> 인기 인디게임 **Downwell** 을 초등학생용으로 옮긴 **세로 우물 낙하 슈팅 로그라이트.** 소년(건부츠)을 조종해 끝없이 깊은 우물로 **떨어진다.** 화면은 위로 스크롤되고(=내려감), 좌우로 움직이며 아래의 적·가시를 피한다. **건부츠는 아래로 쏜다** — 아래로 쏘면 낙하가 잠깐 느려지고 살짝 떠오른다(부양/반동). 단 **탄약이 제한**이라 마구 못 쏜다. **땅이나 적을 밟으면(스톰프) 탄약이 가득 충전**된다. 적은 3종 — **밟아 죽이는 점프벌레 / 반드시 쏴야 하는 박쥐 / 밟으면 가시에 찔리니 꼭 쏴야 하는 가시고슴도치.** 땅에 안 닿고 연속으로 처치하면 **콤보 배수**가 붙어 점수가 곱으로 불어난다. 일정 깊이마다 **강화 택1**(탄약+/연사+/부양력+/이동+)로 점점 강해진다. 체력 0 → GAME OVER, 깃발 재클릭으로 능력치 기본값·우물 꼭대기부터. **점수 = 깊이(내려간 거리).**
> 베이스: `games/rogue-knight/build.py`(VY·중력·점프·바닥/발판 착지·클론 스포너·복제됨 가드·체력/무적·폭발 연출·플로팅 데미지 숫자·강화 택1·게임오버 배너·한글 튜닝 변수 일괄 초기화) + `games/magic-survivor/build.py`(좌표 전달 채널·시간/깊이 기반 스폰·**전용 합성 효과음 `synth_*`/`_wav_bytes`**·노란 코멘트 가이드 투어 `add_comment`·강화 택1 패널·데미지 팝업 sprite).
> 학습 콘셉트 없음(보너스 액션 게임 — [[feedback-game-design]] 준수, 데시벨/음파/산성 같은 추상 콘셉트 금지). **존재 이유는 magic-survivor·rogue-knight 과 똑같이 "아이가 코드의 숫자·규칙을 직접 바꾸며 노는 것"** — 모든 조절 가능한 값을 한글 전역 변수로 노출, 코드 어디에도 매직넘버 없음. 부수적으로 **콤보=곱셈**이 점수로 드러난다. ★★★☆☆.

## 1. 게임 한 줄

건부츠 소년이 끝없는 우물로 떨어진다. 화면은 계속 위로 흘러(=내려감) 아래에서 적·가시·보석이 솟아오른다. 좌우로 피하고, **아래로 쏘면** 잠깐 떠올라(체공) 위기를 넘기지만 **탄약이 제한**이라 함부로 못 쏜다 — **땅이나 적을 밟으면 탄약이 가득 찬다.** 점프벌레는 밟아 죽이고, 박쥐와 가시고슴도치는 반드시 쏴야 한다. 땅에 안 닿고 연속 처치하면 **콤보 배수**로 점수가 곱으로 불어난다. 깊이 내려갈수록 강화 택1로 강해지고, 체력 0이면 GAME OVER. **내려간 깊이가 곧 점수.**

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

```
 (-240,180)                                            (240,180)
   ┌──┬────────────────────────────────────────────┬──┐
   │██│ 깊이:０   콤보×０   탄약▮▮▮▮▮▮   ♥♥♥♥        │██│ ← 좌상단 HUD 4종
   │██│                                            │██│   좌우 ██ = 우물 벽(정적, 배경)
   │██│           [소년▼]  ← 카메라선 y=+40        │██│   소년: x만 좌우, y 고정(카메라)
   │██│              ▼탄                            │██│   탄: 아래로 직진
   │██│   [가시]                  [발판══]          │██│   적·발판·보석: 아래서 위로 스크롤
   │██│            [박쥐]   ◆보석                   │██│
   │██│      [점프벌레]                 [가시]      │██│
   │██│   ←우물 안쪽 채널(x: -150~150)→             │██│
   └──┴────────────────────────────────────────────┴──┘
 (-240,-180)  ↑ 적·보석·발판 여기(y≈-210)서 스폰  (240,-180)

  깊이 % 강화깊이 == 0 → 화면 가운데 [강화카드]: 1.탄약+  2.연사+  3.부양력+  4.이동+
                                          (1/2/3/4 키로 택1 → 낙하 재개)
  체력 0 → 화면 가운데 GAME OVER 배너(깊이=점수 표시) → 깃발 재클릭으로 처음부터(능력치 리셋).
```

- **스크롤 모델(택1 결정 — 빌더는 이대로)**: **플레이어 y 고정(카메라) + 월드 스크롤.** 소년은 화면 `카메라선`(Scratch y=+40)에 y 고정, **x 만 좌우**로 움직인다. 낙하는 소년을 내리는 게 아니라 **월드(적·보석·발판 클론)를 매 틱 `낙하속도`만큼 위로 올려** 표현한다(= 내려가는 효과). `낙하속도`가 음수면 월드가 내려가 소년이 우물에서 솟는다(부양·점프). **깊이(점수)는 `낙하속도>0`일 때만 누적.**
- **우물 벽**: 좌우 세로 벽은 배경 SVG 에 **정적**으로 그린다(스크롤 안 함). 소년 x 를 안쪽 채널(±150)로 clamp. (스크롤되는 가로 **발판**은 별도 `발판` 스프라이트.)
- HUD: 좌상단에 `깊이`(점수) `콤보` `탄약` `체력` 모니터 4종. 튜닝 변수(중력·부양력 등)는 **모니터 숨김**(코드에서만 본다).
- 적·보석·발판은 화면 **아래 가장자리(y≈−210)에서 스폰**되어 위로 스크롤되며, 위 가장자리(y>200)를 넘으면 자기 삭제.

> **아트 방향**: Downwell 풍 **저채도 + 한두 색 강조**(우물=짙은 남보라, 강조=노랑). 적 3종은 **각각 다른 코스튬**으로 한눈에 구분 — 점프벌레=둥글고 초록(위가 안전해 보이게 말랑), 박쥐=보라 날개(공중), 가시고슴도치=빨강+위로 솟은 **가시**(밟으면 위험 신호). 모두 `don't rotate`.
> **스크롤 모델(이 게임의 핵심 패턴)**: 소년은 `카메라선`에 y 고정, x 만 움직인다. `낙하속도`(전역, px/틱) = 월드가 위로 스크롤되는 속도. 적·보석·발판 클론은 각자 forever 안에서 `change y by 낙하속도`(양수→위로=하강). 소년 물리 루프가 매 틱 `낙하속도`를 갱신(중력으로 증가, 점프·부양으로 감소). 클론들이 같은 전역값을 읽으므로 카메라가 한 번에 흔들림 없이 따라온다.
> **탄 분리 패턴(rogue-knight 검판정·magic-survivor 마법탄과 동형)**: 데미지 판정을 소년 본체가 아니라 별도 `총알` 스프라이트가 담당. 적은 `touching 총알`만 보면 되고 friendly-fire 분기가 없다.
> **플로팅 데미지 숫자(say 미사용)**: magic-survivor `데미지` 스프라이트 **그대로 복사** — 숫자 코스튬 0~9 를 자릿수별 클론으로 렌더. `총알공격력`을 바꾸면 통! 뜨는 숫자도 따라 바뀐다.

## 4. 사운드 (전용 합성 — `build.py`에서 결정적 생성)

magic-survivor 처럼 `_wav_bytes(samples, rate)` + `synth_*()` 로 WAV 를 **코드에서 합성**한다(에셋 복사 아님). 이벤트별로 **구분되는 음색**을 만들고, 같은 계열은 pitch 이펙트로 변주. 모든 합성은 고정 시드/수식 → 결정적.

| 합성 함수 | 음색(스펙) | 쓰이는 이벤트 | pitch 변주 |
|-----------|-----------|---------------|------------|
| `synth_zap()` | 짧은 하강 처프 "퓽"(레이저, 0.10s, 빠른 감쇠) — magic-survivor `synth_zap` 재사용 | **아래 발사** | 0 |
| `synth_boom()` | 노이즈 버스트+저음 thump "펑"(폭발, 0.30s) — magic-survivor `synth_boom` 재사용 | **적 처치** | 0 |
| `synth_stomp()` | **신규.** 둔탁한 저음 thud + 짧은 상승 블립(0.14s) — "쿵! 통" 밟는 손맛 | **스톰프/착지(탄약충전)** | 0 |
| `synth_combo()` | **신규.** 맑은 사인 핑(0.08s) — 콤보 올라갈수록 pitch 올려 음이 점점 높아지게 | **공중 연속 처치(콤보+1)** | `콤보 × 25` (상한 ~300) |
| `synth_coin()` | **신규.** 2음 코인 블립(0.10s, 상승 2단) | **보석 획득** | 0 |
| `synth_hurt()` | **신규.** 짧은 저음 buzz(0.16s) — 아픈 느낌 | **소년 피격(체력−1)** | 0 |
| `synth_levelup()` | 밝은 상승 아르페지오(0.25s) | **강화 선택** | 0 |

> 합성 함수는 6~7개. `synth_stomp`/`synth_combo`/`synth_coin`/`synth_hurt`/`synth_levelup` 5개는 신규 작성, `synth_zap`/`synth_boom` 2개는 magic-survivor 에서 그대로 가져온다. **콤보음만 pitch 를 `콤보` 값으로 변조**해 "점점 높아지는 소리 = 점수가 곱으로 커지는 직관"을 귀로 전달.

## 5. 변수 / 리스트 / 메시지

### 5.1 ⚙️ 개조 손잡이 (튜닝 변수) — 한글 전역 변수, ID `V_*` / `var*`

> **이 표가 이 게임의 심장이다.** 모든 조절 가능한 값은 여기 한글 변수로만 존재하고, 코드 어디서도 매직넘버를 쓰지 않는다(예: `change y by 낙하속도` 이지 `change y by 6` 아님). 변수 초기화는 **전부 Stage 깃발 클릭 한 스크립트**에 모은다. (무적쿨 카운트, `repeat 5` 폭발 같은 연출용 소수 상수만 인라인 — magic-survivor 와 동일 기준.)

```

   ├ 소년: 접지여유 −1 → ←→ 좌우 이동(clamp ±150) /

   │       발판/적 위에서 닿고 낙하속도>0 → 스톰프/착지 처리(아래) /

   ├ 적 스포너: 유효스폰간격마다 아래(y≈−210) 1마리(단계로 종류 결정), 적수 관리
   ├ 적 클론: change y by 낙하속도(스크롤) + 좌우 sin 흔들 + 접촉 데미지 +
   │          touching 총알 → 내체력 −총알공격력 / 0이하면 폭발(보석 드롭)+데미지 팝업+공중이면 콤보+1
   ├ 보석 클론: change y by 낙하속도(스크롤), 소년에 닿으면 깊이 += 내값(코인음)
   ├ 발판 클론: change y by 낙하속도(스크롤), 위 가장자리 넘으면 삭제
   └ Stage: 단계 = floor(깊이/난이도깊이) ; 유효스폰간격 = clamp(스폰간격 − 단계×스폰감소)
   ▼
[스톰프/착지]  (소년 물리 루프 안)
   │  ● 적 스톰프(스톰프가능=1 & 위에서 & 낙하속도>0):
   │     → 그 적 처치(보석·폭발·데미지팝업) ; 콤보+1 ; 점수 += 처치점수×콤보 ;
   │       낙하속도 = −착지반동(통! 튐) ; 탄약=탄약최대 ; 접지여유=코요테 ; "쿵·통"
   │  ● 발판 착지(위에서 & 낙하속도>0):
   │     → 낙하속도 = 0 ; 탄약=탄약최대 ; 접지여유=코요테 ; **콤보 = 0(리셋)** ; "쿵"
   │  ● 가시/박쥐를 위에서 밟음(스톰프가능=0):
   │     → 스톰프 처치 안 됨. 일반 접촉 데미지 분기로 떨어져 체력−1(무적)
   ▼
[강화선택중]  (게임상태=2)   ← 낙하·스폰·물리 모두 일시정지(각 루프가 게임상태=1만 처리)
   │  강화카드 show. 1/2/3/4 키:
   │   1 → 탄약최대 += 강화량 ; 탄약 = 탄약최대
   │   2 → 연사간격 −= (0.02 × 강화량)   (최소 0.04 클램프)
   │   3 → 부양력 += (0.3 × 강화량)
   │   4 → 이동속도 += 강화량
   │  다음강화깊이 += 강화깊이 ; "레벨업음"
   │  선택 후 [강화완료] → 게임상태=1
   ▼  (낙하중 루프로 복귀)
[종료]  체력 ≤ 0 → 게임상태=0
   │  게임오버 배너 show(깊이 표시) / 잔여 적·보석·발판·탄 클론 자기 삭제
   ▼
[다시 시작]  초록 깃발 재클릭(능력치 기본값으로 — 깊이 점수 리셋)
```

> **빌더 주의(스크롤 부호)**: `낙하속도` 양수 = 하강(월드 위로). 월드 클론은 전부 `change y by 낙하속도`. 깊이는 `if 낙하속도>0`일 때만 누적(부양 중엔 점수 안 줄게). 점프·부양은 `낙하속도`를 음수로 만들어 월드가 내려가 소년이 우물에서 솟는 연출.
> **콤보 분기(중요)**: **스톰프 처치·공중 사격 처치는 콤보 유지/증가**, **발판 착지만 콤보 리셋.** 이게 "땅에 안 닿고 통통 튀며 연속 처치 = 곱 점수" 라는 Downwell 콤보의 핵심.

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait` 넣어 CPU 폭주 방지. 물리·스크롤 틱은 0.025초(≈40fps). **매직넘버 금지** — 항상 튜닝 변수 참조.

### Stage

**(A) 깃발 클릭 → 변수 60개 전부 초기화(한 곳에 모음) → 게임시작**

when green flag clicked
  # ── 개조 손잡이(튜닝 32): 아이가 여기 숫자만 바꾸면 게임이 바뀐다 ──
  set 이동속도=5 ; set 중력=0.55 ; set 최대낙하=9 ; set 점프력=7
  set 부양력=1.6 ; set 부양상한=5 ; set 카메라선=40 ; set 코요테=6 ; set 착지반동=1.5
  set 탄약최대=6 ; set 연사간격=0.12 ; set 총알속도=14 ; set 총알공격력=1
  set 최대체력=4 ; set 무적시간=30 ; set 처치점수=5
  set 점프벌레_체력=1 ; set 점프벌레_속도=0.8 ; set 점프벌레_보석=1
  set 박쥐_체력=2 ; set 박쥐_속도=1.6 ; set 박쥐_보석=2
  set 가시_체력=3 ; set 가시_속도=0.4 ; set 가시_보석=3
  set 스폰간격=1.0 ; set 난이도깊이=900 ; set 스폰감소=0.07 ; set 스폰간격최소=0.35
  set 발판간격=1.6 ; set 강화깊이=1000 ; set 강화량=1
  # ── 진행 상태(28) ──
  set 게임상태=1 ; set 깊이=0 ; set 콤보=0 ; set 콤보최고=0

  set 데미지표시값=0 ; set 데미지표시x=0 ; set 데미지표시y=0
  set 데미지숫자=0 ; set 데미지오프셋=0 ; set 데미지글자수=0 ; set 데미지자리=0
  wait 0.3
  broadcast 게임시작
```

**(B) 단계/유효스폰간격 계산 forever**
```
when I receive 게임시작
  forever
    if (게임상태 = 1)
        set 단계 = floor(깊이 / 난이도깊이)
        set 유효스폰간격 = (스폰간격 - (단계 * 스폰감소))
        if (유효스폰간격 < 스폰간격최소) : set 유효스폰간격 = 스폰간격최소
    wait 0.1
```

**(C) 무적 타이머 감소 forever**
```
when green flag clicked
  forever
    if 무적 > 0 : change 무적 by -1
    wait 0.025
```

**(D) 강화 / 게임오버 감시 forever**
```
when green flag clicked
  wait until (게임상태 = 1)
  forever
    if (깊이 >= 다음강화깊이) and (게임상태 = 1)
        set 게임상태 = 2
        broadcast 강화
    if (체력 < 1) and (게임상태 = 1)
        set 게임상태 = 0          ← 게임오버
    wait 0.05
```
> 콤보최고 갱신은 소년 처치 처리에서(`if 콤보 > 콤보최고 : set 콤보최고 = 콤보`). 강화 처리(다음강화깊이 += 강화깊이)는 강화카드 스크립트에서.

### 소년 (카메라 + 물리 + 사격 + 스톰프)

**(A) 깃발 초기화**
```
when green flag clicked
  show ; set size 65 ; set rotation style [don't rotate]
  point in direction 90 ; go to front
  set y to 카메라선 ; set x to 0 ; set [ghost] effect to 0
```

**(B) 메인 물리 forever**  (이 게임의 핵심)
```
when I receive 게임시작
  forever
    if (게임상태 = 1)
        if 접지여유 > 0 : change 접지여유 by -1
        # ── 좌우 이동 (x만, y는 카메라선 고정) ──
        if key [right arrow] : change x by 이동속도
        if key [left arrow]  : change x by (0 - 이동속도)
        clamp x to ±150
        set y to 카메라선                  ← 항상 카메라선 고정
        # ── 점프(접지 여유 동안) ──
        if (key [space] or key [up arrow]) and (접지여유 > 0)
            set 낙하속도 = (0 - 점프력) ; set 접지여유 = 0
        # ── 중력 ──
        change 낙하속도 by 중력
        if (낙하속도 > 최대낙하) : set 낙하속도 = 최대낙하
        # ── 깊이 누적(하강할 때만) ──
        if (낙하속도 > 0) : change 깊이 by 낙하속도
        # ── 스톰프/착지 판정 (아래 (C)에서 broadcast 없이 인라인) ──
        ...스톰프·착지·피격 판정...
    wait 0.025
```
> 빌더 노트: 소년은 클론 없음. `set y to 카메라선`을 루프 안에서 매 틱 강제해 카메라 고정. 좌우 clamp 는 rogue-knight `clamp` 헬퍼 재사용(±150).

**(C) 스톰프·착지·피격 판정** (위 forever 안, 게임상태=1 블록 끝)
```
        # 1) 적 스톰프(밟기): 위에서 닿고 낙하 중 + 그 적이 스톰프가능
        #    → 적 클론이 'touching 소년 & 소년이 위'를 직접 판정해 처치하는 게 간단(아래 적 (C)-4).
        #    소년은 스톰프 성공 시 적이 세팅하는 신호로 튕김. (구현: 적 클론이 스톰프 처치 때
        #    낙하속도=−착지반동, 탄약=탄약최대, 접지여유=코요테 를 세팅 → 소년은 별도 처리 불필요)
        # 2) 발판 착지: 소년이 발판에 닿고 낙하 중
        if (touching [발판]) and (낙하속도 > 0)
            set 낙하속도 = 0
            set 탄약 = 탄약최대
            set 접지여유 = 코요테
            set 콤보 = 0                    ← 발판 착지만 콤보 리셋!
            play sound stomp(쿵)
        # 3) 피격: 적에 닿고 무적 아님 (스톰프로 처리 안 된 접촉)
        if (touching [적]) and (무적 = 0)
            change 체력 by -1
            set 무적 = 무적시간
            play sound hurt
```
> 빌더 노트(스톰프 위치 판정): "위에서 밟음"은 **적 클론 쪽에서** 판정하는 게 깔끔하다(소년 y는 카메라선 고정이라 적 클론의 y 가 소년보다 충분히 아래일 때 = 소년이 위). 적 (C)-4 참조. 소년 (C)-3 의 일반 접촉 피격은 박쥐·가시를 위에서 밟거나 옆/아래로 닿았을 때 발동(스톰프 실패 = 데미지). 스톰프 성공 적은 그 틱에 즉시 삭제되므로 (C)-3 의 `touching 적`에 안 걸린다.

**(D) 아래로 발사 forever**
```
when I receive 게임시작
  forever
    if (게임상태 = 1)
        if (key [down arrow] or key [z]) and (탄약 > 0) and (발사쿨 = 0)
            set 발사X = x position
            broadcast 발사                      ← 총알 원본이 그 자리에 탄 클론 생성
            change 탄약 by -1
            change 낙하속도 by (0 - 부양력)      ← 반동: 낙하 느려지고 떠오름
            if (낙하속도 < (0 - 부양상한)) : set 낙하속도 = (0 - 부양상한)
            set 발사쿨 = 연사간격                ← (틱이 아니라 초 → 아래 감소 루프는 시간 기반)
            play sound zap(퓽)
    wait 0.01
# 발사쿨 시간 감소(별도 forever, 0.01초마다 0.01 줄임 → 연사간격(초) 그대로 사용)
when I receive 게임시작
  forever
    if 발사쿨 > 0 : change 발사쿨 by -0.01
    wait 0.01
```
> 빌더 노트: 발사키는 ↓ 또는 z(시후가 막 누르기 좋게 둘 다). 탄약 0이면 발사·부양 불가 → "땅/적 밟아 충전" 동기. 부양은 발사할 때만 들어가므로 **연사 = 체공**이 자연스럽게 성립(magic-survivor 자동조준과 달리 여기선 수동 난사가 곧 재미).

### 적 (스포너 + 클론 본체)

**(A) 깃발 초기화**
```
when green flag clicked
  hide ; set rotation style [don't rotate] ; set 복제됨 = 0
```

**(B) 스폰 forever → 아래에서 1마리씩** (원본만)
```
when I receive 게임시작
  if (복제됨 = 0)
      forever
          if (게임상태 = 1)
              set 적생성X = pick random -140 to 140
              # ── 단계로 종류 결정(점프벌레 → +박쥐 → 박쥐·가시) ──
              if (단계 = 0)
                  set 적생성종류 = 1
              else if (단계 = 1)
                  set 적생성종류 = (1 + (스폰카운트 mod 2))     ← 1·2 번갈아
              else
                  set 적생성종류 = (2 + (pick random 0 to 1))   ← 2·3 섞임

              create clone of [myself]

**(C) 클론 본체**
```
when I start as a clone
  set 복제됨 = 1
  set 적종류 = 적생성종류
  set 흔들위상 = pick random 0 to 100
  # ── 종류별 능력치·외형·스톰프가능(매직넘버 없음) ──
  if (적종류 = 1)               ← 점프벌레(약, 밟기 가능)
      set 내체력 = 점프벌레_체력 ; set 내속도 = 점프벌레_속도
      set 스톰프가능 = 1 ; switch costume to [점프벌레] ; set size 50
  else if (적종류 = 2)          ← 박쥐(중, 쏘기)
      set 내체력 = 박쥐_체력 ; set 내속도 = 박쥐_속도
      set 스톰프가능 = 0 ; switch costume to [박쥐] ; set size 55
  else                          ← 가시고슴도치(강, 밟기 위험)
      set 내체력 = 가시_체력 ; set 내속도 = 가시_속도
      set 스톰프가능 = 0 ; switch costume to [가시고슴도치] ; set size 55
  set 피격쿨 = 0
  go to x:(적생성X) y:(-210) ; show          ← 화면 아래에서 스폰
  forever
    # 0) 게임오버 정리
    if (게임상태 = 0) : change 적수 by -1 ; delete this clone
    if (게임상태 = 1)
        # 1) 스크롤(위로) + 좌우 흔들
        change y by 낙하속도
        change 흔들위상 by 4
        change x by ((sin of 흔들위상) * (내속도 / 10))     ← 살랑살랑 흔들
        if (y position > 200) : change 적수 by -1 ; delete this clone   ← 위로 벗어남
        # 2) 스톰프 판정: 스톰프가능 & 소년이 닿고 소년이 위(소년 y > 내 y + 여유) & 낙하속도>0
        if (스톰프가능 = 1) and (touching [소년]) and ((소년 y) > (y position + 10)) and (낙하속도 > 0)
            # 처치 + 콤보(공중) + 소년 튕김 + 탄약충전
            broadcast 보석생성 채널 세팅(보석X/Y=내위치, 보석값=점프벌레_보석)
            change 콤보 by 1 ; if 콤보 > 콤보최고 : set 콤보최고 = 콤보
            change 깊이 by (처치점수 * 콤보)         ← ✖️ 콤보 곱 점수!
            set 낙하속도 = (0 - 착지반동) ; set 탄약 = 탄약최대 ; set 접지여유 = 코요테
            play sound combo (pitch = 콤보*25)
            처치연출(폭발 코스튬·size/ghost repeat) → change 적수 by -1 → delete this clone
        # 3) 총알 피격(중복 방지 쿨)
        if (touching [총알]) and (피격쿨 = 0)
            change 내체력 by (0 - 총알공격력)
            set 피격쿨 = 6
            set 데미지표시값 = 총알공격력
            set 데미지표시x = x position ; set 데미지표시y = y position
            broadcast 데미지표시
            play sound (적 피격: zap pitch -200 또는 짧은 tick)
        if 피격쿨 > 0 : change 피격쿨 by -1
        # 4) 처치(체력 0 — 총알로 죽음): 보석 드롭 + 공중 콤보 + 폭발
        if (내체력 < 1)
            set 보석X = x position ; set 보석Y = y position
            if (적종류 = 1) : set 보석값 = 점프벌레_보석
            else if (적종류 = 2) : set 보석값 = 박쥐_보석
            else : set 보석값 = 가시_보석
            broadcast 보석생성
            change 콤보 by 1 ; if 콤보 > 콤보최고 : set 콤보최고 = 콤보
            change 깊이 by (처치점수 * 콤보)         ← ✖️ 콤보 곱 점수!
            play sound combo (pitch = 콤보*25)
            change 적수 by -1
            switch costume to [폭발] ; play sound boom
            repeat 5 : change size by 12 ; change [ghost] effect by 20 ; wait 0.02
            delete this clone
    wait 0.025
```
> 빌더 노트: "소년 y"는 `[소년 v] 의 y좌표` (sensing_of) 또는 소년이 카메라선 고정이므로 `카메라선`을 직접 써도 된다(`(카메라선) > (y position + 10)`) — 후자가 더 간단·안정. 스톰프 처치와 총알 처치 둘 다 **콤보+1·콤보곱 점수**(공중 연속 처치). 발판 착지만 콤보 0(소년 (C)-2). `sin of` 흔들은 박쥐·가시도 좌우로 살랑이게 해 정적이지 않게.

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식·조건 1~2층", "새 메커닉 4층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화 스크립트만 건드린다. 망가져도 초록 깃발만 다시 누르면 됨.

### 0층 — 숫자 하나 바꾸기 (무대 초록 깃발 스크립트)
- **둥둥 우주인**: `중력` 을 `0.55` → `0.2` 로. 천천히 떨어져 한참 떠 있다.
- **번개 추락**: `최대낙하` 를 `9` → `16` 으로. 쏜살같이 떨어진다(점수 빨리 쌓이지만 위험).
- **기관총 건부츠**: `연사간격` 을 `0.12` → `0.04` 로. 아래로 따발총처럼 쏟아지며 둥실 떠오른다.
- **탄창 가득**: `탄약최대` 를 `6` → `20` 으로. 공중에서 한참 난사하며 체공.
- **세진 총알**: `총알공격력` 을 `1` → `3` 으로. **적을 때려서 뜨는 데미지 숫자가 `1`→`3` 으로 바뀌는지** 직접 확인!
- **부양 슈퍼**: `부양력`·`부양상한` 을 키워 한 발 쏠 때마다 훅 떠오르게.
- **두부 멘탈**: `최대체력` 을 `4` → `1` 로. 한 대만 맞아도 위험!
- **콤보 대박**: `처치점수` 를 `5` → `20` 으로. 콤보 ×N 의 점수 폭발이 더 짜릿해진다.
- **지옥문**: `스폰간격` 을 `1.0` → `0.4` 로. 적이 두세 배로 솟아오른다.

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **쏠수록 강한 반동**: 소년 (D) 발사에서 `change 낙하속도 by (0 - 부양력)` 을 `change 낙하속도 by (0 - (부양력 + (탄약 / 5)))` 로 — 탄약이 많을수록 더 세게 떠오르게.
- **깊을수록 빠른 적**: 적 (C)-1 흔들에서 `내속도` 대신 `(내속도 + (단계 / 3))` 로 — 깊이 갈수록 적이 더 흔들·빠르게.
- **콤보가 곧 화력**: 적 (C)-3 총알 피격에서 `change 내체력 by (0 - 총알공격력)` 을 `change 내체력 by (0 - (총알공격력 + (콤보 / 3)))` 로 — 콤보가 높을수록 총알도 세진다.

### 2층 — if 조건 더하기
- **막판 회복**: 소년 (C)-2 발판 착지에 `if 체력 < 최대체력 : change 체력 by 1` 추가 — 착지마다 체력 1 회복(쉬운 모드).
- **콤보 무적**: 적 처치(콤보+1) 직후 `if 콤보 >= 5 : set 무적 = 무적시간` 추가 — 5콤보 넘으면 잠깐 무적으로 폭주.
- **바닥 가시 함정**: 적 (C)-1 에 `if (단계 >= 3) and (적종류 = 1) : set 스톰프가능 = 0` 을 넣어, 후반엔 점프벌레도 못 밟게(난이도↑).

### 3층 — 규칙 자체를 바꾸기 (분기)
- **강화 5번 추가(점프력+)**: 강화카드에 `if key [5] : change 점프력 by 강화량` 선택지를 더하고 카드 그림에 5번 칸을 그린다.
- **강화 6번 추가(총알공격력+/최대체력+)**: `if key [6] : change 총알공격력 by 강화량`(또는 `change 최대체력 by 강화량 ; change 체력 by 강화량`) 추가.
- **랜덤 강화**: 강화 선택을 키 입력 대신 `pick random 1 to 4` 로 자동 결정하게 바꿔 운에 맡기기.

### 4층 — 새 메커닉 만들기 (구조)
- **3연발 산탄 건부츠**: 총알 (B)를 `repeat 3 : create clone` 으로 바꾸고, 클론마다 x 를 조금씩 벌려(`-12/0/+12`) 아래로 부채꼴 산탄.
- **회복 보석 추가**: `보석` 을 복제해 `체력보석` 스프라이트를 만들고, 적 처치 시 가끔(`pick random 1 to 5 = 1`) 떨어뜨려 닿으면 `change 체력 by 1`.
- **분열 점프벌레**: 적 처치 시 `if 적종류 = 1 : (작은 점프벌레 2마리 스폰)` 으로 약한 적이 죽으며 둘로 쪼개지게(스폰 채널 재사용).
- **콤보 화면연출**: 콤보가 오를 때 `데미지표시값` 에 `콤보` 를 넣어 큰 숫자를 띄우거나, 콤보음 pitch 식(`콤보 × 25`)을 바꿔 더 짜릿하게.

## 10. 학습 포인트

보너스 액션 게임 — 수학 학습 콘셉트 없음([[feedback-game-design]] 준수: 데시벨/음파/산성 같은 추상 콘셉트 금지). **핵심 가치는 magic-survivor·rogue-knight 과 똑같이 "코딩 학습"**: 모든 능력치가 한글 튜닝 변수(60개 중 32개가 개조 손잡이)로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (4층) 새 스프라이트를 추가하며 **변수·연산·조건·클론·방송·좌표/스크롤** 이라는 프로그래밍 개념을 게임을 망가뜨리고 고치며 체득한다. 코드 곳곳의 **노란 코멘트(가이드 투어)** 로 각 부분이 무엇을 하는지 바로 읽을 수 있다.

부수적으로 액션의 *결과로* 수학이 슬쩍 드러난다:
- **✖️ 곱셈(콤보 배수)** — 땅에 안 닿고 연속 처치하면 `콤보`가 1씩 늘고 점수 = `처치점수 × 콤보`. 콤보가 오를수록 같은 적인데 점수가 **곱으로** 불어나고, 콤보음 pitch 도 같이 높아진다 → "곱하기는 더하기를 빠르게 한 것"을 손과 귀로 만난다.
- **🧭 좌표·부호(스크롤)** — `낙하속도` 양수=하강/음수=부양. 같은 변수의 부호 하나로 화면 전체가 위로 또는 아래로 흐르는 걸 보며 양수·음수와 방향을 체감.
- 그 밖에 **누적**(깊이=낙하속도 합), **확률적 스폰**(난수 x·종류), **점점 세지는 난이도 곡선**(단계 = 깊이÷난이도깊이).
