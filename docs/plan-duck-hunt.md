# 오리 사냥 (duck-hunt) — Plan

> 1984 닌텐도 Duck Hunt 클래식 클론.
> 화면 위를 날아가는 오리를 십자선으로 조준해 사격한다. 탄약 한정 (한 라운드당 3발), 라운드별 오리 N마리.
> 베이스: `games/missile-command/build.py` (마우스 추적 십자선 + 클릭 → 클론 폭발/이펙트 + 라운드 진행 + 게임오버 배너).
> 학습 콘셉트 절대 없음. 순수 액션 (조준 정확도 + 타이밍).

## 1. 컨셉 / 메커닉

- 화면 하단 1/3 은 풀밭/덤불 실루엣 (오리가 지나갈 수 없는 사각지대).
- 상단 2/3 (하늘) 에서 오리가 좌→우 또는 우→좌 로 비스듬히 비행. 가끔 위/아래 흔들림.
- 플레이어는 마우스로 십자선을 움직이고 **클릭** 시 사격.
- 사격 효과: 클릭 위치에 짧은 노란 섬광 (muzzle flash 클론) 이 0.15초 표시 → 자동 삭제.
- 오리에 명중(섬광이 오리에 닿음) → 오리는 "맞음" 코스튬으로 잠시 → 떨어지며 사라짐 → 점수+100.
- 한 마리당 탄약 3발. 3발 다 빗나가면 오리는 화면 밖으로 도망 → 라이프 -1.
- 라이프 3개. 0 되면 게임오버.
- 라운드 = 오리 N마리(라운드 1 = 6마리, 라운드↑ → 마리 +2, 속도×1.15).

## 2. 화면 레이아웃 (Scratch 480×360)

```
y=180   ───────────────── 하늘 (그라데이션 푸른색→연주황) ─────────────────
                                                  ← 오리 비행 영역 (-220..220, 30..160)
                                                     좌우 진입 → 반대편 탈출

y=0     · · · · · 구름 몇 개 · · · · ·

y=-60   ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒  ← 풀밭/덤불 시작 (실루엣)
y=-180  ░░░░ 나무 실루엣 ░░░░░░░░░░░░░░░░░░░  ← 바닥 (오리는 여기 아래로 안 옴)
```

스코어/탄약/라이프/라운드는 화면 좌상단 변수 모니터로 표시.

> 오리 한 마리는 **단일 스프라이트의 메인 스크립트** 로 처리. 클론을 쓰지 않고 한 번에 한 마리만 무대에 등장 → 처리 후 다음 오리 `오리등장` 방송. 단순화·디버그 용이.

## 6. 스프라이트별 블록 흐름

### Stage
1. **깃발 클릭** → 모든 변수 초기화 → 0.5초 대기 → `게임시작` 방송.
2. **게임시작 받으면** → `오리등장` 방송 (첫 오리).
3. **다음라운드 받으면** → 라운드 +1, 오리속도 ×1.15, 라운드오리수 +2, 잡은수=0 → 1.5초 대기 → `오리등장` 방송.

### 조준선 (crosshair)
1. **깃발 클릭** → 크기 80, 맨 앞으로, 보이기, forever: 마우스 X/Y 로 이동.
2. **이 스프라이트 클릭** → if 게임상태=1 and 탄약>0:
   - 탄약 -1
   - 섬광X=mouseX, 섬광Y=mouseY
   - "섬광" 클론 생성
   - pop 사운드 (피치 200)

### 섬광 (flash)
1. **깃발 클릭** → 숨김, 크기 65.
2. **클론으로 시작** → 섬광X/Y 로 이동 → 보이기 → 0.12초 대기 → 숨김 → 클론 삭제.

> 오리는 자기 forever 루프에서 `touching 섬광?` 으로 명중 체크 (clone 동안 닿으면 명중).

### 오리 (duck)
1. **깃발 클릭** → 숨김, 크기 90, 코스튬=fly1.
2. **오리등장 받으면**:
   - 오리상태=0, 탄약=3
   - 오리방향 = (random 1..2 = 1 이면 1, else -1) → 시작 X = -220 또는 220 (반대편 끝)
   - 시작 Y = random 60..160
   - 회전 스타일 = 좌우 (left-right)
   - 90° (오른쪽 가는 경우) 또는 -90° (왼쪽) 방향 설정
   - 보이기, 코스튬=fly1
   - **메인 루프** `repeat until 오리상태≠0`:
     - 코스튬 = (fly1↔fly2 토글, 0.18초 간격 → 단순화: 매 5틱마다 next costume 호출)
     - x by (오리속도 × 오리방향) — 단, 매 8틱 한 번 y 도 ±sin(시간) 흔들림 → 단순화: `change y by (random -3..3)`
     - if `touching 섬광` → 오리상태=1, 점수+100, 잡은수+1, `명중` 방송, 코스튬=hit
     - if x position > 230 OR x position < -230 → 오리상태=2 (도망)
     - wait 0.03
   - **분기 처리**:
     - if 오리상태=1: 코스튬=hit 유지, 0.2초 대기 → repeat 15회 (change y by -10, wait 0.03) → 숨김
     - if 오리상태=2: 숨김, 라이프 -1
   - **공통 후처리**:
     - 잡은수 + 도망수(= 처리완료 오리 수)가 라운드오리수에 도달했는지 검사. 단순화: 잡은수 + (라이프 감소량 누적) 으로 카운트하기 복잡 → 별도 변수 `처리완료수` 를 두지 않고, 그냥 매 오리마다 0.6초 대기 후 :
       - if (잡은수 + (라운드오리수 - 잡은수 - 남은오리수)) ≥ 라운드오리수 → 다음라운드
     - **더 단순한 방식 채택**: Stage 에 변수 `처리완료` 두지 말고, 오리 스프라이트 로컬에 `발사횟수` 카운터 만들지도 말고 → **각 오리 종료 시점에 잡은수 + 도망수 비교**.
     - 변수 추가: `varDone13 = 라운드처리완료` (Stage), 매 오리 종료 시 +1.
     - if 라운드처리완료 ≥ 라운드오리수 → 라운드처리완료=0 → `다음라운드` 방송
     - else → 0.7초 대기 → `오리등장` 방송 (다음 오리)
     - if 라이프 ≤ 0 → 게임상태=0

### 게임오버
- missile-command 와 동일 패턴: 깃발→숨김→게임상태=1까지 대기→게임상태=0까지 대기→show.
