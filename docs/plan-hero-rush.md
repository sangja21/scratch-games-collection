# 용사 진격전 (hero-rush) — Plan

> **"내가 조종하는 용사 1명 vs 몰려오는 적 군단"** — Scratch 가 확실히 감당하는 성공작 구조(rogue-knight/magic-survivor/castle-defense 계보). 전쟁 판타지를 뒤집어, 군대를 소환하는 대신 **용사 한 명이 적 병사들을 베며 오른쪽 적 성으로 진격**한다. 화살표로 이동, 스페이스(또는 마우스 클릭)로 **검격**, X 로 **광역 스킬**(쿨다운 게이지). 용사가 앞으로 밀고 나가면 `진격도`가 차오르고, 적 잡졸이 오른쪽에서 **웨이브로 몰려온다**(밀리면 뒤로 후퇴하며 싸우는 전선 감각). `진격도`가 가득 차면 **적 성문이 밀려 들어오고 → 성문을 부수는 공성 피날레 → 스테이지 클리어 → 강화 택1**(공격력/체력/속도/스킬) → 다음 스테이지는 **지수 스케일**(`적배율 = 적성장배율^(스테이지−1)`). 용사 체력이 0이면 GAME OVER(로그라이트, 처음부터). **점수 = 도달한 스테이지.**
> 베이스: `games/rogue-knight/build.py`(좌우 이동·VY/중력/점프·검판정 히트박스 분리·클론 스포너·복제됨 가드·폭발 연출·강화 택1·플로팅 데미지 숫자·게임오버 배너) + `games/castle-defense/build.py`(지수 스케일·전용 합성 효과음 `_wav_bytes`·`synth_*`·`add_comment` 가이드 투어).
> **chess-war 실패 교훈 반영**: 다대다 유닛 자율 전투는 Scratch 한계 밖("최전방 절충"이 멀뚱멀뚱하게 보임)이라 폐기. hero-rush 는 **적 클론 로직을 극단적으로 단순화**(용사를 향해 전진 + 접촉 데미지, 히트박스에 맞으면 죽음 — **클론끼리 상호작용 전혀 없음**)해 그 함정을 원천 차단한다. **템포는 기본값부터 빠르게**(약졸 1~2타 처치·용사 이동 시원·한 스테이지 30~60초), **모든 공격이 눈에 보이게**(검격 휘두름 모션+반달 이펙트·스킬 회전베기 링·처치 폭발·데미지 숫자), **쿨/자원은 UI로**(체력바·스킬 쿨 게이지·진격 진행바 상시 표시).
> 학습 콘셉트 없음(직관적 액션 게임). **진짜 가치는 "아이가 코드의 숫자·규칙을 직접 바꾸며 노는 것"** — 모든 조절 값을 한글 전역 변수로 노출, 매직넘버 0. ★★★★.

- **주제**: 1인 용사 진격형 횡스크롤 액션 로그라이트 (하얀 용사 vs 검은 군단·공성 피날레)
- **카테고리**: 액션
- **난이도**: ★★★★☆
- **폴더**: `games/hero-rush/`

## 1. 게임 한 줄

용사 한 명으로 오른쪽 적 성을 향해 진격한다. 앞으로 밀고 나가면 진격도가 차오르고, 적 병사들이 웨이브로 몰려온다. 스페이스로 검을 휘둘러 베고(잡졸은 1~2타), X 로 광역 스킬(쿨다운 게이지가 다 차야 발동)을 쓴다. 밀리면 왼쪽으로 후퇴하며 싸울 수도 있다. 진격도가 가득 차면 적 성문이 밀려 들어오고, 성문을 부수면 스테이지 클리어 → 강화 셋 중 하나를 골라 강해진 채 다음 스테이지로. 스테이지가 오를수록 적이 곱하기로 세진다(지수). 용사 체력이 0이면 1스테이지부터 다시(도달 스테이지가 곧 점수).

- 평면 사이드뷰. 바닥선 `레인Y≈-110`(용사·적 발이 여기에 정지). 용사는 점프(↑, 회피용) 시 중력으로 낙하. 적은 지상만(공중 이동 없음).
- **진격 모델(treadmill·Scratch 안전 선택)**: 실제 배경 스크롤 대신 추상 `진격도`로 진행을 표현한다. 적은 항상 오른쪽 끝(`x≈235`)에서 스폰돼 용사를 향해 전진하므로 "앞에서 밀려오는" 진격감이 나고, 성문은 공성 시 슬라이드 인으로 등장한다. 진행은 **진격 진행바 + 성문 등장**으로 눈에 보인다(스크롤 버그 회피 — chess-war 교훈처럼 Scratch 강점 안에서).

## 4. 사운드 — 전용 합성(build.py `_wav_bytes` + `synth_*`), orphan 0

|-----------|------|-------------------|------------------|--------|
| `synth_swing` | swing | 짧은 "쉭" 검 휘두름(1.5kHz→400Hz 하강 처프, 0.08초) | 용사 | 검격 |

| `synth_hit` | hit | 아주 짧은 "틱"(2kHz 클릭, 0.03초) | 적 | 적 피격(히트박스) |

## 5. 변수 / 리스트 / 메시지

### 5.1 ⚙️ 개조 손잡이 (튜닝 변수) — 한글 전역 변수, ID `V_*` / `var*`

   ▼

   │  스테이지 += 1 ; 적배율 = 적성장배율^(스테이지−1) ; 성문체력 = 성문기본체력×적배율
   │  진격도 = 0 ; 공성중 = 0 ; 성문 hide ; 적수 = 0 ; 웨이브타이머 = 0 ; 게임상태 = 1
   │  [스테이지시작](뿔피리·용사 (-120,-110) 리셋) → 플레이 루프 복귀
[종료]  체력 ≤ 0 → 게임상태=0
   │  게임오버 배너 show(스테이지) / 잔여 적 클론 삭제
   ▼
[다시 시작]  초록 깃발 재클릭(능력치·스테이지 기본값으로 — 로그라이트)
```

> **템포(교훈 1)**: 이동속도 6·검격쿨 8틱·잡졸 1~2타·진격속도 1.8 로 **첫 감부터 빠르게**. 강화 카드는 1/2/3/4 즉시 선택 → 바로 다음 스테이지. 공성 피날레는 성문을 신나게 부수는 짧고 굵은 클라이맥스.

## 7. 블록 흐름 (스프라이트별 의사코드)

### 용사

**(A) 깃발 초기화 + 스테이지 배치**

**(B) 이동 + 진격 + 점프 + 중력 forever**
```
when green flag clicked
  forever
    if (게임상태 = 1)
      # ── 좌우 이동 ──
      if key [right arrow] : point in direction 90  ; change x by 이동속도
      if key [left arrow]  : point in direction -90 ; change x by (0 - 이동속도)
      clamp x to ±220
      # ── 진격 밀기(앞에서 → 누를 때, 공성 아닐 때) ──
      if (공성중 = 0) and (key [right arrow]) and ((x position) >= 진격기준X)

      set 점프이전키 = 점프키
      # ── 중력 + 수직 ──
      change VY by 중력 ; change y by VY
      if (y position < 레인Y) : set y to 레인Y ; set VY = 0 ; set 점프남음 = 1

**(C) 검격 입력(Space + 마우스) forever**  — 에지+쿨

**(E) 피격 감시 forever**
```
when green flag clicked
  forever
    if (무적 > 0) : change 무적 by -1
    if (touching [적]) and (무적 = 0) and (게임상태 = 1)
        change 체력 by (0 - 접촉데미지대표)     # 아래 노트: 실제론 적 클론이 자기 내공격력으로 깎음
        set 무적 = 무적시간
        set sound effect pitch 0 ; play sound hurt
    wait 0.03
```
> **접촉 데미지 처리 방식**: 용사가 `touching 적`을 감지하되, **깎는 양은 적 클론의 `내공격력`이라야 종류별로 다르다.** 단일 스프라이트 `touching`으론 어떤 적인지 모르므로 → **적 클론 쪽에서 처리**한다: 적 (C) 루프에서 `if touching 용사 and 용사무적=0` 일 때 `change 체력 by (0−내공격력)`·`set 무적=무적시간`·hurt음. 용사 (E)는 무적 타이머 감소만 담당(또는 잡졸 기준 대표값으로 단순화 가능). **빌더는 적 클론 처리 방식을 채택**(종류별 데미지 반영). `무적`이 공유 게이트라 한 프레임에 한 번만 깎임.

### 적 (스포너 + 클론 본체)

**(A) 깃발 초기화**

**(B) 적생성 → 클론 1기** (원본만)

**(C) 클론 본체** (종류 세팅 → 용사 향해 전진 → 접촉/피격/처치)
```
when I start as a clone
  set 복제됨 = 1 ; set 적종류 = 적생성종류
  if (적종류 = 1)       # 잡졸
      set 내체력 = (잡졸_체력 * 적배율) ; set 내속도 = 잡졸_속도 ; set 내공격력 = 잡졸_공격력
      switch costume to [잡졸] ; set size 45
  else if (적종류 = 2)  # 전사
      set 내체력 = (전사_체력 * 적배율) ; set 내속도 = 전사_속도 ; set 내공격력 = 전사_공격력
      switch costume to [전사] ; set size 55
  else                  # 기사
      set 내체력 = (기사_체력 * 적배율) ; set 내속도 = 기사_속도 ; set 내공격력 = 기사_공격력
      switch costume to [기사] ; set size 70
  set 피격쿨 = 0
  goto (적생성X, 적생성Y) ; show
  forever
    if (게임상태 = 0) or (게임상태 = 2) : change 적수 by -1 ; delete this clone   # 게임오버/강화 → 전장 정리
    if (게임상태 = 1)
        # 1) 용사 향해 전진(단순 — 클론끼리 상호작용 없음)
        if ((x position) > (용사 x)) : point in direction -90 ; change x by (0 - 내속도)
        else                         : point in direction 90  ; change x by 내속도
        set y to 레인Y
        # 2) 접촉 데미지(용사 무적 게이트 공유)
        if (touching [용사]) and (무적 = 0) and (게임상태 = 1)
            change 체력 by (0 - 내공격력) ; set 무적 = 무적시간
            set sound effect pitch 0 ; play sound hurt
        # 3) 히트박스 피격(중복 방지 쿨)
        if (피격쿨 = 0) and ( (touching [검판정]) or (touching [스킬판정]) )
            if (touching [스킬판정]) : change 내체력 by (0 - 스킬공격력) ; set 데미지표시값 = 스킬공격력
            else                     : change 내체력 by (0 - 공격력)     ; set 데미지표시값 = 공격력
            set 피격쿨 = 6
            set sound effect pitch 0 ; play sound hit
            set 데미지표시x = x position ; set 데미지표시y = y position ; set 팝업종류 = 0
            broadcast 데미지표시
        if (피격쿨 > 0) : change 피격쿨 by -1
        # 4) 처치
        if (내체력 < 1)
            change 골드 by 처치골드 ; change 적수 by -1
            set 데미지표시값 = 처치골드 ; set 데미지표시x = x position ; set 데미지표시y = (y position + 20)
            set 팝업종류 = 1 ; broadcast 데미지표시                        # 금색 +골드
            play sound coin
            switch costume to [폭발]
            repeat 5 : change size by 10 ; change [ghost] effect by 20 ; wait 0.02
            delete this clone
    wait 0.025
```
> 빌더 노트: **적 로직은 여기서 끝**(용사 추적 전진 + 접촉 + 히트박스 피격 + 처치). **클론이 다른 클론을 참조/충돌하는 코드 없음** — chess-war 를 죽인 다대다 자율 전투를 원천 배제. 이동은 `change x by 내속도`(vrep 리포터 불필요·리터럴 상수 아님, 변수 그대로 사용) — move-steps 미사용이라 `b_movesteps` 변수 id 함정과 무관.

  broadcast 다음스테이지

when I receive 다음스테이지
  change 스테이지 by 1
  set 적배율 = 1 ; repeat (스테이지 - 1) : set 적배율 = (적배율 * 적성장배율)   # ★ 지수(곱셈 루프)
  set 진격도 = 0 ; set 공성중 = 0 ; set 적수 = 0 ; set 웨이브타이머 = 0
  set 게임상태 = 1
  broadcast 스테이지시작                       # 뿔피리 + 용사 리셋
```
> `적성장배율^(스테이지−1)` 은 지수 블록이 없어 **repeat 곱셈 루프**로 계산. 성문 체력은 `성문등장` 시 `성문기본체력×적배율`로 세팅되므로 여기선 진격도·상태만 리셋.

### 데미지 (플로팅 숫자 — say 미사용)

> rogue-knight 데미지 스프라이트와 동일 + `팝업종류` 금색 분기.
```
when green flag clicked
  hide ; set 복제됨 = 0 ; set rotation style [don't rotate]
when I receive 데미지표시
  if (복제됨 = 0)
      set 데미지글자수 = (length of 데미지표시값) ; set 데미지자리 = 1
      repeat 데미지글자수
          set 데미지숫자 = (letter 데미지자리 of 데미지표시값)
          set 데미지오프셋 = ((데미지자리 - 1) * 14) - ((데미지글자수 - 1) * 7)
          create clone of [myself] ; change 데미지자리 by 1
when I start as a clone
  set 복제됨 = 1 ; go to front
  switch costume to (데미지숫자)                       # 코스튬 이름 "0"~"9" 매칭
  if (팝업종류 = 1) : set [color] effect to 30         # 금색(골드)
  else : clear graphic effects
  goto ((데미지표시x + 데미지오프셋), 데미지표시y) ; set [ghost] effect to 0 ; show
  repeat 12 : change y by 4 ; change [ghost] effect by 8 ; wait 0.02
  delete this clone
```

### 체력바 / 스킬게이지 / 진격바 (상시 UI)

```
# 체력바
when green flag clicked
  show ; goto (-150,150) ; go to front
  forever
    set 비율 = (체력 / 최대체력)
    if (비율 < 0) : set 비율 = 0
    switch costume to (round(비율 * 10) + 1)      # hp_0..hp_10 (코스튬 1~11)
    wait 0.05

# 스킬게이지 (충전율)
when green flag clicked
  show ; goto (-40,150) ; go to front
  forever
    switch costume to (round(((스킬쿨 - 스킬타이머) / 스킬쿨) * 10) + 1)   # sk_0..sk_10, 가득이면 READY 광
    wait 0.05

# 진격바
when green flag clicked
  show ; goto (-90,128) ; go to front
  forever
    set 비율 = (진격도 / 스테이지길이)
    if (비율 > 1) : set 비율 = 1
    switch costume to (round(비율 * 10) + 1)      # mr_0..mr_10
    wait 0.05
```
> `비율`은 렌더 임시값 — 진행 변수에 `비율`(varRatio) 하나 추가하거나 인라인 계산. 코스튬 인덱스는 1-기반 보정(+1). **상태(체력·스킬쿨·진격)가 항상 눈에 보임**(교훈 3).

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화만 건드린다.

### 0층 — 숫자 하나 바꾸기
- **세진 용사**: `공격력` 을 `3` → `8` 로. 잡졸이 한 방에!
- **번개 이동**: `이동속도` 를 `6` → `10` 으로. 순식간에 전선을 오간다.
- **두부 멘탈**: `최대체력` 을 `6` → `2` 로. 한두 대만 맞아도 위험(난이도 급상승).
- **속사 검**: `검격쿨` 을 `8` → `3` 으로. 검을 따발총처럼 휘두른다.
- **광역 지배자**: `스킬범위` 를 `130` → `220` 으로, `스킬쿨` 을 `200` → `80` 으로. 스킬을 자주·넓게.
- **거북이 기사**: `기사_속도` 를 `1.0` → `0.3` 으로. 기사가 더 느릿.
- **짧은 스테이지**: `스테이지길이` 를 `900` → `400` 으로. 금방 성문에 도달.
- **순한 적 / 지옥 적**: `적성장배율` 을 `1.22` → `1.1`(쉬움) 또는 `1.5`(지옥) 로. **한 숫자로 난이도 곡선 전체가 바뀐다!**
- **물량 지옥**: `웨이브간격` 을 `1.2` → `0.4` 로, `최대적수` 를 `8` → `16` 으로.

### 1층 — 값을 식으로 묶기
- **진격할수록 세지는 검**: 용사 검격에서 `공격력` 대신 `공격력 + floor(진격도 / 300)` — 밀고 나갈수록 강해지게.
- **후반 보상**: 적 처치 골드를 `처치골드` → `처치골드 × 스테이지` 로.
- **묵직 점프**: `set 중력 = (0 - 점프력) / 12` 로 점프력에 비례.

### 2층 — if 조건 더하기
- **최후의 발악**: 용사 피격에 `if 체력 = 1 : set 무적 = 60` — 위기 시 잠깐 무적.
- **광전사**: `if 체력 <= 2 : set 이동속도 = 10` (위험하면 빨라짐)/회복 시 복구.
- **엘리트 웨이브**: 스포너에 `if (스테이지 mod 4 = 0) : set 적생성종류 = 3` — 4의 배수 스테이지는 기사 러시.

### 3층 — 규칙 바꾸기
- **2단 점프**: `점프남음` 리필을 `1` → `2` 로(용사 (B) 착지·초기화), 공중 회피 강화.
- **랜덤 강화**: 강화 선택을 `pick random 1 to 4` 자동으로.
- **스킬로 성문 즉파**: `스킬공격력` 을 성문에 2배로 적용하는 조건 추가(성문피해에서 스킬이면 ×2).

### 4층 — 새 메커닉
- **원거리 적(궁수)**: 적 4번째 종류로 화살 쏘는 궁수 추가(스프라이트 1개·적사격 방송) — 클론끼리는 여전히 상호작용 없음.
- **보스 스테이지**: `스테이지 mod 5 = 0` 이면 성문 대신 거대 보스(성문 스프라이트 재활용·체력 크게).
- **연속 콤보**: 검격을 짧게 연속 입력하면 3타째 데미지 2배(콤보 카운터 변수 추가).
- **골드 상점**: 스테이지 사이 골드로 능력치 구매(강화패널 확장).

## 10. 학습 포인트

보너스 직관 액션 게임 — 수학/추상 학습 콘셉트를 전면에 내세우지 않음([[feedback-game-design]] 준수). **rogue-knight/castle-defense 와 똑같이 "코딩 학습"이 핵심 가치**: 모든 능력치(용사 공격력/체력/이속/점프·검격/스킬 쿨과 범위·진격속도·적 3종 체력/속도/공격력·`적성장배율`·웨이브·강화량)가 한글 튜닝 변수로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (4층) 새 적/메커닉을 추가하며 **변수·연산·조건·클론·에지 디텍션·타이머·broadcast** 를 게임을 망가뜨리고 고치며 체득한다.
부수적으로 **메커닉의 결과로** 만나는 것: **지수 스케일링**(적이 곱하기로 세지는 벽을 강화로 따라잡기), **비율/게이지**(체력·스킬 충전율·진격도를 0~1로 정규화해 바로 렌더), **타이머·쿨다운**(에지 디텍션 + 쿨 카운트). 코드 곳곳의 **노란 코멘트(가이드 투어)** 가 각 부분을 설명한다 — 어디까지나 직관적 진격 액션으로 포장된 채.
