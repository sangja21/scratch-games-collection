# 마법 생존 (magic-survivor) — Plan

> 한 화면짜리 탑다운 아레나에서 **끝없이 몰려오는 적 무리를 자동 마법으로 쓸어담는 뱀파이어-서바이벌 라이트.** 마법사 한 명이 화살표키로 8방향을 누비며 피하기만 하면 되고(조준 불필요), 마법탄은 일정 간격(`발사간격`)마다 **가장 가까운 적**에게 자동으로 날아간다. 적은 **약·중·강 3종**(슬라임/박쥐/골렘)이며, **생존 시간이 길어질수록**(단계 = 생존시간÷난이도주기) 더 세고 빠른 적이 섞여 나온다. 적을 잡으면 경험치 보석이 떨어지고, 가까이 가면 빨아들인다. 경험치 바가 차면 **레벨업 → 강화 택1(공격력+/발사간격-/이동속도+)**. 적에 닿으면 체력 1 감소(+잠깐 무적), 체력 0이면 GAME OVER. **점수 = 생존 시간(초).**
> 베이스: `games/rogue-knight/build.py`(클론 스포너·복제됨 가드·체력/무적·폭발 연출·플로팅 데미지 숫자·강화 택1·게임오버 배너·한글 튜닝 변수 일괄 초기화) + `games/tank-battle/build.py`(클론 스포너·좌표 전달 채널·게임오버).
> 학습 콘셉트 없음(보너스 액션 게임). **이 게임의 진짜 존재 이유는 로그 나이트와 똑같이 "아이가 코드의 숫자·규칙을 직접 바꾸며 노는 것"** — 모든 조절 가능한 값을 한글 전역 변수로 노출하고, 코드 어디에도 매직넘버를 흩뿌리지 않는다. ★★★☆☆.

- **주제**: 자동 공격 생존형 액션(뱀파이어 서바이벌 라이트)
- **카테고리**: 액션
- **난이도**: ★★★☆☆
- **폴더**: `games/magic-survivor/`
- **출력 파일**: `마법_생존.sb3`
- **빌드**: `python3 games/magic-survivor/build.py`

## 1. 게임 한 줄

마법사 한 명으로 한 화면짜리 아레나에 갇혀, 사방에서 끝없이 몰려오는 적을 자동 마법으로 막아낸다. 마법탄은 `발사간격`마다 가장 가까운 적에게 알아서 날아가므로 아이는 **움직여서 피하는 데만** 집중한다. 적은 약·중·강 3종이며 생존 시간이 길어질수록 센 적이 섞여 나온다. 적을 잡으면 경험치 보석이 떨어지고, 모아서 레벨업하면 강화 셋 중 하나를 골라 점점 강해진다. 체력이 0이 되면 GAME OVER, 깃발을 다시 누르면 능력치가 기본값으로 리셋된다(생존 시간이 곧 점수).

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

```
 (-240,180)                                            (240,180)
   ┌──────────────────────────────────────────────────────┐
   │ ⏱생존:０초   Lv.１   ♥♥♥♥♥   EXP:０/５                 │  ← 좌상단 모니터 4종
   │   [적]                              [적]               │  적: 사방 가장자리에서 스폰
   │        ◆(보석)                                         │  보석: 처치 자리에 떨어짐
   │              [적]    ✦→     [적]                       │  마법탄: 가장 가까운 적으로 자동 발사
   │                  [마법사]                              │  마법사: 화면 중앙 (0,0) 시작
   │   [적]                       ◆(보석)        [적]       │
   │            [적]                       [적]             │
   │   ← ↑ ↓ → 8방향 이동 (피하기만!)                       │
   └──────────────────────────────────────────────────────┘
 (-240,-180)                                          (240,-180)

  EXP 가득 → 화면 중앙에 [강화카드] 표시: 1.마법공격력+  2.발사간격-  3.이동속도+
                                          (1/2/3 키로 선택 → 전투 재개)
  체력 0 → 화면 중앙에 GAME OVER 배너 → 깃발 재클릭으로 처음부터(능력치 리셋).
```

- HUD: 좌상단에 `생존시간`(점수) `레벨` `체력` `경험치`(/`레벨업경험치`) 모니터 4종. 튜닝 변수(마법공격력 등)는 **모니터 숨김**(코드에서만 본다).
- 평면 탑다운 — 중력·점프·바닥선 없음. 마법사·적·보석 모두 480×360 평면을 자유 이동. 마법사 시작 위치 (0,0).
- 적은 화면 4변(상/하/좌/우) 바깥 가장자리에서 스폰되어 마법사 쪽으로 직선 추적.

> **아트 방향**: 모두 정면 탑다운, `don't rotate`(이동·추적은 `point towards` 로 방향만 돌고 코스튬은 안 뒤집힘). 적 3종은 **각각 다른 코스튬**(슬라임=작고 초록 / 박쥐=보라 / 골렘=크고 회갈색)으로 한눈에 구분 — 로그 나이트는 한 코스튬을 color 이펙트로 재색했지만, 여기선 코스튬 3장으로 분리해 더 또렷하게.
> **자동 조준 핸드셰이크(이 게임의 핵심 패턴)**: 마법사가 발사 직전 `조준요청` 을 **broadcast and wait** 로 보낸다. Scratch 는 단일 스레드 협력형이라, wait 없는 수신 스크립트들이 **클론 순서대로 하나씩 원자적으로** 실행된다 → 각 적 클론이 "내 거리가 지금까지 최소면 `조준X/Y` 를 내 위치로 갱신"하는 **최솟값 리덕션**이 경쟁 조건 없이 안전하게 성립한다. 끝나면 마법사가 `조준있음=1` 일 때만 그 좌표로 탄을 쏜다.
> **탄 분리 패턴(로그 나이트 검판정과 동형)**: 데미지 판정을 마법사 본체가 아니라 별도 `마법탄` 스프라이트가 담당. 적은 `touching 마법탄` 만 보면 되고 friendly-fire 분기가 없다. 관통(`관통`)은 탄이 자기 `남은관통` 카운터로 관리.
> **플로팅 데미지 숫자(say 미사용)**: 로그 나이트와 동일하게 숫자 코스튬 0~9 를 자릿수별 클론으로 렌더해 항상 같은 자리에 또렷이 띄운다. `마법공격력` 을 바꾸면 통! 뜨는 숫자도 따라 바뀐다.

## 6. 씬 / 상태머신

```
[깃발 클릭]
   │  Stage: 튜닝 변수 24개(적 종류별 9 포함) + 진행 변수 전부 초기화
   │         (체력=최대체력, 게임상태=1, 생존시간=0, 레벨=1, 경험치=0, 단계=0 …)
   │  마법사: 위치(0,0), 무적=0
   ▼
[게임시작 방송]
   │  적 스포너 forever 가동 / 마법사 자동발사 forever 가동 / Stage 타이머 가동
   ▼
[전투중 forever]  (게임상태=1)
   ├ 마법사: ←↑↓→ 8방향 이동(clamp) / 적 접촉→체력−1(무적) /
   │         발사간격마다: 조준요청(broadcast&wait)→조준있음이면 발사X/Y 세팅 후 발사
   ├ 마법탄: 발사 받으면 추가발사 개수 탄 클론 → 목표로 직진, 적 닿으면 관통 −1, 0이면 삭제
   ├ 적 스포너: 스폰간격마다 가장자리 1마리 스폰(단계로 종류 결정), 적수 관리
   ├ 적 클론: 마법사 직선 추적 + 접촉 데미지 + touching 마법탄 → 내체력 −마법공격력 /
   │          0이하면 폭발(보석 드롭: 보석생성) + 데미지 팝업(데미지표시)
   ├ 보석 클론: 흡수범위 안이면 마법사로 빨려옴, 닿으면 경험치 +내경험치
   ├ Stage 타이머: 1초마다 생존시간+1, 단계 = floor(생존시간/난이도주기)
   └ Stage 감시: 경험치 ≥ 레벨업경험치 → 게임상태=2 + 레벨업 / 체력<1 → 게임상태=0
   ▼
[강화선택중]  (게임상태=2)   ← 전투·스폰·타이머 모두 일시정지(각 루프가 게임상태=1만 처리)
   │  강화카드 show. 경험치 −= 레벨업경험치 ; 레벨 += 1 (감시에서 처리)
   │  1/2/3 키 입력:
   │   1 → 마법공격력 += 강화량
   │   2 → 발사간격 -= (0.05 * 강화량)   (최소 0.1 클램프)
   │   3 → 이동속도 += 강화량
   │  선택 후 [강화완료] → 게임상태=1
   ▼  (전투중 루프로 복귀)
[종료]  체력 ≤ 0 → 게임상태=0
   │  게임오버 배너 show(생존시간 표시) / 잔여 적·보석·탄 클론 자기 삭제
   ▼
[다시 시작]  초록 깃발 재클릭(능력치 기본값으로 — 생존 점수 리셋)
```

> 난이도 곡선(빌더 주의): 적은 방 단위가 아니라 **시간 단위로 끝없이** 나온다. `단계` 가 0→1→2로 오를 때 종류 풀이 약→약·중→중·강으로 바뀌고, `스폰간격` 을 줄이면 더 자주 쏟아진다(미션 사다리에서 `스폰간격 = 1.2 - (단계 * 0.1)` 식으로 가속 가능).

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait` 넣어 CPU 폭주 방지. 이동·충돌 틱은 0.025초(≈40fps). **매직넘버 금지** — 항상 튜닝 변수 참조.

### Stage

**(A) 깃발 클릭 → 변수 전부 초기화(한 곳에 모음) → 게임시작**
```
when green flag clicked
  # ── 개조 손잡이(튜닝) 기본값: 아이가 여기 숫자만 바꾸면 게임이 바뀐다 ──
  set 마법공격력=1 ; set 발사간격=0.6 ; set 마법탄속도=8 ; set 관통=1 ; set 추가발사=1
  set 이동속도=4 ; set 최대체력=5 ; set 무적시간=25
  set 흡수범위=90 ; set 흡수속도=6 ; set 레벨업경험치=5 ; set 강화량=1
  set 스폰간격=1.2 ; set 난이도주기=15
  # ── 적 종류별 능력치(약/중/강) ──
  set 약한적_체력=1 ; set 약한적_속도=1.2 ; set 약한적_경험치=1
  set 중간적_체력=3 ; set 중간적_속도=0.9 ; set 중간적_경험치=3
  set 강한적_체력=6 ; set 강한적_속도=0.6 ; set 강한적_경험치=6
  # ── 진행 상태 ──
  set 체력=최대체력 ; set 게임상태=1 ; set 생존시간=0 ; set 레벨=1 ; set 경험치=0
  set 무적=0 ; set 단계=0 ; set 스폰카운트=0 ; set 적수=0
  set 조준거리=99999 ; set 조준있음=0
  wait 0.3
  broadcast 게임시작
```

**(B) 생존시간 타이머 + 난이도 단계 forever**
```
when I receive 게임시작
  forever
    wait 1
    if (게임상태 = 1)
        change 생존시간 by 1
        set 단계 = floor(생존시간 / 난이도주기)
```

**(C) 무적 타이머 감소 forever**
```
when green flag clicked
  forever
    if 무적 > 0 : change 무적 by -1
    wait 0.025
```

**(D) 레벨업 / 게임오버 감시 forever**
```
when green flag clicked
  wait until (게임상태 = 1)
  forever
    if (경험치 >= 레벨업경험치) and (게임상태 = 1)
        change 경험치 by (0 - 레벨업경험치)
        change 레벨 by 1
        set 게임상태 = 2
        broadcast 레벨업
    if (체력 < 1) and (게임상태 = 1)
        set 게임상태 = 0          ← 게임오버
    wait 0.05
```

### 마법사

**(A) 깃발 초기화**
```
when green flag clicked
  show ; set size 60 ; set rotation style [don't rotate]
  point in direction 90 ; goto (0,0) ; go to front
```

**(B) 8방향 이동 forever**  (전투중일 때만)
```
when green flag clicked
  forever
    if (게임상태 = 1)
      if key [right arrow] : change x by 이동속도
      if key [left arrow]  : change x by (0 - 이동속도)
      if key [up arrow]    : change y by 이동속도
      if key [down arrow]  : change y by (0 - 이동속도)
      clamp x to ±230 ; clamp y to ±170
    wait 0.025
```

**(C) 자동 조준 발사 forever**  (이 게임의 핵심)
```
when I receive 게임시작
  forever
    if (게임상태 = 1)
        set 조준거리 = 99999 ; set 조준있음 = 0
        broadcast 조준요청 and wait        ← 모든 적 클론이 최솟값 리덕션으로 가장 가까운 적 보고
        if (조준있음 = 1)
            set 발사X = 조준X ; set 발사Y = 조준Y
            set sound effect [pitch] to 120 ; play sound pop
            broadcast 발사                  ← 마법탄 원본이 추가발사 개수만큼 탄 생성
        wait 발사간격
    else
        wait 0.05
```
> 빌더 노트: `broadcast and wait` 가 핵심. 적 클론들의 `조준요청` 수신 스크립트는 **wait 가 없어** 순차·원자 실행되므로 `조준거리` 최솟값 갱신에 경쟁 조건이 없다. 적이 0마리면 `조준있음=0` 이라 발사 생략.

**(D) 피격 감시 forever**
```
when green flag clicked
  forever
    if (touching [적]) and (무적 = 0) and (게임상태 = 1)
        change 체력 by -1
        set 무적 = 무적시간
        set sound effect [pitch] to -300 ; play sound pop
    wait 0.04
```
> 무적 동안 깜빡임(선택): `무적 > 0` 이면 ghost 토글. 연출 옵션.

### 마법탄 (탄 스포너 + 클론 본체)

**(A) 깃발 초기화**

```

**(B) 발사 → 추가발사 개수만큼 탄 클론** (원본만)
```
when I receive 발사
  if (복제됨 = 0)
      set 발사i = 0
      repeat 추가발사
          create clone of [myself]
          change 발사i by 1
```

**(C) 클론 본체** (목표로 직진, 관통 처리)
```
when I start as a clone
  set 복제됨 = 1
  set 남은관통 = 관통 ; set 관통쿨 = 0
  goto (마법사 x, 마법사 y)
  # ── 목표 좌표로 방향 조준(부채꼴 스프레드 포함) ──
  point in direction ( (atan((발사X - x position)/(발사Y - y position))) + ((y position > 발사Y) * 180)
                       + ((발사i - ((추가발사 - 1) / 2)) * 12) )      ← 12도 간격 부채꼴
  go to front ; show
  repeat until (touching edge) or (게임상태 = 0)
      move 마법탄속도 steps
      if (touching [적]) and (관통쿨 = 0)
          change 남은관통 by -1
          set 관통쿨 = 4                ← 같은 적 중복 차감 방지(적은 자기 HP를 깎음)
          if (남은관통 < 1) : delete this clone
      if 관통쿨 > 0 : change 관통쿨 by -1
      wait 0.01
  delete this clone
```
> 빌더 노트: 좌표 조준 공식 `atan(dx/dy) + (dy<0 ? 180 : 0)` 는 Scratch 표준 "좌표 향하기"다. `(y position > 발사Y)` 가 참(1)/거짓(0)이라 `*180` 으로 분기. 데미지 차감은 **적이 `touching 마법탄` 으로 자기 내체력을 깎는다**(아래 적 (C)) — 탄은 관통 횟수만 센다(로그 나이트 검판정과 동일 역할 분리).

### 적 (스포너 + 클론 본체)

**(A) 깃발 초기화**

```

**(B) 스폰 forever → 가장자리에서 1마리씩** (원본만)
```
when I receive 게임시작
  if (복제됨 = 0)
      forever
          if (게임상태 = 1)
              # ── 가장자리(상/하/좌/우) 랜덤 스폰 좌표 ──
              set 변 = pick random 1 to 4
              if 변=1 : set 적생성X = pick random -230 to 230 ; set 적생성Y = 175
              if 변=2 : set 적생성X = pick random -230 to 230 ; set 적생성Y = -175
              if 변=3 : set 적생성X = -235 ; set 적생성Y = pick random -170 to 170
              if 변=4 : set 적생성X = 235  ; set 적생성Y = pick random -170 to 170
              # ── 단계로 종류 결정(약→약·중→중·강) ──
              if (단계 = 0)
                  set 적생성종류 = 1                              ← 약만
              else if (단계 = 1)
                  set 적생성종류 = (1 + (스폰카운트 mod 2))        ← 약(1)·중(2) 번갈아
              else
                  set 적생성종류 = (2 + (pick random 0 to 1))     ← 중(2)·강(3) 섞임
              change 스폰카운트 by 1
              change 적수 by 1
              create clone of [myself]
          wait 스폰간격
```
> 빌더 노트: `변` 은 인라인 임시(또는 진행 변수로 추가). 본 plan 변수표에는 별도로 안 넣었으니 `변`을 진행 변수에 추가하거나 `pick random` 결과를 if 4개로 직접 분기해도 됨(권장: if 안에서 `pick random 1 to 4` 한 번만 평가하도록 임시 변수 사용).

**(C) 클론 본체**
```
when I start as a clone
  set 복제됨 = 1
  set 적종류 = 적생성종류
  # ── 종류별 능력치·외형(매직넘버 없음: 종류 튜닝 변수 참조) ──
  if (적종류 = 1)               ← 약(슬라임)
      set 내체력 = 약한적_체력 ; set 내속도 = 약한적_속도
      switch costume to [슬라임] ; set size 60
  else if (적종류 = 2)          ← 중(박쥐)
      set 내체력 = 중간적_체력 ; set 내속도 = 중간적_속도
      switch costume to [박쥐] ; set size 70
  else                          ← 강(골렘)
      set 내체력 = 강한적_체력 ; set 내속도 = 강한적_속도
      switch costume to [골렘] ; set size 95
  set 피격쿨 = 0
  goto (적생성X, 적생성Y) ; show
  forever
    # 1) 게임오버 정리
    if (게임상태 = 0) : change 적수 by -1 ; delete this clone
    # 2) 추적(전투중일 때만 다가옴)
    if (게임상태 = 1)
        point towards [마법사] ; move 내속도 steps
    # 3) 마법탄 피격(중복 방지 쿨)
    if (touching [마법탄]) and (피격쿨 = 0)
        change 내체력 by (0 - 마법공격력)
        set 피격쿨 = 8
        set sound effect [pitch] to 0 ; play sound pop
        # ── 플로팅 데미지 팝업(맞은 자리에 마법공격력 숫자) ──
        set 데미지표시값 = 마법공격력
        set 데미지표시x = x position ; set 데미지표시y = y position
        broadcast 데미지표시
    if 피격쿨 > 0 : change 피격쿨 by -1
    # 4) 처치(보석 드롭 + 폭발 연출 후 삭제)
    if (내체력 < 1)
        set 보석X = x position ; set 보석Y = y position
        if (적종류 = 1) : set 보석값 = 약한적_경험치
        else if (적종류 = 2) : set 보석값 = 중간적_경험치
        else : set 보석값 = 강한적_경험치
        broadcast 보석생성
        change 적수 by -1
        switch costume to [폭발]
        set sound effect [pitch] to -50 ; play sound pop
        repeat 5 : change size by 12 ; change [ghost] effect by 20 ; wait 0.02
        delete this clone
    wait 0.025
```

**(D) 조준 보고(최솟값 리덕션)**  — wait 없는 원자 실행이 핵심

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식·조건 1~2층", "새 메커닉 4층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화 스크립트만 건드린다.

### 0층 — 숫자 바꾸기 (한 곳만 고친다)
- **속사포 마법사**: `발사간격` 을 `0.6` → `0.2` 로. 마법탄이 따발총처럼 쏟아진다.
- **관통 마법**: `관통` 을 `1` → `5` 로. 한 발이 일렬의 적 다섯을 뚫는다.
- **삼연발**: `추가발사` 를 `1` → `3` 으로. 부채꼴 3발이 동시에 날아간다.
- **세진 마법**: `마법공격력` 을 `1` → `3` 으로. **떠오르는 데미지 숫자가 `1`→`3` 으로 바뀌는지** 직접 확인! (데미지 팝업은 `마법공격력` 을 그대로 보여준다)
- **두부 멘탈**: `최대체력` 을 `5` → `1` 로. 한 대만 맞아도 위험!
- **거북이 골렘**: `강한적_속도` 를 `0.6` → `0.2` 로. 골렘이 더 느릿느릿.
- **자석 장갑**: `흡수범위` 를 `90` → `200` 으로. 보석이 화면 멀리서도 빨려온다.
- **지옥문**: `스폰간격` 을 `1.2` → `0.4` 로. 적이 세 배로 쏟아진다(난이도 급상승).

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **잡을수록 빨라지는 손**: 무대 (A)의 `발사간격` 을 그대로 두고, 강화카드 2번을 `change 발사간격 by (0 - (발사간격 / 10))` 로 바꿔 **현재 간격의 10%씩** 줄게(곱셈 감소).
- **레벨 비례 공격**: 마법탄 피격에서 `마법공격력` 대신 `마법공격력 + (레벨 / 2)` 로 데미지 계산 — 레벨이 오를수록 기본 화력↑.
  - 적 (C)-3 의 `change 내체력 by (0 - 마법공격력)` → `change 내체력 by (0 - (마법공격력 + (레벨 / 2)))`.
- **시간이 갈수록 빨라지는 적**: 적 (C)-2 의 `move 내속도 steps` → `move (내속도 + (단계 / 5)) steps`. 오래 버틸수록 적이 가속.

### 2층 — if 조건 더하기
- **마지막 발악**: 마법사 (D) 피격 감시에 `if 체력 = 1 : set 무적 = 60` 추가 — 체력 1 남으면 잠깐 무적.
- **레벨업 폭발**: Stage (D) 레벨업 처리에 `set 무적 = 무적시간` 추가 — 레벨업 순간 잠깐 무적으로 숨 돌리기.
- **보스 타임**: 적 스폰 (B)에 `if (단계 >= 3) : set 강한적_체력 = 강한적_체력 + 단계` 를 넣어, 후반 골렘을 점점 더 단단하게.

### 3층 — 규칙 자체를 바꾸기 (분기)
- **강화 4번 추가(추가발사+)**: 강화카드에 `if key [4] : change 추가발사 by 1` 선택지를 추가하고 카드 그림에 4번 칸을 그린다.
- **강화 5번 추가(관통+/최대체력+)**: `if key [5] : change 관통 by 1` (또는 `change 최대체력 by 강화량 ; change 체력 by 강화량`) 추가.
- **랜덤 강화**: 강화 선택을 키 입력 대신 `pick random 1 to 3` 으로 자동 결정하게 바꿔 운에 맡기기.

### 4층 — 새 메커닉 만들기 (구조)
- **유도탄**: 마법탄 (C)에서 직진 대신 매 틱 `point towards [적]` 를 한 번 섞어 살짝 휘어 날아가는 유도 마법으로.
- **회복 보석 추가**: `경험치보석` 을 복제해 `체력보석` 스프라이트를 만들고, 적 처치 시 가끔(`pick random 1 to 5 = 1`) 떨어뜨려 닿으면 `change 체력 by 1`.
- **분열 슬라임**: 적 (C) 처치 시 `if 적종류 = 1 : (작은 슬라임 2마리 스폰)` 을 넣어, 약한 적이 죽으며 둘로 쪼개지게(스폰 채널 재사용).
- **화려한 데미지 숫자**: `데미지` 클론의 떠오르는 속도(`change y by 4`)·페이드(`change ghost by 8`)를 바꾸거나, `데미지표시값` 에 `× 2` 를 곱해 "표시만 두 배"인 가짜 폭딜 숫자 연출.

## 10. 학습 포인트

보너스 액션 게임 — 수학 학습 콘셉트 없음([[feedback-game-design]] 준수: 데시벨/음파/산성 같은 추상 콘셉트 금지). **대신 로그 나이트와 똑같이 "코딩 학습"이 핵심 가치**: 모든 능력치가 한글 튜닝 변수로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (4층) 새 스프라이트를 추가하며, 변수·연산·조건·클론·**broadcast and wait** 라는 프로그래밍 개념을 게임을 망가뜨리고 고치며 체득한다. 부수적으로 **타이머/누적**(생존시간), **확률적 스폰**(가장자리·종류 난수), **증가하는 난이도 곡선**(단계 = 생존시간÷난이도주기)을 게임 행동으로 자연스럽게 만난다 — 어디까지나 직관적 액션으로 포장된 채.
