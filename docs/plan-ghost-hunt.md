# 유령 사냥 (ghost-hunt) — Plan

> 어두운 방에 깜빡이며 나타나는 만화풍 유령을 회중전등 십자선으로 조준 사격.
> 60초 타임어택. 라운드 없음, 라이프 없음, 탄약 무한 — 순수 점수 모으기.
> 베이스: `games/duck-hunt/build.py` (마우스 추적 십자선 + 클릭 → 섬광 클론 + 적 명중 판정)
>         + `games/whack-a-prime/build.py` (적 클론 무작위 등장 → 짧은 표시 시간 → 자동 삭제).
> 학습 콘셉트 없음. 직관적 사격 액션.
> 분위기: 어두운 보라/검정 톤이지만 유령은 만화풍 (Casper 스타일, 동그란 눈 + 미소). 초등학생이 무서워하지 않게.

## 1. 컨셉 / 메커닉

- 배경: 짙은 보라/검정 톤의 방. 벽지 무늬·창문 실루엣·바닥 그림자 정도. 색은 어둡지만 만화풍 (귀신의 집을 단순화).
- 회중전등 십자선: 마우스 따라 이동. 십자선 스프라이트 = 십자선 + 그 주변에 큰 반투명 노란 원(라이트 영역). 라이트 안만 살짝 밝게 보이는 듯한 연출.
- 유령 클론: 화면 무작위 위치(천장/벽 근처 우선)에 1초간 깜빡이며 등장 → 자동 삭제. 1.0~1.4초 간격으로 새 유령 스폰.
- 사격: 십자선 클릭 → 클릭 위치에 작은 노란 섬광 클론 0.15초 표시. 섬광이 유령 클론에 닿으면 점수 +10, 유령은 "맞음" 코스튬 0.15초 → 삭제.
- 60초 타이머 후 게임오버 배너 표시 (총 점수 표시).
- 탄약/라이프/라운드 없음 — 빠른 손맛 중심.

## 2. 화면 레이아웃 (Scratch 480×360, Stage 좌표 -240..240 × -180..180)

```
y=180   ───────── 방 천장 (짙은 보라) ─────────
                                                    유령 스폰 영역
                                                    x ∈ [-200..200]
                                                    y ∈ [-100..140]
                       (창문 실루엣)

y=-180  ───────── 방 바닥 (그림자 그라데이션) ─────────
```

좌상단: 점수, 남은시간 변수 모니터.

## 6. 스프라이트별 블록 흐름

### Stage
1. **깃발 클릭**:
   - 점수=0, 남은시간=60, 게임상태=1, 섬광X/Y=0, 유령X/Y=0
   - 0.5초 대기 → `게임시작` 방송
2. **게임시작 받으면 (병렬 스크립트 A — 타이머)**:
   - repeat until 남은시간 ≤ 0:
     - wait 1
     - 남은시간 -1
   - 게임상태 = 0
   - `게임종료` 방송
3. **게임시작 받으면 (병렬 스크립트 B — 스폰 루프)**:
   - repeat until 게임상태 = 0:
     - 유령X = random -200..200
     - 유령Y = random -100..140
     - `유령등장` 방송
     - wait (random 0.7..1.2 초)

### 회중전등 (flashlight)
1. **깃발 클릭** → 크기 100, 맨 앞으로, 보이기, forever: goto (mouseX, mouseY)
2. **이 스프라이트 클릭** → if 게임상태=1:
   - 섬광X=mouseX, 섬광Y=mouseY
   - "섬광" 클론 생성
   - pop @ 피치 200

### 섬광 (flash)
1. **깃발 클릭** → 숨김, 크기 60
2. **클론으로 시작** → goto (섬광X, 섬광Y) → 맨 앞으로 → 보이기 → wait 0.12 → 숨김 → 클론 삭제

### 유령 (ghost) — 클론 기반
1. **깃발 클릭** → 숨김, 크기 90, 코스튬=a
2. **유령등장 받으면** → "유령" 클론 생성 (Stage 의 유령X/Y 사용)
3. **클론으로 시작**:
   - goto (유령X, 유령Y)
   - 코스튬 = a
   - 보이기
   - 깜빡임 루프: `repeat 4` { next costume; wait 0.12 } — a↔b 토글 (코스튬 2개 사이)
   - touching 섬광 검사: 깜빡임 루프 안에서 매 반복 마다 if touching 섬광 → 코스튬 = hit, 점수+10, pop @ 피치 80 → wait 0.12 → 삭제 (stop this script)
   - 깜빡임 끝 → 숨김 → 클론 삭제
4. (단순화) 단일 클론 수명 ≈ 0.5초. 0.7~1.2초 간격으로 새 클론 → 동시에 1~2 유령 떠 있을 수 있음.

### 게임오버
1. **깃발 클릭** → 숨김, goto (0,0), 크기 100, 맨 앞으로
2. **게임종료 받으면** → 보이기

## 7. 예상 블록 카운트

- Stage: ~40 (init + 타이머 루프 + 스폰 루프)
- 회중전등: ~25 (mouse follow + click → 섬광 클론)
- 섬광: ~12 (클론 → 위치 → 표시 → 삭제)
- 유령: ~60 (등장 핸들러 + 클론 생성 + 깜빡임 + 명중 분기 + 삭제)
- 게임오버: ~10
- **총합 예상: ~140~180 블록**

## 8. 자산

- `room_bg.svg` — 어두운 보라(#2A1B3D) 배경 + 창문 실루엣 2개 + 가구(소파/책장) 실루엣 + 바닥 그라데이션. 만화풍 단순 도형.
- `flashlight.svg` — 큰 반투명 노란 원(반경 80, radial gradient: 중심 #FFEB3B 0.45 → 가장자리 0) + 가운데 흰 십자선 + 작은 빨간 점.
- `flash.svg` — 노란 8각 별 (duck-hunt 의 flash 재사용 가능, 색 동일).
- `ghost_a.svg` — 만화 유령 자세 1: 흰색 둥근 몸 + 아래 물결, 큰 동그란 검은 눈 2개, 작은 곡선 미소. 색 #F5F5F5.
- `ghost_b.svg` — 만화 유령 자세 2: 살짝 더 작거나 투명도 0.7 (깜빡임 second frame).
- `ghost_hit.svg` — 유령 맞음: 눈 X 표시 + "POOF" 같은 노란 작은 별 흩어짐 (혹은 단순히 색 변경).
- `gameover.svg` — 검은 배너 + "TIME UP!" + 총 점수 안내 (점수 모니터 그대로 표시되므로 텍스트만).
- `pop.wav` — 기존 duck-hunt/whack-a-prime 의 pop.wav 복사.
