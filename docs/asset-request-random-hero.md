# 🎨 에셋 요청서 v2 (애니메이션) — 「낭만 고양이」 랜덤 성장 엔들리스 RPG

> **v2 변경**: 캐릭터/이펙트를 **스프라이트 애니메이션**으로 만듭니다. 정적 한 장이 아니라 **여러 프레임**이 필요해요.
> Grok 이미지 생성은 프레임을 따로 뽑으면 캐릭터가 매번 달라져서 애니가 울렁거립니다.
> 그래서 **"한 번의 생성으로 여러 프레임을 한 장에" = 스프라이트 시트** 방식으로 요청하세요. 그러면 한 그림 안이라 프레임끼리 캐릭터가 일관됩니다.
> 받은 시트는 제가 build.py에서 **N등분으로 잘라 코스튬**으로 넣고 애니 루프를 겁니다.

## 🌙 세계관 (동일)
슬픈 도시 뒷골목, 달과 별 아래 홀로 살아가는 낭만적인 길고양이 **낭만 고양이**. 발톱과 꼬리로 밤골목 훼방꾼을 물리치고 생선을 먹으며 바다로 떠나길 꿈꾼다.

## 📐 공통 스타일 가이드 (Grok에 먼저 알려주기)
```
스타일: 감성적이고 몽환적인 2D 게임 아트, 귀엽지만 살짝 쓸쓸한 '낭만적인 밤' 무드,
부드러운 셀 셰이딩, 굵고 매끈한 외곽선. 색감: 깊은 남보라 밤하늘, 은은한 달빛/별빛,
따뜻한 가로등 주황 포인트. 차분하고 분위기 있는 야경 팔레트.
시점: 사이드뷰(옆모습). 배경 완전 투명(PNG 알파). 텍스트/워터마크 없음, 사실 사진풍 금지.
```

## 🎞️ 스프라이트 시트 공통 규칙 (모든 애니 항목에 필수!)
Grok에게 각 시트 요청 시 아래를 **꼭** 포함하세요:
```
스프라이트 시트로 만들어줘: 같은 캐릭터를 한 장의 가로 이미지에 프레임 N개로,
왼쪽→오른쪽 한 줄로 균등 간격 배치. 각 칸 크기·캐릭터 크기·바닥선·바라보는 방향
전부 동일하게 유지하고, 프레임마다 포즈만 다르게. 프레임 사이·배경은 완전 투명.
캐릭터 색·외곽선·비율은 모든 프레임에서 똑같이. 격자선이나 숫자는 넣지 말 것.
```
- **방향**: 낭만 고양이는 **오른쪽**, 훼방꾼/보스는 **왼쪽**을 봄.
- **해상도**: 가로로 긴 이미지(예: 프레임 4개면 4096×1024 정도). 각 프레임은 정사각 칸, 캐릭터 중앙.
- 프레임이 어긋나거나 캐릭터가 프레임마다 달라지면 → 그 시트만 다시 생성. (일관성이 생명)

## 🐱 애니메이션 에셋 (스프라이트 시트, 총 6종)

### 1. 낭만 고양이 — 4프레임 시트 `hero_sheet.png`
- **프레임 순서(좌→우)**: ① 대기A(가만히) ② 대기B(숨쉬며 살짝 웅크림) ③ 공격A(앞발 뒤로 젖힘) ④ 공격B(앞발로 앞을 힘차게 할큄)
- **용도**: ①②는 idle 바운스 루프, ③④는 스킬 시전 순간
- **프롬프트**:
```
[공통 스타일 + 스프라이트 시트 규칙]
목에 낡은 체크무늬 목도리를 두른 감성적인 회색 길고양이, 두 발로 섬, 오른쪽을 바라봄.
프레임 4개: (1) 편히 선 대기, (2) 살짝 웅크려 숨쉬는 대기, (3) 오른 앞발을 뒤로 젖힌
공격 준비, (4) 앞발로 앞을 세게 할퀴는 공격. 4프레임 모두 같은 고양이·같은 목도리·같은 크기.
```

### 2. 심술 쥐 — 3프레임 시트 `enemy_mouse_sheet.png`

- **프롬프트**:
```

밤골목의 짓궂은 회색 생쥐, 두 발로 서서 왼쪽을 바라봄, 심술궂지만 귀여움.
프레임 3개: (1) 걸어오는 자세, (2) 반대 발로 걷는 자세, (3) 입을 벌려 깨무는 공격.
3프레임 모두 같은 쥐·같은 크기.
```

### 3. 들개 — 3프레임 시트 `enemy_dog_sheet.png`

프레임 3개: (1) 걸어오는 자세, (2) 반대 발로 걷기, (3) 입 벌려 달려드는 공격.
같은 개·같은 크기.

### 4. 골목대장 불독(보스) — 4프레임 시트 `boss_bulldog_sheet.png`
- **프레임 순서**: ① 걸음A ② 걸음B ③ 포효(고개 젖힘) ④ 내려찍기(앞발 강타)

프레임 4개: (1) 묵직하게 걷기, (2) 반대 발 걷기, (3) 고개 젖혀 포효, (4) 앞발로 앞을 내려찍는 공격.
4프레임 모두 같은 불독·같은 크기, 다른 적보다 크게.

## 🏃 v3 추가 요청 (오토러너 전환) — 3종

> 게임이 **오토러너**로 바뀌었어요: 고양이가 **화면 가운데**에서 달리고 배경이 무한 스크롤됩니다. 그래서 아래 3종이 새로 필요해요.

### A. 무한 스크롤 배경 — **2레이어, 가로 이음새 없음(seamless)** ⭐가장 중요
> 기존 `background.png`는 한 장짜리 풍경이라 무한 반복하면 이음선이 보여요. 그래서 **좌우 끝이 딱 맞아 끝없이 이어지는(타일링 가능한)** 배경을 레이어로 나눠 받습니다. 고양이는 **화면 정중앙**에서 달리니, 캐릭터가 가릴 가운데를 비워두고 **길/바닥은 화면 아래쪽**에 오게.

**A-1. 먼 배경 `bg_far.png`** (하늘 레이어, 천천히 스크롤)
```
[공통 스타일] 가로로 이음새 없이 무한 반복되는(좌우 끝이 완벽히 맞물리는, seamless tileable)
밤하늘 배경 스트립. 깊은 남보라 하늘, 둥근 달과 별, 아래쪽에 멀리 반짝이는 바다 수평선.
캐릭터·건물 없이 하늘/바다만. 가로로 아주 긴 이미지(예 2048×720). 반복해도 이음선 안 보이게.
```

**A-2. 가까운 거리 `bg_near.png`** (길/건물 레이어, 빠르게 스크롤, 위쪽 투명)
```
[공통 스타일] 가로로 이음새 없이 무한 반복되는(좌우 끝이 맞물리는, seamless tileable) 밤 뒷골목
'가까운' 레이어: 화면 아래쪽에 돌바닥 길과 인도, 양옆으로 이어지는 낮은 건물 아랫부분·가로등·쓰레기통.
**이미지 위쪽 절반은 완전 투명**(먼 하늘이 비치게), 아래쪽에만 길과 건물. 가로로 아주 긴 이미지, 투명 PNG.
반복해도 이음선 안 보이게. 가운데에 캐릭터가 달릴 평평한 길.
```

### B. 고양이 달리기 시트 `hero_run_sheet.png` (진짜 달리기 모션)
> 지금은 대기 프레임을 흔들어서 "달리는 것 같지 않다"는 피드백. 실제 **달리기 사이클**이 필요해요.
```
[공통 스타일 + 스프라이트 시트 규칙] 목도리 두른 회색 길고양이(hero_sheet와 같은 캐릭터)가
오른쪽을 보고 **달리는 애니메이션**, 프레임 6개로 다리를 번갈아 내딛는 자연스러운 질주 사이클
(앞다리·뒷다리 교차, 몸 살짝 상하). 6프레임 모두 같은 고양이·같은 크기·같은 목도리. 투명 배경, 한 줄 균등 배치.
```

### C. 적 처치 이펙트 시트 `fx_poof_sheet.png` (전용 — 회오리와 다름)
> 지금 적이 죽을 때 '꼬리 회오리' 스킬 이펙트를 재사용 중 → **처치 전용** 연출이 필요해요.
```
[공통 스타일 + 스프라이트 시트 규칙] 적이 쓰러질 때 터지는 처치 이펙트 애니메이션,
'펑' 하는 먼지 연기 구름 + 사방으로 튀는 작은 별. 프레임 4개로 작게 터짐→확 퍼짐→옅어짐→소멸.
캐릭터 없이 이펙트만. 은은한 별빛 색감, 투명 배경. (회오리 바람과는 다른 느낌 — 둥근 폭발/먼지)
```
