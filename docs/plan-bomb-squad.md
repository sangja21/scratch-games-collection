# 폭탄 스쿼드 (bomb-squad) — Plan

> 하단 슬링샷을 마우스로 **당겨(드래그)** 포탄을 **포물선**으로 쏴 올려, 하늘에서 **낙하산 타고 내려오는 적(몬스터)**을 요격하는 **드래그 조준 물리 방어 게임**(닌텐도 *Bob-omb Squad* 오마주). 발사체를 당기면 당긴 반대 방향으로 발사 벡터가 정해지고, 당기는 동안 **점선 궤적(펜)** 이 실시간으로 휜다. 마우스를 놓으면 그 궤적대로 포탄이 날아가 적을 맞춘다. **포탄은 여러 발 동시 발사(연사)되며 적을 관통**한다. 바닥엔 **꽃밭(꽃 = 라이프)** — 적이 꽃에 닿으면 꽃이 터지고, 꽃이 전멸하면 게임오버. **연속 명중 콤보**로 점수를 불리고, **엔들리스 최고기록**에 도전한다.
> *(v2: 표적 폭탄→낙하산 적 몬스터 리스킨 · 앵커Y 올림 · 관통샷 · 여러 발 연사. §5.2 하단 v2 개조 요약 참조.)*
> 베이스: `games/magic-survivor/build.py`(한글 튜닝 변수 **일괄 초기화**·매직넘버 0 / 클론 스포너 + `복제됨` 가드 / 폭발 연출 / say 미사용 플로팅 숫자 / 전용 합성 효과음 `_wav_bytes`·`synth_*` / `add_comment` 가이드 투어) + `games/fish-tank/build.py`(**엔들리스 생존/최고기록** + 난이도 램프) + `docs/plan-cannon-siege.md`(**드래그 슬링샷 물리·펜 궤적 미리보기** 골격 그대로 재사용). 드래그 입력은 스크래치 `마우스를 누르고 있는가` + `마우스 x/y`.
> 학습 콘셉트 있음(**포물선 운동·속도 성분 분해·중력의 벡터 영향 + 이동표적 리드(예측/상대운동)**) — 추상 그래프가 아니라 **손으로 당겨 떨어지는 적을 맞히는 물리 놀이**. `중력` 손잡이 하나가 **손맛과 난이도를 동시에** 조절(낮으면 직선 슬링샷=쉬움, 높으면 활 모양 로빙=어려움). [[feedback-game-design]]·[[game-success-shape]] 준수: **단일 초점(슬링샷 하나) + 경계 명확한 전투(포탄 vs 낙하 폭탄)** — 자율 난전 금지. 시후(초등·성격 급함)를 위해 **당기는 손맛**·**콤보 배증 점수**·**즉시 재조준**(놓으면 바로 다음 발). ★★★☆☆.

## 1. 게임 한 줄

하단 슬링샷을 마우스로 **누르고 당겨(드래그)** 포탄을 포물선으로 쏘고, 하늘에서 낙하산 타고 천천히 내려오는 **폭탄을 맞혀 터뜨린다**. 당기는 동안 점선 궤적이 실시간으로 보여 떨어지는 폭탄의 앞을 겨눠 요격한다(놓으면 발사). 바닥 꽃밭에 폭탄이 닿으면 꽃이 하나 터지고, 꽃이 다 사라지면 게임오버. 연속으로 맞히면 콤보 점수가 배로 불어나고, 시간이 갈수록 폭탄이 더 빨리·더 자주 떨어진다 — **최고기록에 도전하는 엔들리스**.

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

```
 (-240,180)                                            (240,180)
   ┌──────────────────────────────────────────────────────┐
   │ ⭐점수:０   🔥콤보:x2   🏆최고:０                        │  ← 좌상단 HUD 모니터 3종
   │ 당김X -40  당김Y 60  크기 72  각도 56°                   │  ← 드래그 벡터 수치 모니터(실시간)
   │   ☂         ☂                      ☂                  │  낙하 폭탄(낙하산): 위 랜덤 X에서 스폰
   │   💣        💣          ～라키투～    💣                 │  라키투: 가끔 화면 가로지름(맞히면 전멸)
   │        ·                    ·  ·                       │  점선 궤적(펜): 당김 반대방향 예측 포물선
   │      ·   ·  ·              ·      ·                     │
   │    ·         ·  ·        ·          ·                  │  포탄: 앵커에서 발사, 궤적 따라 비행
   │  ·             ·  ·    ·             💣  ← 요격!         │
   │ ●╲  ← 발사체 당김(앵커→마우스)                           │  발사대: 하단중앙 고정 앵커. 마우스로 당김
   │ ╱◉ 마우스(당기는 손)                                     │
   │ 🌸  🌸  🌸  🌸   ← 꽃밭(라이프 4개)                      │  꽃밭: 바닥 꽃 = 라이프. 폭탄 닿으면 −1
   │████████████████ 지면 ████████████████████████████████████│  ← 지면Y (꽃 라인)
   └──────────────────────────────────────────────────────┘
 (-240,-180)                                          (240,-180)

  조준: 발사체를 마우스로 누르고 당김 → 당김 벡터(앵커−마우스) 실시간 → 점선 궤적이 다시 그려짐
  발사: 마우스를 놓으면 발사 → 포탄이 포물선으로 올라가 낙하 폭탄에 명중 → 폭탄 폭발·콤보 점수
  놓침: 폭탄이 꽃밭 라인 도달 → 가장 가까운 꽃 1개 파괴 + 콤보 리셋. 꽃 전멸 → 게임오버(최고기록 갱신)
```

- **발사대(앵커)**: 하단 **중앙** 고정(Scratch `(0, -120)` = 앵커 `앵커X`/`앵커Y`). 사방(좌·우·위)에서 떨어지는 폭탄을 겨눠야 하므로 중앙 배치(cannon-siege 는 좌하단이었음 — 방어 컨셉이라 중앙으로 변경). 발사체는 앵커에 얹혀 있다가 드래그하면 마우스를 따라 뒤로 당겨진다.
- **HUD 모니터**: **좌상단에 `점수` `콤보배수` `최고점수` 3종만**(적 시야 확보). **드래그 벡터 수치** `당김X`·`당김Y`·`당김크기`·`당김각도` 4종(벡터 분해 실시간 노출 — 학습 핵심)은 **화면 하단으로 이동**(y≈302, 가로 정렬 — 적 낙하 경로를 안 가림). 나머지 튜닝·내부 변수는 숨김. *(v3: 드래그 벡터 4종 상단→하단 이동)*
- **드래그 조준(슬라이더·발사버튼 없음)**: 발사체 스프라이트를 `마우스를 누르고 있는가` + `마우스 x/y` 로 잡아 당긴다. 당김 벡터 = `(앵커X − 마우스x, 앵커Y − 마우스y)`(당긴 반대 방향). **`최대당김거리` 로 클램프** 후 `발사력배율` 을 곱해 초기 속도로. 마우스 놓으면 발사. 앵커↔발사체 사이 당김선(펜/코스튬)이 손맛.
- **궤적 미리보기**: `궤적펜` 이 드래그 중 매 프레임 **당김 벡터·중력·바람으로 포탄 물리를 dry-run** 해 점선을 그린다(포탄 실제 궤적과 동일 공식). 드래그 안 하면 안 그림.
- **낙하 폭탄(적)**: 화면 위(`스폰Y` ≈ 170)에서 랜덤 X로 스폰돼 **낙하산 등속**(살짝 좌우 흔들림)으로 내려온다. 포탄에 맞으면 폭발(콤보 점수), 꽃밭 라인(`지면Y`) 도달 시 꽃 1개 파괴 후 소멸.
- **꽃밭(라이프)**: 바닥에 꽃 `꽃개수`(기본 4)개가 나란히. 폭탄이 라인에 닿으면 **가장 가까운 살아있는 꽃**이 터진다. 남은 꽃 0 → 게임오버.
- 평면 2D 물리(옆에서 본 시점, 무대 전체가 하늘). 중력은 아래(−y), 바람은 수평(±x). 당김·궤적·포탄이 같은 물리 상수(`중력`,`바람`,`발사력배율`,`최대당김거리`)·같은 당김 벡터를 공유 → **점선과 실제 비행이 정확히 일치**(핵심 요건).

> **엔들리스 — 스테이지 없음, 시간/처치 기반 난이도 램프.** cannon-siege 의 "스테이지 클리어/블록 리스트" 구조를 버리고, fish-tank 식 **무한 생존 + 최고기록**으로 교체. 시간이 흐르거나 폭탄을 잡을수록 `낙하속도`↑·`스폰간격`↓(`난이도증가율` 손잡이). 꽃 전멸까지 계속.

## 4. 사운드 — 전용 합성(build.py `_wav_bytes` + `synth_*`), orphan 0

magic-survivor 처럼 **결정적 합성 WAV** 를 만들어, 각 스프라이트에 **실제 재생하는 것만** 등록(고아 0). 모든 합성 함수는 `synth_zap`/`synth_boom` 형태(샘플 리스트 → `_wav_bytes(samples)` → md5). 재생은 대부분 `play sound`(대기 안 함).

| 합성 함수 | 이름 | 음색 스펙(결정적) | 등록 스프라이트 | 트리거 |
|-----------|------|-------------------|------------------|--------|
| `synth_stretch` | stretch | 고무줄 당김 "삐긋"(상승 피치 200→500Hz, 0.1초) | 발사체 | 드래그 시작 시 1회 |
| `synth_fire` | fire | 놓으며 튕기는 "핑!"(노이즈버스트+80Hz thump, 0.18초, `synth_boom` 변형) | 포탄 | 발사(마우스 놓는) 순간 |
| `synth_pop` | pop | 폭탄 요격 "펑"(짧은 노이즈 폭발+중역 thump, 0.14초) | 폭탄 | 포탄에 맞아 폭발 |
| `synth_combo` | combo | 콤보 상승 "띠링↑"(콤보 단계마다 피치 ↑ 정현파, 0.1초, pitch 이펙트 병용) | 폭탄 | 콤보 배증될 때 |
| `synth_wilt` | wilt | 꽃 파괴 "시들"(하강 정현파 500→200Hz + 낮은 buzz, 0.2초) | 꽃 | 폭탄이 꽃밭 도달·꽃 파괴 |
| `synth_lakitu` | lakitu | 라키투 등장 "삐요~"(빠른 상하 비브라토, 0.3초) | 라키투 | 라키투 스폰 |
| `synth_clear` | clear | 라키투 명중 "콰광 화르륵"(큰 노이즈 폭발+상승 화음, 0.35초) | 라키투 | 라키투 맞혀 전멸 보너스 |
| `synth_gameover` | gameover | 하강 버저(400→120Hz 사각파, 0.3초) | 배너 | 꽃 전멸 게임오버 |
| `synth_record` | record | 신기록 팡파르(523→659→784→1047Hz 상승, 0.4초) | 배너 | 최고기록 갱신 |
| *(사용자 제공 mp3)* | **bgm** | **배경 음악 `assets/bgm.mp3`(≈124초, mp3 64kbps 48kHz stereo). 합성 아님 — 파일 그대로 `open(...,'rb')` 읽어 md5→`<md5>.mp3` 로 아카이브, `dataFormat:"mp3"` 등록** | **Stage** | **초록 깃발 → 무한 반복(v4 추가)** |

> 등록 규칙: 발사체=stretch, 포탄=fire, 폭탄(적)=pop/combo, 꽃=wilt, 라키투=lakitu/clear, 배너=gameover/record, **Stage=bgm(mp3)**. 다른 스프라이트엔 사운드 없음(고아 0). 합성 효과음 **9종** + BGM mp3 **1개**.
>

> **⚙️ v3 개조 요약(사용자 플레이 피드백 반영)**: ① **"더 못 쏘는" 잠김 버그 근본 해결** — 원인은 화면 밖으로 나가거나 어떤 경로로도 삭제 안 된 포탄 클론이 전역 `포탄수` 를 도로 감소시키지 않아 `포탄수 < 최대포탄수` 에 계속 막힌 것. **포탄 삭제 경로를 단일 출구로 통일**(관통소진·화면밖·라키투·수명만료·게임오버 전부 같은 `repeat until` 탈출 → `포탄수 −1` → `delete`)해 **삭제와 카운터 감소가 항상 짝**이 되게 함. 추가로 **수명 타이머** `포탄수명`(기본 150틱≈3초, 클론-로컬 `남은수명` 매 틱 −1, 0이면 소멸) 안전망 도입. ② **포탄 레이어를 발사대보다 위로** — `발사대`(frame) 를 `go to back` 으로 내려, 발사되는 포탄(`go to front`)이 슬링샷 프레임에 안 가리게. ③ **모니터 정리** — HUD 3종(`점수`·`콤보배수`·`최고점수`)만 좌상단 유지, 드래그 벡터 4종(`당김X`/`당김Y`/`당김크기`/`당김각도`)은 **화면 하단(y≈302, x 4/128/252/366 가로 정렬)**으로 이동해 적 낙하 경로(상단~중앙) 시야 확보. 값은 계속 보임(숨김 아님, 위치만 하단). **궤적펜=포탄 동일 공식 불변 유지.**

> **⚙️ v4 개조 요약(사용자 제공 BGM)**: 사용자가 만든 배경음악 `assets/bgm.mp3`(≈124초, 64kbps 48kHz stereo)를 **바이너리 그대로**(재인코딩 금지) Stage 사운드로 등록(`open(...,'rb')`→md5→`<md5>.mp3`, `dataFormat:"mp3"`, `rate:48000`). Stage 에 **별도 병렬 깃발 스크립트**로 `set volume (브금볼륨)% → forever { play bgm until done }` 무한 반복. 튜닝 변수 `브금볼륨`(기본 70) 노출. 기존 합성 wav 효과음 9종·게임 로직 전부 불변, sb3 용량은 mp3 때문에 ≈78KB→≈2.98MB.

> 모니터로 띄울 것: `점수`·`콤보배수`·`최고점수`(HUD 3종, **좌상단** y=5/29/53) + `당김X`·`당김Y`·`당김크기`·`당김각도`(드래그 벡터 4종, **하단** y=302 x=4/128/252/366) = **7개**. 나머지는 숨김. (`꽃남음` 은 꽃 스프라이트 자체가 시각화하므로 모니터 선택 — 원하면 추가)

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait`(0.02~0.03초)로 CPU 폭주 방지. **매직넘버 금지** — 항상 튜닝 변수 참조. 드래그 입력은 `마우스를 누르고 있는가`(`sensing_mousedown`) + `마우스 x`/`마우스 y`(`sensing_mousex/y`).

> **공통 ①: 당김 벡터 계산 + 클램프**(발사체 조준 & 궤적 dry-run 공통 — cannon-siege 와 동일, 전역 `당김X/Y/크기/각도` 세팅)
> ```
> 당김X = 앵커X - (마우스 x)          # 당긴 반대 방향
> 당김Y = 앵커Y - (마우스 y)
> 당김크기 = sqrt(당김X*당김X + 당김Y*당김Y)
> if 당김크기 > 최대당김거리:          # 최대 당김 상한 클램프
>   당김X = 당김X * 최대당김거리 / 당김크기
>   당김Y = 당김Y * 최대당김거리 / 당김크기
>   당김크기 = 최대당김거리
> 당김각도 = atan(당김Y / 당김X)        # (당김X=0 가드: 0 이면 90) — HUD 관찰용
> ```
> **공통 ②: 속도 성분 + 물리 적분**(포탄 발사 & 궤적 dry-run 공통 — 반드시 동일 공식, 위 클램프된 당김 벡터 사용)
> ```
> 속도X = 당김X * 발사력배율      # 수평 성분 = 당김 X 성분 × 배율
> 속도Y = 당김Y * 발사력배율      # 수직 성분 = 당김 Y 성분 × 배율
> # 매 스텝:
> 속도Y = 속도Y - 중력           # 중력이 위 성분을 깎음(정점 후 하강)
> 속도X = 속도X + 바람            # 바람이 옆으로 밈
> x = x + 속도X ; y = y + 속도Y
> ```
> > **당김 벡터 X·Y 성분이 곧 속도 성분** → 벡터 분해를 손으로 만진다. 대각선 45°로 당기면 속도X·속도Y 가 같아져 최대 사거리(바람0). **중력을 낮추면 정점이 얕고 궤적이 곧아져 직선 슬링샷 느낌 / 높이면 활처럼 로빙.**

> **공통 ③: 난이도 램프 파생값**(스폰 매니저·폭탄 스폰에서 사용)
> ```
> 현재낙하속도 = 낙하속도 + (처치수 * 난이도증가율)
> 현재스폰간격 = 스폰간격 - (처치수 * 난이도증가율)      # 하한 클램프
> if 현재스폰간격 < 0.4 : 현재스폰간격 = 0.4              # 너무 빨라지지 않게(연출 상수 인라인 허용)
> ```

### 발사체 (ball) — 드래그 조준 + 놓으면 발사 (cannon-siege 그대로, 탄약 무제한)
```
when green flag clicked
  go to x:앵커X y:앵커Y ; show ; switch costume to ball
when I receive 게임시작
  forever
    if <<게임상태 = 1> and <비행중 = 0>>          # (탄약 조건 없음 — 무제한)
      if <<mouse down?> and <touching mouse-pointer?>>
        set 드래그중 = 1 ; play sound stretch
        repeat until <not <mouse down?>>          # 당기는 동안 (공통①)
          set 당김X = 앵커X - (mouse x) ; set 당김Y = 앵커Y - (mouse y)
          set 당김크기 = sqrt((당김X*당김X)+(당김Y*당김Y))
          if <당김크기 > 최대당김거리>
            set 당김X = 당김X * 최대당김거리 / 당김크기
            set 당김Y = 당김Y * 최대당김거리 / 당김크기
            set 당김크기 = 최대당김거리
          end
          if <당김X = 0> : set 당김각도 = 90 else : set 당김각도 = (abs of (atan(당김Y / 당김X)))
          go to x:(앵커X - 당김X) y:(앵커Y - 당김Y)   # 당겨진 위치로 이동
          wait 0.01
        end
        set 드래그중 = 0
        if <당김크기 > 4>                          # 살짝 눌렀다 뗀 오발 방지
          broadcast 발사
        end
        go to x:앵커X y:앵커Y                        # 앵커로 복귀
      end
    end
    wait 0.02
```
> cannon-siege 와 동일하되 **`남은포탄` 조건·차감 제거**(엔들리스라 탄약 무제한). `비행중=0` 가드로 중복 클론 방지.

### 폭탄 (bomb) — 낙하산 스포너 + 클론 본체
```
when green flag clicked : set 복제됨 = 0 ; hide ; set 폭탄수 = 0
when I receive 폭탄생성
  if <복제됨 = 0>
    create clone of myself
when I start as a clone
  set 복제됨 = 1
  change 폭탄수 by 1
  set 흔들기준X = pick random -200 to 200          # 스폰 X
  set 흔들위상 = pick random 0 to 360
  set 내낙하 = 낙하속도 + (처치수 * 난이도증가율)    # 공통③ 램프
  go to x:흔들기준X y:170 ; switch costume to bomb ; show
  forever
    # 하강 + 좌우 흔들림
    change 흔들위상 by 4
    set x to (흔들기준X + (흔들폭 * (sin of 흔들위상)))
    change y by (0 - 내낙하)
    # ── 요격 판정 ──
    if <touching 포탄?>
      # 콤보 증가·점수
      change 콤보수 by 1
      set 콤보배수 = 콤보배수 * 콤보배율
      if <콤보배수 > 콤보최대> : set 콤보배수 = 콤보최대
      set 이번점수 = 폭탄점수 * 콤보배수
      change 점수 by 이번점수
      change 처치수 by 1
      play sound pop ; play sound combo          # combo 는 pitch=콤보수 기반 변주(선택)
      set 명중점수값 = 이번점수 ; set 명중점수x = x position ; set 명중점수y = y position
      set 팝업종류 = 1                             # 금색(콤보 점수)
      broadcast 점수표시
      switch costume to boom ; repeat 6 (change size/ghost)
      change 폭탄수 by -1
      delete this clone
    end
    # ── 꽃밭 도달(놓침) ──
    if <(y position) < 지면Y>
      set 파괴꽃X = x position                      # 채널: 어느 X 근처 꽃을 없앨지
      broadcast 꽃파괴                              # 꽃이 자기 X 와 비교해 가장 가까운 1개 처리
      set 콤보수 = 0 ; set 콤보배수 = 1             # 놓치면 콤보 리셋
      switch costume to boom ; repeat 4 (ghost)
      change 폭탄수 by -1
      delete this clone
    end
    # ── 라키투 전멸 보너스 ──
    if <전멸신호 = 1>
      change 처치수 by 1                            # 전멸도 처치로 인정(점수는 라키투가 일괄? — 여기선 개별 소량)
      play sound pop
      switch costume to boom ; repeat 4 (size/ghost)
      change 폭탄수 by -1
      delete this clone
    end
    wait 0.02
```
> **꽃 파괴 채널**: 폭탄이 꽃밭 라인 도달 시 `파괴꽃X = 자기 x` 세팅 후 `꽃파괴` 방송. 꽃 클론들이 각자 `|내X − 파괴꽃X|` 를 재서 **가장 가까운 살아있는 꽃 1개만** 시든다(아래 꽃 스크립트에서 최솟값 리덕션, magic-survivor 자동조준 패턴 축소판). 단순화: 폭탄 x 에 관계없이 "오른쪽에서부터/왼쪽에서부터 살아있는 첫 꽃"을 없애도 됨(builder 재량) — 단 **꽃남음 정확히 −1**.
> **전멸신호**: 라키투 명중 시 라키투가 `전멸신호=1`(잠깐) → 모든 폭탄이 이 프레임에 폭발. Stage 가 다음 프레임 `전멸신호=0` 복귀(또는 라키투가 짧은 wait 후 0).

```
> **가장 가까운 꽃 판정(권장 구현)**: `꽃파괴` 를 broadcast-and-wait 로 2패스 처리 — (1패스) 각 꽃이 살아있으면 `min(전역 최소거리, |내X−파괴꽃X|)` 갱신, (2패스) 자기 거리 == 최소거리인 꽃 1개만 시들기. magic-survivor 자동조준 리덕션 축소판. **단순 대안**(블록 적어 안전): 폭탄이 닿은 x 와 무관하게 "가장자리부터 살아있는 첫 꽃"을 없애도 게임성 충분 — builder 재량. 어느 쪽이든 **꽃남음 −1 정확·중복 파괴 금지**.

### 숫자팝업 (플로팅 점수) — magic-survivor 데미지 스프라이트 패턴
```
when I receive 점수표시
  # 명중점수값 자릿수만큼 코스튬 클론 렌더(say 미사용), 팝업종류로 흰/금 코스튬 오프셋
  # 위로 떠오르며 ghost 페이드 후 삭제 (magic-survivor 그대로)
```
> **재사용**: magic-survivor 숫자팝업 전체 복사 → 색 세트 2개(흰=일반 / 금=콤보·보너스), `팝업종류` 오프셋만 추가.

### 배너 (banner) — 게임오버 + 최고기록 (fish-tank 패턴)

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식 1층", "조건 2층", "새 메커닉 4층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화/꽃X 리스트만 건드린다.

### 0층 — 숫자 하나 바꾸기 (무대 초록 깃발 스크립트)
- **달나라 로빙 요격**: `중력` 을 `0.30` → `0.10` 으로. 포탄이 거의 직선으로 멀리 날아가 조준이 쉬워진다(Bob-omb 느낌).
- **목성 활쏘기**: `중력` 을 `0.30` → `0.70` 으로. 포탄이 활처럼 크게 휘어 리드 조준이 필요(하드).
- **하드모드 꽃 1개**: `꽃개수` 를 `4` → `1` 로. 한 번만 뚫려도 게임오버 — 목숨을 건 요격.
- **여유 꽃밭**: `꽃개수` 를 `4` → `8` 로. 실수해도 오래 버틴다.
- **폭탄비**: `스폰간격` 을 `1.6` → `0.6` 으로. 폭탄이 우르르 쏟아진다(난이도 급상승).
- **슬로 폭탄**: `낙하속도` 를 `0.7` → `0.3` 으로. 폭탄이 아주 천천히 내려와 초보도 쉽게.
- **점수 폭발**: `콤보배율` 을 `2` → `3` 으로(+`콤보최대` 를 `81` 로). 콤보가 붙으면 점수가 어마어마하게.
- **지그재그 폭탄**: `흔들폭` 을 `12` → `40` 으로. 폭탄이 심하게 좌우로 흔들려 맞히기 어렵다.
- **라키투 자주**: `라키투확률` 을 `40` → `10` 으로. 라키투가 자주 나와 전멸 보너스 찬스↑.
- **힘센 슬링샷**: `발사력배율` 을 `0.22` → `0.35` 로. 살짝만 당겨도 포탄이 쭉.

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **콤보 비례 데미지 연출**: 팝업 색/크기를 `콤보배수` 에 비례시켜(예: 팝업 크기 = 30 + 콤보배수×2) 콤보가 클수록 숫자가 커 보이게.
- **가팔라지는 난이도**: `난이도증가율` 을 `0.02` → `0.05` 로. 후반이 훨씬 빡세짐(실제 낙하 = 낙하속도 + 처치수×이 값).
- **처치 비례 보너스**: 요격 점수에 `+ 처치수` 를 더해(오래 살수록 한 방 점수↑) — Stage 없이도 성장감.

### 2층 — if 조건 더하기
- **황금 폭탄**: 폭탄 스폰에 `if (pick random 1 to 12 = 1) : 코스튬=금색·이번점수×3` 분기(가끔 3배 폭탄).
- **위기 경고**: Stage 에 `if 꽃남음 = 1 : (배경 붉게/사이렌음)` 로 마지막 꽃 경고.
- **콤보 마일스톤**: 폭탄 요격에서 `if 콤보수 mod 10 = 0 : 점수 +1000` 보너스.

### 3층 — 규칙 자체를 바꾸기 (분기)
- **바람 부는 날**: `바람` 을 `0` → `0.08` 로(또는 `set 바람 = pick random -8 to 8 / 100` 매판 랜덤) — 궤적이 휘어 예측이 달라진다.
- **튕기는 포탄**: 포탄 비행에 `if y>190 : set 속도Y = 속도Y × -0.6`(1회) → 천장 튕김으로 위쪽 폭탄 노리기.
- **꽃 부활**: `if 처치수 mod 30 = 0 and 꽃남음 < 꽃개수 : (시든 꽃 1개 되살리기)` — 오래 버티면 라이프 회복.

### 4층 — 새 메커닉 만들기 (구조)
- **꽃밭 재배치**: `꽃X` 리스트 좌표·`꽃개수` 를 바꿔 꽃을 몰아 두거나 넓게 퍼뜨려 방어 난이도를 바꾸자.
- **2단 낙하산**: 폭탄이 아래 절반에서 낙하산을 접고 **가속 낙하**(내낙하 ×2)하게 만들어 후반 압박↑.
- **연쇄 폭발**: 폭탄 요격 시 주변(반경) 폭탄도 같이 터지는 광역(콤보 폭증) 메커닉 추가.
- **당김 가이드 원**: 발사대에 `최대당김거리` 반경 원(펜/코스튬)을 그려 최대 힘 지점을 시각화.

## 10. 학습 포인트

**포물선 운동·벡터·이동표적 리드를 손으로 당겨 쏘며 만나는 물리 방어 게임.** 발사체를 당기면 **당김 벡터가 곧 발사 벡터**(당긴 반대 방향)가 되고, 놓으면 그 점선 궤적대로 포탄이 날아가 **떨어지는 폭탄**을 요격한다. 다음 직관이 게임 행동 자체로 체득된다:

- **속도 성분 분해(벡터)**: 당김 벡터 `(당김X, 당김Y)` 가 그대로 초기 속도 성분 `속도X = 당김X·발사력배율`(옆)·`속도Y = 당김Y·발사력배율`(위)이 된다. 위로 당기면 위 성분↑·옆 성분↓ — **드래그하며 X·Y 성분을 직접 만지고** HUD `당김X`/`당김Y` 로 확인.
- **중력의 적분적 영향 = 손맛/난이도**: 매 틱 `속도Y ← 속도Y − 중력` 이라 포탄이 올라갔다 떨어진다. **`중력` 하나로 게임 성격이 바뀐다** — 낮으면 정점이 얕고 궤적이 곧아 **직선 슬링샷(쉬움)**, 높으면 크게 휘는 **로빙(어려움)**. 물리 상수가 곧 난이도라는 걸 몸으로 안다.
- **이동표적 리드(예측 / 상대운동)** ★신규: 폭탄은 계속 **내려오는 표적**이라, 지금 폭탄 위치가 아니라 **포탄이 도달할 때쯤 폭탄이 있을 자리(앞)** 를 겨눠야 맞는다. 발사부터 명중까지 시간 동안 폭탄이 얼마나 더 내려올지 예측하는 **리드 사격**을 자연히 익힌다 — 점선 궤적과 폭탄 낙하선이 **만나는 지점**을 눈으로 맞추는 놀이.
- **각도-사거리(45° 최적)**: 바람=0에서 당김각도 45° 근처일 때 사거리 최대. 좌·우 멀리 오는 폭탄을 칠 때 체감(HUD `당김각도`).
- **바람의 벡터적 영향(고급)**: `속도X ← 속도X + 바람` 이라 궤적이 수평으로 휜다 — 중력(세로)·바람(가로)의 **벡터 합성**을 눈으로 본다.
- **예측 = 시뮬레이션**: 궤적펜은 실제 포탄과 **똑같은 물리식을 미리 돌려(dry-run)** 점선으로 보여준다 — "계산해서 미리 안다"는 물리·프로그래밍 공통 아이디어.

부수적으로 **변수·리스트·조건·클론·펜·벡터 연산·최솟값 선택(가장 가까운 꽃)·콤보 배증(지수적 증가)** 을 게임을 튜닝하며(0→4층) 코딩 학습으로 이어진다. 코드 곳곳의 **노란 코멘트(가이드 투어)** 가 각 부분을 설명한다. 어디까지나 **직관적 드래그 방어 놀이**로 포장된 채 — 단일 슬링샷·명확한 표적(낙하 폭탄)이라는 성공 게임 모양([[game-success-shape]])을 지킨다.
