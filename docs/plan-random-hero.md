# 낭만 고양이 (random-hero) — Plan

## 1. 게임 한 줄

낭만적인 길고양이가 달과 별 아래 도시 뒷골목에서 밀려오는 밤골목 훼방꾼을 한 마리씩 상대한다. 기본 전투는 자동 — 고양이와 훼방꾼이 서로 `공격속도`마다 상대 HP를 깎는다(발톱 할퀴기). 아이가 하는 건 **액션 3개(별빛 발톱·꼬리 회오리·생선)를 쓸 수 있을 때 1·2·3 키로 누르는** 것뿐(쿠키런 리듬). 스킬 2개는 쿨이 차면, 생선은 남아 있으면 먹어서 HP 회복(1개 소비). 훼방꾼을 잡으면 경험치가 차고, 레벨업하면 **랜덤 강화 카드 3장** 중 하나를 골라 즉시 강해진다 — 카드는 **스킬/생선 강화**(별빛 발톱/꼬리 레벨↑, 생선 회복↑)와 **능력치 강화**(공격/체력/속도/치명)가 섞여 나오고, 어떤 카드가 뜰지·얼마나 오를지 전부 랜덤에 가끔 잭팟 대박. **레벨업하면 생선도 몇 개 보충**된다. 한 적을 잡으면 다시 달리기 재개 → 다음 조우(달리고→싸우고 무한 반복). 달릴수록 훼방꾼은 지수로 세지고 멀리 가면 들개(강적)가 섞이며 `보스주기`마다 골목대장 불독(보스)이 길을 막는다. HP가 0이 되면 게임오버, 깃발을 다시 누르면 처음부터(단, **최고 도달 거리·레벨·처치수는 세션 내 유지** — 하이스코어).

- **HUD(좌상단)**: `레벨` `처치수` `거리`(진행도, m) 3종 + **경험치 바**(EXP/`레벨업경험치`). 우상단에 **최고기록** 3종(`최고거리`·`최고레벨`·`최고처치`). 튜닝 변수는 모니터 숨김(코드에서만).
- **고양이(왼쪽~중앙 고정)**: `영웅X`(≈-140)에 늘 고정. **달리기 페이즈엔 제자리 달림**(배경 스크롤로 전진감), **전투 페이즈엔 그 자리서 자동 공격**. 절대 화면을 가로질러 이동하지 않음(온레일).
- **훼방꾼(오른쪽)**: 달리기 페이즈에 오른쪽 밖(`스폰X`≈260)에서 등장해 왼쪽으로 다가옴(배경보다 조금 빠르게). `조우사거리` 안에 들면 **전투 페이즈 전환**(스크롤 정지, 훼방꾼 `교전Y`≈100 에 정지). **한 조우에 1마리**(보스도 1마리). 처치되면 달리기 재개 → 다음 훼방꾼 접근.
- **HP 바 2개**: 고양이 HP(왼쪽 상단, 항상 표시)·훼방꾼 HP(오른쪽 상단, 전투 중만 표시)를 **펜/코스튬 스케일 바**로.
- **액션 3칸(하단, 항상 표시)**: `[1]별빛 발톱`·`[2]꼬리 회오리`(스킬 2종, **쿨 게이지**)와 `[3]생선`(**소지 개수 표시**). 스킬은 쿨이 차면 반짝, 생선은 `물약개수>0`이면 먹을 수 있고 `×N`으로 남은 개수(0이면 회색 비활성). **전투 페이즈에서 키(1·2·3 또는 Z·X·C)·클릭으로 발동**(달리는 중엔 적이 없어 스킬은 대기, 생선은 달리는 중에도 먹을 수 있게 허용). **필살기 게이지 1개 대신 스킬 2 + 소비 아이템 1**(쿠키런식).
- **레벨업 오버레이**: 페이즈=3 동안 달리기·전투·스폰·스킬쿨·타이머 정지. **강화 카드 3장**(스킬/생선 강화+능력치강화 통합 풀에서 가중 랜덤·중복 없음)을 보여주고 1/2/3 키(또는 카드 클릭)로 택1. 레벨업 순간 **생선 `레벨업생선보상`개 자동 보충**. 선택 후 **직전 페이즈로 복귀**(전투 중이었으면 전투, 아니면 달리기).
- 평면 옆뷰(중력·점프 없음 — 러너지만 점프 아님, 좌우 온레일). 고양이·훼방꾼·HP바·팝업 모두 480×360 평면에 배치. **자율 난전 없음** — 언제나 고양이 1 vs 훼방꾼 1(또는 접근 중 1).

> **오토러너 + 조우전투 — 방/스테이지 클리어 대기 없음.** `거리`는 달리기 페이즈에 `달리기속도`만큼 계속 증가(진행도=점수 축). 난이도는 `거리`/`처치수`로 지수 스케일(`몬스터_성장률`)되고, `조우간격`(다음 적까지 거리)만큼 달린 뒤 다음 훼방꾼이 접근한다. `보스주기`마다(또는 일정 거리마다) 골목대장 불독(보스)이 길을 막는다. 고양이 HP 0까지 계속(fish-tank식 최고기록 도전).
> **🏃 달리기 표현(새 에셋 0 — 사용자 확정)**: 스프라이트 시트에 **러닝 프레임이 없다**(대기A/B·공격만). 달리는 느낌은 **"기존 `background.png` 재사용 + 스크래치 스크롤 + 고양이 `catIdle`↔`catIdle2`(대기A↔B) 빠른 바운스(다리 흔드는 느낌)"** 로 표현 — **새 배경 에셋은 만들지 않는다.**
> - **무한 스크롤**: 배경(또는 스크롤용 레이어)을 **2장 이어붙여**(같은 코스튬 2 스프라이트/2 클론) 왼쪽으로 흘리고, 화면 밖으로 나간 쪽을 오른쪽으로 **wrap**. 야경이라 이음선은 거의 안 보임(허용).
> - **전경 스크롤(전진감 강화)**: 추가로 **스크래치가 펜/도형으로 그리는 바닥 스트라이프·가로등·작은 소품**을 왼쪽으로 흘린다(배경보다 살짝 빠르게 = 시차). 고양이는 제자리 바운스.
> - **전투 페이즈**: 스크롤 정지(또는 감속). **러닝 프레임은 나중 확장**(Grok 러닝 시트 추가 시 교체, §3.1 노트).
> **🌊 바다로 흐르는 배경(무드 연출, 확장 옵션)**: `거리`가 오를수록 배경이 골목→항구→바다 쪽으로 흐르는 느낌을 주면 "언젠가 바다로" 서사가 산다. 기본은 무한 스크롤 1코스튬(야경)으로 충분하고, 여유 있으면 거리 구간별로 배경 코스튬을 바꾸거나(예: 3장) 배경색을 단계적으로 이동. **부담되면 확장 옵션으로 남김**(핵심 메커닉과 무관).

> **전투 판정 분리(rogue-knight 검판정과 동형)**: 데미지 판정을 좌표/`touching` 이 아니라 **타이머·확률 계산**으로 처리(한 조우에 1마리라 대상이 명확 — 항상 "그 훼방꾼"). 고양이 공격 타이머·스킬이 `훼방꾼에게 데미지`, 훼방꾼 공격 타이머가 `고양이에게 데미지`. **미시조작·조준 없음** → 성공-모양의 "온레일 전투".
> **액션 3칸(쿠키런식)**: 고양이가 스킬 2개를 `남은쿨1/2` 로 관리(매 틱 감소, 쿨 0이면 발동 가능·버튼 반짝), 생선은 `물약개수` 로 관리(>0이면 발동 가능). 발동 시 스킬은 효과(별빛 발톱=단일 큰 데미지 / 꼬리 회오리=여러 번 연타) 후 `남은쿨N = 스킬N_쿨` 리셋, 생선은 `체력 += 물약_회복` 후 `물약개수 -1`. 스킬은 각자 **레벨**(`스킬N_레벨`)이 있어 레벨업 카드로 개별 성장(데미지↑·쿨↓), 생선은 카드로 `물약_회복`·최대소지 성장.
> **플로팅 데미지 숫자(say 미사용)**: magic-survivor/rogue-knight 그대로. 색 5세트로 **일반/치명/스킬/피격/회복**을 구분해, 치명타는 금색·스킬은 주황·생선 회복은 초록(+숫자) 큰 숫자가 뜨는 손맛(기댓값·스킬 데미지·회복량의 시각화).

> 2. **처치→보상 경로 확실히 연결(detached clone 금지)**: 훼방꾼 클론은 `몬스터HP≤0` 이면 **먼저 전역(경험치·처치수·거리진행 신호)에 결과를 반영하고 `달리기재개` 방송**한 뒤 **마지막에 `delete this clone`**. 즉 삭제 전에 보상이 끝난다. 카운터를 클론 삭제 이후에 갱신하지 않는다.

> 4. **시작 즉사/즉종료 방지**: 시작은 **페이즈=1(달리기)** 로 시작해 첫 적이 `조우간격` 달린 뒤 등장(즉시 전투 아님). 첫 적은 항상 쥐(약)·기본 배수. 초기 HP 80·`몬스터_기본공격 4`·`몬스터_공격속도 1.4` 라 시작 직후 죽지 않는다. 게임오버는 `체력<1` 에서만.

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait`(0.02~0.03초)로 CPU 폭주 방지. **매직넘버 금지** — 항상 튜닝 변수 참조. 게임상태=1 일 때만 전투 로직이 돈다(=2,0 이면 정지).

> **공통 ①: 가중 랜덤 종류 1개 뽑기**(통합 풀 — 리스트 없이 누적합 7단 사다리. 스킬/생선강화 3 + 능력치강화 4)
> ```
> set 뽑기굴림 = pick random 1 to (발톱확률+회오리확률+생선확률+공격확률+체력확률+속도확률+치명확률가중)
> set 뽑기누적 = 0
> set 뽑기누적 = 뽑기누적 + 발톱확률       ; if 뽑기굴림 ≤ 뽑기누적 : set 뽑기종류 = 1 ; stop   # 별빛 발톱 강화
> set 뽑기누적 = 뽑기누적 + 회오리확률     ; if 뽑기굴림 ≤ 뽑기누적 : set 뽑기종류 = 2 ; stop   # 꼬리 회오리 강화
> set 뽑기누적 = 뽑기누적 + 생선확률       ; if 뽑기굴림 ≤ 뽑기누적 : set 뽑기종류 = 3 ; stop   # 생선 강화
> set 뽑기누적 = 뽑기누적 + 공격확률       ; if 뽑기굴림 ≤ 뽑기누적 : set 뽑기종류 = 4 ; stop   # 공격력+
> set 뽑기누적 = 뽑기누적 + 체력확률       ; if 뽑기굴림 ≤ 뽑기누적 : set 뽑기종류 = 5 ; stop   # 최대체력+
> set 뽑기누적 = 뽑기누적 + 속도확률       ; if 뽑기굴림 ≤ 뽑기누적 : set 뽑기종류 = 6 ; stop   # 공격속도+
> set 뽑기종류 = 7   # 나머지 = 치명확률+
> ```
> > **가중 랜덤의 핵심**: 가중치가 큰 종류일수록 `뽑기굴림`이 그 구간에 떨어질 확률이 크다 → `발톱확률`을 올리면 종류1(발톱 카드)이 자주 나온다(확률이 튜닝값대로 분포 — 학습 성립). 이 로직은 **커스텀 블록(정의: 종류뽑기)** 으로 빼면 3장 뽑기에서 재사용 편함.
>
> **공통 ②: 카드 1장 완성(종류→값→잭팟)**
> ```
> (공통① 으로 뽑기종류 결정)
> if (뽑기종류 ≤ 2):                                     # 스킬 강화 카드(1발톱/2회오리)
>   set 이_값 = 스킬레벨증가                             # 보통 1레벨 (잭팟이면 아래서 ×잭팟배수)
> else if (뽑기종류 = 3):                               # 생선 강화 카드(생선_회복↑)
>   set 이_배율 = (pick random (랜덤성장_최소*100) to (랜덤성장_최대*100)) / 100
>   set 이_값 = 생선회복증가 * 이_배율
> else:                                                  # 능력치 강화 카드(4공격/5체력/6속도/7치명)
>   set 이_기준 = (뽑기종류: 4→공격증가 5→체력증가 6→속도증가 7→치명확률증가)
>   set 이_배율 = (pick random (랜덤성장_최소*100) to (랜덤성장_최대*100)) / 100   # 소수 랜덤
>   set 이_값 = 이_기준 * 이_배율
> set 이_잭 = 0
> if (pick random 1 to 100) ≤ 잭팟확률 : set 이_값 = 이_값 * 잭팟배수 ; set 이_잭 = 1
> # (속도 종류는 소수라 반올림 생략, 나머지 능력치·생선·스킬레벨은 round 권장)
> ```
>
> **공통 ③: 3장 중복 없이 뽑기**(카드1/2/3 채우기 — 오버레이 진입 시 1회)
> ```
> repeat until (카드종류1≠0 and 카드종류2≠0 and 카드종류3≠0):
>   공통① → 뽑기종류
>   if (카드1 비었으면) : (공통② 값·잭 계산) 카드종류1/카드값1/카드잭1 세팅
>   else if (뽑기종류 ≠ 카드종류1) and (카드2 비었으면) : 카드종류2/… 세팅
>   else if (뽑기종류 ≠ 카드종류1,2) and (카드3 비었으면) : 카드종류3/… 세팅
>   # 이미 뽑힌 종류면 그냥 다시 굴림(종류 7개라 금방 3개 채움)
> ```
>
> **공통 ④: 종류 결정 + 지수 난이도 × 종별 배수**(훼방꾼 스폰 시 — 적 3종 전부 사용)
> ```
> # 1) 종류 결정 (거리/처치수 기반)
> if <<처치수 > 0> and <((처치수) mod 보스주기) = 0>>:      # 보스 주기
>   set 몬스터종류 = 3    # 보스 = 골목대장 불독
> else if <거리 ≥ 들개등장거리> and <(pick random 1 to 100) ≤ 들개확률>:
>   set 몬스터종류 = 2    # 들개(강적)  — (원하면 들개확률에 +거리보정)
> else:
>   set 몬스터종류 = 1    # 심술 쥐(기본/약) — 초반·자주
> # 2) 지수 스케일(공통 기저)
> set 기저HP   = 몬스터_기본HP   * (e^(처치수 * (ln 몬스터_성장률)))   # a^b = e^(b·ln a)
> set 기저공격 = 몬스터_기본공격 * (e^(처치수 * (ln 몬스터_성장률)))
> # 3) 종별 배수 적용
> if <몬스터종류 = 1>:   # 쥐
>   set 몬스터최대HP = round(기저HP * 쥐_체력배수)   ; set 몬스터공격 = round(기저공격 * 쥐_공격배수)   ; set 보스임 = 0
> if <몬스터종류 = 2>:   # 들개
>   set 몬스터최대HP = round(기저HP * 들개_체력배수) ; set 몬스터공격 = round(기저공격 * 들개_공격배수) ; set 보스임 = 0
> if <몬스터종류 = 3>:   # 보스 불독
>   set 몬스터최대HP = round(기저HP * 보스배수)      ; set 몬스터공격 = round(기저공격 * 보스배수)      ; set 보스임 = 1 ; play sound boss
> set 몬스터HP = 몬스터최대HP
> ```
> > 종류가 스폰 시 확정되고 클론이 `몬스터종류`로 자기 시트·크기·경험배수를 정한다. **처치 보상 경험치** = `처치경험치 × (종류1→1 / 종류2→들개_경험배수 / 종류3→보스배수)`.

**(B) 스킬 쿨 타이머 감소 forever** (페이즈 무관 항상 감소 — 전투 진입 즉시 발동 가능하게)
```
when I receive 게임시작
  forever
    if <게임상태 = 1>
      if <남은쿨1 > 0> : change 남은쿨1 by -0.05
      if <남은쿨2 > 0> : change 남은쿨2 by -0.05
    wait 0.05
```
> **빌더 노트**: 스킬 쿨은 초 단위 카운트다운(매 0.05초에 −0.05). `남은쿨N ≤ 0` 이면 발동 가능. 발동 시 고양이가 `남은쿨N = 실제쿨`(레벨 반영, 공통②-쿨식)로 리셋. 생선(슬롯3)은 쿨이 없고 `물약개수` 로만 게이팅. **쿨은 달리는 중에도 계속 차서** 전투 진입 시 바로 쓸 수 있다.

**(C) 레벨업 / 게임오버 감시 forever** (페이즈 1·2 어디서든)
```
when I receive 게임시작
  wait until <게임상태 = 1>
  forever
    if <<경험치 ≥ 레벨업경험치> and <게임상태 = 1> and <<페이즈 = 1> or <페이즈 = 2>>>
      set 직전페이즈 = 페이즈                                        # 복귀용(1 or 2)
      change 레벨 by 1
      change 경험치 by (0 - 레벨업경험치)
      set 레벨업경험치 = round( 기본경험치 * (경험치곡선 ^ (레벨 - 1)) )   # 지수 곡선
      set 페이즈 = 3
      broadcast 레벨업
    if <<체력 < 1> and <게임상태 = 1>>
      set 게임상태 = 0
      broadcast 게임오버
    wait 0.05
```
> **빌더 노트**: `직전페이즈` 는 진행 변수(또는 인라인). 강화완료 시 이 값으로 복귀. 레벨업 감시는 달리기(1)·전투(2) 어디서든 걸린다(경계에서 딱 1회).

**(D) 페이즈 워치독 forever** (데드락 안전망 — 전투인데 적이 사라졌으면 달리기 복귀)
```
when I receive 게임시작
  forever
    if <<게임상태 = 1> and <페이즈 = 2>>
      # 훼방꾼 클론이 하나도 없는데 전투 페이즈면(비정상) 일정 시간 후 달리기 복귀
      wait 0.5
      if <<페이즈 = 2> and <적존재 = 0>> : set 페이즈 = 1 ; set 조우거리 = 0
    wait 0.1
```
> **빌더 노트**: `적존재` = 훼방꾼 클론이 살아있으면 1(클론 start 에 1, delete 직전 0). 정상 흐름에선 처치 시 `달리기재개` 가 즉시 페이즈=1 로 돌리므로 워치독은 거의 안 탄다 — **detached/스폰 실패로 멈추는 최악을 막는 안전망**.

**(E) 게임시작 → 킥오프** (액션 버튼 생성; 첫 훼방꾼은 달리기 페이즈의 `조우거리` 트리거로 자동 등장)
```
when I receive 게임시작
  broadcast 액션버튼생성      # 하단 액션 3칸(발톱·꼬리 쿨바 + 생선 개수) 생성
  # 첫 적은 고양이(달리기 루프)가 조우거리 ≥ 조우간격 될 때 [몬스터등장] 방송 → 즉시 전투 아님(시작 여유)
```

### 배경스크롤 (기존 background.png 재사용 · 2장 무한 스크롤 — 새 에셋 0)

**(A) 스포너: 2조각 클론**
```
when I receive 게임시작
  if <복제됨 = 0>
    switch costume to background ; set 복제됨 = 1
    set 배경순번 = 0 ; create clone of myself   # 왼쪽 조각(x=0)
    set 배경순번 = 1 ; create clone of myself   # 오른쪽 조각(x=480)
    hide   # 원본 숨김
```
**(B) 클론: 왼쪽 스크롤 + wrap** (달리기 페이즈만 흐름)
```
when I start as a clone
  set x to (배경순번 * 480) - 240 ; set y to 0 ; go to back ; show   # 두 조각 이어붙임
  forever
    if <<게임상태 = 1> and <페이즈 = 1>>
      change x by (0 - 달리기속도)
      if <(x position) < -720> : change x by 960   # 480×2 만큼 오른쪽으로 wrap(두 조각 폭)
    else if <<게임상태 = 1> and <페이즈 = 2>>
      change x by (0 - 전투감속)                    # 0이면 정지(전투 중 스크롤 멈춤)
    wait 0.02
```
> **빌더 노트**: 무대 좌표라 실제 wrap 임계·오프셋은 배경 폭에 맞춰 조정(2장이 화면을 늘 덮게). 야경이라 이음선은 거의 안 보임(허용). 배경은 `go to back`.

### 전경스크롤 (바닥 스트라이프·가로등·소품 — 전진감, 시차)

```
when I receive 게임시작
  set pen ... (or 소품 코스튬)
  forever
    if <<게임상태 = 1> and <페이즈 = 1>>
      erase all ; (바닥 스트라이프/가로등/소품을 x 오프셋만큼 왼쪽 이동해 그림)
      change 전경오프셋 by (0 - (달리기속도 * 전경시차))   # 배경보다 빠르게 = 시차
      if <전경오프셋 < -(소품간격)> : change 전경오프셋 by 소품간격   # wrap

> **빌더 노트**: `내남은쿨`·`내전체쿨`·`이전준비`·`물약표시` 는 액션버튼 스크립트-로컬 임시(클론-로컬로 추가하거나 인라인). 슬롯1·2는 쿨 게이지(`1 - 남은쿨/전체쿨`), 막 끝난 엣지에서 ready 음 1회. 슬롯3(생선)은 쿨 없이 `물약개수` 만 표시(0이면 `empty` 회색). 실제 발동/소비는 고양이 (C)가 처리(버튼은 표시 전용).

### 강화카드 (레벨업 3장 — 가중 랜덤 · 중복 없음)

**(A) 깃발 초기화**
```
when green flag clicked
  hide ; set 복제됨 = 0
```

**(B) 레벨업 받으면 → 3장 뽑고 클론 3개** (원본만)
```
when I receive 레벨업
  if <복제됨 = 0>
    # ★ 레벨업 생선 보급(최대생선수 클램프) — 레벨업당 1회
    change 물약개수 by 레벨업생선보상
    if <물약개수 > 최대생선수> : set 물약개수 = 최대생선수
    set 카드종류1=0 ; set 카드종류2=0 ; set 카드종류3=0
    (공통③: 3장 중복 없이 뽑기 → 카드종류/값/잭 1·2·3 채움)
    play sound level
    if <<카드잭1=1> or <카드잭2=1> or <카드잭3=1>> : play sound jackpot
    set 카드번호 = 1 ; create clone of myself
    set 카드번호 = 2 ; create clone of myself
    set 카드번호 = 3 ; create clone of myself
```
> **빌더 노트**: 생선 보급은 여기(레벨업 수신 원본, 카드 뽑기 전)에서 **딱 1회**. 카드 클론들은 `복제됨=1` 이라 이 블록을 안 타므로 중복 보급 없음(원본만 실행).

**(C) 클론 본체** (자기 카드 렌더 + 1/2/3 키·클릭 선택)
```
when I start as a clone
  set 복제됨 = 1
  go to x:(-140 + (카드번호-1)*140) y:0           # 3장 가로 배치
  # ── 렌더: 카드종류로 아이콘, 카드값 표시(say 미사용 — 코스튬/자식 팝업 or 값 모니터) ──
  (switch costume to (카드종류i 아이콘) ; 잭팟이면 jackpot 금테 오버레이 ; 값 표시)
  show
  wait until <<페이즈 = 3> and <
       <key [카드번호] pressed?> or <<mouse down?> and <touching mouse-pointer?>>>>
  # ── 이 카드 적용(카드종류i → 대상 스탯/스킬레벨/생선). 아래 T=카드종류(카드번호), V=카드값(카드번호), J=카드잭(카드번호) ──
  # 스킬 강화(1~2): 해당 스킬 레벨 += V (쿨은 발동식에서 자동 반영)
  if <T = 1> : change 스킬1_레벨 by V       # 별빛 발톱 강화
  if <T = 2> : change 스킬2_레벨 by V       # 꼬리 회오리 강화
  # 생선 강화(3): 생선_회복 += V (잭팟이면 최대소지도 +1)
  if <T = 3> : change 생선_회복 by V
               if <J = 1> : change 최대생선수 by 1     # 잭팟 생선 카드는 소지 상한도 늘림
  # 능력치 강화(4~7)
  if <T = 4> : change 공격력 by V           # 공격력+
  if <T = 5> : change 최대체력 by V ; change 체력 by V     # 최대체력+ (현재 체력도 회복)
  if <T = 6> : change 공격속도 by (0 - V)   # 공격속도+
               if <공격속도 < 속도하한> : set 공격속도 = 속도하한
  if <T = 7> : change 치명확률 by V         # 치명확률+
               if <치명확률 > 치명확률상한> : set 치명확률 = 치명확률상한
  play sound pick
  set 페이즈 = 직전페이즈       # 레벨업 진입 전 페이즈로 복귀(전투였으면 2, 아니면 달리기 1)
  broadcast 강화완료          # 다른 두 카드 클론도 이 방송에 hide/delete
```
> **빌더 노트**: `T = 카드종류(카드번호)`, `V = 카드값(카드번호)`, `J = 카드잭(카드번호)` — 카드번호=1이면 `카드종류1`/`카드값1`/`카드잭1`… 를 읽는다는 뜻(3-way if 로 로컬 `T`/`V`/`J` 에 담거나 커스텀 블록). 카드 선택 대기는 **`페이즈 = 3`**(레벨업 페이즈)에서만. 스킬 카드(1~2)는 `스킬N_레벨` 을 올리고, 쿨 단축은 고양이 (C) 발동식 `max(스킬N_쿨 − (레벨-1)*0.5, 스킬쿨하한)` 에서 자동 반영. 생선 카드(3)는 `생선_회복`↑(+잭팟 시 `최대생선수`+1). 한 카드가 선택되면 **`페이즈 = 직전페이즈`(1 or 2 복귀)** 후 `강화완료` 방송 → **모든 카드 클론이 즉시 delete**·오버레이 hide. 선택키는 `카드번호`(1/2/3 키)·클릭. **생선 보급(`물약개수 += 레벨업생선보상`)은 위 (B) 레벨업 수신 원본에서 1회**(카드 선택과 별개).

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식 1층", "조건 2층", "새 메커닉 4층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화만 건드린다.

### 0층 — 숫자 하나 바꾸기 (무대 초록 깃발 스크립트)
- **치명타 잔치**: `치명확률` 을 `15` → `50` 으로. 금색 치명타 숫자가 절반이나 터진다(확률을 눈으로 확인!).
- **한 방 로망**: `치명타배수` 를 `2.0` → `5.0` 으로. 치명타가 뜨면 엄청난 데미지.
- **발톱 카드 홍수**: `발톱확률` 을 `15` → `100` 으로(+다른 가중치 `5`씩으로). 강화 카드가 거의 발톱 강화만 나온다(가중 랜덤 체감).
- **잭팟 천국**: `잭팟확률` 을 `8` → `40` 으로. 강화 카드가 자주 초대형 잭팟(금테)!
- **질주 모드**: `달리기속도` 를 `4` → `8` 로. 배경이 쌩쌩 흐르고 거리가 빨리 오른다(속도감↑).
- **연전 지옥**: `조우간격` 을 `60` → `20` 으로. 쉴 틈 없이 적을 계속 만난다(달리는 시간↓).
- **들개 러시**: `들개등장거리` 를 `150` → `0`, `들개확률` 을 `30` → `70` 으로. 처음부터 들개(강적)가 자주 나온다.
- **순둥 들개**: `들개_체력배수` 를 `2.2` → `1.2`, `들개_공격배수` 를 `1.6` → `1.1` 로. 들개가 쥐와 비슷해진다.
- **지옥 성장**: `몬스터_성장률` 을 `1.04` → `1.12` 로. 금방 벽에 부딪혀 진짜 RPG 손맛.
- **발톱 쿨 제로**: `강타_쿨` 을 `5` → `1.5` 로. 별빛 발톱을 거의 쉴 새 없이 난사(스킬쿨하한까지).
- **회오리 태풍**: `회오리_타수` 를 `4` → `10` 으로. 꼬리 회오리가 열 번 연타해 광역감·데미지 폭발.
- **생선 잔치**: `생선_회복` 을 `25` → `60` 으로, `레벨업생선보상`·`시작생선수`·`최대생선수` 를 넉넉히. 생선으로 계속 버틴다.
- **생선 기근**: `시작생선수` 를 `3` → `0`, `레벨업생선보상` 을 `2` → `0` 으로. 생선이 거의 없어 하드모드(보스 보급만).
- **보스 러시**: `보스주기` 를 `10` → `3` 으로. 몇 마리마다 골목대장 불독이 와서 긴장감↑(+생선 보급↑).

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **레벨 비례 공격**: 고양이 (B-battle)에서 `이번데미지 = 공격력` → `공격력 + (레벨 × 2)` 로. 레벨이 곧 화력.
- **잡을수록 커지는 보상**: `처치경험치` 를 `2` → `2 + floor(처치수/10)` 로 묶어, 후반에 경험치가 더 들어와 성장 가속.
- **멀리 갈수록 빨라지는 질주**: `달리기속도` 를 고정 대신 `4 + floor(거리/500)` 로 묶어, 오래 달릴수록 배경이 빨라진다.
- **거리 비례 들개**: `들개확률` 을 고정 대신 `30 + floor(거리/200)` 로 묶어, 멀리 갈수록 들개가 점점 더 자주.

### 2층 — if 조건 더하기
- **위기의 각성**: 고양이 (B-battle)에 `if 체력 < (최대체력/4) : set 이번데미지 = 이번데미지 × 2` — HP 낮으면 분노 버프.
- **보스 특급 보상**: 훼방꾼 처치에서 `if 보스임=1 : set 남은쿨1=0 ; set 남은쿨2=0` — 보스 잡으면 스킬 쿨 즉시 리셋(생선 보급은 이미 있음).
- **연속 치명 보너스**: 치명타가 뜰 때 `if 이번치명=1 : change 남은쿨1 by -0.5` — 치명타가 발톱 쿨을 앞당긴다.

### 3층 — 규칙 자체를 바꾸기 (분기)
- **강화 4장으로**: 강화카드 (B)의 3장 뽑기를 **4장**으로(카드종류4·클론4·4번 키 추가). 선택지가 넓어짐.
- **종류별 훼방꾼 성격**: 훼방꾼 (C)에서 `if 몬스터종류=1(심술 쥐) : set 몬스터_공격속도 = 몬스터_공격속도 × 0.7`(빠르게), 들개는 느리고 세게 — 종류마다 다른 위협.
- **더블 조우(온레일 유지)**: 처치 후 가끔 곧바로 다음 적을 붙여(짧은 `조우간격`) 연전을 만든다 — **동시 1마리 규칙은 유지**(순차).
- **바다로 흐르는 배경**: `거리` 구간별로 배경 코스튬(골목→항구→바다)을 바꿔 "언젠가 바다로" 서사를 연출.

### 4층 — 새 메커닉 만들기 (구조)
- **3번째 스킬(방패)**: 액션 풀에 `방패`(잠깐 무적/피해감소)를 추가 — 튜닝 변수(`방패_시간`·`방패_쿨`)·강화 가중치·발동 키(4/V)·`남은쿨3` 부활, 공통① 사다리에 한 단.
- **스킬 시너지**: 회오리 발동 중 발톱을 쓰면 데미지 2배 같은 콤보 조건을 고양이 (C)에 추가(오버차지 메커닉).
- **엘리트 훼방꾼**: 보스 외에 `pick random 1 to 20 = 1` 로 가끔 금색 엘리트(HP↑·보상↑·생선↑) 스폰(보스임 로직 재사용).
- **길 위 아이템**: 달리기 중 가끔 생선/코인이 오른쪽에서 흘러와 지나가며 줍는 러너식 픽업 추가.

## 10. 학습 포인트

**가중 확률·기댓값·지수 성장·위험/보상 선택을 밤거리를 달리는 랜덤 강화 미니 RPG(오토러너)로 몸으로 만나는 게임.** 추상 그래프가 아니라, 달리며 조우하고 레벨업마다 **카드를 직접 뽑고 고르며** 다음 직관이 게임 행동으로 체득된다:

- **가중 확률(weighted random)**: 강화 카드는 `발톱확률`·`회오리확률`·`생선확률`·`공격확률`… 7개 가중치에 비례해 뽑힌다(§7 공통① 누적합). **`발톱확률` 을 올리면 발톱 카드가 눈에 띄게 자주** 뜬다 — "확률을 바꾸면 결과 분포가 바뀐다"를 카드 뭉치로 관찰. 잭팟도 `잭팟확률`(%) 확률 판정이라, 숫자를 올리면 대박이 자주 난다.
- **기댓값(expected value)**: 치명타는 `치명확률`(%) 확률로 `치명타배수` 만큼 세게 때린다. **평균 데미지 = 공격력 × (1 + 치명확률 × (치명배수−1))**. 치명확률 15%·배수 2배면 평균 15% 더 세다 — 확률×배수가 평균 기여로 합쳐지는 기댓값을 데미지 숫자로 체감(금색 큰 숫자가 가끔 = 평균을 끌어올림). "치명확률 조금 vs 치명배수 크게, 어느 쪽이 이득?" 을 직접 계산·실험.
- **지수 성장 곡선**: 훼방꾼 HP·공격은 `몬스터_기본HP × 몬스터_성장률 ^ 처치수` 로 **지수적으로** 세진다. `몬스터_성장률` 을 1.04→1.10 으로 올리면 처음엔 비슷하다가 뒤로 갈수록 급격히 벅차진다 — 지수 곡선의 "처음 완만, 나중 폭발"을 난이도로 느낀다. `경험치곡선` 도 같은 지수(레벨업이 점점 느려짐). 스킬 데미지도 `스킬N_레벨` 로 층층이 자라(선형 성장) 지수 훼방꾼과의 힘겨루기가 된다.
- **위험/보상 선택 + 자원 관리(risk vs reward + resource)**: 카드 3장 중 무엇을 고를까 — 발톱 몰빵(한 방은 세지만 쿨 대기)? 회오리 광역(연타지만 쿨 김)? 생선 회복 키우기(딜은 못 늘지만 생존)? 능력치 안정(공격/체력/속도/치명)? 잭팟 카드를 노릴까 안전하게 갈까. **매 레벨업이 작은 의사결정**이라 성장 경로가 매 판 달라진다(로그라이크·쿠키런의 핵심 재미). 게다가 **생선을 지금 먹을까, 위기까지 아낄까** — 한정된 소비 자원(개수)을 언제 쓰느냐가 **자원 관리** 감각을 키운다. 스킬을 한 쪽에 몰면(발톱 특화) 폭딜 빌드, 고루 키우면 균형 빌드 — 아이가 자기 전략을 세운다.

부수적으로 **변수·조건·클론·누적합·거듭제곱·타이머(쿨다운)·자원 카운터(소지/소비/상한 클램프)·상태머신(페이즈 전이)·무한 스크롤(모듈러 wrap)·최댓값 갱신(하이스코어)** 을 게임을 튜닝하며(0→4층) 코딩 학습으로 이어간다. 코드 곳곳의 **노란 코멘트(가이드 투어)** 가 각 부분을 설명한다. 어디까지나 **낭만 고양이가 바다로 달리는 랜덤 성장 RPG 놀이**로 포장된 채 — 단일 초점(고양이)·1대1 온레일 조우전투라는 성공 게임 모양([[game-success-shape]])을 지킨다.
