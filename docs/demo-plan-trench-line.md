# 데모: 참호전선 (trench-line)

> **⚠ lane-siege 대체.** 앞서 만든 `docs/demo-plan-lane-siege.md`(단일 라인 근접 냥코식)는 사용자가 방향을 바꿔 **폐기**됐다. 이 문서가 그 대체 데모다. 콘셉트가 근본적으로 다르다 — **한 줄 근접 격돌이 아니라 3구역 참호 총격전(사격 교환)**이다.

> **데모 스코프** — 풀게임 아님. "3구역 참호전(밀고 밀리는 전선 + 사격 교환 + 데미지 팝업)이 Scratch에서 손맛이 나는가"만 검증받기 위한 빌드. 폴리시·밸런싱·컨텐츠(스테이지/보스/유닛 다양성) 전부 생략. 목표 규모 300~550 블록. 폴더 `demos/trench-line/`, 출력 `참호전선.sb3`.

> **존재 이유 = chess-war 실패 재발 방지 (1급 원칙).** 과거 chess-war 는 **per-unit 듀얼**(유닛마다 모든 적을 각자 탐색·전투)로 **O(n²)** 붕괴 → 폐기됐다. 이 데모는 그 함정을 **구조적으로** 피하는 **"구역 분할 + 이벤트 기반 공유 카운터 + Stage 상수시간 심판"** 모델이 실제로 손맛이 나는지 확인하는 것이 전부다. 아래 §성능 제약이 이 plan 의 1급 원칙이며, 빌더는 이를 절대 위반하지 말 것.

## 콘셉트
세로 컬럼 **3구역(좌/중/우)**. 각 구역이 **독립된 전선(theater)**이다. **화면 아래 = 아군 본진, 위 = 적 본진.** 그 사이는 노맨스랜드이고, **구역마다 교전선(전선)이 하나씩** 있다. 유닛은 배치되면 자기 구역 전선 부근에 자리 잡고 **노맨스랜드 너머 같은 구역의 적을 원거리 사격**한다 — 근접 격돌이 아니라 **사격 교환이 메인**. 명중하면 **데미지 숫자가 팝업**으로 떠올랐다 사라진다(이 플로팅 텍스트가 핵심 연출).

플레이어는 조준하지 않는다. **유닛 종류 버튼을 누른 뒤 좌/중/우 구역을 클릭**해서 그 구역 전선에 투입한다. "어느 전선을 보강하느냐"가 유일한 전략. 각 구역 전선은 그 구역 양측 전력차로 밀고 밀린다. 한 구역 전선을 적 본진까지 밀면 **돌파** → 적 본진 HP 감소. 적 본진 HP 0 → 승리, 아군 본진 HP 0 → 패배.

## ★ 핵심 설계 = "구역 분할 + 공유 카운터 심판" 모델 (이 구조를 반드시 그대로 구현)
per-unit 듀얼을 금지하고, **전투를 유닛 클론이 아니라 Stage 심판이 구역 전력 변수만 보고 상수 시간에 판정**한다.

1. **구역 전력 = 이벤트 기반 공유 카운터.** 유닛이 스폰될 때 `아군전력[구역] += 자기전력`, 사망 시 `-= 자기전력`. 매 틱 유닛을 순회해 합산하지 **말 것**(정 필요하면 프레임당 단일 O(n) 패스 1회만 허용, 유닛끼리 적을 보는 이중 스캔은 절대 금지).
2. **심판은 Stage 1곳.** 구역별 전력 변수 **6개**(3구역 × 2진영) + 전선위치 **3개**만 보고 **상수 시간**에 전선 이동·본진 딜을 처리. 클론 순회로 전투 판정하지 **말 것**.
3. **유닛 클론은** 자기 구역/전선 부근으로 이동 + 사격 애니메이션 + 데미지 팝업 생성 + **자기 사망 보고**만. **다른 유닛 클론을 `touching`/`sensing_of`/리스트로 참조하는 코드 0개** (검증 항목).
4. **밀고 밀리는 전선.** 각 구역 전선위치 `전선[z]` 는 그 구역 전력차 `아군전력[z] - 적전력[z]` 부호·크기에 비례해 매 틱 이동. 우세측이 전선을 상대 쪽으로 **전진**, 열세면 **후퇴**. 전선을 상대 본진 라인까지 밀면 **돌파**.
5. **돌파 = 본진 HP 딜.** `전선[z]` 이 적 본진 라인(`적진y`)에 닿으면 그 구역 돌파 → 적 본진 HP 초당 딜(돌파 지속 시). 대칭으로 아군 본진 라인(`아군진y`)에 닿으면 아군 본진 HP 딜.
6. **누가 죽나(HP 감소) = 구역별 선두 채널.** 열세 구역에서 심판이 주기적으로 **그 구역 전선에 가장 가까운(최전방) 열세측 유닛 1기**를 처치/피해. 구역별 "현재 피격 대상" 채널(구역당 선두 채널)로 O(1) 유지 — lane-siege 의 선두 채널 기법을 **구역별 ×3**로 확장. 유닛은 자기 y 를 구역 선두 채널(`아군선두y[z]`/`적선두y[z]`)에 min/max 로 기록만 하고, 심판이 채널값으로 대상을 지정(`구역별 피격신호 z`)해 그 좌표의 선두 1기가 피해를 받는다.

> **구현 힌트(빌더 재량)**: 배열형 변수 3개는 (a) 이름을 `아군전력L/M/R` 처럼 펼치거나 (b) 리스트 `L_아군전력`(index 1/2/3)로. 리스트로 하면 심판 루프가 `1~3 반복`으로 짧아짐(3회 고정 = 상수). 어느 쪽이든 **"모든 유닛이 모든 적을 본다"가 절대 나오면 안 된다.**

## 성능 제약 (plan 1급 원칙 — 빌더가 수치로 못박아 구현)
1. **구역당 진영별 동시 유닛 캡 = 6기** (`구역캡 6`). → 3구역 × 2진영 = 최대 36기지만 전체 캡도 별도로 **`전체캡 24`**. 캡 도달 시 그 구역 소환 거부(버튼/구역 회색·적 AI 스폰 스킵). 살아있는 클론 수를 `아군수[z]`/`적수[z]` + `아군총수`/`적총수` 로 카운트.
2. **전투 판정은 Stage 심판이 공유 변수로 상수 시간.** 전역 O(n²) 탐색·유닛 전체 `touching` 스캔·리스트 전투 순회 **0개.** 심판은 6전력 + 3전선 + 6선두채널만 본다(모두 상수 개수).
3. **데미지 숫자 팝업 클론: 총량 캡 `팝업캡 20` + 짧은 수명.** 팝업 클론은 떠올랐다 **0.4~0.6초 후 fade + `delete this clone`**. 동시 팝업 ≥ `팝업캡` 이면 **팝업 생략(전투는 계속)**. 이게 새 클론 부담원이니 특히 엄격히 — `팝업수` 카운터로 생성 게이트.
4. **사격은 히트스캔 권장(탄 클론 아님).** 참호전은 사격 빈도가 높으니 실제 탄 클론을 쏘면 폭증한다. **즉시 판정 + 트레이서(짧은 선/점멸)만** + 데미지 팝업. (빌더가 탄 클론을 택하면 **투사체 캡 12 + 화면 밖/명중 즉시 삭제** 필수. 데모 권장은 히트스캔.)
5. **유닛 3종만**(아래). 스테이지/진화/다양성 없음.

## 스프라이트 (8개 + Stage)
- **Stage**: 배경(3구역 컬럼 구분선 + 양쪽 본진 라인 + 노맨스랜드). **모든 전역 변수 초기화 한 곳** + **돈 자동 충전 루프** + **적 AI 스폰 타이머** + **★ 전선 심판 루프**(구역 1~3: 전력차로 `전선[z]` 이동, 돌파 시 본진 HP 딜, 열세 구역 선두 처치 신호 발행) + **승패 감시**(본진 HP≤0). 심판은 클론을 순회하지 않고 **상수 개수 변수만** 본다.
- **아군유닛**: 스포너+클론 본체. `아군소환`(구역=`소환구역`, 종류=`소환종류`) 받으면 클론 1기. 클론은 종류별 스탯 세팅 후 **자기 구역 x + 자기 구역 전선 부근 y** 로 이동해 대기. 자기 y 를 `아군선두y[z]` 채널에 갱신(전선에 가장 가까운 = 최전방). 사격 애니메이션(트레이서/총구 점멸) + **데미지 팝업 생성 방송**(캡·주기 제한). `구역별 피격신호 z` 로 자신이 선두 지목되면 피해(내HP−). 스폰 시 `아군전력[z]+=전력`·`아군수[z]+1`·`아군총수+1`, HP≤0 → 팝 이펙트·`아군전력[z]-=전력`·`아군수[z]-1`·`아군총수-1`·삭제.
- **적유닛**: 위 아군유닛의 미러(사격 방향·타깃 진영 반전, `적소환`/`적수[z]`/`적전력[z]`/`적선두y[z]`). 같은 3종 로직 재활용.
- **데미지팝업**: 플로팅 텍스트 클론(★핵심 연출). `팝업요청`(위치=`팝업x`/`팝업y`, 값=`팝업값`) 받으면 `팝업수<팝업캡` 일 때만 클론 1기 생성. 생성 즉시 위로 살짝 떠오르며 ghost↑(fade), **0.4~0.6초 후 `팝업수-1`·`delete this clone`**. 숫자 표시는 costume(0~9 조합) 또는 간단히 3~4단 세기 costume(약/중/강 데미지 색). say 금지 → 숫자 costume 스프라이트로.
- **유닛버튼**: 하단 3버튼(1 스프라이트에 3클론 또는 3스프라이트). 클릭 폴링+디바운스 → `대기소환종류` 세팅 + 버튼 하이라이트(선택 코스튬). `돈≥코스트 & 소환쿨끝` 아니면 회색.
- **구역클릭영역**: 3구역 투명 클릭 히트박스(1 스프라이트 3클론 또는 배경 마우스존 판정). `대기소환종류` 있고 그 구역 클릭 시 `아군소환`(소환구역=클릭구역·소환종류=대기값) 방송 + 조건검사(캡·쿨·돈). (빌더 재량: 구역별 소환버튼 3×3=9버튼으로 단순화 가능하나 '소환 후 구역 지정' 흐름 유지 권장.)
- **본진HP바 2개**: 아군/적 각 11단 costume-fill(상단·하단).
- **지갑바**: `돈/돈상한` 11단 costume-fill(최하단).
- **전선표시(선택)**: 구역별 전선 위치를 보여주는 마커 3클론(전선[z] y 따라 이동). 없어도 유닛 위치로 전선이 보이면 생략 가능(스프라이트 절약).

## 핵심 변수 (한글 손잡이)
리스트 3구역 배열 권장(index 1=좌,2=중,3=우): `L_아군전력` / `L_적전력` / `L_전선` / `L_아군수` / `L_적수` / `L_아군선두y` / `L_적선두y`(각 길이 3, 초기화 시 3칸 세팅). 스칼라: `돈` 40시작 / `돈상한` 200 / `돈충전율` 4(초당) / `구역캡` 6 / `전체캡` 24 / `팝업캡` 20 / `팝업수` 0 / `아군본진HP` 100 / `적본진HP` 100 / `돌파딜` 6(돌파 구역 초당 본진 딜) / `전선속도` 0.4(전력차 1당 전선 y 이동/틱, 상한 클램프) / `선두처치주기`(열세구역 최전방 처치 간격 틱) / `적스폰간격` 2.5(초) / `적진y` 150 / `아군진y` -130 / `구역x` L/M/R(-140/0/140) / `대기소환종류`(0=없음) / `소환구역` / `소환종류` / `게임상태`(1=플레이 0=종료).

> 클론-로컬(유닛): `복제됨`(증식 가드) / `내구역`(1~3) / `내종류` / `내전력` / `내HP` / `내사거리감` / `사격쿨`. 클론끼리 **참조 금지**(전선·선두·피격은 전부 공유 변수/리스트 경유). 클론-로컬(팝업): `복제됨` / `수명`.

## 방송 (~6개)
`게임시작`(초기화 후 루프 가동) / `아군소환`(구역=`소환구역`,종류=`소환종류`) / `적소환`(구역·종류=적 전용 변수) / `팝업요청`(위치·값 변수) / `구역별피격` L/M/R(심판이 열세 구역 선두 처치 신호 — 리스트/변수 `피격신호구역` 1값으로 통합 가능) / `게임끝`(승/패 배너).

## 템포 (손맛 = 데모의 전부)
- 시작 즉시 돈 40 → 첫 유닛을 5초 안에 한 구역에 투입 가능.
- 첫 사격 교환 + **데미지 팝업**이 10초 내 떠야 함 — "숫자가 튀는" 참호전 손맛이 빨리 보여야 한다.
- **구역 배분의 맛**: 3구역에 나눠 넣으면 전선이 셋 다 팽팽, 한 구역에 몰빵하면 그 구역만 빠르게 밀림(돌파) — "어디를 밀까"의 선택이 느껴지게.
- 제압사수로 한 구역 전선을 밀어 **돌파 → 적 본진 HP 깎기**가 15~30초 내 가능하게.
- 저격수로 열세 구역 견제(장거리)하는 조합의 맛.
- 한 판 40~90초. 이기거나 지거나 확실히 끝남.

## 사운드/연출 (데모 최소)
- 합성 효과음 4종: 소환(pop), 사격(pew/tk 짧게), 데미지팝업(tick), 돌파·본진 타격(crack). `_wav_bytes`+`synth_*`, 고아 0. 사격은 빈도 높으니 아주 짧게(연사 시 소리 겹침 부담 → 확률적 재생 허용).
- 데미지 팝업(떠오르며 fade)이 **핵심 연출**. 유닛 격파 시 팝(size↑+ghost), 돌파 시 그 구역 플래시/본진 흔들림 정도. 그 이상 폴리시 없음.

## 검증 (행동 시나리오 — headless scratch-vm 실측)
1. **소환+구역지정 게이트**: 유닛버튼 클릭→`대기소환종류` 세팅→구역 클릭 시 `돈≥코스트 & 소환쿨끝 & 아군수[z]<구역캡 & 아군총수<전체캡` 일 때만 그 구역 스폰 + `돈`−코스트 + 쿨 리셋 + `아군전력[z]`·카운트 증가. 조건 미달 시 스폰 안 됨(버튼/구역 회색).
2. **★ 유닛 클론이 다른 클론 스캔 0개 (chess-war 회피 — 핵심)**: 유닛 클론 스크립트에 **다른 유닛 클론을 `touching`/`sensing_of`/리스트 순회로 참조하는 코드 0개.** 유닛은 이동·정렬(선두 채널 갱신)·사격연출·팝업요청·자기 피격/사망 보고만. 전투 판정은 **Stage 심판이 공유 전력/전선/선두 변수(상수 개수)로** 처리함을 코드 구조로 확인.
3. **데미지 팝업 클론 총량 캡 + 자동삭제 (신규)**: `팝업요청` 시 `팝업수<팝업캡 20` 일 때만 생성, 생성 시 `팝업수+1`, **0.4~0.6초 후 fade + `delete this clone` + `팝업수-1`.** 사격 폭주 상황에서도 동시 팝업 ≤ `팝업캡`(초과 요청은 팝업 생략, 전투는 계속), 팝업 클론 누수 0.
4. **3구역 전선 밀고밀림이 전력차로 작동 (신규)**: 한 구역에만 아군 유닛 투입 → 그 구역 `아군전력[z] > 적전력[z]` → `전선[z]` 이 적진 쪽으로 전진, 반대면 후퇴. 심판이 **6전력+3전선만 보고 상수 시간**에 이동(클론 순회 없음). 세 구역 전선이 각자 독립적으로 움직임.
5. **돌파 시 본진 HP 감소 (신규)**: `전선[z]` 이 `적진y` 에 닿으면(돌파) 그 구역 지속 시 적 본진 HP `돌파딜`/초 감소, `적본진HP≤0` → 승리. 대칭으로 아군진y 돌파 시 아군 본진 HP 감소·패배. 본진 HP바 costume-fill 반영.
6. **열세 구역 선두 처치 O(1)**: 열세 구역에서 심판이 `선두처치주기` 마다 그 구역 **최전방(전선에 가장 가까운) 열세측 유닛 1기**만 피해/처치(전체 스캔 아님 — 구역 선두 채널 `선두y[z]` 경유). 처치 시 전력·카운트 감소.
7. **경제 루프 + 캡**: `돈`이 `돈충전율`로 차오르고 `돈상한`에서 멈춤, 소환마다 코스트 차감, 지갑바 반영. 구역당 유닛 ≤ `구역캡 6` & 전체 ≤ `전체캡 24`(초과 소환·적 스폰 거부).
8. **자동 플레이 60초 안정성**: 3구역에서 양측 유닛·팝업이 계속 스폰/삭제되는 동안 프레임 붕괴/클론 누수(유닛·팝업 모두) 없이 승패 결착. 캡·삭제가 실제로 총량을 억제하는지 클론 수 관찰.

## 기술 규칙
클론끼리 상호작용 금지(전선·선두·피격은 공유 변수/리스트 경유) / 전투는 Stage 심판 상수 시간(구역 6전력+3전선+6선두채널만, O(n²) 금지) / 유닛 순회 합산 금지(전력은 스폰/사망 이벤트로 ±) / 데미지 팝업 총량 캡 + 짧은 수명 자동삭제 + 초과 시 생략 / 사격 히트스캔 권장(탄 클론 시 캡+즉시삭제) / 유닛 구역캡·전체캡 / 돌파 판정은 전선 y vs 본진 라인 / vrep 리포터(`change x/y by 변수`) / 클릭 폴링+디바운스(`이 스프라이트 클릭했을 때` 지양) / say 금지(팝업 숫자는 costume) / `복제됨` 증식 가드(유닛·팝업 둘 다) / add_comment 3~5개(구역 분할 심판 모델·전선 밀고밀림·팝업 캡·경제 루프 설명).
