# 어항 포식자 (fish-tank) — Plan

> 평화로운 어항 안에서 **물고기 한 마리를 마우스로 조종**하며, 나보다 **작은 물고기만 골라 먹고**(닿으면 조금 성장 + 점수), 나보다 **큰 물고기는 피하는**(닿으면 원히트 게임오버) 피딩-프렌지 라이트. 다른 물고기들은 **플레이어를 추격하지 않고** 각자 자유롭게 배회만 한다(추격 AI 0). 먹어서 크기가 **목표크기**에 도달하면 CLEAR, 큰 물고기에 먹히면 GAME OVER.
> 베이스: `games/magic-survivor/build.py`(클론 스포너 · `복제됨` 증식 가드 · 한글 튜닝 변수 일괄 초기화 · 게임오버/클리어 배너 · pop.wav pitch 멀티유즈) + `games/gold-miner/build.py` 또는 `games/ghost-hunt/build.py`(마우스 추종 조작).
> 학습 콘셉트 없음(보너스 액션 게임). **이 게임의 진짜 존재 이유 = "놀이를 통한 배움" 튜닝 샌드박스** — 성장량·내속도·적속도·스폰빈도·큰물고기비율·먹기기준을 **깨끗이 이름 붙인 최상위 한글 변수**로 노출해, 숫자 하나만 바꿔도 난이도·손맛이 눈에 띄게 달라지게 한다(시후가 만지며 관찰). ★★☆☆☆.
> **성능 구조(핵심 설계 이유)**: 단일 초점 + 1대다. 물고기끼리 서로를 보지 않고(상호참조 0) 각자 배회만 하며, **먹기/피격 판정은 플레이어 스크립트 한 곳**에서만 일어난다 → 프레임당 O(n), O(n²) 자율 난전 없음. 동시 물고기 캡(`최대물고기`≈16) + 먹히면 제거·주기 리스폰으로 개체수 유지 → 클론 300 하드리밋 훨씬 아래.

## 1. 게임 한 줄

물고기 한 마리를 마우스로 몰고 다니며, 나보다 작은 물고기에 닿아 골라 먹으면 조금씩 커지고(점수 +), 나보다 큰 물고기에 닿으면 그 자리에서 잡아먹혀 GAME OVER. 다른 물고기는 나를 쫓지 않고 각자 어항을 유유히 배회한다. 내 크기가 `목표크기`에 닿으면 CLEAR. 깃발을 다시 누르면 크기가 기본값으로 리셋된다.

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

```
 (-240,180)                                            (240,180)
   ┌──────────────────────────────────────────────────────┐
   │ 🐟크기:60/120   ⭐점수:0                              │  ← 좌상단 모니터 2종
   │      🐠(작은·초록)          🦈(큰·빨강경고톤)          │  큰 물고기 = 위험(피해!)
   │                    🐟(내 물고기·강조 테두리)           │  내 물고기: 마우스 따라 부드럽게 헤엄
   │   🐠(작은)                       🐡(중간·노랑)         │  작은/중간 = 먹기 후보
   │            🐡(중간)      🐠(작은)                      │  모두 각자 자유 배회(추격 X)
   │  🦈(큰)                             🐠(작은)          │
   │        🐠(작은)             🐡(중간)                   │
   │   마우스 커서로 헤집으며 작은 걸 골라 먹기!            │
   └──────────────────────────────────────────────────────┘
 (-240,-180)                                          (240,-180)

  내 크기 ≥ 목표크기  → 화면 중앙에 CLEAR 배너.
  큰 물고기에 닿음     → 화면 중앙에 GAME OVER 배너 → 깃발 재클릭으로 처음부터(크기 리셋).
```

- HUD: 좌상단에 `내크기`(/`목표크기`) · `점수` 모니터 2종만. 튜닝 변수(성장량 등)는 **모니터 숨김**(코드에서만 본다).
- 배경: 물빛 그라데이션 어항 + 바닥 자갈/수초 몇 개 + 살짝의 기포. 평면(중력 없음), 480×360 전 영역이 헤엄 공간.
- **가독성이 핵심 재미 요건**: 티어별 색/톤으로 "먹어도 돼 / 피해"가 한눈에 구분되어야 한다.
  - 작은(초록/파랑, 둥글고 귀여움) → 안전·먹이 톤
  - 중간(노랑/주황) → 중립 톤
  - 큰(빨강·이빨/경고 실루엣) → 위험 톤
  - **내 물고기**는 밝은 색 + 흰 테두리(아웃라인) 강조로 남의 물고기와 즉시 구분.
- 실제 위험/먹이 여부는 **색이 아니라 크기 비교**로 판정하되(성장하면 관계가 뒤집힘), 스폰 티어 색이 크기와 상관관계를 갖게 만들어 초기 직관을 돕는다.

총 5 타깃(Stage 포함).

> **아트 방향**: 모두 옆모습(측면), `rotationStyle: left-right`(왼→오 헤엄 시 코스튬만 좌우 반전, 뒤집힘 없음). 티어 3종은 **각각 다른 코스튬 + 다른 색**으로 한눈에 구분(작=초록·둥글 / 중=노랑 / 대=빨강·이빨). 내 물고기는 흰 아웃라인으로 강조.
> **1대다 · 단일 상호작용 지점(이 게임의 핵심 패턴)**: 물고기 클론끼리는 서로 `touching`/`distance`를 **절대 보지 않는다**. 각 클론은 오직 (a) 배회하고 (b) 벽에서 튕기고 (c) 게임오버 시 자기를 정리할 뿐이다. **먹기·피격 판정은 전부 `내물고기` 스크립트 한 곳**에서 `touching [물고기]` + 크기 비교로 처리 → 프레임당 비용이 클론 수에 선형(O(n)).
> **크기 비교 채널**: `내물고기`가 어떤 클론에 닿았을 때 "그 클론의 크기"를 알아야 한다. Scratch에서 `touching`은 어느 클론에 닿았는지 알려주지 않으므로, **닿은 클론 스스로가 자기 크기를 전역 `접촉물고기크기`에 써 보고**하는 핸드셰이크를 쓴다(§7 참조). 이 값을 내 크기와 비교해 먹기/피격을 결정.

### 5.1 ⚙️ 개조 손잡이 (튜닝 변수) — 한글 전역 변수, ID `V_*` / `var*`

> **이 표가 이 게임의 심장이다.** 모든 조절 가능한 값은 여기 한글 변수로만 존재하고, 코드 어디서도 매직넘버를 쓰지 않는다(예: `change size by 성장량`이지 `change size by 4` 아님). 변수 초기화는 **전부 Stage 깃발 클릭 한 스크립트**에 모은다. (배회 방향전환 `repeat`, 파티클 `repeat 6` 같은 연출용 소수 상수만 인라인.)

총 튜닝 전역 변수 **14개**.

> **먹기 판정 규칙(핵심)**: 내물고기가 어떤 클론에 닿았을 때 —
> - `접촉물고기크기 ≤ (내크기 × 먹기기준)` → **먹는다**: 그 클론 삭제, `내크기 += 성장량`, `점수 += 점수당먹기`, 뽁 연출.
> - 그 외(상대가 나보다 크거나 먹기기준 초과) → **잡아먹힘**: 원히트 게임오버.
> - `먹기기준`은 "상대가 내 크기의 몇 % 이하이면 먹히나"를 의미(예 0.9 = 90% 이하면 먹음). 이 한 값이 난이도·손맛을 크게 좌우한다.
>
> **성장으로 관계가 뒤집힌다**: 처음엔 큰 물고기(size 100)를 못 먹지만, 내 크기가 커져(`내크기` > 111 이면 100×먹기기준 통과) 나중엔 먹게 된다 → 골라먹기의 성장 재미.

> 물고기 3개(복제됨/내물고기크기/배회각), 뽁 1개(복제됨). 클론-로컬 합계 **4개**.

### 5.4 메시지(방송) — ID `BR_*` / `br*`

| 한국어 | ID | 트리거 → 동작 |
|--------|----|---------------|
| 게임시작 | `brStart01` | Stage 깃발 초기화 직후 → 물고기 스포너 가동, 내물고기 판정 루프 가동 |
| 뽁생성 | `brPop02` | 내물고기가 물고기를 먹는 순간(`뽁X/Y`=먹은 위치) → 뽁 원본이 그 자리에 파티클 클론 생성 |

총 방송 **2개**.

> v1은 방송 최소. 스폰은 물고기 원본의 자체 forever 루프(게임시작 수신)로 돌리므로 별도 스폰 방송 불필요.

## 6. 씬 / 상태머신

```
[깃발 클릭]
   │  Stage: 튜닝 변수 14개 + 진행 변수 전부 초기화
   │         (내크기=시작크기, 게임상태=1, 점수=0, 물고기수=0 …)
   │  내물고기: 위치(0,0), size=시작크기
   ▼
[게임시작 방송]
   │  물고기 스포너 forever 가동 / 내물고기 마우스추종·판정 forever 가동
   ▼
[플레이중 forever]  (게임상태=1)
   ├ 내물고기: 매 틱 마우스 쪽으로 내속도만큼 이동(부드럽게) + size=내크기 반영
   │            touching [물고기] → 닿은 클론이 접촉물고기크기 보고 → 비교:
   │              ≤ 내크기×먹기기준 → 먹기(클론 삭제/성장/점수/뽁)  else → 게임오버
   ├ 물고기 스포너: 물고기수 < 최대물고기 이면 스폰간격마다 1마리(큰물고기비율로 티어)
   ├ 물고기 클론: 자유 배회(배회각 방향으로 적속도 이동, 벽에서 방향 전환, 가끔 각 살짝 틀기)
   │              — 플레이어 추격/타 클론 참조 절대 없음
   └ Stage 감시: 내크기 ≥ 목표크기 → 게임상태=2(클리어)
   ▼
[클리어]  (게임상태=2)   ← 결과 배너(CLEAR) show, 물고기·스포너 정지(각 루프가 게임상태=1만 처리)
   ▼
[게임오버] 큰 물고기에 먹힘 → 게임상태=0
   │  결과 배너(GAME OVER) show / 잔여 물고기·뽁 클론 자기 삭제
   ▼
[다시 시작]  초록 깃발 재클릭(크기 기본값으로 — 점수·크기 리셋)
```

> v1 승패 결착은 **목표크기 도달=승 / 큰 물고기 접촉=패** 로 명확히 고정(엔들리스 아님). 빌더 구현이 단순한 쪽.

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait` 넣어 CPU 폭주 방지. 이동·판정 틱은 0.025초(≈40fps). **매직넘버 금지** — 항상 튜닝 변수 참조.

### Stage

**(A) 깃발 클릭 → 변수 전부 초기화(한 곳에 모음) → 게임시작**
```
when green flag clicked
  # ── 개조 손잡이(튜닝) 기본값: 아이가 여기 숫자만 바꾸면 게임이 바뀐다 ──
  set 성장량=3 ; set 내속도=6 ; set 적속도=1.6 ; set 스폰간격=0.7
  set 큰물고기비율=0.2 ; set 중간물고기비율=0.3 ; set 먹기기준=0.9
  set 최대물고기=16 ; set 시작크기=60 ; set 목표크기=120
  set 작은크기=35 ; set 중간크기=65 ; set 큰크기=100 ; set 점수당먹기=1
  # ── 진행 상태 ──
  set 게임상태=1 ; set 점수=0 ; set 내크기=시작크기 ; set 물고기수=0
  set 접촉물고기크기=0
  wait 0.3
  broadcast 게임시작
```

**(B) 클리어 감시 forever**
```
when I receive 게임시작
  wait until (게임상태 = 1)
  forever
    if (내크기 >= 목표크기) and (게임상태 = 1)
        set 게임상태 = 2                ← 클리어
        set sound effect [pitch] to 240 ; play sound pop
    wait 0.05
```
> 게임오버(게임상태=0)는 내물고기 판정에서 직접 세팅하므로 여기선 클리어만 감시.

### 내물고기 (마우스 추종 + 유일한 상호작용 지점)

**(A) 깃발 초기화**
```
when green flag clicked
  show ; set rotation style [left-right] ; point in direction -90
  goto (0,0) ; go to front
```

**(B) 마우스 추종 이동 + 크기 반영 forever**  (부드럽게 헤엄)
```
when I receive 게임시작
  forever
    if (게임상태 = 1)
        set size to 내크기 %                       ← 성장 반영
        # ── 마우스 쪽으로 부드럽게(스냅 아님) ──
        if ((distance to [mouse-pointer]) > 4)
            point towards [mouse-pointer]
            move 내속도 steps
    wait 0.025
```
> `point towards [mouse-pointer] + move 내속도`가 부드러운 추종. `rotationStyle left-right`라 오른쪽/왼쪽 헤엄에 코스튬만 반전(위아래로 몸이 뒤집히지 않음). `distance>4` 가드로 커서 위 미세 떨림 방지.

**(C) 먹기/피격 판정 forever**  (이 게임의 유일한 상호작용 — 물고기끼리는 서로 안 봄)
```
when I receive 게임시작
  forever
    if (게임상태 = 1) and (touching [물고기])
        # 닿은 클론이 자기 크기를 접촉물고기크기에 방금 써 두었음(물고기 (C) 참조)
        if (접촉물고기크기 <= (내크기 * 먹기기준))
            # ── 먹는다 ──
            set 뽁X = x position ; set 뽁Y = y position
            broadcast 뽁생성
            change 내크기 by 성장량
            change 점수 by 점수당먹기
            set sound effect [pitch] to 150 ; play sound pop     ← 뽁
            set sound effect [pitch] to 60  ; play sound pop      ← 성장(선택; 합쳐도 됨)
            # (닿은 클론 삭제는 물고기 (C)에서 자기가 처리 — 아래 노트)
        else
            # ── 나보다 큼 → 잡아먹힘 ──
            set sound effect [pitch] to -300 ; play sound pop
            set 게임상태 = 0                 ← 게임오버
    wait 0.02
```
> **빌더 핵심 노트 — 접촉 클론 식별·삭제**: `touching [물고기]`는 "어느 클론"인지 못 준다. 그래서 판정을 **물고기 클론 쪽에서 완결**한다(아래 물고기 (C)): 각 클론이 매 틱 `touching [내물고기]`를 검사해, 닿았으면 자기 크기를 `접촉물고기크기`에 쓰고, 자기가 먹힐 크기면(`내물고기크기 ≤ 내크기×먹기기준`) **스스로 삭제**한다. 내물고기 (C)는 성장·점수·연출만 담당. 이 **두 스크립트가 같은 "먹을 수 있나?" 조건을 각자 평가**해 서로 자기 몫만 처리(중복참조 없이 단일 프레임에 성립). 게임오버 조건(내가 작음)은 내물고기 (C)가 세팅.
> - 정리하면: **먹힐 조건이면** → 클론은 자기 삭제 + `접촉물고기크기` 보고 / 내물고기는 성장·점수·뽁. **먹힐 조건 아니면(내가 작음)** → 내물고기가 게임오버. 클론은 삭제 안 함(살아남음).

### 물고기 (스포너 + 배회 클론 본체)

**(A) 깃발 초기화**
```
when green flag clicked
  hide ; set rotation style [left-right] ; set 복제됨 = 0
```

**(B) 스폰 forever → 캡 아래에서 티어별 1마리씩** (원본만)
```
when I receive 게임시작
  if (복제됨 = 0)
      forever
          if (게임상태 = 1) and (물고기수 < 최대물고기)
              change 물고기수 by 1
              create clone of [myself]
          wait 스폰간격
```
> 캡(`물고기수 < 최대물고기`)이 있어 클론이 무한 증식하지 않음. 먹히거나 게임오버로 클론이 줄면 다시 채워짐.

**(C) 클론 본체** (티어 결정 → 자유 배회 → 접촉 시 자기 몫 처리)
```
when I start as a clone
  set 복제됨 = 1
  # ── 티어(작/중/대) 결정: 큰물고기비율/중간물고기비율 확률 ──
  set r = pick random 0 to 1                         ← r: 인라인/임시(진행변수 추가 가능)
  if (r < 큰물고기비율)
      set 내물고기크기 = 큰크기 ; switch costume to [큰]
  else if (r < (큰물고기비율 + 중간물고기비율))
      set 내물고기크기 = 중간크기 ; switch costume to [중간]
  else
      set 내물고기크기 = 작은크기 ; switch costume to [작은]
  set size to 내물고기크기 %
  # ── 화면 가장자리 한 곳에서 등장 + 초기 배회 방향 ──
  goto (pick random -230 to 230, pick random -160 to 160)
  set 배회각 = pick random 0 to 359
  point in direction 배회각
  show
  forever
    # 1) 게임오버/클리어 시 정리
    if (게임상태 = 0) or (게임상태 = 2)
        change 물고기수 by -1 ; delete this clone
    # 2) 자유 배회(추격 없음! 각자 유유히)  — 게임상태=1
    if (게임상태 = 1)
        point in direction 배회각
        move 적속도 steps
        # 벽에서 방향 전환(가장자리 튕김)
        if on edge, bounce
        set 배회각 = direction                         ← 튕긴 뒤 새 방향 저장
        # 가끔 살짝 방향을 틀어 자연스러운 방황
        if (pick random 1 to 30) = 1
            change 배회각 by (pick random -25 to 25)
        # 3) 내물고기 접촉 처리(자기 몫만) — 물고기끼리는 서로 안 봄
        if (touching [내물고기])
            set 접촉물고기크기 = 내물고기크기            ← 크기 보고 채널
            if (내물고기크기 <= (내크기 * 먹기기준))
                change 물고기수 by -1
                delete this clone                       ← 내가 작아 먹힘 → 사라짐
            # (내가 크면: 삭제 안 함. 내물고기 (C)가 게임오버 처리)
    wait 0.025
```
> 빌더 노트: `r`은 티어 확률 판정용 임시(진행 변수 `티어난수`로 추가하거나, `pick random 1 to 100` 정수로 바꿔 `< 큰물고기비율*100` 식으로 비교해도 됨). `on edge, bounce`는 `left-right` 스타일이라 코스튬이 자연스럽게 헤엄 방향으로 반전됨. **어떤 클론도 `내물고기`를 향해 `point towards` 하지 않는다 — 추격 AI 0** (검증 4-2 항목).

### 뽁 (먹기 파티클 — 최소 폴리시)

**(A) 깃발 초기화**
```
when green flag clicked
  hide ; set 복제됨 = 0 ; set rotation style [don't rotate]
```

**(B) 뽁생성 → 그 자리에 파티클 클론** (원본만)
```
when I receive 뽁생성
  if (복제됨 = 0)
      create clone of [myself]
```

**(C) 클론 본체** (그 자리에서 커지며 페이드 후 삭제)
```
when I start as a clone
  set 복제됨 = 1
  goto (뽁X, 뽁Y) ; set size to 60 % ; set [ghost] effect to 0
  go to front ; show
  repeat 6
      change size by 12 ; change [ghost] effect by 15 ; wait 0.02
  delete this clone
```

### 결과 (CLEAR / GAME OVER 배너)

```
when green flag clicked
  hide ; goto (0,0) ; set size to 100 % ; go to front

when I receive 게임시작
  wait until (게임상태 = 0) or (게임상태 = 2)
  if (게임상태 = 2) : switch costume to [clear]
  else              : switch costume to [over]
  show          ← 배너에 점수·크기 모니터가 함께 보여 결과 확인
```

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식 1층", "새 메커닉 3층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화 스크립트만 건드린다.

### 0층 — 숫자 바꾸기 (한 곳만 고친다)
- **쑥쑥 성장**: `성장량`을 `3` → `10`으로. 한 마리만 먹어도 훌쩍 커진다.
- **느긋한 어항**: `적속도`를 `1.6` → `0.5`로. 물고기들이 느릿느릿, 골라먹기 쉬움.
- **상어 소굴**: `큰물고기비율`을 `0.2` → `0.6`으로. 위험한 큰 물고기가 절반 넘게! 살 떨리는 난이도.
- **대식가 판정**: `먹기기준`을 `0.9` → `1.2`로. 나보다 살짝 큰 물고기까지 삼킨다(쉬워짐). 반대로 `0.6`으로 낮추면 확실히 작아야만 먹혀 까다롭다. **먹히는/못 먹는 경계가 바뀌는 걸 직접 관찰!**
- **초스피드 물고기**: `내속도`를 `6` → `12`로. 손맛이 쌩쌩(단, 예민해져 큰 물고기 실수 접촉 주의).
- **북적북적**: `최대물고기`를 `16` → `28`로. 어항이 물 반 고기 반(밀도·난이도↑).
- **금방 클리어**: `목표크기`를 `120` → `80`으로. 몇 마리만 먹어도 CLEAR.

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **먹을수록 빨라지는 나**: 내물고기 (B)의 `move 내속도` → `move (내속도 + (내크기 / 20))`. 커질수록 더 빨리 헤엄.
- **커질수록 둔해지는 나**: 반대로 `move (내속도 - (내크기 / 40))`(최소 클램프). 거대해질수록 무거워지는 손맛.
- **점수 배율**: 큰 물고기를 먹을 때 더 큰 점수 — 내물고기 (C) `change 점수 by 점수당먹기` → `change 점수 by (접촉물고기크기 / 30)`(큰 걸 먹을수록 점수↑).

### 2층 — if 조건 더하기
- **큰 물고기는 느리게**: 물고기 (C) 배회에서 `if (내물고기크기 = 큰크기) : move (적속도 / 2) steps`를 섞어 큰 물고기만 굼뜨게(피하기 여유).
- **미니 무적 시작**: 게임시작 후 `wait 2` 동안 내물고기 (C) 판정을 건너뛰어(초반 유예), 시작하자마자 억울하게 안 죽게.

### 3층 — 규칙 자체를 바꾸기 (분기/새 메커닉)
- **엔들리스 모드**: Stage (B) 클리어 감시를 지우고, 대신 목표크기 도달 시 `내크기`를 절반으로 줄이고 계속 진행 → 최고 점수 도전 모드로.
- **황금 물고기**: 물고기 (C)에 4번째 티어(작은 확률의 반짝이 코스튬)를 추가해 먹으면 `성장량 × 3` 성장.
- **가시복 함정**: 어떤 티어를 "작아 보이지만 먹으면 게임오버"인 함정으로 만들어(먹기 조건 통과해도 별도 플래그로 즉사), 색만 보고 방심하지 않게.

## 10. 학습 포인트

보너스 액션 게임 — 수학 학습 콘셉트 없음([[feedback-game-design]] 준수). **이 게임의 진짜 존재 이유 = "놀이를 통한 배움" 튜닝 샌드박스**: 성장량·내속도·적속도·스폰간격·큰물고기비율·먹기기준 같은 손잡이가 전부 한글 전역 변수로 노출되어, 아이가 (0층) 숫자를 바꾸고 → (1층) 값을 식으로 묶고 → (2층) if 조건을 더하고 → (3층) 새 티어/모드를 만들며, 변수·연산·비율(확률)·조건·클론이라는 프로그래밍 개념을 게임을 망가뜨리고 고치며 체득한다. 특히 **`먹기기준`(비율 비교)·`큰물고기비율`(확률)** 같은 값은 바꾸는 즉시 거동이 눈에 띄게 달라져(먹히던 게 안 먹히고, 위험 개체가 늘고) "숫자가 규칙을 만든다"는 감각을 직관적으로 준다 — 어디까지나 귀여운 어항 액션으로 포장된 채.

## 11. 빌드 / 검증 체크리스트

- [ ] zip 열기 성공, `project.json` 파싱 성공, 모든 자산 `(md5).svg`/`.wav` 존재
- [ ] 타깃 5개: Stage / 내물고기 / 물고기 / 뽁 / 결과
- [ ] 전역 변수 **정확히 21개**(튜닝 14 + 진행 7), 클론-로컬 **4개**(물고기 3 / 뽁 1), 방송 **정확히 2개**(게임시작/뽁생성)
- [ ] **튜닝 변수 14개가 Stage 깃발 한 스크립트에서 전부 초기화**(매직넘버 산재 없음). 이동/먹기/성장/스폰/배회가 해당 변수 참조(`move 내속도 steps`, `move 적속도 steps`, `change 내크기 by 성장량`, `wait 스폰간격`, `set size to 내물고기크기` 등)
- [ ] **(1) 먹기/피격 규칙**: 작은 물고기(≤ 내크기×먹기기준)에 닿으면 → 클론 삭제 + `내크기 += 성장량` + `점수 += 점수당먹기` + 뽁 연출. 큰 물고기(> 내크기×먹기기준)에 닿으면 → `게임상태=0`(원히트 게임오버). 성장으로 내크기가 커지면 이전에 못 먹던 큰크기(100)도 먹게 됨(관계 역전)
- [ ] **(2) 배회 AI(추격 없음)**: 물고기 클론 스크립트에 **`point towards [내물고기]`·`distance to [내물고기]`(추격/조준) 및 타 클론 참조가 0개**. 배회는 `배회각` 방향 이동 + `on edge, bounce` + 가끔 각 틀기뿐. 물고기끼리 `touching`/`distance` 상호참조 없음(먹기 판정은 내물고기 스크립트 + 각 클론의 `touching 내물고기` 자기처리만)
- [ ] **(3) 동시 캡·리스폰·클론 누수 0**: 스포너가 `물고기수 < 최대물고기`일 때만 스폰. 먹힘·게임오버·클리어 시 `물고기수` 감소 + 클론 자기 삭제. 60초 후에도 클론 총량이 캡(≈16) 부근 유지(폭주·누수 없음)
- [ ] **(4) 튜닝 변수 실효**: `먹기기준`을 낮추면 같은 크기에서 덜 먹히고(까다로움)·올리면 더 먹힘, `성장량`을 올리면 더 빨리 커짐, `큰물고기비율`을 올리면 큰(위험) 티어 스폰 비율이 실제로 증가 — headless로 스폰 티어 분포/성장 반영 확인
- [ ] **(5) 클리어/게임오버 결착**: `내크기 ≥ 목표크기` → 게임상태=2 → CLEAR 배너. 큰 물고기 접촉 → 게임상태=0 → GAME OVER 배너. 배너 코스튬(clear/over) 분기, 잔여 클론 정리. 깃발 재클릭으로 크기·점수 기본값 리셋
- [ ] **(6) 60초 안정성**: 60초 연속 실행 시 클론 총량 캡 유지, 변수 폭주·에러 없음, 게임상태 전이(1→2 / 1→0) 정상
- [ ] **가독성**: 티어 3종이 서로 다른 코스튬·색(작=초록/중=노랑/대=빨강경고), 내물고기는 흰 아웃라인 강조로 남의 물고기와 즉시 구분
- [ ] 말풍선(say) 0개, 사운드 4종(또는 최소 3종) 전부 `pop.wav` pitch 재사용(고아 사운드 0)
- [ ] headless scratch-vm 실행: 변수 초기화·스폰(캡 준수)·물고기수 증감·게임상태 전이 확인. (touching 기반 먹기/피격·배회·뽁 연출은 headless 한계 → opcode/구조 존재 + 추격 opcode 부재로 확인)
