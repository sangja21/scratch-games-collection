# 데모: 스커미시 (skirmish)

> **데모 스코프** — 풀게임 아님. "여러 영역에 유닛이 자율적으로 기어나와 서로 알아서 싸우는 소규모 난전"이 Scratch에서 성립하는가만 검증. 폴더 `demos/skirmish/`, 출력 `스커미시.sb3`.

> **승리 조건 = 중앙 본부(HQ) 격파.** 양측 본부(아군=하단 y≈-155, 적=상단 y≈155)는 **총알이 touching 하면 HP가 깎이는 히트박스 스프라이트**(아군본부/적본부). 유닛은 근처 적이 없거나 사거리 밖이면 적 본부로 전진해 사격 + 교전선이 느리게 본부 쪽으로 밀려 올라가 교착을 방지. 먼저 상대 본부를 깨면 승리. (총알→본부, 총알→유닛 모두 동일한 collision O(1) 방식.)

> ⚠️ **배경 = trench-line(3구역 참호전) 폐기.** 구역·전선·심판 매크로 전부 버림. 대신 chess-war를 O(n²)로 죽였던 **per-unit 자율 전투**를 부활시키되, **유닛 총량을 하드캡으로 아주 적게 묶어(아군≤7·적≤7) O(n²)를 감당 가능하게** 만드는 게 설계 핵심. "클론끼리 스캔 0개" 규칙을 **이번엔 캡 조건 하에 의도적으로 허용**한다.

## 새 모델 = 소규모 자율 난전
- **구역·전선·심판 없음.** 열린 전장: 화면 아래=아군 본진(y≈-150), 위=적 본진(y≈150), 사이는 자유 필드.
- 유닛은 자기 본진 부근 랜덤 x 에서 스폰돼 **자율적으로** 움직인다.
- **하드캡(성능 안전장치 — 절대 준수): 아군 동시 ≤7, 적 동시 ≤7(합 ≤14).** 캡 도달 시 소환/적 스폰 거부. 이 캡이 O(n²)를 감당 가능하게 하는 유일한 근거.

## 공유 로스터 (슬롯 7칸 고정, 병렬 리스트)
각 진영은 **길이 7 고정** 병렬 리스트로 로스터 유지:
- 적 진영(아군이 스캔): `L_적x` `L_적y` `L_적hp` `L_적활성`(1=살아있음 0=빈슬롯)
- 아군 진영(적이 스캔): `L_아군x` `L_아군y` `L_아군hp` `L_아군활성`
- 스폰 시 유닛이 `활성==0`인 첫 슬롯을 찾아 자기 index 로 등록(`내슬롯`). 매 프레임 자기 x/y/hp 를 슬롯에 기록. 사망 시 `활성[슬롯]=0`.
- **스캔은 index 1..7 반복(상수 7회)** — 캡으로 묶여 있어 싸다.

## 유닛 자율 AI (per-clone, throttle 스캔)
- 유닛은 **~0.3초마다(재조준타이머)** 상대 로스터(7슬롯)를 스캔해 활성 슬롯 중 **가장 가까운 적 1기를 조준**(`목표x`/`목표y` 로컬 저장). 매 프레임 스캔 금지(throttle 필수).
- 조준 대상이 사거리 밖이면 접근(그쪽으로 이동), 사거리 안이면 정지·견제하며 **실탄 발사**. 살아있는 적이 없으면 적 본진(y=150)으로 전진해 본진 사격.

## 실탄 총격 (collision 명중)
- 사격 시 **총알 클론**을 조준 방향으로 발사. 아군총알/적총알 = 별도 스프라이트 2개(collision 판정용).
- **명중 판정 = O(1) collision**: 유닛이 매 프레임 `touching [적총알]?` 검사 → 맞으면 자기 HP 감소 + `팝업요청`(자기 위치·데미지). 총알은 `touching [적유닛]?` 이면 `delete this clone`. (리스트 스캔은 조준용에만; 명중은 collision 으로.)
- **총알캡 ≤30**(진영별 카운터) + 발사쿨 + 화면 밖(y>185/<-185)·명중 즉시 `delete this clone`. 누수 0.

## 유닛 3종(원거리)
| 종류 | 이름 | 코스트 | 소환쿨 | HP | 이동 | 사거리 | 발사쿨 | 성격 |
|---|---|---|---|---|---|---|---|---|
| 1 | 소총병 | 30 | 0.7s | 20 | 보통 | 중(110) | 중 | 균형 |
| 2 | 제압사수 | 55 | 1.8s | 26 | 느림 | 근중(85) | 짧(고DPS) | 근중거리 압박 |
| 3 | 저격수 | 50 | 2.0s | 8 | 보통 | 장(190) | 김 | 장거리 물몸 |
> 적 유닛 = 같은 3종 재활용(색 반전). 적 AI 타이머로 자동 스폰(적수<7).

## 스프라이트 (~11 + Stage)
- **Stage**: 전역 초기화(로스터 리스트 포함) + 돈 자동충전 + 소환쿨 감소 + 적 AI 스폰 타이머(적수<7) + 승패 감시.
- **아군유닛 / 적유닛**: 스포너+클론 본체. 슬롯 등록, throttle 재조준, 접근/사격(총알 클론 발사), collision 피격(touching 적총알), 사망(슬롯 해제·팝·삭제). 진영 미러.
- **아군총알 / 적총알**: 총알 클론. 조준 방향 직진, `touching [상대유닛]`→명중 삭제, 화면 밖→삭제, 진영 총알수 캡.
- **데미지팝업**: 팝업요청 시 팝업수<캡20 일 때만 클론, 떠오르며 fade, 0.5초 자동삭제.
- **버튼소총/버튼제압/버튼저격**: 클릭 폴링+디바운스. 돈≥코스트 & 소환쿨끝 & 아군수<7 이면 즉시 소환(구역 지정 없음 — 열린 필드라 버튼만으로 스폰).
- **아군본진바/적본진바/지갑바**: 11단 costume-fill.
- **결과배너**: 승/패.

## 검증 (verify_vm.js, headless 실측)
1. 소환 게이트(돈·쿨·캡) + 돈 차감.
2. **하드캡 강제**: 아군·적 각각 동시 유닛 절대 ≤7(초과 소환/스폰 거부) — 실측 관찰.
3. **자율 조준·사격**: 유닛이 가까운 적 조준→총알 발사, 재조준이 throttle(재조준타이머, 매 프레임 아님)임을 코드 구조로 확인.
4. **실탄 명중**: 총알이 touching 으로 명중→대상 HP 감소+팝업, 명중/화면밖 즉시 삭제, 동시 총알 ≤30, 누수 0.
5. 팝업 캡20 + 자동삭제.
6. 승패: 적 본진HP 0 승리 / 아군 0 패배, 배너.
7. 경제 루프.
8. **60초 자동 안정성**: 유닛 ≤14·총알 ≤30·팝업 ≤20 캡 준수하며 프레임 붕괴/클론 누수 없이 승패 결착.

## 기술 규칙
캡 하에 리스트 스캔 허용(단 index 1..7 상수 반복·throttle) / 명중은 collision(touching) O(1) / 총알 화면밖·명중 즉시 삭제 + 진영별 총량 캡 / 유닛 동시 캡7 / vrep 리포터 / 클릭 폴링+디바운스 / say 금지 / `복제됨` 증식 가드 / synth 사운드(pop/pew/tick/crack) / add_comment 4~5개(캡·throttle 스캔·collision 명중·경제 설명).
