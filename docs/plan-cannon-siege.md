# 대포 공성전 (cannon-siege) — Plan

> 좌하단 고정 발사대(슬링샷/대포)에서 발사체를 **포물선**으로 쏴, 우측의 성벽 블록·과녁을 맞춰 파괴하는 **드래그 조준 물리 게임**(앵그리버드 단순화판). 발사체를 마우스로 **누르고 당기면(드래그)** 당긴 반대 방향으로 발사 벡터가 정해지고, 당기는 동안 **점선 궤적(펜)** 이 실시간으로 휘어 보인다. 마우스를 놓으면 그 궤적 그대로 포탄이 날아간다. 더 세게·다른 방향으로 당길수록 궤적이 눈에 띄게 달라져 **당김 벡터 → 궤적** 물리 인과가 선명하게 보인다.
> 베이스: `games/magic-survivor/build.py`(한글 튜닝 변수 **일괄 초기화**·매직넘버 0 / 클론 스포너 + `복제됨` 가드 / 폭발 연출 / say 미사용 플로팅 숫자 / 전용 합성 효과음 `_wav_bytes`·`synth_*` / `add_comment` 가이드 투어) + **펜 곡선 미리보기 + 수치 모니터 패턴**(발사 전 점선 궤적·당김 벡터 수치 노출). 드래그 입력은 스크래치 `마우스를 누르고 있는가` + `마우스 x/y` 로 구현.
> 학습 콘셉트 있음(**포물선 운동·각도-사거리·속도 성분 분해·중력/바람의 벡터 영향**) — 단, 추상 그래프가 아니라 **손으로 당겨 쏘는 물리 놀이**로 포장. 당김 벡터 자체가 벡터라 **드래그하며 X·Y 성분을 직접 만진다**. [[feedback-game-design]]·[[game-success-shape]] 준수: **단일 초점(발사대 하나) + 경계 명확한 전투(포탄 vs 표적)** — 자율 난전·다수 대 다수 금지. 시후(초등·성격 급함)를 위해 **당기는 손맛**(드래그 중 궤적·벡터 수치 실시간)과 **탄착·파괴 손맛**(명중음·플로팅 점수), **즉시 재조준**(놓으면 바로 다음 발 준비). ★★★☆☆.

- **주제**: 포물선 드래그(슬링샷) 조준 물리 (앵그리버드류 공성전)
- **카테고리**: 벡터 / 물리 (포물선 운동)
- **난이도**: ★★★☆☆
- **폴더**: `games/cannon-siege/`
- **출력 파일**: `대포_공성전.sb3`

## 1. 게임 한 줄

좌하단 발사대에서 발사체를 마우스로 **누르고 당겨(드래그)** 포탄을 포물선으로 쏘고, 우측 성벽 블록과 과녁을 맞춰 부순다. 당긴 반대 방향으로 날아가며, 당기는 동안 점선 궤적이 실시간으로 휘어 보여 어디로 날아갈지 확인하며 조준한다(마우스를 놓으면 발사). 한 스테이지의 표적을 정해진 포탄 수 안에 모두 부수면 다음 스테이지로. 중력·바람 값을 바꾸면 같은 당김이라도 궤적이 확 달라진다.

## 2. 화면 레이아웃 (480×360)

좌표계: SVG/무대 픽셀(0~480, 0~360) ↔ Scratch 좌표(-240~240, -180~180). 변환: `scratchX = svgX - 240`, `scratchY = 180 - svgY`.

```
 (-240,180)                                            (240,180)
   ┌──────────────────────────────────────────────────────┐
   │ 🎯스테이지:１  💣남은포탄:５  ⭐점수:０                   │  ← 좌상단 HUD 모니터 3종
   │ 당김X -60  당김Y 48  크기 77  각도 39°                   │  ← 드래그 벡터 수치 모니터(실시간)
   │        ·                                                │
   │      ·   ·  ·                        ▓▓                 │  점선 궤적(펜): 당김 반대방향 예측 포물선
   │    ·         ·  ·                    ▓▓  ← 성벽 블록     │  블록: 우측에 세로로 쌓인 클론들
   │   ·            ·  ·                  ▓▓▒▒                │  ▒▒=과녁(맞히면 보너스 점수)
   │  ·   포탄 궤적    ·  ·             ▓▓▓▓                  │  포탄: 앵커점에서 발사, 궤적 따라 비행
   │ ▟●╲  ← 발사체를 당김(앵커→마우스)    ·  ·  ▓▓▓▓          │  발사대: 좌하단 고정 앵커. 마우스로 당김
   │████╲═══◉ 마우스 위치(당기는 손)══════════════════════════│  ← 지면(y=-150). 포탄이 닿으면 착탄
   │  드래그: 발사체를 누르고 당기면 반대방향 벡터로 발사 준비   │  ← 슬라이더/버튼 없음(전부 드래그로)
   └──────────────────────────────────────────────────────┘
 (-240,-180)                                          (240,-180)

  조준: 발사체를 마우스로 누르고 당김 → 당김 벡터(앵커−마우스) 실시간 → 점선 궤적이 다시 그려짐
  발사: 마우스를 놓으면 발사 → 포탄이 궤적 따라 비행 → 블록/과녁/지면/화면밖 닿으면 착탄
  블록 전부 파괴 → 스테이지 클리어(다음 스테이지, 표적 배치 강화) / 포탄 소진+블록 잔존 → 스테이지 재시도
```

- **발사대(앵커)**: 좌하단 고정(Scratch `(-190, -110)` = 앵커 `앵커X`/`앵커Y`). 발사체는 조준 중 앵커에 얹혀 있다가, 드래그하면 마우스를 따라 뒤로 당겨진다. 발사 원점은 항상 앵커.
- **HUD 모니터**: 좌상단에 `스테이지`(진행) `남은포탄` `점수` 3종 + **드래그 벡터 수치** `당김X`·`당김Y`·`당김크기`·`당김각도` 4종(벡터 분해를 실시간 노출 — 학습 핵심). 나머지 튜닝·내부 변수는 모니터 숨김.
- **드래그 조준(슬라이더·발사버튼 없음)**: 발사체 스프라이트를 `마우스를 누르고 있는가` + `마우스 x/y` 로 잡아 당긴다. 당김 벡터 = `(앵커X − 마우스x, 앵커Y − 마우스y)`(당긴 반대 방향으로 날아감). 이 벡터를 **`최대당김거리` 로 클램프**한 뒤 `발사력배율` 을 곱해 초기 속도로 변환. 마우스를 놓는 순간 발사. 발사체가 앵커에서 당겨진 위치로 움직이는 것 + 앵커↔발사체 사이 당김선(펜/코스튬)이 손맛을 준다.
- **궤적 미리보기**: `궤적펜` 스프라이트가 드래그 중 매 프레임 **현재 당김 벡터·중력·바람으로 포탄 물리를 dry-run** 하며 점선을 그린다(펜 곡선 미리보기 골격 — 포물선 물리 적분을 스텝별로 찍음). 드래그하지 않을 땐 그리지 않음.
- 평면 2D 물리(옆에서 본 시점). 중력은 아래(−y), 바람은 수평(±x) 가속. 당김·궤적·포탄 모두 같은 물리 상수(`중력`,`바람`,`발사력배율`,`최대당김거리`)와 같은 당김 벡터를 공유하므로 **점선과 실제 비행이 정확히 일치**한다(핵심 요건).

> **성벽 블록 배치 — 스테이지별 리스트로 좌표·타입 정의.** Stage 깃발에서 `블록X`/`블록Y`/`블록타입` 리스트에 순서대로 add. 블록 원본이 리스트를 훑어 클론을 찍는다. **표적을 바꾸려면 이 리스트만 고치면 됨**(개조 4층 미션).
>
> | 스테이지 | 블록 배치(예) | 과녁 위치 |
> |----------|---------------|-----------|
> | 1 | 3층 세로 벽 1개(x≈150) | 맨 위 1칸 |
> | 2 | 벽 2개(x≈120, 190) + 사이 빈 공간 | 오른쪽 벽 중간 |
> | 3 | 계단식 3층 + 공중 과녁(점프 궤적 필요) | 공중 우상단 |
> | 4+ | 랜덤 높이 벽 + 과녁 2개 | 랜덤 |

## 3. 스프라이트 / 코스튬 / 초기 상태

| # | 스프라이트 | 역할 | 코스튬 | 초기 상태 |
|---|-----------|------|--------|-----------|
| 0 | Stage | 배경(하늘·언덕·지면) + 전역 변수 전부 초기화 + 스테이지 매니저 + 블록 리스트 | `bg` | — |
| 1 | 발사대 | 좌하단 슬링샷/대포 몸체(고정 앵커). 앵커점을 시각화(Y자 슬링샷 프레임) | `frame` | (-190,-125) 표시 |
| 2 | 발사체 | **드래그로 잡아 당기는 발사체**(조준 중 앵커에 얹힘). 마우스로 당기면 따라오고, 놓으면 발사·포탄 클론 생성 | `ball`, `boom` | 앵커 위 표시, 드래그 대기 |
| 3 | 당김선 | 앵커↔발사체 사이 고무줄(당기는 손맛). 드래그 중만 펜/코스튬으로 표시 | `band`(펜 또는 늘어나는 코스튬) | 숨김, 드래그 중만 |
| 4 | 궤적펜 | 드래그 중 점선 포물선 미리보기(펜). 드래그 안 할 땐 안 그림 | `dot`(작은 점, 숨김·펜만) | 숨김, 펜만 사용 |
| 5 | 포탄 | 발사되어 궤적 따라 비행하는 탄(클론 1개씩). 착탄 시 폭발·삭제 | `ball`, `boom` | 숨김(발사 때 클론) |
| 6 | 블록 | 성벽 블록·과녁(스포너 + 클론 본체). 포탄 맞으면 체력↓·파괴 | `wood`, `stone`, `target`, `rubble` | 숨김(스테이지마다 클론) |
| 7 | 숫자팝업 | 명중 시 뜨는 플로팅 점수 숫자(say 미사용, 코스튬 0~9) | `d0`~`d9`(흰/금 20장) | 숨김(클론) |
| 8 | 배너 | 스테이지 클리어 / 스테이지 실패(재시도) / 최종 배너 | `clear`, `retry`, `win` | 숨김 |

> 타깃 **9개**(슬라이더 손잡이 2개·발사버튼 제거, 당김선 1개 추가). 조준 안내 텍스트는 배경(`bg`)에 그려 스프라이트 수 절약.
>
> **발사체 vs 포탄 분리 이유**: `발사체`(스프라이트 2)는 조준 중 앵커에 얹혀 드래그당하는 "당김 표시용" 본체이고, 놓는 순간 `포탄`(스프라이트 5) 클론 1개를 생성해 그 클론이 물리 비행을 담당한다. 발사체 본체는 즉시 앵커로 되돌아와 다음 발을 대기(발사체는 절대 자기가 날지 않음 — 클론 관리 단순화). 코스튬은 둘 다 같은 `ball` 공유 가능.

## 4. 사운드 — 전용 합성(build.py `_wav_bytes` + `synth_*`), orphan 0

magic-survivor 처럼 **결정적 합성 WAV** 를 만들어, 각 스프라이트에 **실제 재생하는 것만** 등록(고아 0). 모든 합성 함수는 `synth_zap`/`synth_boom` 형태(샘플 리스트 → `_wav_bytes(samples)` → md5). 재생은 대부분 `play sound`(대기 안 함).

| 합성 함수 | 이름 | 음색 스펙(결정적) | 등록 스프라이트 | 트리거 |
|-----------|------|-------------------|------------------|--------|
| `synth_stretch` | stretch | 고무줄 당김 "삐긋"(상승 피치 200→500Hz 짧은 톤, 0.1초) | 발사체 | 드래그 시작 시 1회(선택·손맛) |
| `synth_fire` | fire | 놓으며 튕기는 "핑!"(노이즈버스트+80Hz thump, 0.18초, `synth_boom` 변형) | 포탄 | 발사(마우스 놓는) 순간 |
| `synth_whoosh` | whoosh | 짧은 바람 스침(밴드패스 노이즈 400→900Hz, 0.15초, 페이드) | 포탄 | 비행 시작 시 1회(선택) |
| `synth_hitwood` | hitwood | 나무 "딱"(1.2kHz 클릭+짧은 링, 0.06초) | 블록 | 나무 블록 피격 |
| `synth_hitstone` | hitstone | 돌 "탁"(둔탁 저역 클릭 300Hz, 0.08초) | 블록 | 돌 블록 피격 |
| `synth_break` | break | 블록 붕괴 "와르르"(노이즈 하강+thump, 0.22초) | 블록 | 블록 파괴 |
| `synth_ding` | ding | 과녁 명중 "딩동"(두 톤 988→1319Hz 정현파, 0.15초) | 블록 | 과녁 명중(보너스) |
| `synth_ground` | ground | 지면 착탄 "털썩"(저역 thump 120Hz, 0.12초) | 포탄 | 지면/화면밖 착탄(빗나감) |
| `synth_clear` | clear | 상승 아르페지오(523→659→784→1047Hz, 0.35초) | 배너 | 스테이지 클리어 |
| `synth_fail` | fail | 하강 버저(400→150Hz 사각파, 0.25초) | 배너 | 포탄 소진 실패 |

> 등록 규칙: 발사체=stretch, 포탄=fire/whoosh/ground, 블록=hitwood/hitstone/break/ding, 배너=clear/fail. 다른 스프라이트엔 사운드 없음(고아 0). pitch 이펙트는 콤보 명중 시 살짝 변주 가능(선택). 총 **10종**.

## 5. 변수 / 리스트 / 메시지

### 5.1 ⚙️ 개조 손잡이 (튜닝 변수) — 한글 전역 변수, ID `V_*` / `var*`

> **이 표가 게임의 심장.** 모든 조절 값은 여기 한글 변수로만 존재하고 코드 어디서도 매직넘버를 쓰지 않는다(예: `change 속도Y by (0 - 중력)`, `change 속도X by 바람`, `set 속도X = 당김X × 발사력배율`). 초기화는 **전부 Stage 깃발 클릭 한 스크립트**에 모은다. (연출용 `repeat 8` 폭발 같은 소수 상수만 인라인.)

#### 조준 / 물리 손잡이 (★ 이 4개가 학습의 핵심 — 하나 바꾸면 궤적이 눈에 띄게 달라짐)
| 한국어 | ID | 기본값 | 의미 / 바꾸면 일어나는 일 |
|--------|----|--------|---------------------------|
| 발사력배율 | `varPowMul01` | 0.16 | **당김 거리 → 초기 속도 환산 배율.** 초기 속도 = 당김 벡터 × 이 값. ↑ 하면 같은 힘으로 당겨도 더 멀리 날아감(기본 클램프 80px × 0.16 ≈ 12.8 px/tick) |

> - 당김 벡터: `당김X = 앵커X − 마우스x`, `당김Y = 앵커Y − 마우스y` (당긴 반대 방향)

> - 참고: 각도는 이제 **당김에서 파생**된다(`당김각도 = atan(당김Y / 당김X)`). 45° 근처에서 사거리 최대는 여전히 성립 — HUD 의 `당김각도` 모니터로 관찰

[다시 시작]  초록 깃발 재클릭(스테이지=1, 점수=0 리셋)
```

> **템포 설계(시후용)**: 발사와 조준 사이 대기 없음 — 착탄하면 즉시 조준 상태로 돌아가 발사체가 앵커에 다시 얹혀 당길 수 있다. 발사·명중·붕괴에 각각 효과음+연출(폭발·플로팅 점수)로 손맛. 드래그는 조준중 항상 반응, 당김선·벡터 수치·궤적 점선이 실시간으로 휘어 인과가 즉각 보인다. **드래그 한 동작으로 조준+발사가 끝나므로**(당겼다 놓기) 슬라이더 여러 개를 만지던 것보다 빠르고 직관적.

## 7. 블록 흐름 (스프라이트별 의사코드)

> 모든 forever 루프 끝에 `wait`(0.02~0.03초)로 CPU 폭주 방지. **매직넘버 금지** — 항상 튜닝 변수 참조. 드래그 입력은 `마우스를 누르고 있는가`(`sensing_mousedown`) + `마우스 x`/`마우스 y`(`sensing_mousex/y`) 로 구현.

> **공통 ①: 당김 벡터 계산 + 클램프**(발사체 조준 & 궤적 dry-run 이 모두 이 결과를 씀 — 전역 `당김X`/`당김Y`/`당김크기`/`당김각도`에 세팅)
> ```
> 당김X = 앵커X - (마우스 x)          # 당긴 반대 방향
> 당김Y = 앵커Y - (마우스 y)
> 당김크기 = sqrt(당김X*당김X + 당김Y*당김Y)
> if 당김크기 > 최대당김거리:          # 최대 당김 상한 클램프
>   당김X = 당김X * 최대당김거리 / 당김크기
>   당김Y = 당김Y * 최대당김거리 / 당김크기
>   당김크기 = 최대당김거리
> 당김각도 = atan(당김Y / 당김X)        # (당김X=0 가드: 0 이면 90) — HUD 관찰용
> ```
> **공통 ②: 속도 성분 + 물리 적분**(포탄 발사 & 궤적 dry-run 공통 — 반드시 동일 공식, 위 클램프된 당김 벡터 사용)
> ```
> 속도X = 당김X * 발사력배율      # 수평 성분 = 당김 X 성분 × 배율
> 속도Y = 당김Y * 발사력배율      # 수직 성분 = 당김 Y 성분 × 배율
> # 매 스텝:
> 속도Y = 속도Y - 중력           # 중력이 위 성분을 깎음
> 속도X = 속도X + 바람            # 바람이 옆으로 밈
> x = x + 속도X ; y = y + 속도Y
> ```
> > **당김 벡터 X·Y 성분이 곧 속도 성분** → 벡터 분해를 손으로 만진다. 대각선 45°로 당기면 속도X·속도Y 가 같아져(코스·사인 반반) 사거리 최대(바람0·평지).

### 발사체 (ball) — 드래그 조준 + 놓으면 발사
```
when green flag clicked
  go to x:앵커X y:앵커Y ; show ; switch costume to ball
when I receive 게임시작
  forever
    if <<<게임상태 = 1> and <비행중 = 0>> and <남은포탄 > 0>>
      # ── 이 발사체 위에서 마우스 다운 시작 → 드래그 잡기 ──
      if <<mouse down?> and <touching mouse-pointer?>>
        set 드래그중 = 1 ; play sound stretch
        repeat until <not <mouse down?>>          # 당기는 동안
          # 공통① 당김 벡터 계산 + 클램프 (당김X/Y/크기/각도 세팅)
          set 당김X = 앵커X - (mouse x) ; set 당김Y = 앵커Y - (mouse y)
          set 당김크기 = sqrt((당김X*당김X)+(당김Y*당김Y))
          if <당김크기 > 최대당김거리>
            set 당김X = 당김X * 최대당김거리 / 당김크기
            set 당김Y = 당김Y * 최대당김거리 / 당김크기
            set 당김크기 = 최대당김거리
          end
          if <당김X = 0> : set 당김각도 = 90 else : set 당김각도 = (abs of (atan(당김Y / 당김X)))
          # 발사체를 당겨진 위치(앵커에서 당김 반대쪽 = 마우스 쪽, 클램프 반영)로 이동
          go to x:(앵커X - 당김X) y:(앵커Y - 당김Y)
          wait 0.01
        end
        # ── 마우스 놓음 → 발사 ──
        set 드래그중 = 0
        if <당김크기 > 4>                          # 살짝만 눌렀다 뗀 오발 방지(소수 상수 인라인 허용)
          change 남은포탄 by -1
          broadcast 발사                            # 포탄 클론 생성(현재 당김X/Y 로 발사)
        end
        go to x:앵커X y:앵커Y                        # 발사체 본체는 앵커로 복귀(다음 발 대기)
      end
    end
    wait 0.02
```
> **드래그 잡기**: `touching mouse-pointer?` + `mouse down?` 로 발사체를 누르기 시작했을 때만 당기기 시작. 당기는 동안 `당김X/Y`(공통①)를 매 틱 세팅 → HUD 벡터 모니터·궤적펜이 그 값을 읽음. **비행중=0·게임상태=1·남은포탄>0 가드**로 비행 중 재발사·오발 방지(안정성). 발사체 본체는 절대 날지 않고 놓으면 앵커로 복귀 — 실제 비행은 `포탄` 클론이 담당.

### 당김선 (band) — 앵커↔발사체 고무줄(드래그 손맛)
```
when green flag clicked
  hide ; set pen size 5 ; set pen color 갈색
when I receive 게임시작
  forever
    erase all           # (당김선 전용 펜 레이어 — 궤적펜과 별개면 각자 erase; 한 레이어면 궤적펜이 담당)
    if <드래그중 = 1>
      go to x:앵커X y:앵커Y ; pen down
      go to x:(앵커X - 당김X) y:(앵커Y - 당김Y)   # 당겨진 발사체 위치까지 선
      pen up
    end
    wait 0.02
```
> 단순화: 당김선을 펜 대신 **앵커~발사체 사이를 잇는 늘어나는 코스튬**(중점 배치+회전+세로 stretch)으로 그려도 됨(builder 재량). 펜 레이어를 궤적펜과 공유하면 erase 충돌 주의 — 별도 스프라이트로 분리했으니 각자 erase.

### 궤적펜 (dry-run 포물선 점선 미리보기) — 펜 미리보기 패턴 재사용
```
when green flag clicked
  hide ; set pen size 3 ; set pen color 흰반투명
when I receive 게임시작
  forever
    erase all
    if <<<게임상태 = 1> and <비행중 = 0>> and <드래그중 = 1>>   # 당기고 있을 때만 미리보기
      # 공통②: 당김 벡터(공통①에서 이미 세팅됨) × 발사력배율 → dry-run
      set 미리X = 앵커X ; set 미리Y = 앵커Y
      set 미리VX = 당김X * 발사력배율
      set 미리VY = 당김Y * 발사력배율
      pen up ; go to x:미리X y:미리Y ; pen down
      set i = 0
      repeat 미리보기점수                          # 최대 미리보기점수 스텝 dry-run
        change 미리VY by (0 - 중력)                # 실제 포탄과 완전히 동일한 물리
        change 미리VX by 바람
        change 미리X by 미리VX
        change 미리Y by 미리VY
        change i by 1
        if <(i mod 미리보기간격) = 0>              # 간격마다 점 하나(점선 효과)
          pen up ; go to x:미리X y:미리Y ; pen down
        else
          pen up ; go to x:미리X y:미리Y            # 점 사이는 펜 안 찍고 이동만
        end
        if <<미리Y < 지면Y> or <(abs of 미리X) > 착탄허용밖>>
          stop this script’s repeat (exit)          # 지면/화면밖 도달하면 그만 그림
        end
      end
    end
    wait 0.03
```
> **핵심**: dry-run 이 포탄과 **똑같은 공통② 4식**(속도Y−중력, 속도X+바람, x+속도X, y+속도Y)을, **같은 클램프된 당김 벡터**(`당김X/Y × 발사력배율`)로 돈다 → 점선이 실제 궤적과 정확히 일치. 점선 효과는 `미리보기간격` 마다만 `pen down` 흔적을 남김(간단히는 `stamp`/짧은 선도 OK — builder 재량). 드래그를 놓으면(`드래그중=0`) `erase all` 만 돌아 점선이 지워지고, 발사되면 포탄만 남는다.

### 포탄 (cannonball) — 스포너 + 클론 본체(실제 물리 비행)
```
when green flag clicked
  set 복제됨 = 0 ; hide
when I receive 발사
  if <복제됨 = 0>
    create clone of myself             # 원본만 클론 1개 생성
when I start as a clone
  set 복제됨 = 1
  set 게임상태 = 2 ; set 비행중 = 1
  go to x:앵커X y:앵커Y ; switch costume to ball ; show ; play sound fire
  # 공통②: 발사 순간의 당김 벡터로 초기 속도 성분 (궤적펜과 동일)
  set 속도X = 당김X * 발사력배율
  set 속도Y = 당김Y * 발사력배율
  repeat until <<<y position < 지면Y> or <(abs of x position) > 착탄허용밖>> or <touching 블록? within 포탄반경>>
    change 속도Y by (0 - 중력)          # 매 틱 물리 적분(궤적펜과 동일)
    change 속도X by 바람
    change x by 속도X
    change y by 속도Y
    point in direction towards (x+속도X, y+속도Y)   # 진행 방향으로 회전(연출)
    wait 0.02
  end
  # ── 착탄 처리 ──
  if <touching 블록?>                   # 블록 명중
    set 폭발X = x ; set 폭발Y = y        # (블록타격 채널)
    broadcast 블록타격 and wait          # 반경 안 블록이 데미지·팝업 처리
  else
    play sound ground                    # 지면/화면밖 = 빗나감
  end
  switch costume to boom ; repeat 8 (change size, ghost) ; hide
  set 비행중 = 0
  broadcast 착탄
  delete this clone
```
> `touching 블록? within 포탄반경`: 실제로는 `touching [블록]?`(코스튬 크기로 근사) 또는 블록 쪽에서 거리 판정. **광역 아님 — 포탄이 닿은 그 지점**. `블록타격 and wait` 로 반경 안 블록에 데미지(대개 1~2칸). 착탄 후 `비행중=0`·`착탄` 방송 → Stage 감시 forever 가 클리어/실패/계속을 판정.

### 블록 (성벽·과녁) — 스포너 + 클론 본체
```
when green flag clicked
  set 복제됨 = 0 ; hide ; set 블록남음 = 0
when I receive 스테이지준비
  if <복제됨 = 0>
    wait 0.05                            # Stage 가 리스트 다 채우도록 양보
    set 블록남음 = 0
    set i = 1
    repeat (length of 블록X)
      set 다음X = item i of 블록X
      set 다음Y = item i of 블록Y
      set 다음T = item i of 블록타입
      create clone of myself
      change 블록남음 by 1
      change i by 1
    end
when I start as a clone
  set 복제됨 = 1
  set 내타입 = 다음T                      # (스포너가 직전에 세팅한 채널)
  go to x:다음X y:다음Y
  if <내타입 = 1> : switch costume to wood  ; set 내체력 = 표적_체력
  if <내타입 = 2> : switch costume to stone ; set 내체력 = 표적_체력 * 돌체력배수
  if <내타입 = 3> : switch costume to target; set 내체력 = 표적_체력
  show
  # 피격 감시
  forever
    if <touching 포탄?>
      change 내체력 by (0 - 명중데미지)
      if <내타입 = 2> : play sound hitstone else play sound hitwood
      if <내타입 = 3> : play sound ding
      if <내체력 < 1>
        # 점수 + 팝업
        if <내타입 = 3> : change 점수 by 과녁점수 ; set 명중점수값=과녁점수 ; set 팝업종류=1
                     else : change 점수 by 블록점수 ; set 명중점수값=블록점수 ; set 팝업종류=0
        set 명중점수x = x position ; set 명중점수y = y position
        broadcast 점수표시
        play sound break
        switch costume to rubble ; repeat 6 (change ghost) 
        change 블록남음 by -1
        delete this clone
      else
        # 살아남음: 살짝 흔들림 연출(선택)
      end
      wait until <not <touching 포탄?>>   # 한 번 닿을 때 한 번만 처리(연타 방지)
    end
    wait 0.03
```
> **주의(스포너 채널 경쟁)**: 리스트 인덱스별 `다음X/다음Y/다음T` 를 전역 채널로 쓰면 클론 생성이 매우 빠르면 값이 덮일 수 있음 — magic-survivor 처럼 `create clone` **직후 클론이 즉시 로컬로 복사**하도록 스포너 루프에 `wait 0`(또는 아주 짧은 wait) 넣거나, `다음X/Y/T` 를 세팅→`create clone`→클론이 `start as clone` 첫 줄에서 로컬 저장 순서 보장. 블록 수가 적어(≤6) 안전하나 builder 는 순서 지킬 것.

## 9. 🎯 개조 미션 사다리 (README 에 수록)

> "숫자 바꾸기 0층"에서 "관계식 1층", "조건 2층", "새 메커닉 4층"으로. 각 미션은 5.1 튜닝 변수 또는 Stage (A) 초기화/블록 리스트만 건드린다.

### 0층 — 숫자 하나 바꾸기 (무대 초록 깃발 스크립트)
- **달나라 대포**: `중력` 을 `0.35` → `0.1` 로. 포탄이 둥실 멀리 날아간다(달 중력).
- **목성 대포**: `중력` 을 `0.35` → `0.8` 로. 포탄이 뚝뚝 금방 떨어진다.
- **강풍 주의보**: `바람` 을 `0` → `0.15` 로. 궤적이 오른쪽으로 쭉 휜다(점선도 휘는지 확인!).
- **역풍**: `바람` 을 `0` → `-0.15` 로. 이번엔 왼쪽으로 휜다.
- **힘센 대포**: `발사력배율` 을 `0.16` → `0.28` 로. 같은 세기로 당겨도 훨씬 멀리 날아간다.
- **멀리 당기기**: `최대당김거리` 를 `80` → `140` 으로. 더 많이 뒤로 당길 수 있어 최대 사거리가 늘어난다.
- **넉넉한 포탄**: `시작포탄수` 를 `5` → `10` 으로. 실수해도 여유롭게.
- **단단한 성**: `돌체력배수` 를 `3` → `6` 으로. 돌 블록을 여러 번 쳐야 부서진다.
- **철벽 표적**: `표적_체력` 을 `1` → `3` 으로. 모든 블록·과녁이 여러 번 맞아야 부서진다.
- **큰 포탄**: `포탄반경` 을 `10` → `25` 로. 스치듯 지나가도 명중.
- **45° 실험**: 같은 세기로 당기되, **거의 수평(낮게)·45°(대각선)·거의 수직(높이)** 세 방향으로 당겨 쏴 보자. HUD `당김각도` 를 보며 **45° 근처에서 제일 멀리 가는지** 눈으로 확인!

### 1층 — 값을 식으로 묶기 (관계 만들기)
- **비례 점수**: 블록(D) 팝업 점수를 `블록점수` 대신 `블록점수 × 스테이지` 로 — 후반 스테이지일수록 점수 팍팍.
- **바람 연동 난이도**: Stage (C) 에서 `set 바람 = 스테이지 × 0.05` 로 — 스테이지가 오를수록 바람이 세져 조준이 어려워진다.
- **힘 비례 데미지**: 포탄 착탄 시 `명중데미지 = 1 + floor(당김크기/40)` 로 — 세게 당길수록 더 큰 데미지(원하면).

### 2층 — if 조건 더하기
- **직격 보너스**: 블록(D)에 `if (내타입=3) and (남은포탄 = 시작포탄수 - 1)` (첫 발에 과녁) → 점수 2배 보너스.
- **바람 경고등**: Stage 에 `if (abs of 바람 > 0.1) : (배경/텍스트로 '강풍!' 표시)`.
- **콤보**: 한 발로 블록 2개 이상 부수면(카운트 세서) 추가 점수.

### 3층 — 규칙 자체를 바꾸기 (분기)
- **랜덤 바람**: Stage (C)에서 `set 바람 = pick random -15 to 15 / 100` 로 매 스테이지 바람이 랜덤 — 매번 새로 조준.
- **튕기는 포탄**: 포탄 비행 루프에서 `if (y position < 지면Y) : set 속도Y = 속도Y × -0.5`(1회 한정) → 지면에서 한 번 튕겨 굴러가게(반발 계수 실험).
- **폭발 포탄**: 착탄 시 `포탄반경` 을 잠깐 크게(범위 폭발) → 주변 블록 여러 개 동시 파괴.

### 4층 — 새 메커닉 만들기 (구조)
- **성벽 짓기**: Stage (C) 의 `블록X`/`블록Y`/`블록타입` 리스트에 `add` 를 더해 나만의 성벽·과녁 배치를 만들자.
- **움직이는 과녁**: 과녁 클론(내타입=3)에 좌우로 왔다갔다 하는 forever 이동을 더해, 조준이 어려운 움직이는 표적으로.
- **당김 최대치 눈금**: 발사대에 `최대당김거리` 반경의 얇은 원(가이드)을 그려, 어디까지 당기면 최대 힘인지 보이게(펜/코스튬).
- **경사면**: 지면을 평지 대신 언덕(색 감지)으로 바꿔 착탄 판정을 `touching color 지면색` 으로 — 지형이 있는 사격.

## 10. 학습 포인트

**포물선 운동과 벡터를 손으로 당겨 쏘며 만나는 물리 게임.** 추상 그래프가 아니라, 발사체를 마우스로 당기면 **당김 벡터가 곧 발사 벡터가 되고**(당긴 반대 방향) 점선 궤적이 실시간으로 휘며, 놓으면 그 궤적대로 포탄이 날아간다. 다음 직관이 게임 행동 자체로 체득된다:

- **속도 성분 분해(벡터)**: 당김 벡터 `(당김X, 당김Y)` 가 그대로 초기 속도 성분 `속도X = 당김X·발사력배율`(옆)·`속도Y = 당김Y·발사력배율`(위) 이 된다. 더 위로 당기면 위 성분↑·옆 성분↓ — **드래그하며 X·Y 성분을 직접 만지고**, HUD `당김X`/`당김Y` 수치로 분해를 눈으로 확인한다(슬라이더보다 벡터를 더 직접적으로 체감).
- **각도-사거리 관계(45° 최적)**: 바람=0·평지에서 **당김각도 45° 근처일 때 사거리가 최대**. 당김의 옆·위 성분이 딱 반반이 되는 지점이 곧 최적임을, 낮게·대각선·높이 세 방향으로 같은 세기로 당겨 쏴 보며 발견한다(0층 미션 "45° 실험", HUD `당김각도` 관찰).
- **중력의 적분적 영향**: 매 틱 `속도Y ← 속도Y − 중력` 이라 포탄이 **올라갈수록 느려지다 정점에서 멈추고 다시 빨라지며 떨어진다**(등가속 운동). `중력` 을 0.1(달)↔0.8(목성)로 바꾸면 포물선이 납작해지거나 뾰족해진다.
- **바람의 벡터적 영향**: `속도X ← 속도X + 바람` 이라 궤적이 수평으로 휜다(±). 중력이 세로 벡터라면 바람은 가로 벡터 — 두 힘의 합이 실제 궤적을 만든다는 **벡터 합성**을 눈으로 본다.
- **예측 = 시뮬레이션**: 궤적펜은 실제 포탄과 **똑같은 물리식을 미리 한 번 돌려(dry-run)** 결과를 점선으로 보여준다. "계산해서 미리 안다"는 물리·프로그래밍 공통 아이디어를 체감.

부수적으로 **변수·리스트·조건·클론·펜·벡터 연산(당김X/Y·크기·각도)** 을 게임을 튜닝하며(0→4층 미션 사다리) 코딩 학습으로 이어진다. 코드 곳곳의 **노란 코멘트(가이드 투어)** 가 각 부분을 설명한다. 어디까지나 **직관적 드래그 조준 놀이**로 포장된 채 — 단일 발사대·명확한 표적이라는 성공 게임 모양([[game-success-shape]])을 지킨다.
